## Overview

Bootstrap 5 modals are flexible dialog prompts that sit on top of the main content. They consist of three primary elements: `.modal` (the wrapper), `.modal-dialog` (the positioning container), and `.modal-content` (the inner content). Modals are triggered via `data-bs-toggle="modal"` and `data-bs-target="#id"` attributes, and dismissed with `data-bs-dismiss="modal"`. The component supports scrollable content, size variants (sm, lg, xl), fullscreen modes, and vertical centering out of the box.

## Basic Implementation

```html
<!-- Trigger Button -->
<button type="button" class="btn btn-primary" data-bs-toggle="modal" data-bs-target="#basicModal">
  Open Modal
</button>

<!-- Modal Structure -->
<div class="modal fade" id="basicModal" tabindex="-1" aria-labelledby="basicModalLabel" aria-hidden="true">
  <div class="modal-dialog">
    <div class="modal-content">
      <div class="modal-header">
        <h5 class="modal-title" id="basicModalLabel">Modal Title</h5>
        <button type="button" class="btn-close" data-bs-dismiss="modal" aria-label="Close"></button>
      </div>
      <div class="modal-body">
        <p>This is the modal body content.</p>
      </div>
      <div class="modal-footer">
        <button type="button" class="btn btn-secondary" data-bs-dismiss="modal">Close</button>
        <button type="button" class="btn btn-primary">Save Changes</button>
      </div>
    </div>
  </div>
</div>
```

## Advanced Variations

### Size Variants

Use `.modal-sm`, `.modal-lg`, or `.modal-xl` on `.modal-dialog` to control modal width. The default size adapts to the viewport. Fullscreen modals use `.modal-fullscreen` or breakpoint-prefixed variants like `.modal-fullscreen-sm-down`.

```html
<!-- Small Modal -->
<div class="modal-dialog modal-sm">...</div>

<!-- Large Modal -->
<div class="modal-dialog modal-lg">...</div>

<!-- Extra Large Modal -->
<div class="modal-dialog modal-xl">...</div>

<!-- Fullscreen below lg breakpoint -->
<div class="modal-dialog modal-fullscreen-lg-down">...</div>
```

### Scrollable Modal

When content exceeds viewport height, use `.modal-dialog-scrollable` on `.modal-dialog` to scroll only the body. Combine with `.modal-dialog-centered` to vertically center the modal.

```html
<!-- Scrollable Modal Body -->
<div class="modal-dialog modal-dialog-scrollable">
  <div class="modal-content">
    <div class="modal-header">
      <h5 class="modal-title">Scrollable Modal</h5>
      <button type="button" class="btn-close" data-bs-dismiss="modal"></button>
    </div>
    <div class="modal-body">
      <p><!-- Long content here --></p>
    </div>
    <div class="modal-footer">
      <button type="button" class="btn btn-secondary" data-bs-dismiss="modal">Close</button>
    </div>
  </div>
</div>

<!-- Centered Scrollable Modal -->
<div class="modal-dialog modal-dialog-centered modal-dialog-scrollable">...</div>
```

### Centered Modal

```html
<div class="modal-dialog modal-dialog-centered">
  <div class="modal-content">
    <div class="modal-body text-center">
      <p>Vertically centered modal dialog.</p>
      <button type="button" class="btn btn-primary" data-bs-dismiss="modal">Got it</button>
    </div>
  </div>
</div>
```

## Best Practices

1. Always include `tabindex="-1"` on the `.modal` element for correct focus management.
2. Set `aria-hidden="true"` on the `.modal` element when hidden.
3. Link the modal title to `aria-labelledby` on the `.modal` element.
4. Use `aria-describedby` to reference a brief modal description for screen readers.
5. Always include a `.btn-close` button with `data-bs-dismiss="modal"` in the header.
6. Place focusable action buttons in the `.modal-footer` for consistent UX.
7. Use `backdrop: 'static'` when the user must interact with the modal before proceeding.
8. Set `keyboard: false` only when closing on Escape would cause data loss.
9. Add `.fade` class for smooth entrance/exit transitions.
10. Limit modal body content to keep dialogs focused and scannable.
11. Use `aria-modal="true"` to indicate the modal blocks interaction with content behind it.
12. Keep modals shallow in nesting depth; prefer offcanvas or full-page views for complex workflows.

## Common Pitfalls

- **Missing `tabindex`:** Without `tabindex="-1"` on `.modal`, focus does not move into the dialog on open.
- **Incorrect `data-bs-target`:** If the selector does not match the modal `id`, the modal will not open.
- **Placing modals inside other components:** Modals should be direct children of `<body>`. Nesting inside navbars or cards can break z-index and layout.
- **Forgetting `aria-labelledby`:** Screen readers will not announce the modal title without this attribute.
- **Overriding `z-index` carelessly:** Bootstrap modals use `z-index: 1055` (backdrop `1050`). Custom CSS that sets higher values can bury modals.
- **Multiple modals without management:** Opening modals on top of each other requires manual `z-index` stacking and focus handling.
- **Not calling `.hide()` before DOM removal:** Removing a modal element from the DOM without hiding it first leaves orphaned backdrops.

## Accessibility Considerations

Bootstrap automatically sets `role="dialog"` on `.modal` and manages `aria-hidden` toggling. The `aria-labelledby` attribute should reference the `.modal-title` element. When the modal opens, focus moves to the first focusable element inside the dialog. Escape key closes the modal by default. Screen readers announce the dialog title and its content as a region.

## Responsive Behavior

Bootstrap modals are responsive by default. On viewports wider than 576px, `.modal-dialog` has a max-width. Below that, modals expand to full width. Use `.modal-fullscreen-sm-down`, `.modal-fullscreen-md-down`, or similar classes to make modals fullscreen at specific breakpoints. The `.modal-sm` and `.modal-lg` sizes adjust their widths relative to the viewport using `max-width` percentages.
