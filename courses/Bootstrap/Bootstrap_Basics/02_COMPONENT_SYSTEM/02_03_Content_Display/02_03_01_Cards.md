# Cards

## Overview

Cards are one of Bootstrap's most versatile and widely-used components. A card is a flexible and extensible content container that supports a wide variety of content including images, text, list groups, links, and more. Cards replace the older panels, wells, and thumbnails components from Bootstrap 3, consolidating them into a single, unified component.

The card system in Bootstrap 5 is built around a simple markup structure. Every card starts with the `.card` class applied to a container element, typically a `div`. Inside, you use `.card-body` for the main content area, and optional elements like `.card-header`, `.card-footer`, `.card-title`, `.card-subtitle`, and `.card-text` for structured content. Media elements use `.card-img-top`, `.card-img-bottom`, or `.card-img-overlay` for image placement.

Cards are designed to work seamlessly with Bootstrap's grid system. You can place cards inside column classes to create responsive card layouts that adapt from single-column mobile views to multi-column desktop displays. The `.card-group` class provides an additional way to render cards as a unified, attached set with equal height and width.

The flexibility of cards makes them suitable for dashboards, product listings, blog post previews, user profiles, feature highlights, and countless other UI patterns. Their built-in support for headers, footers, colors, and navigation tabs or pills means you can create complex card designs with minimal markup.

Understanding the card component thoroughly is essential because cards serve as building blocks for many common UI patterns. Mastering cards means mastering a significant portion of Bootstrap's layout capabilities.

## Basic Implementation

The simplest card requires just two classes: `.card` on the wrapper and `.card-body` on the content area.

### Minimal Card

```html
<div class="card">
  <div class="card-body">
    This is a basic card with minimal markup.
  </div>
</div>
```

This produces a bordered container with padding inside the body area. The card stretches to the full width of its parent container.

### Card with Title, Text, and Button

Most cards include a title, descriptive text, and an action button. Bootstrap provides dedicated classes for these elements.

```html
<div class="card" style="width: 18rem;">
  <div class="card-body">
    <h5 class="card-title">Card Title</h5>
    <h6 class="card-subtitle mb-2 text-body-secondary">Card subtitle</h6>
    <p class="card-text">
      Some quick example text to build on the card title and make up the
      bulk of the card's content.
    </p>
    <a href="#" class="btn btn-primary">Go somewhere</a>
  </div>
</div>
```

The `.card-title` and `.card-subtitle` classes apply appropriate margin and sizing. The `.card-text` class can be used on one or more paragraphs. Note the inline `style="width: 18rem;"` to constrain the card width since cards are naturally block-level and full-width.

### Card with Header and Footer

Headers and footers provide visual separation for additional information or actions.

```html
<div class="card">
  <div class="card-header">Featured</div>
  <div class="card-body">
    <h5 class="card-title">Special title treatment</h5>
    <p class="card-text">With supporting text below as a natural lead-in to additional content.</p>
    <a href="#" class="btn btn-primary">Go somewhere</a>
  </div>
  <div class="card-footer text-body-secondary">2 days ago</div>
</div>
```

### Card with Images

Images can be placed at the top or bottom of a card using `.card-img-top` or `.card-img-bottom`. These classes round the corners of the image to match the card's border radius.

```html
<div class="card" style="width: 18rem;">
  <img src="https://via.placeholder.com/286x180" class="card-img-top" alt="Card image">
  <div class="card-body">
    <h5 class="card-title">Card with Image</h5>
    <p class="card-text">
      A quick example of a card with an image at the top.
    </p>
    <a href="#" class="btn btn-primary">View Details</a>
  </div>
</div>
```

### Card with List Groups

Cards can contain list groups, which work well for navigation-style cards or displaying collections.

```html
<div class="card" style="width: 18rem;">
  <div class="card-header">Categories</div>
  <ul class="list-group list-group-flush">
    <li class="list-group-item">Cras justo odio</li>
    <li class="list-group-item">Dapibus ac facilisis in</li>
    <li class="list-group-item">Vestibulum at eros</li>
  </ul>
  <div class="card-body">
    <a href="#" class="card-link">Card link</a>
    <a href="#" class="card-link">Another link</a>
  </div>
</div>
```

The `.list-group-flush` class removes borders and rounded corners so the list group integrates cleanly with the card.

## Advanced Variations

### Card Image Overlay

The `.card-img-overlay` class allows you to place card content directly on top of an image, similar to a background image effect. The card body becomes absolutely positioned over the image.

```html
<div class="card text-bg-dark">
  <img src="https://via.placeholder.com/500x250" class="card-img" alt="Background image">
  <div class="card-img-overlay">
    <h5 class="card-title">Card title</h5>
    <p class="card-text">
      This is a wider card with supporting text below as a natural lead-in
      to additional content. This content is a little bit longer.
    </p>
    <p class="card-text"><small>Last updated 3 mins ago</small></p>
  </div>
</div>
```

When using overlay, the image should be a sibling of the `.card-img-overlay` div, both inside the `.card` container. Add `.text-bg-dark` for light text on dark images, or `.text-bg-light` for dark text on light images.

### Horizontal Cards

Cards can be displayed horizontally by combining the card layout with Bootstrap's grid system inside the card body.

```html
<div class="card mb-3" style="max-width: 540px;">
  <div class="row g-0">
    <div class="col-md-4">
      <img src="https://via.placeholder.com/250x250" class="img-fluid rounded-start" alt="Horizontal card image">
    </div>
    <div class="col-md-8">
      <div class="card-body">
        <h5 class="card-title">Horizontal Card</h5>
        <p class="card-text">
          This is a horizontal card layout using Bootstrap's row and column
          classes inside the card container.
        </p>
        <p class="card-text"><small class="text-body-secondary">Last updated 3 mins ago</small></p>
      </div>
    </div>
  </div>
</div>
```

The `.g-0` class removes gutters between columns, and `.rounded-start` ensures the image rounds only on the left side to match the card border.

### Card Grid with Row and Columns

To create a responsive grid of cards, wrap cards in `.row` containers with `.col` classes.

```html
<div class="row row-cols-1 row-cols-md-3 g-4">
  <div class="col">
    <div class="card h-100">
      <img src="https://via.placeholder.com/300x150" class="card-img-top" alt="...">
      <div class="card-body">
        <h5 class="card-title">Card 1</h5>
        <p class="card-text">Content for the first card in the grid.</p>
      </div>
    </div>
  </div>
  <div class="col">
    <div class="card h-100">
      <img src="https://via.placeholder.com/300x150" class="card-img-top" alt="...">
      <div class="card-body">
        <h5 class="card-title">Card 2</h5>
        <p class="card-text">Content for the second card in the grid.</p>
      </div>
    </div>
  </div>
  <div class="col">
    <div class="card h-100">
      <img src="https://via.placeholder.com/300x150" class="card-img-top" alt="...">
      <div class="card-body">
        <h5 class="card-title">Card 3</h5>
        <p class="card-text">Content for the third card in the grid.</p>
      </div>
    </div>
  </div>
</div>
```

The `.h-100` class ensures all cards in a row have equal height. The `.row-cols-md-3` class makes three cards appear per row on medium screens and above.

### Card Groups

Card groups render cards as a single attached element with shared borders and equal width.

```html
<div class="card-group">
  <div class="card">
    <img src="https://via.placeholder.com/300x150" class="card-img-top" alt="...">
    <div class="card-body">
      <h5 class="card-title">Card title</h5>
      <p class="card-text">This is a wider card with supporting text.</p>
    </div>
    <div class="card-footer"><small class="text-body-secondary">Last updated 3 mins ago</small></div>
  </div>
  <div class="card">
    <img src="https://via.placeholder.com/300x150" class="card-img-top" alt="...">
    <div class="card-body">
      <h5 class="card-title">Card title</h5>
      <p class="card-text">This card has supporting text below.</p>
    </div>
    <div class="card-footer"><small class="text-body-secondary">Last updated 3 mins ago</small></div>
  </div>
  <div class="card">
    <img src="https://via.placeholder.com/300x150" class="card-img-top" alt="...">
    <div class="card-body">
      <h5 class="card-title">Card title</h5>
      <p class="card-text">This is a wider card with more content.</p>
    </div>
    <div class="card-footer"><small class="text-body-secondary">Last updated 3 mins ago</small></div>
  </div>
</div>
```

### Colored and Styled Cards

Bootstrap provides contextual background and text classes that work with cards.

```html
<div class="card text-bg-primary mb-3" style="max-width: 18rem;">
  <div class="card-header">Header</div>
  <div class="card-body">
    <h5 class="card-title">Primary card</h5>
    <p class="card-text">Some quick example text to build on the card title.</p>
  </div>
</div>

<div class="card text-bg-success mb-3" style="max-width: 18rem;">
  <div class="card-body">
    <h5 class="card-title">Success card</h5>
    <p class="card-text">Some quick example text to build on the card title.</p>
  </div>
</div>

<div class="card text-bg-danger mb-3" style="max-width: 18rem;">
  <div class="card-body">
    <h5 class="card-title">Danger card</h5>
    <p class="card-text">Some quick example text to build on the card title.</p>
  </div>
</div>
```

The `.text-bg-{color}` classes set both the background and text color simultaneously. Available colors include `primary`, `secondary`, `success`, `danger`, `warning`, `info`, `light`, and `dark`.

### Navigation Tabs and Pills in Cards

Cards can include tab or pill navigation in the header area.

```html
<div class="card text-center">
  <div class="card-header">
    <ul class="nav nav-tabs card-header-tabs">
      <li class="nav-item">
        <a class="nav-link active" aria-current="true" href="#">Active</a>
      </li>
      <li class="nav-item">
        <a class="nav-link" href="#">Link</a>
      </li>
      <li class="nav-item">
        <a class="nav-link disabled" aria-disabled="true">Disabled</a>
      </li>
    </ul>
  </div>
  <div class="card-body">
    <h5 class="card-title">Card with Nav Tabs</h5>
    <p class="card-text">Navigation tabs integrated into a card header.</p>
    <a href="#" class="btn btn-primary">Go somewhere</a>
  </div>
</div>
```

## Best Practices

1. **Use `.card-body` consistently.** Always wrap card content in `.card-body` to ensure proper padding and spacing. Content outside `.card-body` will bleed to the card edges.

2. **Use `.h-100` for equal-height cards.** When placing cards in a grid row, add `.h-100` to each card to ensure uniform height regardless of content length.

3. **Use `.row-cols-*` for responsive grids.** Instead of wrapping each card in an individual column, use `.row-cols-1`, `.row-cols-md-2`, `.row-cols-lg-3` on the row to control column count responsively.

4. **Use `.g-*` utilities for spacing.** Apply gutter classes like `.g-4` on `.row` containers to control spacing between cards in a grid.

5. **Use `.list-group-flush` for list groups in cards.** This removes the outer borders of list groups so they integrate seamlessly within the card structure.

6. **Apply `.text-truncate` on long titles.** Prevent card titles from wrapping onto multiple lines by adding `.text-truncate` for single-line display with ellipsis.

7. **Use semantic heading levels.** Maintain proper heading hierarchy in cards. If your page structure calls for `h5` in cards, use `h5` consistently across all cards for screen reader navigation.

8. **Add `alt` text to card images.** Every image in a card should have a descriptive `alt` attribute. Use empty `alt=""` only for purely decorative images.

9. **Use `.rounded-start` or `.rounded-end` for horizontal cards.** Match the image rounding to the card border to avoid visual inconsistencies.

10. **Consider using `<article>` for standalone cards.** If a card represents a standalone piece of content (blog post, product), wrapping it in `<article>` adds semantic meaning.

11. **Use `.card-img-overlay` carefully.** Ensure sufficient contrast between overlay text and background image. Use darkened overlays or text shadows when needed.

12. **Avoid nesting cards deeply.** While technically possible, nesting cards more than one level deep creates confusing layouts and accessibility challenges.

## Common Pitfalls

1. **Forgetting `.card-body` and wondering why padding is missing.** Content placed directly inside `.card` without `.card-body` will have no padding. This is intentional for images and overlays but catches many developers off guard.

2. **Cards not stretching to equal height in a row.** Without `.h-100` on each card, cards in the same row will have different heights based on their content. Always add `.h-100` when cards sit in a grid.

3. **Using `.card-group` and expecting wrapping behavior.** `.card-group` renders all cards in a single row. On smaller screens, cards will shrink rather than wrap. Use `.row-cols-*` grid instead when wrapping is needed.

4. **Placing images without `.card-img-top` or `.card-img-bottom`.** Images without these classes will not have their corners rounded to match the card, creating visual mismatches at the borders.

5. **Overusing inline styles for card width.** Setting `style="width: 18rem;"` on every card is repetitive. Use grid column classes or utility classes like `.w-75` instead for consistent sizing.

6. **Not handling long content gracefully.** Long titles, descriptions, or button text can break card layouts. Use `.text-truncate`, `overflow-hidden`, or set `max-height` with `overflow: auto` on `.card-body`.

7. **Confusing `.card-img-overlay` with background images.** `.card-img-overlay` uses an `<img>` element, not CSS `background-image`. The image is part of the DOM, which is better for accessibility and SEO.

8. **Ignoring responsive behavior on horizontal cards.** Horizontal card layouts using `.row` and `.col-*` inside the card will stack on smaller screens by default. Test this behavior and adjust breakpoints as needed.

## Accessibility Considerations

Cards themselves are generic containers and have no inherent ARIA roles. Semantic structure within cards is your responsibility. Use heading elements (`h2`-`h6`) with logical hierarchy for card titles. This allows screen reader users to navigate by headings.

When a card represents an article or standalone piece of content, wrap it in `<article>` and include a heading. This creates a landmark that assistive technology can identify and navigate to.

If the entire card is clickable, wrap the card in a single `<a>` tag or use JavaScript to make the card focusable and clickable. Avoid wrapping just the title in a link when the expectation is that the entire card is interactive, as this confuses keyboard users who may tab past the card image.

Add `aria-label` or `aria-labelledby` to cards that contain navigation tabs or complex interactive elements. When using `.card-img-overlay`, ensure the text contrast meets WCAG 2.1 AA minimum contrast ratios of 4.5:1 for normal text.

For cards with list groups used as navigation, use `<nav aria-label="...">` around the list group and apply proper `aria-current="page"` on the active item.

Images inside cards must have meaningful `alt` attributes describing the image content. Decorative images should use `alt=""` and `role="presentation"`.

## Responsive Behavior

Cards are naturally responsive. By default, a card stretches to fill 100% of its parent container width and its height adjusts to fit the content. This means cards work at any screen size without additional classes.

To create responsive multi-column card layouts, combine cards with Bootstrap's grid system. Use `.row` and `.col-*` breakpoint classes to control how many cards appear per row at different screen sizes.

```html
<div class="row row-cols-1 row-cols-sm-2 row-cols-md-3 row-cols-lg-4 g-4">
  <div class="col">
    <div class="card h-100">...</div>
  </div>
  <!-- More columns -->
</div>
```

The `.row-cols-*` classes define how many equal-width columns appear at each breakpoint. The `g-4` class adds consistent spacing between cards.

For horizontal cards, the internal `.row` and `.col-md-*` structure naturally stacks on screens below the `md` breakpoint. The image moves above the content on mobile, which is the expected behavior.

Card groups use flexbox and will not wrap. On very small screens, cards in a group will become extremely narrow. Consider switching to a stacked layout on small screens by using CSS to override `display: flex` on `.card-group` at small breakpoints, or simply prefer the `.row-cols-*` approach for better responsive control.

The `.card-img-top` and `.card-img-bottom` images are responsive by default through Bootstrap's image handling. Images will scale proportionally within the card boundaries. For overlay cards, the `.card-img` class ensures the image covers the full card area while the overlay content remains accessible on top.
