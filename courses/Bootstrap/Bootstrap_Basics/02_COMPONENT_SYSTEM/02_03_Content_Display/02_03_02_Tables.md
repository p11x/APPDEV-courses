# Tables

## Overview

Tables in Bootstrap 5 provide a clean, consistent way to display tabular data while maintaining readability and responsiveness. Bootstrap does not require any special markup to get started — simply add the `.table` class to any `<table>` element and Bootstrap applies its default styles including padding, borders, and typography.

The core table classes in Bootstrap include `.table` as the base, followed by modifier classes like `.table-striped` for zebra-striped rows, `.table-bordered` for full borders, `.table-hover` for row hover effects, `.table-dark` for dark mode tables, and `.table-sm` for compact tables. These modifiers can be combined freely to achieve the desired appearance.

Color classes like `.table-primary`, `.table-success`, `.table-danger`, `.table-warning`, `.table-info`, and `.table-light` can be applied to entire tables or individual rows and cells. This makes it straightforward to highlight specific data or indicate status within a table.

For handling large datasets on narrow screens, Bootstrap provides `.table-responsive` which wraps the table in a container that allows horizontal scrolling. This can also be applied with breakpoint modifiers like `.table-responsive-md` to activate scrolling only below certain screen widths.

The `.table-group-divider` class adds a thicker divider between table body sections, useful for visually separating grouped rows without adding additional markup structures.

Tables are one of the most data-centric components in Bootstrap, and understanding their full range of options is critical for building admin dashboards, data reports, comparison tables, and any interface that presents structured information.

## Basic Implementation

### Default Table

Apply the `.table` class to any standard HTML table for Bootstrap's default styling.

```html
<table class="table">
  <thead>
    <tr>
      <th scope="col">#</th>
      <th scope="col">First</th>
      <th scope="col">Last</th>
      <th scope="col">Handle</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">1</th>
      <td>Mark</td>
      <td>Otto</td>
      <td>@mdo</td>
    </tr>
    <tr>
      <th scope="row">2</th>
      <td>Jacob</td>
      <td>Thornton</td>
      <td>@fat</td>
    </tr>
    <tr>
      <th scope="row">3</th>
      <td colspan="2">Larry the Bird</td>
      <td>@twitter</td>
    </tr>
  </tbody>
</table>
```

Using `scope="col"` on `<th>` elements in the header and `scope="row"` on row headers improves accessibility by explicitly defining the relationship between headers and data cells.

### Responsive Tables

Wrap any table with `.table-responsive` to enable horizontal scrolling on small viewports. The table will scroll sideways rather than compress columns to unreadable widths.

```html
<div class="table-responsive">
  <table class="table">
    <thead>
      <tr>
        <th scope="col">#</th>
        <th scope="col">Heading</th>
        <th scope="col">Heading</th>
        <th scope="col">Heading</th>
        <th scope="col">Heading</th>
        <th scope="col">Heading</th>
        <th scope="col">Heading</th>
        <th scope="col">Heading</th>
        <th scope="col">Heading</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <th scope="row">1</th>
        <td>Cell</td><td>Cell</td><td>Cell</td>
        <td>Cell</td><td>Cell</td><td>Cell</td>
        <td>Cell</td><td>Cell</td>
      </tr>
    </tbody>
  </table>
</div>
```

To make the table responsive only at certain breakpoints, use `.table-responsive-sm`, `.table-responsive-md`, `.table-responsive-lg`, or `.table-responsive-xl`.

```html
<div class="table-responsive-md">
  <table class="table">
    <!-- Table content -->
  </table>
</div>
```

## Best Practices

1. **Always use `.table` as the base class.** Every styled table must include the `.table` class. Without it, Bootstrap provides no table styling and you get raw browser defaults.

2. **Use `scope` attributes on header cells.** Add `scope="col"` to column headers and `scope="row"` to row headers. This establishes the data-header relationship for assistive technology.

3. **Combine modifiers intentionally.** Classes like `.table-striped`, `.table-bordered`, `.table-hover`, and `.table-dark` can be combined. Use `.table-striped.table-hover` for alternating row colors with hover highlight.

4. **Wrap large tables with `.table-responsive`.** On mobile devices, tables with many columns become unreadable without horizontal scrolling. Always wrap tables that might exceed viewport width.

5. **Use `.table-sm` for dense data displays.** Admin dashboards and data-heavy interfaces benefit from compact tables. Apply `.table-sm` to reduce padding.

6. **Use contextual classes sparingly on cells.** Overusing `.table-success`, `.table-danger`, etc., on every cell creates visual noise. Reserve them for status indicators or alerts.

7. **Add `<caption>` for accessibility.** Screen readers use captions to identify what a table represents. A visible or visually-hidden caption greatly improves accessibility.

8. **Avoid using tables for layout purposes.** Tables are for tabular data. Never use `<table>` to arrange page layout — use Bootstrap's grid system instead.

9. **Use `.table-group-divider` for visual sections.** Instead of nesting multiple `<table>` elements, use `.table-group-divider` on `<tr>` elements to create visual breaks within a single table.

10. **Maintain consistent column alignment.** Apply `.text-start`, `.text-center`, or `.text-end` to columns consistently. Right-align numeric columns for easier scanning and comparison.

11. **Prefer CSS variables for customization.** Bootstrap's tables use CSS variables for colors, borders, and spacing. Override these variables at the `:root` level or on `.table` for theme-level changes rather than writing custom CSS.

## Common Pitfalls

1. **Forgetting to add `.table` to the `<table>` element.** The most common mistake is expecting Bootstrap styles without the base class. The table will render with default browser styles.

2. **Placing `.table-responsive` on the `<table>` element itself.** The responsive wrapper must be a `<div>` around the table, not a class on the table element. Placing it on `<table>` does nothing.

3. **Using `<table>` for non-tabular layout.** Resist the temptation to use tables for form layouts, image grids, or page structure. Use Bootstrap's grid and flex utilities instead.

4. **Striped rows not appearing with dark tables.** When combining `.table-dark` with `.table-striped`, the stripe colors are automatically adjusted. If you do not see stripes, verify both classes are present on the same element.

5. **Not testing responsive tables on actual devices.** Emulators may not accurately represent horizontal scroll behavior. Test on physical devices or resize browser windows to verify the scroll experience.

6. **Applying color classes to `<thead>`.** Contextual color classes like `.table-primary` are designed for `<tr>`, `<td>`, and `<th>` inside `<tbody>`. Applying them to `<thead>` can produce unexpected results.

7. **Forgetting `scope` attributes for accessibility.** Without `scope`, screen readers cannot associate header cells with data cells, making the table difficult to navigate for visually impaired users.

8. **Overriding Bootstrap table styles with `!important`.** Avoid using `!important` to override table styling. Instead, use more specific selectors or Bootstrap's CSS variables for cleaner customization.

## Accessibility Considerations

Tables must be structured with proper semantic HTML. Use `<thead>`, `<tbody>`, and `<tfoot>` to define table sections. Column headers should use `<th scope="col">` and row headers should use `<th scope="row">`. This allows screen readers to announce the relevant header when a user navigates to a data cell.

Add a `<caption>` element to every data table. The caption describes the table's purpose and is announced by screen readers when users encounter the table. If the caption should not be visible, use the `.visually-hidden` class.

For complex tables with multi-level headers or merged cells, use `id` and `headers` attributes on `<th>` and `<td>` elements to explicitly define header-cell relationships. This is necessary when the simple `scope` attribute cannot express the structure.

When using `.table-responsive`, note that horizontal scrolling can be difficult for keyboard-only users. Ensure the wrapper is focusable with `tabindex="0"` and add `role="region"` with `aria-label` to indicate the scrollable area.

```html
<div class="table-responsive" tabindex="0" role="region" aria-label="User data table">
  <table class="table">
    <!-- Table content -->
  </table>
</div>
```

Avoid using `display: none` on table headers to visually hide them. Use `.visually-hidden` instead, which keeps headers available to screen readers while hiding them from view.

## Responsive Behavior

Bootstrap tables are not responsive by default. A table wider than its container will cause the entire page to scroll horizontally, which is rarely the desired behavior.

The `.table-responsive` class wraps the table in a container with `overflow-x: auto` and `overflow-y: hidden`, enabling horizontal scrolling only for the table area. A `max-width: 100%` is applied to the table so it takes its natural width but is clipped by the container.

For responsive behavior tied to specific breakpoints, use `.table-responsive-sm` (scrolls below 576px), `.table-responsive-md` (below 768px), `.table-responsive-lg` (below 992px), `.table-responsive-xl` (below 1200px), or `.table-responsive-xxl` (below 1400px).

```html
<div class="table-responsive-lg">
  <table class="table">
    <!-- Only scrolls on screens narrower than 992px -->
  </table>
</div>
```

When building responsive data interfaces, consider which columns are essential on small screens and which can be hidden using `.d-none .d-md-table-cell` utility classes. This allows you to show a simplified table on mobile and the full table on larger screens.

```html
<th class="d-none d-md-table-cell">Optional Column</th>
```

For tables with many columns, another approach is to transpose rows and columns on mobile using CSS, though this requires custom styles beyond Bootstrap's built-in classes.
