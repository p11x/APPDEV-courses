💡 Pro Tips

  • Use Semantic Colors: Use .text-success for positive messages, .text-danger for errors
  • Ensure Contrast: Always use light text on dark backgrounds and vice versa
  • Don't Overdo It: Too many colors can be overwhelming - use them sparingly
  • Consistency: Use the same color for the same meaning throughout your site
  • Accessibility: Check color contrast for readability (WCAG guidelines)
  • Subtle Backgrounds: Use bg-*-subtle for softer, less intense backgrounds

✅ Best Practices

  1. Semantic Usage: Use colors to convey meaning, not just for decoration
  2. Primary for Main Actions: Use .btn-primary for the main call-to-action
  3. Error States: Always use red/danger colors for form validation errors
  4. Warnings: Use yellow/warning colors for important notices
  5. Backgrounds: Use light backgrounds for content areas, dark for footers
  6. Links: Use .text-primary or custom colors for links (not default blue)
