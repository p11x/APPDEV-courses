Accessible Patterns

Building inclusive UI patterns that meet WCAG 2.1 AA standards.

1. Skip Navigation Link

Allow keyboard users to skip repetitive navigation.

Skip to main content 
<a href="#mainContent" class="visually-hidden-focusable">
    Skip to main content
</a>

2. ARIA Landmarks

Use semantic HTML5 elements and ARIA roles for navigation.

nav aria-label="Main navigation"
main Primary content area
aside Supplementary content
footer Page footer

3. Keyboard Navigation Patterns

Tab Navigation

All interactive elements must be reachable via Tab key.

Tab - Move forward
Shift+Tab - Move backward
Enter/Space - Activate
Arrow Key Navigation

Use arrow keys within component groups.

Content A
Content B
Content C
Arrow Left/Right - Switch tabs

6. Live Region Announcements

Use aria-live regions for dynamic content updates.

<div aria-live="polite" aria-atomic="true">
    <!-- Dynamic content announced by screen readers -->
</div>

/* For urgent announcements use: */
<div aria-live="assertive">Urgent alert!</div>

7. Focus Management

Visible Focus Indicator

Bootstrap 5 includes a visible focus ring by default. Never remove :focus styles without providing an alternative.

Focus Trap (Modal)

Bootstrap modals automatically trap focus within the dialog. Tab cycles through modal elements only.

Accessible Modal

Focus is trapped inside this modal. Press Tab to cycle through elements. Press Escape to close.
