2. Deprecation Policy

Phase 1: Announce

Mark pattern as deprecated. Add warning in docs. Team has 30 days to migrate.

/* @deprecated - Use NewComponent instead */
Phase 2: Remove

Remove from library after migration period. Redirect to replacement pattern.

// Removed in v4.0.0
Phase 3: Clean Up

Remove all references. Update documentation. Run final audit.

// Migration complete
