Deprecation Process

A structured approach to deprecating features in a controlled and communicative manner.

Stage Details

Criteria for Deprecation
  • Feature has a better replacement
  • Feature causes accessibility issues
  • Feature is rarely used (based on analytics)
  • Feature conflicts with modern standards
Documentation Requirements
  • Reason for deprecation
  • Recommended alternative
  • Migration path
  • Timeline for removal

Communication Channels
Documentation

Update official docs with deprecation notice

Release Notes

Include in changelog and release notes

Blog Post

Detailed blog post with migration guide

Warning Implementation

During the deprecation period, the feature should emit console warnings:

console.warn(
  'Bootstrap: .badge-pill is deprecated since v4.0. ' +
  'Use .rounded-pill instead. Will be removed in v6.0.'
);
Minimum Warning Period
Best Practice: Maintain warnings for at least one minor version cycle before removal.

Removal Guidelines
  • Only remove in major version releases
  • Provide codemods for automated migration
  • Update all documentation
  • Mention in migration guide
Warning: Never remove features in minor or patch releases.
