Git Workflow for Migrations

Recommended Git Commands
# Create migration branch from main
git checkout main
git pull origin main
git checkout -b bootstrap-migration/v4-to-v5

# Stage and commit migration changes
git add .
git commit -m "chore(bootstrap): migrate from v4 to v5

- Replace badge-pill with rounded-pill
- Update media components to flexbox
- Migrate form-group to mb-3
- Update data-toggle to data-bs-toggle

Refs: #123"

# Tag release after successful migration
git tag -a v2.0.0 -m "Release: Bootstrap 5 migration complete"

# Push and create pull request
git push origin bootstrap-migration/v4-to-v5

Branch Strategy

main

Production-ready code. Protected branch.

  • Requires PR approval
  • All tests must pass
  • No direct commits
develop

Integration branch for features.

  • Latest development
  • Auto-runs CI
  • Merges to main
feature/*

Individual migration tasks.

  • One task per branch
  • Short-lived
  • PR to develop
hotfix/*

Emergency fixes.

  • From main
  • Urgent patches
  • Fast-tracked PR

Commit Message Conventions

Prefix Usage Example
feat: New feature feat: add dark mode support
fix: Bug fix fix: resolve modal backdrop z-index
chore: Maintenance chore: update bootstrap to 5.3.3
docs: Documentation docs: update migration guide
refactor: Code refactor refactor: migrate form-group to mb-3
test: Tests test: add visual regression tests
