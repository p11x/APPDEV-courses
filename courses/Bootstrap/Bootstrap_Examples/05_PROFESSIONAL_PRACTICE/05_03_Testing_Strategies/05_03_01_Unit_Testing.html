Unit Testing Strategies

Testing individual Bootstrap components in isolation to ensure reliability

What is Unit Testing?

Unit testing involves testing individual components or functions in isolation to verify they work correctly. For Bootstrap, this means testing each UI component independently before integrating them into larger views.

Isolation

Each component is tested independently without dependencies on other components or external systems.

Core Principle
Speed

Unit tests run quickly because they don't require browser rendering or network requests.

Fast Execution
Reliability

Tests produce consistent results and help catch regressions before they reach production.

Consistent

Testing Patterns

Verify that components render with correct HTML structure, CSS classes, and content. Use data-testid attributes to reliably select elements.

// Example: Test that a card renders correctly
test('Card renders with title', () => {
    const card = getByTestId('card-isolation');
    expect(card.querySelector('.card-title').textContent)
        .toBe('Isolation');
});

Test that user interactions trigger the correct event handlers. Simulate clicks, form submissions, and keyboard events.

// Example: Test button click handler
test('CTA button triggers scroll', () => {
    const btn = getByTestId('hero-cta');
    fireEvent.click(btn);
    expect(window.scrollTo).toHaveBeenCalled();
});

Verify that component state changes correctly based on user interactions and prop updates.

// Example: Test accordion state toggle
test('Accordion panel toggles', () => {
    const btn = getByTestId('btn-event');
    fireEvent.click(btn);
    const panel = getByTestId('pattern-event');
    expect(panel.querySelector('.accordion-collapse'))
        .toHaveClass('show');
});
