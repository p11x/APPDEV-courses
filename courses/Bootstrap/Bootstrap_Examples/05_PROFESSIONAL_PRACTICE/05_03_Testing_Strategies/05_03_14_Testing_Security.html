Common Frontend Vulnerabilities

Cross-Site Scripting (XSS)

Injecting malicious scripts into web pages viewed by other users.

Test Payloads:
// Reflected XSS test
<script>alert('XSS')</script>
<img src=x onerror=alert('XSS')>
<svg onload=alert('XSS')>

// DOM-based XSS test
javascript:alert(document.cookie)

// Bootstrap-specific: tooltip content
sanitize: true,  // Always enable
allowList: {}    // Restrict HTML tags
Prevention:
  • Use textContent instead of innerHTML
  • Enable Bootstrap's sanitizer
  • Implement Content Security Policy
  • Use DOMPurify for rich content
Cross-Site Request Forgery (CSRF)

Tricking users into performing unintended actions on authenticated sites.

Test Scenario:
// CSRF attack simulation
// Malicious page contains:
<form action="https://target.com/transfer"
  method="POST">
  <input name="amount" value="1000">
  <input name="to" value="attacker">
</form>
<script>document.forms[0].submit()</script>
Prevention:
  • Include CSRF tokens in all forms
  • Use SameSite cookie attribute
  • Validate Origin header
  • Require re-authentication for sensitive actions
Clickjacking

Embedding a site in an invisible iframe to trick users into clicking hidden elements.

Test:
// Test if site can be iframed
<iframe src="https://target.com"
  style="opacity:0;position:absolute">
</iframe>
Prevention:
  • Set X-Frame-Options: DENY
  • Use CSP frame-ancestors 'none'
  • Bootstrap modals use aria-hidden correctly
Dependency Vulnerabilities

Using packages with known security vulnerabilities.

Audit Commands:
# NPM security audit
npm audit
npm audit fix
npm audit --json > audit.json

# Snyk vulnerability scan
npx snyk test
npx snyk monitor

# Check for outdated packages
npm outdated
Prevention:
  • Run npm audit in CI pipeline
  • Use Snyk or Dependabot for alerts
  • Pin dependency versions
  • Review changelogs before updating

Security Testing Best Practices

Automated Security Checks
# GitHub Actions security workflow
name: Security Scan
on: [push, pull_request]

jobs:
  security:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm audit --audit-level=high
      - uses: snyk/actions/node@master
        env:
          SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
      - uses: github/codeql-action/init@v2
      - uses: github/codeql-action/analyze@v2
Security Headers Checklist
  • Content-Security-Policy Required
  • X-Frame-Options: DENY Required
  • X-Content-Type-Options: nosniff Required
  • Strict-Transport-Security Required
  • Referrer-Policy: strict-origin Recommended
  • Permissions-Policy Recommended
Bootstrap-Specific Security
Sanitizer Configuration
// Always enable sanitizer for tooltips
const tooltip = new bootstrap.Tooltip(
  element, {
    sanitize: true,
    allowList: {
      'b': [],
      'i': [],
      'strong': []
    }
  }
);
