GitHub Actions Configuration

Basic CI Workflow
● Active 
# .github/workflows/ci.yml
name: CI Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node: [18, 20, 22]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node }}
      - run: npm ci
      - run: npm test
      - run: npm run lint
Cross-Browser CI
● Playwright 
# .github/workflows/e2e.yml
name: E2E Tests

on: [push, pull_request]

jobs:
  e2e:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: npx playwright install
      - run: npx playwright test
      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: playwright-report
          path: playwright-report/
Caching

Cache node_modules, build artifacts, and browser binaries for faster runs.

uses: actions/cache@v4
with:
  path: ~/.npm
  key: ${{ runner.os }}-node
  restore-keys: ${{ runner.os }}-
Matrix Strategy

Test across multiple OS and browser combinations simultaneously.

strategy:
  matrix:
    os: [ubuntu, windows, macos]
    browser: [chrome, firefox, safari]
runs-on:
  ${{ matrix.os }}-latest
Artifacts & Reports

Upload test results, screenshots, and coverage reports for review.

uses: actions/upload-artifact@v4
with:
  name: test-results
  path: |
    coverage/
    test-results/
    screenshots/

Pipeline Stage Details

Build Stage

The build stage compiles source code, bundles assets, and prepares the application for testing and deployment.

build:
  runs-on: ubuntu-latest
  steps:
    - run: npm ci
    - run: npm run build
    - run: npm run lint
    - uses: actions/upload-artifact@v4
      with:
        name: dist
        path: dist/
Key Checks
  • Dependency installation npm ci
  • TypeScript compilation tsc
  • Asset bundling webpack/vite
  • Linting eslint
Test Stage
Unit Tests

Isolated function/component tests. Fast execution, no browser needed.

npm run test:unit
Integration Tests

Component interaction tests. Verifies data flow between modules.

npm run test:integration
E2E Tests

Full user journey tests. Browser automation with Playwright/Cypress.

npx playwright test
Deploy Stage
Staging First: Always deploy to staging, run smoke tests, then promote to production. 
deploy-staging:
  needs: [build, test]
  if: github.ref == 'refs/heads/develop'
  runs-on: ubuntu-latest
  steps:
    - run: aws s3 sync dist/ s3://staging-bucket
    - run: npm run test:smoke

deploy-production:
  needs: [build, test, deploy-staging]
  if: github.ref == 'refs/heads/main'
  environment: production
  runs-on: ubuntu-latest
  steps:
    - run: aws s3 sync dist/ s3://prod-bucket

Testing Framework Configuration

Jest Configuration
// jest.config.js
module.exports = {
  testEnvironment: 'jsdom',
  setupFilesAfterSetup: [
    '@testing-library/jest-dom'
  ],
  moduleNameMapper: {
    '\\.(css)$': 'identity-obj-proxy',
    '\\.(jpg|png)$': '<rootDir>/mock.js'
  },
  collectCoverageFrom: [
    'src/**/*.{js,ts}',
    '!src/**/*.test.{js,ts}'
  ],
  coverageThreshold: {
    global: {
      branches: 80,
      functions: 80,
      lines: 80,
      statements: 80
    }
  }
};

});

Automation Best Practices

Do's
  • ✓ Run tests on every pull request
  • ✓ Use parallel test execution
  • ✓ Cache dependencies between runs
  • ✓ Fail fast - run unit tests first
  • ✓ Store test artifacts for debugging
  • ✓ Use matrix builds for browser coverage
Don'ts
  • ✗ Skip tests to speed up CI
  • ✗ Use hardcoded secrets in workflows
  • ✗ Ignore flaky tests
  • ✗ Run full suite on documentation changes
  • ✗ Deploy without staging verification
  • ✗ Commit to main without PR review
