1. Compilation Pipeline

Source SCSS Files

Write modular .scss files with variables, nesting, mixins. Use partials (_filename.scss) for reusable modules.

Sass Compiler

Dart Sass processes imports, resolves variables, expands mixins, flattens nesting, and evaluates functions.

PostCSS Processing

Autoprefixer adds vendor prefixes, cssnano minifies output, additional transforms can be applied.

CSS Output

Final .css file ready for browsers. Source maps link back to original .scss for debugging.

Browser Delivery

Optimized CSS served to browsers, potentially split for critical CSS inlining.

CLI Compilation Commands
// Install Dart Sass (recommended) npm install sass --save-dev // Watch mode - recompiles on file changes npx sass scss/main.scss css/style.css --watch // Compile with source maps npx sass scss/main.scss css/style.css --source-map // Compressed output for production npx sass scss/main.scss css/style.css --style compressed // Compile entire directory npx sass scss/:css/ --watch
package.json Scripts
// package.json { "scripts": { "sass:watch": "sass scss/main.scss css/style.css --watch", "sass:build": "sass scss/main.scss css/style.css --style compressed", "build:css": "npm run sass:build && postcss css/style.css -o css/style.min.css" } }

2. Sass Variables

Defining Variables
// _variables.scss // Color variables $primary: #0d6efd; $secondary: #6c757d; $success: #198754; $danger: #dc3545; // Spacing scale $spacer: 1rem; $spacers: ( 0: 0, 1: calc($spacer * 0.25), 2: calc($spacer * 0.5), 3: $spacer, 4: calc($spacer * 1.5), 5: calc($spacer * 3) ); // Typography $font-family-sans-serif: 'Inter', sans-serif; $font-size-base: 1rem; $line-height-base: 1.5; // Breakpoints $grid-breakpoints: ( xs: 0, sm: 576px, md: 768px, lg: 992px, xl: 1200px, xxl: 1400px );
Overriding Bootstrap Variables
// main.scss - ORDER MATTERS! // Variables must be defined BEFORE Bootstrap is imported // 1. Override Bootstrap defaults FIRST $primary: #6f42c1; $border-radius: 0.75rem; $enable-shadows: true; // 2. Then import Bootstrap @import '../node_modules/bootstrap/scss/bootstrap'; // 3. Add custom styles AFTER Bootstrap .custom-component { // Use Bootstrap's variable padding: map-get($spacers, 3); }

3. Sass Mixins

Defining Mixins
// _mixins.scss // Simple mixin - no parameters @mixin flex-center { display: flex; align-items: center; justify-content: center; } // Mixin with parameters @mixin box-shadow($x: 0, $y: 0, $blur: 4px, $color: rgba(0,0,0,.1)) { box-shadow: $x $y $blur $color; } // Mixin with optional parameters @mixin transition($property: all, $duration: 0.3s, $timing: ease) { transition: $property $duration $timing; } // Mixin with content block @mixin hover-active { &:hover, &:active { @content; } } // Responsive breakpoint mixin @mixin media-up($breakpoint) { $min: map-get($grid-breakpoints, $breakpoint); @media (min-width: $min) { @content; } }
Using Mixins
// Using @include to apply mixins .card-custom { // Include simple mixin @include flex-center; // Include mixin with parameters @include box-shadow(0, 2px, 8px, rgba(0,0,0,.15)); // Include mixin with defaults @include transition(); // Include mixin with content block @include hover-active { transform: translateY(-2px); opacity: 0.9; } // Include responsive mixin @include media-up(md) { padding: 2rem; } }

4. Sass Partials

File Structure
_variables.scss (Partial)
// Defines variables only $primary: #6f42c1; $spacer: 1rem;
_mixins.scss (Partial)
// Defines mixins only @mixin flex-center { ... } @mixin transition() { ... }
_buttons.scss (Partial)
// Button component styles .btn-custom { @include transition(); padding: map-get($spacers, 2); }
main.scss (Entry Point)
// Import all partials - order matters! @import 'variables'; @import 'mixins'; @import 'buttons'; @import 'cards'; @import 'navbar';
Import Rules
@import vs @use

@import is being deprecated in favor of @use and @forward.

// OLD (deprecated) @import 'variables'; @import 'mixins'; // NEW (recommended) @use 'variables' as *; @use 'mixins' as *; // With namespace @use 'variables' as vars; .element { color: vars.$primary; } // Forward partials from index file // _index.scss @forward 'variables'; @forward 'mixins';
Best Practices
  • DO Use underscores for partial filenames
  • DO Keep partials focused (variables, mixins, one component)
  • DO Import variables/mixins before components
  • DO Use @use/@forward for new projects
  • DON'T Compile partials directly (no underscore = compiled)
  • DON'T Create circular dependencies

Key Takeaways
  • Override Bootstrap variables BEFORE importing
  • Use partials for modular architecture
  • Use @use instead of @import in new projects
  • Compile with Dart Sass (not node-sass)
  • Add source maps for development
  • Minify for production builds
