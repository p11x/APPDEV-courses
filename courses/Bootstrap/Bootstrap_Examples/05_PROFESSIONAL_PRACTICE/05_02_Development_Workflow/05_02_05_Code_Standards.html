1. Naming Conventions

Convention Comparison
Convention Example Pros
BEM .card__title--active Clear structure, low specificity
kebab-case .user-profile-card Readable, Bootstrap default
camelCase .userProfileCard JS-friendly
Utility-first .d-flex .p-3 Fast, Bootstrap utilities
OOCSS .media .media-body Reusable objects
Bootstrap's Convention
/* Bootstrap uses kebab-case throughout */ /* Component classes */ .navbar-brand .nav-link .card-body .form-control .btn-primary /* Utility classes */ .d-flex .flex-column .justify-content-center .align-items-center .mt-3 .mb-4 .text-truncate /* Modifier classes */ .btn-lg .btn-outline-primary .navbar-expand-lg .table-striped .form-control-lg /* State classes */ .active .disabled .show .collapsed
Naming Best Practices
Use semantic, meaningful names
.red-box (bad) .alert-error (good)
Describe purpose, not appearance
.big-text (bad) .heading-xl (good)
Keep specificity low
.nav .nav-item .nav-link.active (bad) .nav-link--active (good)
Be consistent with Bootstrap
.cardContainer (bad) .card-wrapper (good)
Avoid abbreviations
.usr-prfl (bad) .user-profile (good)

2. BEM Methodology

BEM Structure
.block__element--modifier
  • Block: Standalone component
    .card, .navbar, .form-group
  • Element (__): Part of a block
    .card__title, .navbar__link
  • Modifier (--): Variation of block/element
    .card--featured, .btn--large
BEM in SCSS
// BEM with SCSS nesting using & .user-card { display: flex; padding: 1rem; border: 1px solid #ddd; border-radius: 0.5rem; // Elements - use &__ prefix &__avatar { width: 48px; height: 48px; border-radius: 50%; } &__name { font-weight: 600; margin-bottom: 0.25rem; } &__email { color: #6c757d; font-size: 0.875rem; } // Modifiers - use &-- prefix &--featured { border-color: gold; background-color: #fffbeb; } &--compact { padding: 0.5rem; // Element inside modifier .user-card__avatar { width: 32px; height: 32px; } } }
Rendered Output
JD
John Doe
john@example.com
JD
John Doe Featured
john@example.com
BEM Rules
  • No tag selectors (never div.card)
  • No more than 1 level of nesting
  • Elements belong to their block only
  • No cascading - each class is independent
  • Block names are unique globally

3. BEM + Bootstrap Integration

Hybrid Approach
// Combining Bootstrap utilities with BEM classes // HTML - Bootstrap for layout/utilities, BEM for components <div class="container"> <div class="row g-4"> <!-- Bootstrap grid --> <div class="col-md-6"> <div class="product-card product-card--featured"> <!-- BEM --> <img class="product-card__image img-fluid" /> <!-- BEM + BS --> <h3 class="product-card__title text-truncate"> <!-- BEM + BS --> <div class="product-card__actions d-flex gap-2"> <!-- BEM + BS --> <button class="btn btn-primary">Buy</button> <!-- BS only --> </div> </div> </div> </div> </div> // SCSS - BEM for component structure .product-card { border: 1px solid #e5e7eb; border-radius: 0.5rem; overflow: hidden; &__image { width: 100%; height: 200px; object-fit: cover; } &__title { padding: 1rem 1rem 0.5rem; font-size: 1.125rem; } &__actions { padding: 0.5rem 1rem 1rem; } &--featured { border-color: var(--bs-primary); box-shadow: 0 0 0 2px rgba(var(--bs-primary-rgb), 0.25); } }

4. HTML Standards

Semantic HTML Rules
<!-- GOOD: Semantic HTML with Bootstrap --> <header class="site-header"> <nav class="navbar" aria-label="Main navigation"> <a class="navbar-brand" href="/">Logo</a> <ul class="navbar-nav"> <li class="nav-item"> <a class="nav-link active" aria-current="page">Home</a> </li> </ul> </nav> </header> <main class="container"> <article class="blog-post"> <h1>Post Title</h1> <section aria-label="Post content"> <p>Content...</p> </section> </article> </main> <footer class="site-footer"> <nav aria-label="Footer navigation">...</nav> </footer> <!-- BAD: Div soup --> <div class="header"> <div class="nav"> <div class="link">Home</div> </div> </div>
Bootstrap Attribute Standards
<!-- Always include accessibility attributes --> <!-- Toggle buttons --> <button class="navbar-toggler" type="button" data-bs-toggle="collapse" data-bs-target="#navMenu" aria-controls="navMenu" aria-expanded="false" aria-label="Toggle navigation"> <!-- Collapse container --> <div class="collapse" id="navMenu"> <!-- Active states --> <a class="nav-link active" aria-current="page"> <!-- Form labels --> <label class="form-label" for="emailInput">Email</label> <input class="form-control" id="emailInput" type="email"> <!-- ARIA labels for icon buttons --> <button class="btn btn-close" aria-label="Close"></button>
