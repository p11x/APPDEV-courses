1. Node.js & Package Managers

Node Version Management

Use nvm (Node Version Manager) to switch between Node versions per project. This ensures all team members use the same Node version.

# Install nvm (macOS/Linux) curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash # Install nvm-windows (Windows) # Download installer from github.com/coreybutler/nvm-windows # Install and use specific Node version nvm install 20 nvm use 20 nvm alias default 20 # Pin Node version in project # .nvmrc 20

2. Editor Configuration

.editorconfig

EditorConfig defines consistent coding styles across different editors and IDEs. Supported by VS Code, WebStorm, Sublime, etc.

# EditorConfig - https://editorconfig.org # Ensures consistent formatting across all editors root = true # All files [*] indent_style = space indent_size = 2 end_of_line = lf charset = utf-8 trim_trailing_whitespace = true insert_final_newline = true # Markdown files - preserve trailing whitespace [*.md] trim_trailing_whitespace = false # HTML files [*.html] indent_size = 2 # SCSS/CSS files [*.{scss,css}] indent_size = 2 # JavaScript files [*.{js,ts}] indent_size = 2 # JSON files [*.json] indent_size = 2 # YAML files [*.{yml,yaml}] indent_size = 2 # Package files {package.json,.eslintrc.json,.prettierrc} indent_size = 2
VS Code Workspace Settings
// .vscode/settings.json { // Formatting "editor.formatOnSave": true, "editor.formatOnPaste": true, "editor.defaultFormatter": "esbenp.prettier-vscode", "editor.tabSize": 2, "editor.wordWrap": "on", // File associations "files.associations": { "*.scss": "scss" }, "files.exclude": { "**/node_modules": true, "**/dist": true, "**/.git": true }, // SCSS/CSS "scss.validate": false, // Emmet (HTML expansion) "emmet.includeLanguages": { "javascript": "html" }, // Git "git.autofetch": true, "git.confirmSync": false }
