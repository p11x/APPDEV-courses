1. Live Reload Options

BrowserSync
Standalone

Synchronized testing across devices. Automatically reloads on file changes with CSS injection support.

# Install globally npm install -g browser-sync # Serve static files with live reload browser-sync start --server --files "css/*.css, *.html" # Watch SCSS and HTML browser-sync start --server --files "**/*.css, **/*.html" # With proxy (for PHP, Node backend) browser-sync start --proxy "localhost:8080" --files "**/*"
Vite Dev Server
Built-in

Native ESM dev server with instant start and sub-50ms hot module replacement.

// vite.config.js export default { server: { port: 5173, host: true, // Expose to network open: true, // Open browser cors: true, // Enable CORS strictPort: true, // Fail if port taken // File watching options watch: { usePolling: true, // For Docker/WSL interval: 100, }, } }; // Run: npx vite
Webpack Dev Server
Built-in

Full-featured dev server with HMR, proxy support, and history API fallback.

// webpack.config.js module.exports = { devServer: { static: './dist', port: 3000, hot: true, // HMR enabled liveReload: true, // Live reload open: true, compress: true, historyApiFallback: true, client: { overlay: true, // Show errors in browser progress: true, // Show build progress }, } };
Live Server (VS Code)
Editor Extension

VS Code extension for quick static file serving. Right-click HTML file and select "Open with Live Server".

// .vscode/settings.json { "liveServer.settings.port": 5500, "liveServer.settings.root": "/", "liveServer.settings.CustomBrowser": "chrome", "liveServer.settings.NoBrowser": false, "liveServer.settings.ignoreFiles": [ "**/*.scss", "**/node_modules/**" ] }

2. Local HTTPS Setup

mkcert (Recommended)

Creates locally-trusted SSL certificates without configuration.

# Install mkcert (macOS) brew install mkcert # Install mkcert (Windows - via Chocolatey) choco install mkcert # Install the local CA mkcert -install # Generate certificates for localhost mkcert localhost 127.0.0.1 ::1 # Creates: localhost+2.pem and localhost+2-key.pem
Vite with HTTPS
// vite.config.js import { defineConfig } from 'vite'; import fs from 'fs'; export default defineConfig({ server: { https: { key: fs.readFileSync('localhost+2-key.pem'), cert: fs.readFileSync('localhost+2.pem'), }, host: 'localhost', port: 5173, }, });
Webpack with HTTPS
// webpack.config.js const fs = require('fs'); module.exports = { devServer: { server: { type: 'https', options: { key: fs.readFileSync('./certs/localhost+2-key.pem'), cert: fs.readFileSync('./certs/localhost+2.pem'), }, }, host: 'localhost', port: 3000, }, };
Self-Signed (Without mkcert)
# Generate self-signed cert with OpenSSL openssl req -x509 -newkey rsa:4096 \ -keyout key.pem -out cert.pem \ -days 365 -nodes \ -subj "/CN=localhost" # Browser will show security warning - click Advanced > Proceed
When Local HTTPS Is Required
  • Service Worker registration
  • getUserMedia (camera/microphone)
  • Geolocation API (some browsers)
  • Payment Request API
  • Secure cookies (SameSite=None)
  • OAuth redirects

3. Network Access

Find Your Local IP
# macOS / Linux ifconfig | grep "inet " # or ip addr show # Windows ipconfig # Look for IPv4 Address under your adapter
Bind to All Interfaces
# Vite - expose to network npx vite --host 0.0.0.0 # Access via: http://192.168.x.x:5173 # Webpack Dev Server // In webpack.config.js devServer: { host: '0.0.0.0', port: 3000, } # BrowserSync browser-sync start --server --host "0.0.0.0"
Testing on Mobile Devices
Steps to Test on Phone
  1. Start dev server with --host 0.0.0.0
  2. Note your computer's local IP (e.g., 192.168.1.100)
  3. Ensure phone is on same WiFi network
  4. Open http://192.168.1.100:5173 on phone
BrowserSync Unique Feature

BrowserSync synchronizes scroll, click, and form input across all connected devices. Great for responsive testing on multiple screen sizes simultaneously.

4. Server Configuration

API Proxy Setup
// Vite - proxy API requests to backend server: { proxy: { '/api': { target: 'http://localhost:8080', changeOrigin: true, rewrite: (path) => path.replace(/^\/api/, ''), }, '/auth': { target: 'https://auth.example.com', changeOrigin: true, secure: false, }, }, } // Webpack - proxy API requests devServer: { proxy: { '/api': { target: 'http://localhost:8080', pathRewrite: { '^/api': '' }, }, }, }
History API Fallback
// For SPA routing - redirect all 404s to index.html // Vite - automatic for SPA mode // Webpack devServer: { historyApiFallback: true, // Or with custom rules historyApiFallback: { rewrites: [ { from: /^\/admin/, to: '/admin.html' }, { from: /./, to: '/index.html' }, ], }, }
Custom Headers
// Add custom headers for development // Vite server: { headers: { 'X-Custom-Header': 'dev-value', 'Access-Control-Allow-Origin': '*', }, } // Webpack devServer: { headers: { 'X-Custom-Header': 'dev-value', }, }
