1. Lock Files

Why Lock Files Matter
Without Lock Files

package.json says "bootstrap": "^5.3.0". Developer A installs 5.3.3, Developer B installs 5.3.1. Different behavior, different bugs.

With Lock Files

package-lock.json pins bootstrap@5.3.3 exactly. Everyone gets the same version. Reproducible builds guaranteed.

Lock File Best Practices
# ALWAYS commit lock files to version control # NEVER manually edit lock files # Regenerate lock file if corrupted rm package-lock.json npm install # Use npm ci for CI/CD (clean install from lock file) npm ci # - Installs exact versions from lock file # - Deletes existing node_modules first # - Faster than npm install # - Fails if lock file is out of sync # Verify lock file is in sync npm ci --dry-run

Version Range Specifiers
PrefixMeaningExample
^ Compatible with (minor updates) ^5.3.0 allows 5.x.x
~ Approximately (patch updates) ~5.3.0 allows 5.3.x
>= Greater than or equal >=5.0.0
* Any version * (dangerous)
exact Exact version (no prefix) 5.3.3
Dependency Types
{ // Production dependencies (shipped with app) "dependencies": { "bootstrap": "^5.3.3", "@popperjs/core": "^2.11.8" }, // Development dependencies (build tools only) "devDependencies": { "sass": "^1.69.0", "vite": "^5.0.0", "eslint": "^9.0.0", "prettier": "^3.0.0" }, // Peer dependencies (required by your package) "peerDependencies": { "bootstrap": ">=5.0.0" }, // Optional dependencies "optionalDependencies": { "fsevents": "^2.3.0" } }

4. Update Strategy

Update Commands
# Check for outdated packages npm outdated # Update specific package npm update bootstrap # Update to latest major (interactive) npx npm-check-updates -u npm install # Update only patch versions npx npm-check-updates -u --target patch # Update only minor versions npx npm-check-updates -u --target minor
npm-check-updates
# Install npm install -g npm-check-updates # Check what would be updated ncu # Update package.json only (don't install) ncu -u # Update and install ncu -u && npm install # Filter by package name ncu --filter "bootstrap, @popperjs/*"
Update Strategy
Recommended Approach
  1. Run npm outdated weekly
  2. Update patch versions frequently
  3. Update minor versions monthly
  4. Update major versions on dedicated branches
  5. Run full test suite after every update
  6. Check changelogs for breaking changes
Bootstrap Update Checklist
  • Read Bootstrap release notes
  • Check for breaking changes in your SCSS overrides
  • Test all components after update
  • Verify JavaScript plugins still work
  • Run npm audit after update
