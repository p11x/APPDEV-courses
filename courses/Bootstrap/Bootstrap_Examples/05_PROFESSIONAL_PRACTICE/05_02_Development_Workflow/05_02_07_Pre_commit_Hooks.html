1. Husky Setup

Installation
# Install Husky npm install --save-dev husky # Initialize Husky (creates .husky/ directory) npx husky init # This creates: # .husky/ - hooks directory # .husky/_/ - internal husky scripts # package.json - adds "prepare" script # Add a pre-commit hook npx husky .husky/pre-commit "npm run lint" # Add a commit-msg hook npx husky .husky/commit-msg 'npx --no -- commitlint --edit "$1"'
Hook File: .husky/pre-commit
# .husky/pre-commit # This file runs before every git commit npm run lint-staged npm run typecheck
Hook File: .husky/commit-msg
# .husky/commit-msg # Validates commit message format npx --no -- commitlint --edit "$1"
Project Structure
my-project/ ├── .husky/ │ ├── _/ # Husky internals (auto-generated) │ ├── pre-commit # Runs before commit │ ├── commit-msg # Validates commit message │ ├── pre-push # Runs before push │ └── pre-commit-msg # Runs before commit-msg ├── src/ ├── package.json # Has "prepare" script └── ...
package.json Scripts
{ "scripts": { // Husky adds this automatically "prepare": "husky", // Scripts used by hooks "lint-staged": "lint-staged", "lint": "eslint 'src/**/*.js'", "format": "prettier --check ." } }
How Husky Works

Husky uses Git's core.hooksPath config to redirect hooks to the .husky directory. The "prepare" script runs on npm install, ensuring hooks are set up for all team members automatically.

2. lint-staged

Installation
npm install --save-dev lint-staged
.lintstagedrc.json
{ // Run different commands based on file type "*.js": [ "eslint --fix", "prettier --write" ], "*.scss": [ "stylelint --fix", "prettier --write" ], "*.html": [ "prettier --write" ], "*.{json,md}": [ "prettier --write" ], // Don't process Bootstrap's vendor files "!node_modules/**", "!dist/**", "!vendor/**" }
package.json Config (Alternative)
{ "lint-staged": { "*.js": [ "eslint --fix", "prettier --write" ], "*.scss": [ "stylelint --fix --allow-empty-input", "prettier --write" ], "*.html": ["prettier --write"], // Run SCSS build if any SCSS files changed "src/scss/**/*.scss": [ "stylelint --fix", { "command": "npm run sass:build", // Only run build if SCSS changed "concurrent": false } ] } }
lint-staged Flow
git commit
↓
Husky triggers pre-commit
↓
lint-staged gets staged files
↓
Run linters on staged files only
↓
Pass → Commit Fail → Block
Advanced Config
// lint-staged config in JS for more control // lint-staged.config.js const path = require('path'); const buildEslintCommand = (filenames) => `eslint --fix ${filenames.map(f => path.relative(process.cwd(), f) ).join(' ')}`; module.exports = { '*.{js,jsx,ts,tsx}': [buildEslintCommand], '*.scss': [ 'stylelint --fix --allow-empty-input', 'prettier --write', ], // Run type-check only if TypeScript files changed '*.{ts,tsx}': [ () => 'tsc --noEmit', ], };

Commitlint for Conventional Commits
# Install commitlint npm install --save-dev @commitlint/cli @commitlint/config-conventional # commitlint.config.js module.exports = { extends: ['@commitlint/config-conventional'], rules: { 'type-enum': [2, 'always', [ 'feat', // New feature 'fix', // Bug fix 'docs', // Documentation 'style', // Formatting (no code change) 'refactor', // Code restructure 'test', // Adding tests 'chore', // Build/tool changes 'perf', // Performance ]], }, }; # Valid commit messages: # feat: add new card component # fix(navbar): resolve mobile collapse issue # docs: update SCSS documentation

4. Git Hooks Reference

Hook Trigger Common Use Can Block?
pre-commit Before commit Lint, format, run tests Yes
commit-msg After message entry Validate message format Yes
pre-push Before push Run full test suite, build Yes
post-commit After commit Notifications, logging No
post-merge After merge Reinstall deps, rebuild No
post-checkout After checkout Clear cache, update deps No
