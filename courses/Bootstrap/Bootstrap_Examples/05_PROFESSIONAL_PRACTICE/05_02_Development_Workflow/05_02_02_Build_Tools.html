1. Build Tool Overview

Webpack
Mature Ecosystem

The most established bundler with extensive plugin ecosystem. Uses a configuration file to define loaders and plugins for processing different file types.

  • ~ 50M weekly npm downloads
  • ~ 63k GitHub stars
  • Highly configurable
  • Rich loader/plugin system
Vite
Modern & Fast

Next-generation build tool using native ES modules during development for instant server start. Uses Rollup for production builds with optimized output.

  • ~ 15M weekly npm downloads
  • ~ 67k GitHub stars
  • Instant dev server start
  • Lightning HMR
Parcel
Zero Config

Zero-configuration bundler that automatically detects and installs dependencies. Great for quick prototyping and simpler projects with minimal setup.

  • ~ 2M weekly npm downloads
  • ~ 43k GitHub stars
  • Zero configuration
  • Auto dependency detection
Architecture Comparison
Webpack Pipeline
Entry Point ↓ Loaders (babel, sass, css) ↓ Plugins (html, mini-css) ↓ Output Bundle
Vite Pipeline
Entry Point ↓ Native ESM (dev) ↓ Rollup (prod) ↓ Optimized Output
Parcel Pipeline
Any File Entry ↓ Auto-detect ↓ Auto-install ↓ Output Bundle

2. Webpack Configuration

Installation
# Core Webpack npm install --save-dev webpack webpack-cli webpack-dev-server # Loaders - process different file types npm install --save-dev sass sass-loader css-loader style-loader npm install --save-dev mini-css-extract-plugin # HTML plugin npm install --save-dev html-webpack-plugin # Bootstrap and its dependencies npm install bootstrap @popperjs/core
webpack.config.js
// webpack.config.js const path = require('path'); const HtmlWebpackPlugin = require('html-webpack-plugin'); const MiniCssExtractPlugin = require('mini-css-extract-plugin'); module.exports = (env, argv) => { const isDev = argv.mode === 'development'; return { // Entry point - where Webpack starts bundling entry: './src/js/main.js', // Output configuration output: { path: path.resolve(__dirname, 'dist'), filename: isDev ? 'js/[name].js' : 'js/[name].[contenthash].js', clean: true, // Clean dist folder before build }, // Module rules - how to process different files module: { rules: [ { // SCSS/SASS processing test: /\.(scss|sass)$/, use: [ isDev ? 'style-loader' : MiniCssExtractPlugin.loader, 'css-loader', 'sass-loader', // Compiles SCSS to CSS ], }, { // Regular CSS (Bootstrap's CSS if using CDN) test: /\.css$/, use: [ isDev ? 'style-loader' : MiniCssExtractPlugin.loader, 'css-loader', ], }, ], }, // Plugins for build optimization plugins: [ // Generates HTML with injected bundles new HtmlWebpackPlugin({ template: './src/index.html', title: 'Bootstrap Webpack Project', }), // Extracts CSS into separate files for production !isDev && new MiniCssExtractPlugin({ filename: 'css/[name].[contenthash].css', }), ].filter(Boolean), // Development server configuration devServer: { static: './dist', port: 3000, hot: true, // Hot Module Replacement open: true, // Open browser on start compress: true, // Gzip compression }, }; };
Entry Point (main.js)
// src/js/main.js // Import Bootstrap's SCSS for full customization import '../scss/main.scss'; // Import Bootstrap JS bundle (includes Popper) import 'bootstrap'; // Or import specific components for tree-shaking import { Tooltip, Toast, Popover } from 'bootstrap'; // Initialize Bootstrap components document.addEventListener('DOMContentLoaded', () => { // Enable all tooltips const tooltipTriggerList = document.querySelectorAll( '[data-bs-toggle="tooltip"]' ); tooltipTriggerList.forEach(el => new Tooltip(el)); });
SCSS Entry (main.scss)
// src/scss/main.scss // Override Bootstrap variables BEFORE import $primary: #6f42c1; $enable-shadows: true; // Import Bootstrap from node_modules @import '~bootstrap/scss/bootstrap'; // Custom styles after Bootstrap .hero-section { padding: 4rem 0; background: linear-gradient(135deg, $primary, darken($primary, 20%)); }
package.json Scripts
{ "scripts": { "dev": "webpack serve --mode development", "build": "webpack --mode production", "build:analyze": "webpack --mode production --stats detailed" } }
Webpack Strengths
  • Massive plugin ecosystem (1000+ plugins)
  • Code splitting and lazy loading
  • Module federation for micro-frontends
  • Extremely configurable for complex needs
Webpack Considerations
  • Slower cold start (bundles everything first)
  • Verbose configuration required
  • Steep learning curve for optimization

3. Vite Configuration

Installation
# Create Vite project npm create vite@latest my-bootstrap-app -- --template vanilla cd my-bootstrap-app npm install # Add Bootstrap and Sass npm install bootstrap @popperjs/core npm install --save-dev sass
vite.config.js
// vite.config.js import { defineConfig } from 'vite'; export default defineConfig({ // Base public path base: '/', // Development server config server: { port: 5173, open: true, // Auto-open browser https: false, // Enable for HTTPS testing host: 'localhost', }, // CSS configuration css: { // Sass preprocessor options preprocessorOptions: { scss: { // Global SCSS variables/mixins injected into every file additionalData: ` $primary: #6f42c1; $enable-shadows: true; `, }, }, // PostCSS plugins (autoprefixer is built-in) postcss: { plugins: [], }, }, // Build output configuration build: { outDir: 'dist', sourcemap: true, // Generate source maps minify: 'terser', // Minification engine rollupOptions: { output: { // Manual chunk splitting manualChunks: { bootstrap: ['bootstrap'], popper: ['@popperjs/core'], }, }, }, }, });
SCSS Entry with Vite
// src/main.js // Vite natively handles SCSS imports import './scss/main.scss'; // Import Bootstrap JS - Vite handles ESM natively import * as bootstrap from 'bootstrap'; // Vite supports dynamic imports for code splitting const loadModal = async () => { const { Modal } = await import('bootstrap'); return Modal; };
index.html (Entry)
<!-- Vite uses index.html as the entry point --> <!-- No HTML plugin needed - it's native --> <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="..."> <title>Vite Bootstrap App</title> </head> <body> <div id="app"></div> <!-- Vite injects the module script --> <script type="module" src="/src/main.js"></script> </body> </html>
package.json Scripts
{ "scripts": { "dev": "vite", "build": "vite build", "preview": "vite preview" } }
Vite Strengths
  • Instant dev server start (no bundling)
  • Blazing fast HMR (sub-50ms updates)
  • Minimal configuration needed
  • Native ESM + optimized Rollup production builds

4. Parcel Configuration

Installation
# Install Parcel npm install --save-dev parcel # Install Bootstrap and Sass npm install bootstrap @popperjs/core npm install --save-dev sass
Minimal Configuration
// .parcelrc (optional - Parcel works with zero config) { "extends": "@parcel/config-default", "transformers": { // Parcel auto-detects .scss files "*.scss": ["@parcel/transformer-sass"], } } // package.json - add source field { "source": "src/index.html", "scripts": { "dev": "parcel", "build": "parcel build" } }
Entry HTML
<!-- src/index.html --> <!-- Parcel uses HTML as the entry point --> <!-- It auto-resolves linked resources --> <!DOCTYPE html> <html> <head> <!-- Parcel processes SCSS linked here --> <link rel="stylesheet" href="./scss/main.scss"> </head> <body> <h1>Parcel Bootstrap App</h1> <!-- Parcel bundles JS linked here --> <script type="module" src="./js/main.js"></script> </body> </html>
Parcel Strengths
  • Truly zero configuration
  • Auto-installs missing dependencies
  • Built-in image optimization
  • Great for quick prototyping

Recommendation for Bootstrap Projects

Vite is recommended for most new Bootstrap 5 projects. It offers instant startup, fast HMR, native SCSS support, and minimal configuration. Use Webpack if you need module federation or have complex legacy build requirements. Use Parcel for quick prototypes where zero-config is preferred.
