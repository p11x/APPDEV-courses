1. Workflow Overview

A professional development workflow consists of 8 interconnected phases. Each phase has specific tools, configurations, and quality gates that ensure code quality from first keystroke to production deployment.

Phase 1
Environment

Node.js, npm, editor config, VS Code extensions

Phase 2
SCSS & Build

Sass compilation, Vite/Webpack, asset pipeline

Phase 3
Dev Server

Live reload, HMR, HTTPS local, network access

Phase 4
Code Quality

ESLint, Stylelint, Prettier, BEM standards

Phase 5
Git & Hooks

Branching, commits, Husky, lint-staged

Phase 6
Testing

Unit tests, E2E, visual regression, coverage

Phase 7
Performance & Security

Lighthouse, bundle analysis, npm audit, Snyk

Phase 8
Deployment

GitHub Actions, Netlify/Vercel, CI/CD pipeline

2. Daily Development Flow

Start: Pull Latest Changes
git checkout main && git pull origin main git checkout -b feature/my-feature # or continue existing branch npm ci # Ensure deps match lock file
Develop: Write Code with Hot Reload
npm run dev # Start dev server with HMR # Edit SCSS/HTML/JS - changes reflect instantly # ESLint/Stylelint show errors in editor # Prettier formats on save
Test: Run Tests Locally
npm run test # Unit tests npm run test:coverage # Check coverage npm run lint # Full lint check
Commit: Pre-commit Hooks Run Automatically
git add . git commit -m "feat: add responsive card component" # Husky triggers pre-commit hook: # 1. lint-staged runs ESLint/Prettier on staged files # 2. Stylelint checks SCSS # 3. commitlint validates message format # If any check fails, commit is blocked
Push: Pre-push Hooks Run Tests
git push -u origin feature/my-feature # Pre-push hook runs full test suite # Create Pull Request on GitHub
Review: CI Pipeline Runs on PR
# GitHub Actions runs automatically: # 1. Lint (ESLint + Stylelint + Prettier) # 2. Unit Tests + Coverage # 3. E2E Tests (Playwright) # 4. Security Audit (npm audit) # 5. Build Verification # 6. Lighthouse Performance Check # 7. Deploy Preview (Netlify/Vercel) # All checks must pass before merge
Deploy: Merge Triggers Production Deploy
# After PR approval and merge to main: # 1. Build runs with production settings # 2. SCSS compiled and minified # 3. JS bundled and tree-shaken # 4. Assets optimized # 5. Deployed to CDN # 6. Cache invalidated # 7. Site live in production

3. New Project Setup Checklist

Step-by-Step Setup
# 1. Create project mkdir my-bootstrap-app && cd my-bootstrap-app npm init -y # 2. Install Bootstrap and build tools npm install bootstrap @popperjs/core npm install --save-dev sass vite # 3. Install linting and formatting npm install --save-dev eslint @eslint/js npm install --save-dev stylelint stylelint-config-standard-scss npm install --save-dev prettier eslint-config-prettier # 4. Install testing tools npm install --save-dev vitest @testing-library/dom npm install --save-dev @testing-library/jest-dom npm install --save-dev jsdom # 5. Install git hooks npm install --save-dev husky lint-staged npm install --save-dev @commitlint/cli @commitlint/config-conventional npx husky init # 6. Create config files touch .editorconfig .prettierrc .stylelintrc.json touch eslint.config.js vite.config.js # 7. Create source structure mkdir -p src/scss/abstracts src/scss/components src/js mkdir -p src/test .vscode .husky # 8. Initialize git git init echo "node_modules/\ndist/" > .gitignore

4. Complete npm Scripts Reference

{ "scripts": { // Development "dev": "vite", "dev:host": "vite --host 0.0.0.0", "dev:https": "vite --host --https", // Building "build": "vite build", "preview": "vite preview", // SCSS "sass:watch": "sass src/scss/main.scss dist/css/style.css --watch", "sass:build": "sass src/scss/main.scss dist/css/style.css --style compressed", // Linting "lint": "npm run lint:js && npm run lint:css", "lint:js": "eslint 'src/**/*.js'", "lint:css": "stylelint 'src/**/*.scss'", "lint:fix": "npm run lint:js:fix && npm run lint:css:fix", "lint:js:fix": "eslint 'src/**/*.js' --fix", "lint:css:fix": "stylelint 'src/**/*.scss' --fix", // Formatting "format": "prettier --write .", "format:check": "prettier --check .", // Testing "test": "vitest", "test:run": "vitest run", "test:coverage": "vitest run --coverage", "test:e2e": "playwright test", // Security "audit": "npm audit --audit-level=high", "audit:fix": "npm audit fix", // Performance "lighthouse": "lhci autorun", "analyze": "source-map-explorer dist/js/*.js", // Validation (all checks) "validate": "npm run format:check && npm run lint && npm run test:run && npm run build", // Git hooks "prepare": "husky", "lint-staged": "lint-staged" } }
