Hot Module Replacement

  • Preserves application state
  • CSS changes: instant, no reload
  • JS changes: module-level swap
  • No loss of form input, scroll position

  • All state is lost

  • Works with any file type
  • Scroll position reset

2. CSS Injection

How CSS Injection Works

When a CSS/SCSS file changes, the dev server sends the new CSS to the browser via WebSocket. The browser replaces the existing <style> tag content without reloading the page, preserving all interactive state.

Vite CSS Configuration
// vite.config.js export default { css: { // Inject CSS as <style> tags (dev) // Extract to files (prod) - automatic devSourcemap: true, // SCSS global variables preprocessorOptions: { scss: { additionalData: ` $enable-gradients: true; $theme-colors: ( "custom": #6f42c1 ); `, }, }, // CSS Modules config modules: { localsConvention: 'camelCase', }, }, };
Webpack CSS HMR
// webpack.config.js module: { rules: [ { test: /\.scss$/, use: [ // style-loader enables CSS HMR // In dev mode - injects <style> tags 'style-loader', 'css-loader', 'sass-loader', ], }, { test: /\.css$/, use: [ { loader: 'style-loader', options: { // Enable HMR for CSS injectType: 'singletonStyleTag', }, }, 'css-loader', ], }, ], }

3. JavaScript HMR

Manual HMR API
// Using the HMR API directly // Works with Vite (import.meta.hot) and Webpack (module.hot) // Vite HMR API if (import.meta.hot) { // Accept updates to this module import.meta.hot.accept((mod) => { // Module has been updated console.log('Module updated:', mod); }); // Accept updates to specific dependency import.meta.hot.accept('./utils.js', (newUtils) => { // Re-run logic with new utils newUtils.initUI(); }); // Handle disposal (cleanup before swap) import.meta.hot.dispose((data) => { // Cleanup: remove event listeners, intervals clearInterval(data.timer); }); } // Webpack HMR API if (module.hot) { module.hot.accept(); }
Bootstrap Component HMR
// HMR with Bootstrap JS components import { Modal, Tooltip } from 'bootstrap'; let activeModals = []; let activeTooltips = []; function initComponents() { // Initialize all tooltips document.querySelectorAll('[data-bs-toggle="tooltip"]') .forEach(el => { activeTooltips.push(new Tooltip(el)); }); } // HMR: cleanup and reinitialize on update if (import.meta.hot) { import.meta.hot.dispose(() => { // Dispose Bootstrap instances before swap activeTooltips.forEach(t => t.dispose()); activeModals.forEach(m => m.dispose()); activeTooltips = []; activeModals = []; }); import.meta.hot.accept(() => { // Reinitialize after module swap initComponents(); }); } initComponents();

SCSS Variables HMR
// _variables.scss - Change these and see instant updates // Theme colors - change any and HMR reflects instantly $primary: #0d6efd; $secondary: #6c757d; $success: #198754; // Spacing scale $spacer: 1rem; // Border radius $border-radius: 0.375rem; // main.scss @import 'variables'; @import '../node_modules/bootstrap/scss/bootstrap'; // With Vite/Webpack + sass-loader, // changing any variable above triggers // instant CSS injection - no reload needed!

HMR Performance Tips
  • CSS changes: ~10-50ms update time
  • Keep SCSS partials small for faster recompilation
  • Use @use instead of @import (faster)
  • Avoid deep nesting (>4 levels) in SCSS
  • Split large SCSS files into focused partials
