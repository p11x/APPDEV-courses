1. npm audit

Running npm audit
# Basic audit - shows vulnerabilities npm audit # JSON output for CI processing npm audit --json > audit-results.json # Show only critical/high vulnerabilities npm audit --audit-level=high # Auto-fix vulnerabilities npm audit fix # Force fix (may include breaking changes) npm audit fix --force # Dry run - see what would change npm audit fix --dry-run
Audit Report Example

CI Integration
# npm scripts for auditing { "scripts": { "audit": "npm audit", "audit:fix": "npm audit fix", "audit:ci": "npm audit --audit-level=high", "audit:json": "npm audit --json > reports/audit.json" } }
GitHub Actions
# .github/workflows/security.yml name: Security Audit on: schedule: - cron: '0 8 * * 1' # Weekly Monday 8am push: branches: [main] jobs: audit: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: actions/setup-node@v4 with: node-version: 20 - run: npm ci - run: npm audit --audit-level=high # Fail if high/critical found

2. Snyk Security

Snyk CLI
# Install Snyk npm install -g snyk # Authenticate snyk auth # Test for vulnerabilities snyk test # Test specific severity snyk test --severity-threshold=high # Monitor project (sends to dashboard) snyk monitor # Fix vulnerabilities snyk fix # Test and ignore specific vuln snyk ignore --id="SNYK-JS-LODASH-567746"
.snyk Policy File
# .snyk version: v1.25.0 ignore: {} patch: {} language-settings: javascript: # Ignore specific vulnerability for 30 days ignore: SNYK-JS-LODASH-567746: - '*': reason: 'Not exploitable in our context' expires: 2025-12-31T00:00:00.000Z
Snyk GitHub Action
# .github/workflows/snyk.yml name: Snyk Security on: [push, pull_request] jobs: security: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: snyk/actions/node@master env: SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }} with: args: --severity-threshold=high # Or use Snyk CLI directly - uses: actions/setup-node@v4 with: node-version: 20 - run: npm ci - run: npx snyk test --severity-threshold=high - run: npx snyk monitor

3. Static Analysis (SAST)

Security-Focused ESLint Rules
npm install --save-dev eslint-plugin-security // eslint.config.js import security from 'eslint-plugin-security'; export default [ { plugins: { security }, rules: { // Detect unsafe regular expressions 'security/detect-unsafe-regex': 'error', // Detect eval() usage 'security/detect-eval-with-expression': 'error', // Detect non-literal require() 'security/detect-non-literal-require': 'warn', // Detect object injection 'security/detect-object-injection': 'warn', // Detect possible timing attacks 'security/detect-possible-timing-attacks': 'warn', // Detect XSS in HTML 'security/detect-bidi-characters': 'error', }, }, ];
Secret Detection
# detect-secrets - find hardcoded secrets pip install detect-secrets # Create baseline detect-secrets scan > .secrets.baseline # Scan for new secrets detect-secrets scan --baseline .secrets.baseline # Pre-commit hook integration # .husky/pre-commit detect-secrets-hook .secrets.baseline
Security Checklist
  • No hardcoded API keys or secrets
  • No eval() or innerHTML with user input
  • All dependencies audited
  • Content Security Policy configured
  • HTTPS enforced in production
  • XSS protection headers set
  • CORS properly configured
