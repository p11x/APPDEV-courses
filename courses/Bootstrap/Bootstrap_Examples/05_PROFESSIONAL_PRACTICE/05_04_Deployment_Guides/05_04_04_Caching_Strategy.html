HTTP Cache Headers

Cache-Control Headers Configuration
# nginx configuration for static assets

# HTML files - always revalidate
location ~* \.html$ {
    add_header Cache-Control "no-cache, must-revalidate";
    add_header Cache-Control "public, max-age=0";
}

# CSS/JS with hash in filename (immutable)
location ~* \.(css|js)$ {
    # If filename contains hash (e.g., app.a1b2c3.js)
    add_header Cache-Control "public, max-age=31536000, immutable";
}

# Images - long cache, stale-while-revalidate
location ~* \.(jpg|jpeg|png|gif|webp|avif|svg|ico)$ {
    add_header Cache-Control "public, max-age=2592000, stale-while-revalidate=86400";
}

# Fonts - immutable (never change once deployed)
location ~* \.(woff|woff2|ttf|otf|eot)$ {
    add_header Cache-Control "public, max-age=31536000, immutable";
    add_header Access-Control-Allow-Origin "*";
}

# Bootstrap CDN assets already have optimal caching
# jsDelivr sets: Cache-Control: public, max-age=31536000, s-maxage=31536000, immutable
Apache .htaccess Configuration
# .htaccess cache configuration

# Enable mod_headers and mod_expires
<IfModule mod_expires.c>
    ExpiresActive On

    # HTML - 0 seconds (always revalidate)
    ExpiresByType text/html "access plus 0 seconds"

    # CSS and JS - 1 year (use cache-busting filenames)
    ExpiresByType text/css "access plus 1 year"
    ExpiresByType application/javascript "access plus 1 year"

    # Images - 30 days
    ExpiresByType image/jpeg "access plus 30 days"
    ExpiresByType image/png "access plus 30 days"
    ExpiresByType image/webp "access plus 30 days"
    ExpiresByType image/svg+xml "access plus 30 days"

    # Fonts - 1 year
    ExpiresByType font/woff2 "access plus 1 year"
    ExpiresByType font/woff "access plus 1 year"
</IfModule>

# Cache-Control headers
<IfModule mod_headers.c>
    <FilesMatch "\.(css|js)$">
        Header set Cache-Control "public, max-age=31536000, immutable"
    </FilesMatch>
</IfModule>

Service Worker Caching

Service Worker Registration
// main.js - Register service worker
if ('serviceWorker' in navigator) {
    window.addEventListener('load', async () => {
        try {
            const reg = await navigator
                .serviceWorker.register(
                    '/sw.js',
                    { scope: '/' }
                );
            console.log(
                'SW registered:', reg.scope
            );
        } catch (err) {
            console.error(
                'SW registration failed:', err
            );
        }
    });
}
Service Worker (Cache-First)
// sw.js - Cache-first strategy
const CACHE_NAME = 'v1.0.0';
const ASSETS = [
    '/',
    '/css/style.css',
    '/js/app.js'
];

self.addEventListener('install', e => {
    e.waitUntil(
        caches.open(CACHE_NAME)
            .then(cache => cache.addAll(ASSETS))
    );
});

self.addEventListener('fetch', e => {
    e.respondWith(
        caches.match(e.request)
            .then(r => r || fetch(e.request))
    );
});

Caching Strategies Comparison

Strategy Behavior Best For Cache-Control
Immutable Never revalidate, use hash-busted filenames Versioned CSS/JS, fonts max-age=31536000, immutable
Stale-While-Revalidate Serve cached, update in background Images, API responses max-age=60, stale-while-revalidate=300
Must-Revalidate Always check with server when stale HTML documents no-cache, must-revalidate
No-Store Never cache Sensitive data, auth tokens no-store
ETag Conditional requests with hash Dynamic content, APIs Server generates ETag header

CDN Cache Configuration

Cache Busting Strategy
// Vite - automatic hash in filename
// Output: app.a1b2c3d4.js
// Output: style.e5f6g7h8.css

// webpack.config.js
output: {
    filename: '[name].[contenthash:8].js',
    chunkFilename: '[name].[contenthash:8].js'
}

// Content hash ensures filename changes
// when file content changes, automatically
// invalidating CDN/browser caches
CDN Purge Strategy
# Cloudflare purge via API
curl -X POST \
  "https://api.cloudflare.com/client/v4/zones/{zone_id}/purge_cache" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "purge_everything": true
  }'

# Or purge specific files
-d '{
  "files": [
    "https://example.com/css/style.css"
  ]
}'
