NPM Package Publishing

Publishing Bootstrap themes, components, and utilities to npm

Initial Setup

1. Create NPM Account
# Register via CLI
npm adduser

# Or login if account exists
npm login

# Verify authentication
npm whoami
2. Initialize Package
# Initialize new npm project
npm init -y

# Interactive setup
npm init

# Scoped package
npm init --scope=@myorg

package.json Configuration

Complete package.json for a Bootstrap Package
{
  "name": "@myorg/bootstrap-theme",
  "version": "1.0.0",
  "description": "Premium Bootstrap 5 theme with custom components",
  "main": "dist/js/bootstrap-theme.min.js",
  "module": "dist/js/bootstrap-theme.esm.js",
  "types": "dist/types/index.d.ts",
  "style": "dist/css/bootstrap-theme.min.css",
  "sass": "scss/bootstrap-theme.scss",
  "exports": {
    ".": {
      "import": "./dist/js/bootstrap-theme.esm.js",
      "require": "./dist/js/bootstrap-theme.min.js",
      "types": "./dist/types/index.d.ts"
    },
    "./css/*": "./dist/css/*",
    "./scss/*": "./scss/*"
  },
  "files": [
    "dist/",
    "scss/",
    "LICENSE"
  ],
  "keywords": [
    "bootstrap", "theme", "css", "components"
  ],
  "author": "Your Name",
  "license": "MIT",
  "repository": {
    "type": "git",
    "url": "https://github.com/myorg/bootstrap-theme"
  },
  "peerDependencies": {
    "bootstrap": ">=5.3.0"
  },
  "devDependencies": {
    "sass": "^1.69.0",
    "rollup": "^4.0.0",
    "postcss": "^8.4.0"
  },
  "scripts": {
    "build": "rollup -c && sass scss:dist/css --style compressed",
    "test": "jest",
    "prepublishOnly": "npm run build && npm test",
    "release": "standard-version"
  }
}
Entry Points
  • main - CommonJS entry (require)
  • module - ES Module entry (import)
  • types - TypeScript definitions
  • exports - Modern conditional exports
Publishing Control
  • files - Whitelist of files to publish
  • .npmignore - Blacklist alternative
  • prepublishOnly - Pre-publish hook
  • private: true - Prevents publishing

Publishing Workflow

Build & Test
# Run build pipeline
npm run build

# Run all tests
npm test

# Check what will be published
npm pack --dry-run
Version Management
# Semantic versioning
npm version patch  # 1.0.0 → 1.0.1 (bugfix)
npm version minor  # 1.0.0 → 1.1.0 (feature)
npm version major  # 1.0.0 → 2.0.0 (breaking)

# Pre-release versions
npm version prerelease --preid=beta
# → 1.0.0-beta.0

# Using standard-version for changelog
npx standard-version
Publish
# Publish to npm registry
npm publish

# Publish scoped package publicly
npm publish --access public

# Publish beta/next tag
npm publish --tag beta

# Dry run (test without publishing)
npm publish --dry-run
NPM Distribution Tags
# Manage distribution tags
npm dist-tag ls                    # List all tags
npm dist-tag add pkg@1.0.0 stable  # Add tag
npm dist-tag rm pkg@1.0.0 beta     # Remove tag

# Tag workflow
npm publish --tag beta       # Publish as beta
npm dist-tag add pkg@1.0.0 latest  # Promote to latest

NPM Publishing Best Practices

Versioning
  • Follow semantic versioning (semver)
  • Use pre-release tags for testing
  • Write changelogs for every release
  • Tag releases in git
Quality
  • Include TypeScript definitions
  • Provide both CJS and ESM builds
  • Add comprehensive README
  • Include LICENSE file
Security
  • Enable 2FA on npm account
  • Use npm provenance (SLSA)
  • Minimize published files
  • Run npm audit before publishing
