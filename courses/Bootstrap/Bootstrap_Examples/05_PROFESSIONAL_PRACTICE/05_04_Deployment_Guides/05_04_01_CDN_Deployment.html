CDN Deployment Strategies

Content Delivery Network setup, optimization, and Bootstrap CDN integration

What is a CDN?

A Content Delivery Network (CDN) distributes your static assets across geographically dispersed edge servers, reducing latency and improving load times for users worldwide.

Key Benefit: CDNs reduce the round-trip time for static assets by serving them from servers closest to the user's location.

Subresource Integrity (SRI)

SRI hashes verify that CDN-served files haven't been tampered with.

SRI Implementation
<!-- Bootstrap CSS with SRI hash -->
<link
  href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css"
  rel="stylesheet"
  integrity="sha384-QWTKZyjpPEjISv5WaRU9OFeRpok6YcnS/1VE1V2WT0mIPln8YfVgIGbYMB8re6U"
  crossorigin="anonymous">

<!-- Bootstrap JS with SRI hash -->
<script
  src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"
  integrity="sha384-YvpcrYf0tY3lHB60NNkmXc5s9fDVZLESaAA55NDzOxhy9GkcIdslK1eN7N6jIeHz"
  crossorigin="anonymous"></script>
Generating SRI Hashes
Using OpenSSL
# Generate SHA-384 hash
cat bootstrap.min.css | \
openssl dgst -sha384 -binary | \
openssl base64 -A
Using Online Tools
  • srihash.org - Generate SRI hashes
  • jsDelivr automatically provides SRI
  • Browser DevTools shows integrity status

Resource Hints & Preloading

<!-- DNS prefetch for CDN domains -->
<link rel="dns-prefetch" href="//cdn.jsdelivr.net">

<!-- Preconnect establishes early connection -->
<link rel="preconnect" href="https://cdn.jsdelivr.net">
<link rel="preconnect" href="https://fonts.googleapis.com">

<!-- Preload critical CSS -->
<link rel="preload" href="css/bootstrap.min.css"
  as="style"
  onload="this.onload=null;this.rel='stylesheet'">

<!-- Preload critical JS -->
<link rel="preload" href="js/bootstrap.bundle.min.js"
  as="script">

CDN Deployment Best Practices

Do's
  • ✓ Always use SRI hashes for CDN resources
  • ✓ Pin to specific versions (avoid @latest)
  • ✓ Use preconnect for primary CDN domains
  • ✓ Set proper cache-control headers
  • ✓ Implement fallback loading strategy
  • ✓ Use HTTP/2 or HTTP/3 for multiplexing
Don'ts
  • ✗ Don't use @latest or @next tags in production
  • ✗ Don't skip SRI verification
  • ✗ Don't rely on a single CDN provider
  • ✗ Don't load unused CSS/JS from CDN
  • ✗ Don't ignore CDN error handling
  • ✗ Don't serve sensitive data via CDN
CDN Fallback Pattern
<!-- Load from CDN with local fallback -->
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css"
  rel="stylesheet"
  integrity="sha384-..."
  crossorigin="anonymous">
<!-- Fallback: check if Bootstrap loaded -->
<script>
  // If Bootstrap CSS fails to load, use local copy
  if (!document.querySelector('.btn')) {
    var link = document.createElement('link');
    link.rel = 'stylesheet';
    link.href = '/css/bootstrap.min.css';
    document.head.appendChild(link);
  }
</script>
