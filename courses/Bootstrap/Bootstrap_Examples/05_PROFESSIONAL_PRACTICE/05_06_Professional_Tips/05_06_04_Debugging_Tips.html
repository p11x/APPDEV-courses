Essential Panels
  • Elements - Inspect/edit DOM & CSS in real-time

2. CSS Debugging Strategies

Outline All Elements

Add to your console or CSS to see every element's boundaries:

// In browser console:
document.querySelectorAll('*').forEach(el => {
    el.style.outline = '1px solid red';
});

/* In CSS (temporary): */
* { outline: 1px solid red !important; }

Inspect Computed Styles

In DevTools Elements panel, check the Computed tab to see final values:

  1. Right-click element > Inspect
  2. Click "Computed" tab
  3. Expand properties to see source
  4. Use "Filter" to search specific properties

Check "Show all" to see inherited and default values.

Content area (click to inspect box model)
Debug Specificity Issues

When a style isn't applying, check the Styles panel:

  • Crossed out = overridden
  • Higher rules win (scroll to see all)
  • Click the file link to jump to source
// Console: check what wins
getComputedStyle(element).backgroundColor;

// See all rules targeting an element
console.table(
    [...element.computedStyleMap()]
);
Visual Debug Classes

Add temporary debug classes to elements:

/* Add to your debug CSS */
.debug-red   { outline: 3px solid red; }
.debug-blue  { outline: 3px solid blue; }
.debug-green { outline: 3px solid green; }

/* Show grid lines */
.debug-grid [class*="col"] {
    outline: 1px dashed rgba(0,0,255,0.5);
}

3. JavaScript Debugging Strategies

Console Techniques
// Styled console output
console.\u0025cStyled text
    'color: blue; font-size: 20px');

// Table for structured data
console.table([
    {name: 'Card', w: 300, h: 200},
    {name: 'Button', w: 100, h: 40}
]);

// Group related logs
console.group('Bootstrap Init');
console.log('Tooltips:', count);
console.log('Modals:', count);
console.groupEnd();

// Time operations
console.time('Render');
// ... expensive code ...
console.timeEnd('Render');

// Trace call stack
console.trace('Where was this called?');
Breakpoint Techniques
// Conditional breakpoint
debugger; // Pauses here

// In Sources panel, right-click line number:
// - "Add breakpoint" - always pause
// - "Add conditional breakpoint"
//   (e.g., i === 5)

// Event listener breakpoints:
// Sources > Event Listener Breakpoints
// Check: click, submit, load

// DOM breakpoint:
// Right-click element in Elements
// > Break on > subtree modifications
Opens DevTools first! Will pause at debugger statement.

4. Bootstrap-Specific Debugging

Common Bootstrap Issues

Cause: Columns exceed 12, or missing .row.
Fix: Check column total = 12, ensure container > row > col structure.

Cause: Bootstrap JS not loaded, or duplicate modal IDs.
Fix: Verify bundle.js loaded, check for duplicate id attributes.

Cause: Missing data-bs-toggle="dropdown" or Popper.js.
Fix: Use bootstrap.bundle.min.js (includes Popper).
Bootstrap Console Checks
// Check Bootstrap version
console.log(
    bootstrap.Tooltip.VERSION
);

// List all active Bootstrap components
document.querySelectorAll(
    '[data-bs-toggle]'
).forEach(el =>
    console.log(
        el.dataset.bsToggle,
        el
    )
);

// Check if Bootstrap CSS loaded
[...document.styleSheets]
    .filter(s => s.href?.includes('bootstrap'))
    .forEach(s =>
        console.log('Loaded:', s.href)
    );

// Manually trigger Bootstrap event
const modal = bootstrap.Modal
    .getOrCreateInstance(
        document.getElementById('myModal')
    );
modal.show();
