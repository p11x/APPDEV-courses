2. Development Workflow

Design Handoff

Designer exports Figma specs with design tokens, component states, and responsive breakpoints. Assets go into shared drive.

Component Planning

Developer reviews design, identifies Bootstrap components to extend, lists custom CSS needs, and creates Jira tickets.

Branch & Build

Create feature branch: feature/component-name. Build component using Bootstrap classes + minimal custom CSS.

Code Review

PR submitted with screenshots. Reviewer checks: naming, responsiveness, a11y, no unnecessary overrides, follows code review checklist.

QA Testing

Test across browsers (Chrome, Firefox, Safari, Edge), devices (mobile, tablet, desktop), and screen readers.

Merge & Deploy

After approval, merge to main. CI runs lint, tests, and builds. Deploy to staging, then production.

3. Design Handoff Best Practices

What Designers Should Provide

Show every state: default, hover, active, disabled, focus, loading, error, empty. Use Figma variants.

Design at 320px (xs), 768px (md), 1024px (lg), and 1440px (xl) minimum. Show how components adapt.

Export colors as hex values, spacing as rem units, font sizes matching Bootstrap scale. Use naming that maps to Bootstrap variables.

Annotate spacing (use Bootstrap's 4px grid), font weights, interactive behaviors, and edge cases (long text, missing images).

4. Shared Component Library

Library Structure
components/
├── _variables.scss     (shared tokens)
├── _mixins.scss        (shared mixins)
├── base/
│   ├── _typography.scss
│   └── _reset.scss
├── layout/
│   ├── _header.scss
│   ├── _footer.scss
│   └── _sidebar.scss
├── ui/
│   ├── _buttons.scss   (extends .btn)
│   ├── _cards.scss     (extends .card)
│   ├── _forms.scss     (extends .form-*)
│   └── _modals.scss
└── pages/
    ├── _home.scss
    └── _dashboard.scss
Component Naming Convention
PatternExample
Block.product-card
Element.product-card__image
Modifier.product-card--featured
State.product-card--loading
Responsive.product-card__image--sm
Rule: Custom component classes use a prefix. Bootstrap classes are used as-is without modification.
Example: Design-to-Code Parity

Figma Design Token:

color-primary: #0d6efd;
spacing-md: 16px;
radius-md: 8px;

Bootstrap Implementation:

class="text-primary p-3 rounded-2"

5. Git Branching Strategy

main - Production-ready code

Protected branch. Merges only from release branches. Auto-deploys to production.

develop - Integration branch

All features merge here first. Auto-deploys to staging for QA testing.

feature/* - New components/features

Branch from develop. Named: feature/product-card, feature/modal-redesign

bugfix/* - Bug fixes

Named: bugfix/grid-overflow-safari, bugfix/modal-z-index

hotfix/* - Production emergencies

Branch from main. Fix critical issues. Merge back to both main and develop.
