🧍 Accessibility Tips

Quick wins, ARIA patterns, and keyboard navigation for inclusive Bootstrap sites.

1. Quick A11y Wins

🌐 lang Attribute
<html>
<html lang="en">

Screen readers need this to use the correct pronunciation.

📷 Image Alt Text
<img src="photo.jpg">
<img src="photo.jpg" alt="Team meeting">

Decorative images use alt="". Meaningful images get descriptive text.

📋 Semantic Elements
<div class="nav">...</div>
<nav aria-label="Main">...</nav>

Use <main>, <nav>, <section>, <article>.

🎨 Color Contrast
Light gray on white
Dark gray on white (4.5:1+)

WCAG AA: 4.5:1 for text, 3:1 for large text. Use WebAIM Contrast Checker.

📝 Form Labels
<input placeholder="Name">
<label for="name">Name</label>
<input id="name">

Placeholders disappear - always use visible or associated labels.

👁 Focus Indicators
:focus { outline: none; }
:focus-visible { outline: 3px solid blue; }

Never remove focus indicators. Use :focus-visible for keyboard-only focus styles.

2. ARIA Patterns with Bootstrap

Bootstrap 5 includes ARIA attributes in its components. Here's how to use them correctly.

Alerts (role="alert")

Bootstrap alerts use role="alert" automatically for screen reader announcements:

Warning! This alert will be announced by screen readers. 
<div class="alert alert-warning"
     role="alert">
  <button aria-label="Close
    warning alert">...</button>
</div>
Live Regions (aria-live)

For dynamic content updates, use aria-live regions:

Status: Ready 
<div aria-live="polite"
     aria-atomic="true">
  Status updates announced
  without interrupting
</div>

<!-- assertive = interrupts -->
<div aria-live="assertive">
  Critical error messages
</div>
Modal A11y

Bootstrap modals handle focus trap and ARIA automatically. Ensure proper labeling:

<!-- aria-labelledby points to
     modal title -->
<div class="modal" role="dialog"
     aria-labelledby="modalTitle"
     aria-modal="true">
  <h5 id="modalTitle">...</h5>
</div>

3. Keyboard Navigation

Essential Keyboard Patterns
KeyAction
TabMove to next interactive element
Shift+TabMove to previous element
EnterActivate links, buttons, submit forms
SpaceActivate buttons, toggle checkboxes
EscClose modals, dropdowns, menus
↓ ↑Navigate dropdown items, tabs

Use Tab to navigate through these elements:

Skip Navigation Link

Press Tab on this page (before interacting with anything else) to see the "Skip to main content" link. This helps keyboard users bypass the navigation.

<!-- First element in body -->
<a href="#main-content"
   class="skip-link">
  Skip to main content
</a>

<!-- Target section -->
<main id="main-content">
  ...
</main>

/* CSS: hide until focused */
.skip-link {
  position: absolute;
  top: -100px;
}
.skip-link:focus {
  top: 0;
}

Key A11y Patterns for Forms
  • <label for="id"> - Associates label with input
  • aria-required="true" - Marks required fields
  • aria-invalid="true" - Indicates validation errors
  • aria-describedby - Links to help/error text
  • <fieldset> + <legend> - Groups related inputs
  • role="alert" on error messages - Announced immediately
