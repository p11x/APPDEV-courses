📋 Code Review Checklist

A comprehensive checklist for reviewing Bootstrap-based projects.

1. HTML Structure Review

✔ Check Severity Details
DOCTYPE declaration present Critical <!DOCTYPE html> must be first line
lang attribute on html tag Critical <html lang="en"> for screen readers
Meta viewport present Critical Required for responsive design
Charset meta tag Warning <meta charset="UTF-8">
Semantic HTML5 elements used Warning Use <main>, <section>, <article>, <nav>, <footer>
No duplicate IDs Critical IDs must be unique - breaks Bootstrap JS
All images have alt attributes Critical A11y requirement - even decorative: alt=""
Scripts at end of body Info Improves perceived load time

2. CSS Review

✔ Check Severity Details
No ID selectors in custom CSS Warning IDs cause specificity issues - use classes
Max 3 levels of nesting Warning Deeper nesting = fragile selectors
!important used sparingly Critical Max 5 instances in custom CSS
No inline styles Warning Except dynamic values from JS
Bootstrap utilities preferred Info Use .d-flex over display: flex
Custom classes are namespaced Info Prefix with project name: .app-
No duplicate CSS rules Warning Use linter to detect duplicates

3. Bootstrap-Specific Review

Grid System
  • container > row > col hierarchy
  • Columns sum to 12 per row
  • Responsive breakpoints defined
  • No empty columns for spacing
  • Gutter classes used (.g-*)
Components
  • Correct data-bs-* attributes
  • ARIA labels on interactive elements
  • Unique IDs for toggled components
  • Bootstrap JS bundle loaded
  • No jQuery dependency
Responsive Design
  • Mobile-first approach (sm first)
  • Images use .img-fluid
  • Tables use .table-responsive
  • Navbar collapses properly
  • No horizontal scroll at any width
Performance
  • CSS loaded in head
  • JS loaded before closing body
  • Images have loading="lazy"
  • Font loading optimized
  • No unused Bootstrap components

4. Accessibility Review

✔ Check Severity Details
All images have meaningful alt text Critical Describe content, not "image of..."
Form inputs have labels Critical Use <label> or aria-label
Color contrast meets WCAG AA Critical 4.5:1 for normal text, 3:1 for large
Keyboard navigation works Critical Tab through all interactive elements
Focus indicators visible Warning Don't remove outline without replacement
Heading hierarchy correct Warning h1 > h2 > h3, no skipping levels
ARIA roles used correctly Warning Bootstrap handles most - verify custom ones
