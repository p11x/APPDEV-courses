⚡ Performance Tips

Quick wins for faster Bootstrap pages: critical CSS, lazy loading, and optimization strategies.

1. Core Web Vitals

2.5s
LCP Target
Largest Contentful Paint

Time until the largest content element is visible. Optimize images, use CDN, reduce server response time.

100ms
FID Target
First Input Delay

Time from first interaction to browser response. Minimize JS execution, defer non-critical scripts.

0.1
CLS Target
Cumulative Layout Shift

Visual stability. Set dimensions on images/embeds, avoid injecting content above the fold.

2. Critical CSS Strategy

Technique: Inline + Async Load

Inline critical CSS in <head>, load full stylesheet asynchronously:

<!-- Critical CSS inlined in head -->
<style>
  /* Only above-the-fold styles */
  .hero { padding: 4rem 0; }
  .navbar { position: sticky; }
</style>

<!-- Full CSS loaded async -->
<link rel="preload"
      href="bootstrap.min.css"
      as="style"
      onload="this.onload=null;this.rel='stylesheet'">

<!-- Fallback for no-JS -->
<noscript>
  <link rel="stylesheet"
        href="bootstrap.min.css">
</noscript>
Tools for Extracting Critical CSS
  • critical (npm)
    Automatically extracts above-the-fold CSS
  • criticalCSS
    CLI tool for critical CSS extraction
  • PurgeCSS
    Removes unused CSS from Bootstrap 
    npx purgecss --css bootstrap.min.css \
  --content *.html \
  --output dist/css/
  • Chrome DevTools
    Coverage tab shows used vs unused CSS bytes

3. Lazy Loading

Native Lazy Loading

Add loading="lazy" to images/iframes below the fold:

<img src="photo.jpg"
     class="img-fluid"
     loading="lazy"
     alt="Lazy loaded image"
     width="800"
     height="600">

<!-- Always set width/height to
     prevent CLS -->
content-visibility: auto

CSS-only lazy rendering. Browser skips off-screen content:

.below-fold-section {
  content-visibility: auto;
  contain-intrinsic-size: 0 500px;
  /* Tells browser the approximate
     height for scrollbar calculation */
}
Browser Support: Chrome 85+, Edge 85+, Opera 71+. Graceful degradation in others.
Skeleton Loading Screens

Show placeholder UI while content loads:

<!-- Replace with real content
     when loaded -->
<div class="skeleton"></div>
Script Loading Strategies
<!-- 1. Defer: Downloads parallel,
     executes after HTML parsed -->
<script src="app.js" defer></script>

<!-- 2. Async: Downloads parallel,
     executes when ready -->
<script src="analytics.js"
        async></script>

<!-- 3. Module: Deferred by default,
     respects import order -->
<script type="module"
        src="app.mjs"></script>

<!-- 4. Dynamic import: Load on
     demand -->
<script>
  btn.addEventListener('click',
    async () => {
      const module = await import(
        './heavy-feature.js'
      );
      module.init();
    }
  );
</script>

4. Quick Performance Wins

💾

Use CDN

Bootstrap CDN is globally cached. One user's visit caches it for all sites using the same CDN URL.

👁

Preload Key Resources

<link rel="preload"> for fonts and critical CSS tells the browser to fetch them early.

🎨

Purge Unused CSS

Bootstrap CSS is ~200KB. PurgeCSS can reduce it to ~10KB by removing unused component styles.

📷

Optimize Images

Use WebP/AVIF formats, srcset for responsive sizes, and always set width/height to prevent CLS.
