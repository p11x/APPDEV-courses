2. Common Browser Quirks

🐦 Safari Quirks

Safari <14.1 doesn't support gap in flexbox.

/* Fallback */ .flex-container > * + * { margin-left: 1rem; } /* Modern browsers override */ @supports (gap: 1rem) { .flex-container { gap: 1rem; } .flex-container > * + * { margin-left: 0; } }

Mobile Safari's 100vh includes the address bar, causing overflow.

/* Use dvh instead */ .full-screen { height: 100vh; /* fallback */ height: 100dvh; /* dynamic */ } /* Or use JS */ document.documentElement.style .setProperty('--vh', window.innerHeight * 0.01 + 'px' );

Safari doesn't support input type="date" on macOS. Use a date picker library or type="text" with validation.

🦌 Firefox Quirks

Firefox uses scrollbar-width instead of ::-webkit-scrollbar.

/* Firefox */ .container { scrollbar-width: thin; scrollbar-color: #888 #f1f1f1; } /* Chrome/Safari */ .container::-webkit-scrollbar { width: 8px; } .container::-webkit-scrollbar-thumb { background: #888; }

Firefox added :focus-visible in v85. Use both :focus and :focus-visible for coverage.

3. Polyfills & Fallbacks

@supports Feature Detection

Use CSS @supports to provide fallbacks:

/* Fallback for older browsers */ .grid-container { display: flex; flex-wrap: wrap; } .grid-container > * { flex: 0 0 50%; } /* Modern browsers get grid */ @supports (display: grid) { .grid-container { display: grid; grid-template-columns: repeat(2, 1fr); } .grid-container > * { flex: none; } }
JavaScript Feature Detection

Detect API support before using:

// Check for IntersectionObserver if ('IntersectionObserver' in window) { // Use modern lazy loading new IntersectionObserver(...) } else { // Fallback: load all images images.forEach(img => img.src = img.dataset.src ); } // Check for CSS.supports() if (CSS.supports('display', 'grid')) { document.body.classList .add('css-grid'); }
Polyfill Services

Load polyfills only for browsers that need them:

<!-- polyfill.io serves only what the browser needs --> <script src="https://polyfill.io/v3/ polyfill.min.js?features= IntersectionObserver, smoothscroll, CSS.supports"> </script>
Note: polyfill.io was sold and is no longer trusted. Use self-hosted polyfills or alternatives like core-js.
Autoprefixer (Build Tool)

Automatically adds vendor prefixes based on your browser support targets:

// package.json or .browserslistrc "browserslist": [ ">= 0.5%", "last 2 versions", "not dead", "not op_mini all" ] // PostCSS config module.exports = { plugins: [ require('autoprefixer') ] };

4. Progressive Enhancement

The Philosophy

Start with a functional base (HTML), enhance with CSS, then add JS interactions. The page works at every level.

📋

Layer 1: HTML

Content is accessible. Forms submit. Links navigate.

🎨

Layer 2: CSS

Bootstrap layout. Responsive grid. Visual polish.

🚀

Layer 3: JS

Modals, tooltips, dynamic content, SPA behavior.

Example: Form Works Without JS
<!-- Form submits normally without JS --> <form action="/api/submit" method="POST"> <input name="email" type="email" required> <button type="submit">Subscribe</button> </form> <!-- JS enhances with AJAX submission --> <script> document.querySelector('form') .addEventListener('submit', e => { e.preventDefault(); fetch('/api/submit', { method: 'POST', body: new FormData(e.target) }); }); </script>
