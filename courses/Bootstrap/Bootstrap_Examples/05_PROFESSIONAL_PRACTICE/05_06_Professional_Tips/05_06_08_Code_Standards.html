📋 Code Standards

Linters, formatters, pre-commit hooks, and CI pipeline configuration for Bootstrap projects.

1. Linters

HTML Linting

Tools: html-validate, HTMLHint

Catches missing attributes, invalid nesting, duplicate IDs, and Bootstrap structure errors.

Install
npm install --save-dev html-validate
.htmlvalidate.json
{ "extends": ["htmlvalidate:recommended"], "rules": { "no-dup-id": "error", "element-required-attributes": "error", "no-trailing-whitespace": "warn" } }
CSS Linting

Tool: Stylelint

Enforces CSS quality: no invalid rules, specificity limits, nesting depth.

Install
npm install --save-dev \
  stylelint \
  stylelint-config-standard
.stylelintrc.json
{ "extends": "stylelint-config-standard", "rules": { "max-nesting-depth": 3, "no-duplicate-selectors": "error", "declaration-no-important": "error", "selector-max-id": 0, "selector-class-pattern": "^[a-z][a-z0-9-]*$" } }
JS Linting

Tool: ESLint

Enforces JS quality, catches errors, ensures consistent coding style.

Install
npm install --save-dev \
  eslint \
  eslint-config-recommended
.eslintrc.json
{ "env": { "browser": true, "es2021": true }, "extends": "eslint:recommended", "rules": { "no-unused-vars": "warn", "no-console": "warn", "eqeqeq": "error" } }

2. Formatters

Prettier - Universal Formatter

Auto-formats HTML, CSS, and JS. Eliminates formatting debates.

Install
npm install --save-dev prettier
.prettierrc
{ "printWidth": 100, "tabWidth": 2, "useTabs": false, "semi": true, "singleQuote": true, "htmlWhitespaceSensitivity": "ignore" }
.prettierignore
# Don't format vendor files css/bootstrap.min.css js/bootstrap.bundle.min.js dist/
EditorConfig - Editor Consistency

Ensures all team members use the same editor settings regardless of IDE.

.editorconfig
# Top-most EditorConfig file root = true # All files [*] indent_style = space indent_size = 2 end_of_line = lf charset = utf-8 trim_trailing_whitespace = true insert_final_newline = true # HTML files - 2 spaces [*.html] indent_size = 2 # CSS files - 2 spaces [*.css] indent_size = 2

3. Pre-Commit Hooks & CI Pipeline

Husky + lint-staged

Run linters only on staged files before each commit.

Setup
# Install Husky
npm install --save-dev husky lint-staged
npx husky init

# Add pre-commit hook
echo "npx lint-staged" > .husky/pre-commit
package.json
{ "lint-staged": { "*.html": ["html-validate", "prettier --write"], "*.css": ["stylelint", "prettier --write"], "*.js": ["eslint", "prettier --write"] } }
GitHub Actions CI

Automate linting and testing on every push/PR.

.github/workflows/ci.yml
name: CI on: [push, pull_request] jobs: lint: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: actions/setup-node@v4 with: node-version: 20 - run: npm ci - run: npm run lint:html - run: npm run lint:css - run: npm run lint:js - run: npm run format:check
Recommended npm scripts
"scripts": { "lint:html": "html-validate '**/*.html'", "lint:css": "stylelint '**/*.css'", "lint:js": "eslint '**/*.js'", "format": "prettier --write '**/*.{html,css,js,md}'", "format:check": "prettier --check '**/*.{html,css,js,md}'", "lint": "npm run lint:html && npm run lint:css && npm run lint:js" }

4. Bootstrap-Specific Lint Rules

RuleToolPurposeConfig
No ID selectors Stylelint Prevents specificity issues "selector-max-id": 0
Max nesting depth Stylelint Keeps selectors flat "max-nesting-depth": 3
No !important Stylelint Forces proper specificity "declaration-no-important": "error"
No duplicate IDs html-validate Bootstrap JS depends on unique IDs "no-dup-id": "error"
Required alt on images html-validate Accessibility requirement "element-required-attributes": "error"
Valid Bootstrap classes Custom Catches typos in class names Custom validation script
