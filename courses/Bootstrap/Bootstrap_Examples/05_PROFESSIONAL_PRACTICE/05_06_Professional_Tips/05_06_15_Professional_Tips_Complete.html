1. Best Practices

1 Naming Conventions
  • Use semantic names: .user-card not .blue-box
  • Prefix custom classes: .app-header
  • BEM for custom components: .block__element--modifier
  • Never use IDs in CSS selectors
2 Structure Rules
  • Always: .container > .row > .col
  • Use semantic HTML5: <main>, <section>, <nav>
  • Keep nesting depth ≤ 3 levels
  • Use Bootstrap utilities over custom CSS

2. Common Mistakes to Avoid

MistakeImpactFix
!important abuse Maintenance nightmare Use flat selectors, CSS variables
Specificity wars (4+ nesting) Fragile, hard to override Single-class selectors
Missing .container Full-width overflow Always container > row > col
Custom media queries Duplicates Bootstrap breakpoints Use .col-md-6 classes
Inline styles Overrides everything, not maintainable Utility classes or custom CSS classes
Columns exceed 12 Unexpected wrapping Ensure col spans sum to 12

3. Pro Techniques

🛠

CSS Custom Properties

Override --bs-primary to retheme without Sass compilation.

/* Instant retheme */
:root { --bs-primary: #ff6600; }

🔄

Cascade Layers

Control specificity order with @layer instead of !important.

/* Priority control */
@layer base, bootstrap, overrides;

📄

Container Queries

Components that adapt to their container width, not viewport.

/* Container-based responsive */
@container (min-width: 400px) { }

4. Debugging Quick Reference

// CSS Debug: outline all elements
* { outline: 1px solid red !important; }

// Console: check computed style
getComputedStyle(el).backgroundColor;

// Console: find Bootstrap components
document.querySelectorAll('[data-bs-toggle]')
  .forEach(el => console.log(el.dataset.bsToggle));

// Check Bootstrap version
console.log(bootstrap.Tooltip.VERSION);
DevTools Shortcuts:
  • Ctrl+Shift+C - Inspect element
  • Ctrl+Shift+I - Open DevTools
  • Esc - Toggle console
  • Ctrl+Shift+P - Command menu

5. Code Review Essentials

Critical
  • DOCTYPE & lang attribute
  • Meta viewport present
  • No duplicate IDs
  • All images have alt text
  • Form inputs have labels
  • Container > Row > Col
Warning
  • No ID selectors in CSS
  • Max 3 nesting levels
  • Semantic HTML elements
  • Responsive breakpoints
  • Heading hierarchy (h1-h6)
  • Focus indicators visible
Info
  • Scripts at end of body
  • Namespaced custom classes
  • CSS loaded in head
  • Lazy loading images
  • No unused Bootstrap components
  • Documentation up to date

7. Security Essentials

XSS Prevention
// SAFE: textContent auto-escapes
el.textContent = userInput;

// SAFE: DOMPurify for rich HTML
el.innerHTML = DOMPurify.sanitize(input);

// DANGEROUS:
el.innerHTML = userInput; // NEVER!
CSP Header
<meta http-equiv="Content-Security-Policy"
  content="
    default-src 'self';
    script-src 'self' https://cdn.jsdelivr.net;
    style-src 'self' 'unsafe-inline';
    img-src 'self' data:;
  ">

11. Framework Integration

FrameworkRecommended LibraryInstall
React react-bootstrap npm install react-bootstrap bootstrap
Vue 3 bootstrap-vue-next npm install bootstrap bootstrap-vue-next
Angular ng-bootstrap npm install @ng-bootstrap/ng-bootstrap

Key Rule: In frameworks, use framework-specific component libraries instead of vanilla Bootstrap JS. Frameworks manage the DOM - don't let Bootstrap JS also manipulate it.
