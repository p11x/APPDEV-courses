1. Cross-Site Scripting (XSS) Prevention

⚠ What is XSS?

Attackers inject malicious scripts into your pages via user input. This can steal cookies, redirect users, or deface content.

✘ Vulnerable Patterns
// NEVER: innerHTML with user input element.innerHTML = userInput; // NEVER: document.write document.write(userInput); // NEVER: eval with user input eval(userInput); // NEVER: href with javascript: `<a href="javascript:${userInput}">` // NEVER: jQuery .html() with user data $('#output').html(userInput); // NEVER: Unquoted attribute values `<div class=${userInput}>`
✔ Safe Patterns
// DO: textContent (auto-escapes) element.textContent = userInput; // DO: createElement + textContent const p = document.createElement('p'); p.textContent = userInput; container.appendChild(p); // DO: Bootstrap's built-in sanitizer // Bootstrap tooltips/popovers sanitize // HTML by default // DO: Use textContent for dynamic content document.getElementById('name') .textContent = userName; // DO: DOMPurify for rich content element.innerHTML = DOMPurify .sanitize(richUserInput);

2. Content Security Policy (CSP)

CSP tells the browser which resources are allowed to load, blocking XSS and data injection attacks.

Recommended CSP for Bootstrap
<!-- Via meta tag --> <meta http-equiv="Content-Security-Policy" content=" default-src 'self'; script-src 'self' https://cdn.jsdelivr.net; style-src 'self' https://cdn.jsdelivr.net 'unsafe-inline'; img-src 'self' https://placehold.co data:; font-src 'self' https://cdn.jsdelivr.net; connect-src 'self'; frame-ancestors 'none'; base-uri 'self'; form-action 'self'; ">
CSP Directives Explained
DirectivePurpose
default-srcFallback for all resource types
script-srcAllowed JS sources (CDN for Bootstrap)
style-srcAllowed CSS sources
img-srcAllowed image sources
font-srcAllowed font sources
connect-srcAllowed fetch/AJAX endpoints
frame-ancestorsPrevents clickjacking
base-uriPrevents base tag injection
form-actionRestricts form submission targets
⚠ Note on 'unsafe-inline'

Bootstrap uses inline styles for some components (e.g., modal positioning). If you can avoid inline styles in your own code, you can use 'unsafe-inline' only for style-src while keeping script-src strict.

3. Input Sanitization

DOMPurify - Client-Side Sanitizer

The gold standard for sanitizing HTML in the browser:

// Install: npm install dompurify
import DOMPurify from 'dompurify';

// Sanitize user HTML
const clean = DOMPurify.sanitize(
  userHTML,
  {
    ALLOWED_TAGS: [
      'b', 'i', 'em', 'strong',
      'a', 'p', 'br'
    ],
    ALLOWED_ATTR: ['href', 'title']
  }
);

// Use sanitized HTML safely
output.innerHTML = clean;
Bootstrap's Built-in Sanitizer

Bootstrap 5 sanitizes tooltip/popover content by default:

// Bootstrap's DefaultAllowList
const ALO = bootstrap.Tooltip
  .Default.allowList;

// Default allowed tags:
// a, abbr, b, br, code, em,
// i, strong, small, span, sub, sup

// Customize the allowlist:
ALO.table = [];
ALO.thead = [];
ALO.tbody = [];
ALO.tr = [];
ALO.td = [];
ALO.th = [];

// Or disable sanitization (NOT recommended):
new bootstrap.Tooltip(el, {
  sanitize: false  // Danger!
});

4. Bootstrap-Specific Security

⚠ Tooltip/Popover XSS

Custom HTML in tooltips can be an XSS vector if not sanitized:

Dangerous: title: userControlledString

Safe approach: Use data-bs-title attribute (auto-escaped) or ensure content is sanitized before passing to Bootstrap JS.

🔒 Modal & Offcanvas Security
  • Always use data-bs-backdrop="static" for critical confirmations
  • Validate form inputs inside modals server-side
  • Don't load modal content from URL parameters without sanitization
  • Use data-bs-focus="true" (default) to maintain focus trap
