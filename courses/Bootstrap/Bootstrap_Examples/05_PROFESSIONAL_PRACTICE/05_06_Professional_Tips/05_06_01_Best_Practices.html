Bootstrap Best Practices

Naming conventions, project structure, and performance optimization for professional Bootstrap development.

Naming Conventions

📌
Semantic Classes

Use .product-card instead of .blue-box. Class names should describe purpose, not appearance.

.user-profile
.big-blue-div
🔒
Namespace Prefixes

Prefix custom classes with a project identifier: .app-header, .my- to prevent Bootstrap conflicts.

.app-navbar
.navbar-custom
🛠
BEM Convention

Block__Element--Modifier keeps custom components organized and predictable.

.card
.card__header
.card--featured

Project Structure

Recommended File Structure
project/
├── css/
│   ├── bootstrap.min.css    (vendor)
│   ├── custom.css           (your styles)
│   └── utilities.css        (overrides)
├── js/
│   ├── bootstrap.bundle.min.js
│   └── app.js
├── img/
│   ├── icons/
│   └── content/
├── index.html
└── components/
    ├── header.html
    └── footer.html
HTML Structure Rules
  • 1. Always wrap content in .container
  • 2. Rows must be direct children of containers
  • 3. Columns must be direct children of rows
  • 4. Use semantic HTML5 elements (<main>, <section>, <article>)
  • 5. Keep nesting depth under 4 levels

Performance Tips

⚡

Responsive Images

Always use .img-fluid and appropriate srcset for different screen sizes.

💾

Lazy Loading

Use loading="lazy" on images below the fold to improve initial page load.

📝

Use Utilities First

Prefer Bootstrap utilities (.d-flex, .p-3) over custom CSS to reduce stylesheet size.

d-flex p-3 gap-2
