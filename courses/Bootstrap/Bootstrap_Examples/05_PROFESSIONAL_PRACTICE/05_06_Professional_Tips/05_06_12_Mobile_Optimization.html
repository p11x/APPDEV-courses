Mobile-first approach, touch optimization, and responsive images for Bootstrap.

1. Mobile-First Approach

Bootstrap's CSS is mobile-first. Base styles apply to all screens. Use responsive prefixes to add styles at larger breakpoints.

Bootstrap Breakpoints
PrefixMin WidthDevice
(none)0Portrait phones
sm576pxLandscape phones
md768pxTablets
lg992pxDesktops
xl1200pxLarge desktops
xxl1400pxExtra large
Mobile-First CSS Pattern
/* Base = mobile (all screens) */
.card-grid {
  display: flex;
  flex-direction: column;
  gap: 1rem;
}

/* Tablet and up: 2 columns */
@media (min-width: 768px) {
  .card-grid {
    flex-direction: row;
    flex-wrap: wrap;
  }
  .card-grid > * {
    flex: 0 0 50%;
  }
}

/* Bootstrap way (in HTML): */
/* <div class="row"> */
/*   <div class="col-md-6"> */

2. Touch Optimization

Touch Target Sizes

WCAG requires minimum 44x44px touch targets:

44x44px (min)

56x56px (better)

.touch-btn {
  min-width: 44px;
  min-height: 44px;
  padding: 12px;
}

/* Bootstrap .btn-lg = ~48px */
/* Bootstrap .btn = ~38px */
Touch-Friendly Spacing

Space interactive elements at least 8px apart:

List item with padding Easy to tap each item 8px+ gap between targets 
/* Use padding, not margin, for
   tap area expansion */
.list-group-item {
  padding: 0.75rem 1rem; /* 12px 16px */
}

/* Bootstrap .list-group-item
   already has good spacing */
CSS Touch Helpers
/* Disable tap highlight on iOS */
.btn {
  -webkit-tap-highlight-color:
    transparent;
}

/* Remove 300ms delay */
html {
  touch-action: manipulation;
}

/* Prevent text selection on
   buttons */
.btn {
  user-select: none;
}

/* Smooth scroll for touch */
html {
  scroll-behavior: smooth;
  -webkit-overflow-scrolling:
    touch;
}
/* Swipeable container */
.swipe-container {
  overflow-x: auto;
  scroll-snap-type: x mandatory;
  display: flex;
  gap: 1rem;
  -webkit-overflow-scrolling:
    touch;
}
.swipe-item {
  scroll-snap-align: start;
  flex: 0 0 80%;
}

/* Hide scrollbar */
.swipe-container::-webkit-scrollbar {
  display: none;
}

3. Responsive Images

img-fluid + srcset
<img
  src="photo-800.jpg"
  srcset="
    photo-400.jpg 400w,
    photo-800.jpg 800w,
    photo-1200.jpg 1200w
  "
  sizes="
    (max-width: 576px) 100vw,
    (max-width: 992px) 50vw,
    33vw
  "
  class="img-fluid rounded"
  loading="lazy"
  width="1200"
  height="800"
  alt="Responsive image"
>

img-fluid = max-width:100%; height:auto
srcset = browser picks best resolution
sizes = tells browser how wide image will be
loading="lazy" = deferred loading
width/height = prevents CLS

picture Element
<picture>
  <!-- AVIF for modern browsers -->
  <source
    srcset="photo.avif"
    type="image/avif"
  >
  <!-- WebP fallback -->
  <source
    srcset="photo.webp"
    type="image/webp"
  >
  <!-- JPG fallback -->
  <img
    src="photo.jpg"
    class="img-fluid"
    alt="Format negotiation"
  >
</picture>

/* Use for art direction too: */
<picture>
  <source
    media="(min-width:768px)"
    srcset="landscape.jpg"
  >
  <img src="portrait.jpg"
       alt="Art directed">
</picture>
