⚠ Common Bootstrap Mistakes

Learn from the most frequent pitfalls and how to avoid them.

1. Specificity Wars

When you fight Bootstrap's CSS specificity, you lose. Here's how to work with it instead.

WRONG
Escalating Specificity
/* DON'T: Chain selectors to override */ body .container .row .col .card .card-body h5 { color: red; } /* Even worse: adding more selectors */ html body main .container div.row div.col-lg-6 div.card div.card-body h5 { color: red !important; }

Problem: Each override requires more specificity, creating an arms race.

RIGHT
Flat, Simple Selectors
/* DO: Use a single, descriptive class */ .cm-card-title { color: #dc3545; } /* Or use CSS custom properties */ :root { --card-title-color: #dc3545; } .card-title { color: var(--card-title-color); }

Solution: One class, one rule. Clean and maintainable.

2. !important Abuse

Every !important declaration is technical debt. Here are alternatives.

🔎

Use Better Selectors

Instead of !important, write a more specific (but still flat) selector.

.card.cm-theme { border-color: blue; }

🛠

CSS Variables

Override Bootstrap's CSS custom properties to change values globally.

:root { --bs-primary: #ff6600; }

📦

Sass Variables

With Sass, override Bootstrap variables before importing to change defaults.

$primary: #ff6600; @import "bootstrap";

🎯

Utility Classes

Bootstrap utilities have the right specificity. Use them directly.

class="bg-primary text-white p-3 rounded"

3. Over-Nesting

Deep nesting makes CSS brittle and hard to debug.

Over-Nested CSS (6 levels deep)
.page-wrapper .main-content
  .content-area .section
  .section-inner .card-group
  .card .card-header
  .card-header .title {
  font-size: 1.25rem;
}

/* If structure changes, this breaks! */
Flat CSS (1 level)
.cm-card-title {
  font-size: 1.25rem;
}

/* Survives any structural change! */
Nesting Depth Guide
Depth Example Verdict
1 level .my-component ✔ Ideal
2 levels .card .card-title ✔ Acceptable (Bootstrap pattern)
3 levels .card .card-body .card-title ⚠ Max recommended
4+ levels .page .section .card .body .title ✘ Refactor needed

4. Grid System Mistakes

✘ Missing container

Without .container, rows have no max-width and content stretches edge-to-edge.

<!-- WRONG: No container -->
<div class="row">
  <div class="col">...</div>
</div>
✔ Proper nesting

Always: container > row > col. No exceptions.

<!-- CORRECT -->
<div class="container">
  <div class="row">
    <div class="col">...</div>
  </div>
</div>
✘ Columns exceed 12

Bootstrap grid = 12 columns. Exceeding wraps to new line unexpectedly.

<!-- WRONG: 13 total -->
<div class="row">
  <div class="col-8">...</div>
  <div class="col-6">...</div>
</div>
✔ Columns total 12

Keep total column span at 12 or use .col for auto-sizing.

<!-- CORRECT: 8 + 4 = 12 -->
<div class="row">
  <div class="col-8">...</div>
  <div class="col-4">...</div>
</div>

Mistake Checklist

⚠ If your code has any of these, refactor!
  • Selectors with 4+ levels of nesting
  • More than 5 !important declarations in your custom CSS
  • Custom media queries matching Bootstrap breakpoints
  • div or span as direct selectors in CSS
  • Inline styles overriding Bootstrap classes
  • IDs in CSS selectors (e.g., #myDiv .card)
  • Overriding .container max-width without good reason
