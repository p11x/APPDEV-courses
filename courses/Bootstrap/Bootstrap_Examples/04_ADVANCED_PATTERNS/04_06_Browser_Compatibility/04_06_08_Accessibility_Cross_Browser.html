ARIA attributes, keyboard navigation, screen reader support, and cross-browser accessibility patterns.

1. ARIA Attributes

ARIA Live Regions

Live regions announce dynamic content changes to screen readers. Different aria-live values control urgency.

aria-live="polite"

Waits for user to pause

Status updates appear here.

aria-live="assertive"

Interrupts immediately

Alerts appear here.

role="status"

Implicit polite live region

Status messages.

ARIA States & Properties

aria-expanded toggle

This content is revealed when aria-expanded becomes true. Screen readers announce the state change.

aria-pressed toggle

aria-pressed indicates toggle button state to screen readers.

2. Keyboard Navigation

Tab Order & Focus Management

Natural tab order follows DOM order. Use tabindex carefully:

tabindex="0"

Adds to natural tab order

Focusable div (tabindex="0")

tabindex="-1"

Programmatic focus only

tabindex="-1" - not in tab order

tabindex > 0 (Avoid!)

Anti-pattern - breaks natural order

Roving Tabindex Pattern

Arrow keys navigate within a group. Only one item is in the tab order at a time.

Use Arrow Left/Right to navigate, Space to toggle.

    4. Media Accessibility

    Image Alt Text

    Informative image

    alt="Description of image content"

    Decorative image

    alt="" (empty - screen reader skips it)

    Complex image

    alt="Chart showing..." + aria-describedby pointing to detailed description

    Color Contrast (WCAG Requirements)

    AAA Pass

    White on #0d6efd
    Ratio: 8.59:1

    AA Pass

    Dark on #ffc107
    Ratio: 1.81:1 (large text only)

    Fail

    Low contrast
    Ratio: 1.63:1

    WCAG AA requires 4.5:1 for normal text, 3:1 for large text (18px+ or 14px bold). Bootstrap's default color combinations are designed to meet AA standards.
