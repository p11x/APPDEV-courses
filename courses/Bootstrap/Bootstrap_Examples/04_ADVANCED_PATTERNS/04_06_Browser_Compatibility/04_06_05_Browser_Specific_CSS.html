Browser-Specific CSS

Common workarounds for Safari, Firefox, Chrome, and Edge rendering quirks. These patterns solve real cross-browser compatibility issues.

1. Safari (WebKit) Workarounds

The 100vh Mobile Safari Bug

Mobile Safari's 100vh includes the address bar, causing content to be hidden behind it. Solutions: 

/* Problem: 100vh includes Safari's address bar */
.full-height { height: 100vh; }

/* Solution 1: -webkit-fill-available */
.full-height {
  height: 100vh;
  height: -webkit-fill-available;
}

/* Solution 2: Use dvh (dynamic viewport height) */
.full-height { height: 100dvh; }

/* Solution 3: JavaScript-based fix */
.full-height { height: calc(var(--vh, 1vh) * 100); }
/* JS: document.documentElement.style.setProperty('--vh',
       window.innerHeight * 0.01 + 'px'); */
Flexbox Gap (Safari <14.1)

Safari versions before 14.1 don't support gap in flexbox. Use margin fallback:

Item 1
Item 2
Item 3
/* Fallback: negative margin on container */
.flex-container {
  display: flex;
  margin: -0.5rem; /* Half the desired gap */
}
.flex-container > * {
  margin: 0.5rem;
}

/* Modern: use @supports to override */
@supports (gap: 1rem) {
  .flex-container { gap: 1rem; margin: 0; }
  .flex-container > * { margin: 0; }
}
Safari Auto-Zoom on Input Focus

Mobile Safari auto-zooms when focusing an input with font-size < 16px. Fix by ensuring 16px minimum: 

@media screen and (max-width: 768px) {
  input, select, textarea {
    font-size: 16px !important;
  }
}

2. Firefox (Gecko) Workarounds

Button Inner Focus Ring

Firefox adds an inner dotted border on focused buttons:

.btn::-moz-focus-inner {
  border: 0;
  padding: 0;
}
Number Input Spinner

Remove spin buttons from number inputs:

/* Chrome */
input::-webkit-inner-spin-button { display:none; }
/* Firefox */
input { -moz-appearance: textfield; }
Custom Select Arrow

Replace native select arrow consistently:

select {
  -moz-appearance: none;
  -webkit-appearance: none;
  appearance: none;
  background-image: url("arrow.svg");
  background-position: right 0.75rem center;
  padding-right: 2.5rem;
}
Scrollbar Styling

Different approaches per browser engine:

/* WebKit (Chrome, Safari, Edge) */
::-webkit-scrollbar { width: 8px; }
::-webkit-scrollbar-thumb { background: #888; }

/* Firefox */
scrollbar-width: thin;
scrollbar-color: #888 #f1f1f1;

/* IE/Edge Legacy */
-ms-overflow-style: -ms-autohiding-scrollbar;

3. Chrome (Blink) Workarounds

Autofill Yellow Background

Chrome applies a yellow background to autofilled inputs. This cannot be removed with background-color. 

input:-webkit-autofill {
  -webkit-box-shadow: 0 0 0 1000px white inset;
  transition: background-color 5000s ease-in-out 0s;
}
CSS Scroll Snap

Scroll snapping works across modern browsers but has subtle differences in behavior.

4. Cross-Browser Consistency Patterns

Consistent Focus Styles

Override default browser focus rings:

:focus {
  outline: none;
  box-shadow: 0 0 0 3px rgba(13,110,253,0.5);
}
/* Or use :focus-visible for keyboard-only */
Consistent Placeholder

Firefox defaults to 0.4 opacity:

::placeholder { color: #999; opacity: 1; }
::-webkit-input-placeholder { color: #999; }
::-moz-placeholder { color: #999; opacity: 1; }
:-ms-input-placeholder { color: #999; }
Best Practice Summary
  • Use @supports for feature detection over browser hacks
  • Use Autoprefixer in your build process
  • Test on real devices and browsers, not just DevTools emulation
  • Apply progressive enhancement - base experience works everywhere
  • Avoid CSS hacks (star hack, \9 hack) - they're brittle and unmaintainable
