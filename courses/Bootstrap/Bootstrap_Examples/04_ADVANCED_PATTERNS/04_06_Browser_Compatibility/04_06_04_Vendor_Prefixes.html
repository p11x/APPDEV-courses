2. Autoprefixer & Build Tools

Autoprefixer automatically adds vendor prefixes based on Can I Use data and your browser support targets. You should never manually write most prefixes in production code.
What You Write
/* Clean, unprefixed CSS */
.container {
  display: grid;
  grid-template-columns: 
    repeat(auto-fit, minmax(200px, 1fr));
  gap: 1rem;
  user-select: none;
  backdrop-filter: blur(10px);
  appearance: none;
}
What Autoprefixer Outputs
/* Auto-prefixed CSS */
.container {
  display: -ms-grid;
  display: grid;
  -ms-grid-columns: 
    (1fr)[auto-fit];
  gap: 1rem;
  -webkit-user-select: none;
     -moz-user-select: none;
      -ms-user-select: none;
          user-select: none;
  -webkit-backdrop-filter: blur(10px);
          backdrop-filter: blur(10px);
  -webkit-appearance: none;
     -moz-appearance: none;
          appearance: none;
}
Browserslist Configuration

Autoprefixer uses browserslist to determine which browsers to support. Common configurations:

package.json
{
  "browserslist": [
    "> 0.5%",
    "last 2 versions",
    "not dead",
    "not ie 11"
  ]
}
.browserslistrc
# Browserslist config
# https://browsersl.ist/

> 0.5%
last 2 versions
not dead
not ie 11
Common Query Examples
QueryMeaning
> 0.5%Browsers with > 0.5% global usage
last 2 versionsLast 2 versions of each browser
not deadExcludes browsers without official support
not ie 11Excludes IE 11
>= 1%Browsers with >= 1% usage
defaultsDefault browserslist config

3. Prefix Reference & Status

Property -webkit- -moz- -ms- -o- Standard Notes
transform No No No No Yes All browsers support standard
transition No No No No Yes All browsers support standard
user-select Safari No No No Yes Safari still needs -webkit-
backdrop-filter Safari No No No Yes Safari needs -webkit-
appearance No No No No Yes Stabilized in most browsers
mask-image Yes No Yes No Partial Needs -webkit- in Chrome/Safari
hyphens Safari Firefox Edge No Partial Language-dependent dictionaries
fit-content Chrome Firefox No No Yes Use -webkit- and -moz- prefixes
linear-gradient No No No No Yes All browsers support standard
scrollbar-* Yes No No No Partial WebKit: ::-webkit-scrollbar; FF: scrollbar-width
Rule of thumb (2024): Most CSS properties no longer need vendor prefixes in modern browsers. Autoprefixer handles the few that do. Focus on writing clean, standard CSS and let build tools add prefixes based on your browserslist config.
