4. Bootstrap 5 Compatibility Details

What Bootstrap 5 Requires
  • CSS Flexbox (grid system)
  • CSS Custom Properties (theming)
  • ES6 JavaScript (let, const, arrow, classes)
  • Native Popper.js for tooltips/dropdowns
  • CSS transitions (animations)
  • CSS calc() (utilities)
What Bootstrap 5 Does NOT Support
  • Internet Explorer (any version)
  • Opera Mini
  • Browsers without ES6 support
  • Browsers without Flexbox support

5. Quick Polyfill Reference

Modern approach: Bootstrap 5 doesn't need polyfills for its target browsers. Only add polyfills if you need to support older browsers beyond Bootstrap's official support. 
<!-- Conditional polyfill loading pattern -->
<script>
// Load polyfills only when needed
(function() {
  var polyfills = [];
  
  if (!window.Promise) polyfills.push('es6-promise');
  if (!window.fetch) polyfills.push('whatwg-fetch');
  if (!CSS.supports || !CSS.supports('color','var(--x)')) polyfills.push('css-vars-ponyfill');
  
  if (polyfills.length > 0) {
    document.write('<script src="polyfills.bundle.js"><\/script>');
  }
})();
</script>

<!-- Then load your app -->
<script src="app.js"></script>

6. Vendor Prefix Quick Reference

Use Autoprefixer! Add it to your build process and never manually write vendor prefixes again. Configure via browserslist.
Prefix Browser Still Needed For
-webkit- Chrome, Safari, Edge backdrop-filter, user-select, mask
-moz- Firefox user-select (rare)
-ms- IE, Legacy Edge Only if supporting IE (not recommended)
-o- Legacy Opera Never needed (2024)

7. Feature Detection Quick Reference

CSS @supports
/* Check for CSS Grid */
@supports (display: grid) {
  .container { display: grid; }
}

/* Check for custom properties */
@supports (color: var(--x)) {
  .themed { color: var(--brand); }
}

/* Compound check */
@supports (display: grid) and 
          (color: var(--x)) {
  .enhanced { /* ... */ }
}

/* Negation */
@supports not (aspect-ratio: 1) {
  .ratio { padding-bottom: 56.25%; }
}
JavaScript Detection
// CSS feature detection
CSS.supports('display', 'grid')
CSS.supports('color', 'var(--x)')

// API detection
typeof fetch === 'function'
'IntersectionObserver' in window
'serviceWorker' in navigator

// Element support
'loading' in HTMLImageElement.prototype

// Media query detection
window.matchMedia('(prefers-reduced-motion: reduce)').matches

8. Cross-Browser Performance Tips

CSS Performance
  • Use class selectors (fast)
  • Avoid deep nesting
  • Use contain property
  • Use content-visibility: auto
  • Animate only transform/opacity
  • Use will-change sparingly
Loading Performance
  • Preconnect to CDN origins
  • Lazy-load below-fold images
  • Use loading="lazy"
  • Defer non-critical JS
  • Inline critical CSS
  • Use skeleton screens
Core Web Vitals
  • LCP < 2.5s
  • INP < 200ms
  • CLS < 0.1
  • Set image dimensions
  • Reserve space for ads/embeds
  • Use font-display: swap

9. Cross-Browser Accessibility

Essential A11y Checklist
  • All images have alt text
  • All form inputs have labels
  • Color contrast meets WCAG AA (4.5:1)
  • All interactive elements are keyboard accessible
  • Focus indicators are visible
  • Heading hierarchy is correct (h1-h6)
  • ARIA attributes used correctly
  • Skip navigation link present
  • Reduced motion is respected
  • Touch targets are at least 44x44px
Browser-Specific A11y Concerns
  • Safari: VoiceOver may not announce all live regions
  • Firefox: Different focus ring style
  • Chrome: Autofill can confuse screen readers
  • All: Test with actual assistive technology
