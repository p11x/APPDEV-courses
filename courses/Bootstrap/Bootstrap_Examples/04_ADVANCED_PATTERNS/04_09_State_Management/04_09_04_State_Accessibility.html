Managing component state with full accessibility: ARIA, focus, screen readers, and keyboard navigation.

1. ARIA Live Regions

Live regions announce dynamic content changes to screen readers. Use polite for updates, assertive for alerts.

Polite Announcement

Waits until the screen reader finishes current speech.

Assertive Announcement

Interrupts current speech immediately.

Move focus when state changes: open modal → focus first element; close → return focus; error → focus error field.

3. Accessible State Indicators (Not Just Color)

Color-only indicators fail colorblind users and screen readers. Use icons, text, and ARIA attributes together.

Bad: Color Only

Screen readers can't distinguish state. Colorblind users can't either.

Good: Icon + Text + ARIA

Icon + text for sighted users. aria-pressed for screen readers.
