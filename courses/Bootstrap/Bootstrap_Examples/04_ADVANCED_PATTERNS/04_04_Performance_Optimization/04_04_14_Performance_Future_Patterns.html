Future Performance Patterns

Emerging APIs and techniques for next-generation web performance.

View Transitions API New

The View Transitions API enables smooth animated transitions between page states without complex JavaScript animation libraries.

// Enable view transitions (CSS)
::view-transition-old(root) {
  animation: fade-out 0.3s ease-out;
}
::view-transition-new(root) {
  animation: fade-in 0.3s ease-in;
}

// Named transitions for specific elements
.hero-image {
  view-transition-name: hero;
}

// JS trigger for SPA navigation
document.startViewTransition(() => {
  updateDOM(); // DOM changes happen here
});

Speculation Rules API New

The Speculation Rules API tells the browser to prefetch or prerender pages the user is likely to navigate to.

<!-- Declarative speculation rules -->
<script type="speculationrules">
{
  "prefetch": [{
    "source": "list",
    "urls": [
      "/products",
      "/about",
      "/contact"
    ]
  }],
  "prerender": [{
    "source": "document",
    "where": {
      "href_matches": "/*"
    }
  }]
}
</script>
Prefetch vs Prerender
StrategySavesCost
Prefetch DNS + TCP + download Low (network only)
Prerender Everything (instant nav) High (full render)
Navigate instantly
Prerendered pages load in <50ms

103 Early Hints Experimental

103 Early Hints allows the server to send preload/preconnect hints before the final response is ready, letting the browser start fetching critical resources during server processing.

Server sends (during processing):
HTTP/1.1 103 Early Hints
Link: </css/bootstrap.min.css>;
     rel=preload; as=style
Link: </js/app.js>;
     rel=preload; as=script
Timeline comparison:
WITHOUT Early Hints:
[Server processing... 800ms]
[Browser receives HTML, parses, fetches CSS]
Total: 800ms + 200ms = 1000ms

WITH Early Hints:
[Server processing... 800ms]
[Browser starts fetching CSS at 50ms]
[CSS ready when HTML arrives]
Total: 800ms (parallel!) = 800ms

CSS Performance Features

Container Queries
Stable

Components adapt to their container width, not viewport. Enables truly reusable responsive components.

.card-container {
  container-type: inline-size;
}
@container (min-width: 400px) {
  .card { flex-direction: row; }
}
CSS Anchor Positioning
Experimental

Position elements relative to other elements without JavaScript. Replaces many Popper.js use cases.

.tooltip {
  position-anchor: --trigger;
  top: anchor(bottom);
  left: anchor(center);
}
Scroll-Driven Animations
Stable

CSS-only scroll-linked animations without JavaScript scroll listeners. Offloaded to compositor thread.

.progress {
  animation: grow linear;
  animation-timeline: scroll();
}
@keyframes grow {
  from { width: 0; }
  to { width: 100%; }
}
