Code Splitting

Load only what users need, when they need it.

Splitting Strategies

Route-Based Splitting

Each page/route loads its own JavaScript chunk. Users only download code for the page they visit.

  • /home home.chunk.js
  • /dashboard dashboard.chunk.js
  • /settings settings.chunk.js
Component-Based Splitting

Heavy components are loaded on demand. A modal with a rich editor loads only when opened.

// Lazy-load heavy modal component const RichEditor = React.lazy( () => import('./RichEditor') );
~40KB saved on initial load
Vendor Splitting

Separate third-party libraries into a vendor chunk. This chunk changes rarely, so it benefits from long-term caching.

// Webpack splitChunks config optimization: { splitChunks: { cacheGroups: { vendor: { test: /[\\/]node_modules[\\/]/, name: 'vendors', chunks: 'all' } } } }
Prefetch & Preload

Hint the browser to prefetch chunks for likely next navigations during idle time.

// Webpack magic comments import( './Dashboard' /* webpackPrefetch: true */ ); // Adds <link rel="prefetch"> to HTML

Benefits
  • ✓ Faster initial load: Only shared + home chunk loaded
  • ✓ Better caching: Vendor chunk cached separately
  • ✓ Parallel downloads: Multiple small chunks load faster
  • ✓ Progressive loading: Users see content sooner

Bundler Config Property Purpose

Webpack splitChunks.maxSize Maximum chunk size before further splitting
Vite build.rollupOptions.output.manualChunks Manually assign modules to specific chunks
Rollup output.manualChunks Function or object mapping module paths to chunk names
