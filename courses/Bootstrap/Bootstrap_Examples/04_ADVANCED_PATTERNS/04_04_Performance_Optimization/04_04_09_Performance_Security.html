Performance & Security

Security measures that enhance performance, not hinder it.

Subresource Integrity (SRI)

How SRI Works
  1. Generate a cryptographic hash of the resource
  2. Add the integrity attribute to the <link> or <script> tag
  3. Browser computes the hash of the downloaded file
  4. If hashes don't match, the resource is blocked
Performance note: SRI adds negligible overhead (hash computation) but prevents loading compromised resources that could inject malicious code and degrade performance.
Generating SRI Hashes
# Generate SHA-384 hash for SRI
cat bootstrap.min.css | openssl dgst \
  -sha384 -binary | openssl base64 \
  -A

# Output: sha384-QWTKZyjp...

# Use in HTML:
<link href="bootstrap.min.css"
  integrity="sha384-QWTKZyjp..."
  crossorigin="anonymous">

Content Security Policy (CSP)

Performance-Friendly CSP
# HTTP header (server-side)
Content-Security-Policy:
  default-src 'self';
  script-src 'self' cdn.jsdelivr.net;
  style-src 'self' cdn.jsdelivr.net
    'unsafe-inline';
  img-src 'self' picsum.photos;
  font-src 'self' cdn.jsdelivr.net;
  connect-src 'self' api.example.com;

A well-configured CSP prevents unauthorized script injection that could add performance overhead.

CSP Performance Impact
Inline scripts Blocked by default. Use nonces or hashes.
Eval Blocked. Forces better code practices.
Third-party resources Must be explicitly allowed.
Overhead Negligible browser-side cost.

Security Headers & Performance

HSTS - HTTP Strict Transport Security

Forces HTTPS, eliminating HTTP redirect overhead.

Strict-Transport-Security: max-age=31536000; includeSubDomains
Cache-Control - Secure Caching

Controls caching behavior to prevent stale sensitive data.

Cache-Control: public, max-age=31536000, immutable
Permissions-Policy

Disables unused browser features, reducing attack surface.

Permissions-Policy: camera=(), microphone=(), geolocation=()
X-Content-Type-Options

Prevents MIME sniffing, avoids unnecessary re-downloads.

X-Content-Type-Options: nosniff

HTTPS & Protocol Performance

Protocol Multiplexing Header Compression Server Push Connection Setup
HTTP/1.1 No (6 conn limit) No No TCP + TLS 1.2
HTTP/2 Yes HPACK Yes (deprecated) TCP + TLS 1.2/1.3
HTTP/3 (QUIC) Yes QPACK Proposed 0-RTT (QUIC)
Key insight: HTTP/3 with QUIC reduces connection setup from 2-3 round trips (TCP+TLS) to 0-1 round trips, significantly improving mobile performance.
