Performance Maintenance

Prevent regressions and maintain fast load times over time.

Regression Prevention

Common Regressions
  • New dependency added
    A library adds 200KB to the bundle unnoticed.
  • Image optimization removed
    Content team uploads 5MB uncompressed images.
  • CSS purge config broken
    Build change disables PurgeCSS silently.
  • Third-party script bloat
    Marketing adds 5 tracking scripts.
  • Cache headers removed
    Server config change loses long cache TTL.
Prevention Strategies
// .github/workflows/perf-guard.yml
name: Performance Guard
on: [pull_request]
jobs:
  perf-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci && npm run build
      - uses: treosh/lighthouse-ci-action
        with:
          budgetPath: ./budget.json
          uploadArtifacts: true
      - uses: andresz1/size-limit-action
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}

Monthly Performance Audit Checklist

Frontend Audit
✓ Run Lighthouse on all key pages
✓ Review bundle size changes (last 30 days)
✓ Check for unused Bootstrap components
✓ Audit third-party scripts and their impact
✓ Verify lazy loading is working correctly
✓ Check image optimization compliance
Infrastructure Audit
✓ Verify CDN cache hit rates
✓ Check Gzip/Brotli compression enabled
✓ Review HTTP/2 or HTTP/3 status
✓ Verify cache headers on static assets
✓ Check TLS certificate and HSTS
✓ Review server response times (TTFB)

Performance Team Workflow

  • Review performance budgets before adding new features
  • Estimate bundle impact of new dependencies
  • Check if existing utilities can replace new library needs

  • Use npm run build -- --analyze to check bundle composition
  • Test on throttled network (Chrome DevTools: Slow 3G)
  • Verify images have proper width/height and lazy loading
  • Use Bootstrap utilities instead of custom CSS when possible

  • CI automatically runs Lighthouse and bundle size checks
  • Reviewer checks for unnecessary new dependencies
  • Verify no inline styles that could use Bootstrap utilities
  • Confirm images have lazy loading and proper sizing

  • Monitor RUM dashboards for 24-48 hours after deploy
  • Set alerts for Web Vitals regression (>10% degradation)
  • Document any performance changes in release notes
