Bundle Optimization

Reduce payload size and improve load performance with proven optimization strategies.

Optimization Strategies

CSS Purging

Remove unused CSS rules from Bootstrap using tools like PurgeCSS. Can reduce CSS payload by 60-90%.

npx purgecss --css bootstrap.min.css \
  --content "**/*.html" \
  --output dist/
Selective Imports

Import only the Bootstrap components you need via Sass. Exclude unused modules entirely.

// Import only needed modules
@import "bootstrap/scss/functions";
@import "bootstrap/scss/variables";
@import "bootstrap/scss/grid";
@import "bootstrap/scss/buttons";
Minification

Minify CSS and JS to remove whitespace, comments, and shorten identifiers.

// Webpack minification config
optimization: {
  minimize: true,
  minimizer: [
    new CssMinimizerPlugin(),
    new TerserPlugin()
  ]
}
Compression

Enable Gzip or Brotli on your server. Reduces transfer size by 70-85% for text-based assets.

# Nginx gzip config
gzip on;
gzip_types text/css
  application/javascript;
gzip_min_length 256;
CDN Caching

Use CDN-hosted Bootstrap with long cache headers. Users benefit from shared browser cache across sites.

<!-- CDN with SRI hash -->
<link rel="stylesheet"
  href="https://cdn.jsdelivr.net/npm/
  bootstrap@5.3.3/dist/css/
  bootstrap.min.css"
  integrity="sha384-..."
  crossorigin="anonymous">
Tree Shaking

Use ES module imports so bundlers can eliminate dead code from the final output.

// ES module tree-shakeable import
import { Tooltip } from 'bootstrap';
// Only Tooltip code is bundled
