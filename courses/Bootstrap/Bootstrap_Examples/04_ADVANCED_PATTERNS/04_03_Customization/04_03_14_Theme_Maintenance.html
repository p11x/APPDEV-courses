Design Token System

Design tokens are the atomic values that define your design system. They bridge design tools and code.

Color Tokens
$primary #4f46e5
$success #059669
--bs-primary-rgb 79, 70, 229
Typography Tokens
$font-family-base Inter
$font-size-base 1rem
$headings-font-weight 700
Spacing Tokens
$spacer 1rem
$border-radius 0.5rem
$card-border-radius 0.75rem
Tokens JSON (for design tool sync)
{ "color": { "primary": { "value": "#4f46e5", "type": "color" }, "success": { "value": "#059669", "type": "color" }, "danger": { "value": "#dc2626", "type": "color" } }, "typography": { "fontFamily": { "value": "Inter, sans-serif" }, "fontSize": { "base": "16px", "lg": "20px" } }, "spacing": { "sm": "0.5rem", "md": "1rem", "lg": "1.5rem", "xl": "2rem" } }

Bootstrap Version Upgrade Strategy

5.2 → 5.3 Upgrade
  • Added color mode system (data-bs-theme)
  • New CSS variables for all components
  • Deprecated .badge.bg-* for .text-bg-*
5.1 → 5.2 Upgrade
  • New CSS grid option
  • Updated offcanvas backdrop
  • New .placeholder component
4.x → 5.0 Migration (Major)
  • Dropped jQuery dependency
  • Renamed data attributes: data-toggle → data-bs-toggle
  • New utility API system
  • Dropped IE support

Team Collaboration Workflow

Theme Update Process
  1. Design Token Update

    Designer updates tokens in Figma/Sketch. Tokens are exported to tokens.json.

  2. Token Sync

    Automated tool (Style Dictionary) converts JSON to SCSS variables and CSS custom properties.

  3. SCSS Compilation

    Updated tokens flow into Bootstrap variable overrides and component styles.

  4. Visual Testing

    BackstopJS runs visual regression tests against baseline screenshots.

  5. Review & Deploy

    PR review includes before/after screenshots. Merge triggers deployment.
