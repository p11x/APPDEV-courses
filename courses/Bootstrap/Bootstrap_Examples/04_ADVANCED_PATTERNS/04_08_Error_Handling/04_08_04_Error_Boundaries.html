Error Boundaries

Isolate failures to prevent one broken component from crashing the entire page.

Without Error Boundaries

One component crashes = entire page breaks.

A single JavaScript error in any widget can cause a blank white screen for the user, losing all page state.

With Error Boundaries

One component fails = graceful fallback shown.

The error is contained to the affected section. Other parts of the page continue to function normally.

Error Boundary Implementation
// Error Boundary wrapper function
function withErrorBoundary(element, fallbackFn) {
  return async function() {
    try {
      // Attempt to render the component
      await renderComponent(element);
    } catch (error) {
      // Log the error for monitoring
      console.error(`Error in ${element.id}:`, error);

      // Show fallback UI
      element.classList.add('has-error');
      const fallback = element.querySelector('.error-fallback');
      if (fallback) {
        fallback.querySelector('.error-message').textContent = error.message;
      }

      // Report to error tracking service
      reportError({
        boundary: element.id,
        error: error.message,
        stack: error.stack,
        timestamp: new Date().toISOString()
      });
    }
  };
}

// Reset function to retry failed component
function resetError(boundaryId) {
  const element = document.getElementById(boundaryId);
  element.classList.remove('has-error');
  // Re-attempt rendering
  renderComponent(element);
}

Page-Level Compartmentalization

Header Zone - Critical: Never fails
Main Content Zone - Isolated boundary
Fallback: Skeleton + retry button
Sidebar Zone - Optional
Fallback: Hidden if broken
Footer Zone - Static: Rarely fails

Each zone has its own error boundary. Critical zones (header, footer) use static HTML that doesn't depend on JavaScript.
