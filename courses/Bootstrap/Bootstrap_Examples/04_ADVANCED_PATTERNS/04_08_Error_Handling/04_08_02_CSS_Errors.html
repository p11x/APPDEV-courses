CSS Fallback Strategies
Custom Property Fallback
/* Fallback chain for CSS variables */
.element {
  color: var(--primary, var(--brand, #0d6efd));
}

/* If --primary is undefined, use --brand.
   If --brand is also undefined, use #0d6efd */
Property Value Fallback
/* Declare fallback BEFORE modern value */
.element {
  display: block;        /* Fallback */
  display: flex;         /* Modern */
  display: grid;         /* Most modern */
}

/* Browsers ignore properties they don't
   understand, using the last one they do */
Font Stack Fallback
/* System font stack with fallbacks */
body {
  font-family:
    -apple-system,
    BlinkMacSystemFont,
    "Segoe UI",
    Roboto,
    "Helvetica Neue",
    Arial,
    sans-serif;
}
Image Fallback
/* Background image with color fallback */
.hero {
  background-color: #1a1a2e; /* Shows if image fails */
  background-image: url('hero.jpg');
  background-size: cover;
}

/* HTML: img with alt text as fallback */
<img src="photo.jpg" alt="Description">
@supports Feature Detection
/* Progressive enhancement with @supports */
.container {
  /* Fallback: flexbox layout */
  display: flex;
  flex-wrap: wrap;
}

@supports (display: grid) {
  .container {
    /* Enhanced: CSS Grid layout */
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  }
}

@supports (backdrop-filter: blur(10px)) {
  .modal-backdrop {
    backdrop-filter: blur(10px);
    background: rgba(255,255,255,0.5);
  }
}

Common CSS Errors & Prevention
Error Problem Prevention
z-index wars Elements layering incorrectly Use a z-index scale (10, 20, 30...)
Overflow hidden clipping Content cut off unexpectedly Use overflow: auto instead
Specificity conflicts Styles not applying Avoid !important, use BEM naming
Missing font fallbacks FOUT/FOIT text rendering Always include generic font family
Fixed dimensions Content overflow on small screens Use max-width and clamp()
