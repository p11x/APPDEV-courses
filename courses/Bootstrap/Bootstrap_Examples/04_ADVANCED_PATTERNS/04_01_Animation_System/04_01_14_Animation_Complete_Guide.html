7. Troubleshooting Guide

Cause: Element starts visible, animation already "completed".
Fix: Add opacity: 0 to the element and animation-fill-mode: forwards. Or use @starting-style for entry animations.

Cause: CSS animations don't replay when class is re-added (browser considers it unchanged).
Fix: Remove the class, force a reflow (void el.offsetWidth), then re-add the class.

Cause: Animating layout-triggering properties (top, left, width, height).
Fix: Use transform: translateX() instead of left. Use transform: scale() instead of width/height. Add will-change for elements that animate frequently.

Cause: transitionend fires once per property. If you transition 3 properties, it fires 3 times.
Fix: Filter by e.propertyName or use e.target === myElement.

Cause: Safari requires -webkit- prefix for some animation properties.
Fix: Use Autoprefixer in your build pipeline, or add @-webkit-keyframes manually.
