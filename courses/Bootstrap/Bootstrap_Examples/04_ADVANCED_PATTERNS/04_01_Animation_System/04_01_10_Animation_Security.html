Animation Security

Preventing XSS, ensuring CSP compliance, and safely handling dynamic animation values.

1. XSS via Style Injection

Vulnerability: Injecting user input directly into style attributes or <style> blocks can execute malicious code via expression(), url(javascript:...), or @import.
❌ Vulnerable Code
// NEVER DO THIS — user input in style el.style.cssText = userInput; el.setAttribute('style', userInput); styleTag.innerHTML = userInput; // attacker input: // "background:url(javascript:alert(1))" // "width:expression(alert(1))"
✅ Secure Code
// SAFE: Use CSS custom properties el.style.setProperty('--user-color', sanitize(color)); // SAFE: Use classList for predefined styles el.classList.add('animate-slide-in'); // SAFE: Whitelist allowed values const allowed = ['red','blue','green']; if (allowed.includes(userInput)) { el.style.color = userInput; }

2. Content Security Policy (CSP) Compliance

CSP can block inline styles and scripts. Use nonce-based or hash-based CSP to allow specific animations while blocking injection attacks.
CSP Header (Restrictive)
Content-Security-Policy: default-src 'self'; style-src 'self' https://cdn.jsdelivr.net 'nonce-abc123'; script-src 'self' https://cdn.jsdelivr.net 'nonce-abc123';
Using Nonces for Inline Styles
<!-- Server generates unique nonce --> <style nonce="abc123"> .animated { animation: fadeIn 0.5s; } </style> <!-- Each request gets a fresh nonce --> <script nonce="abc123"> el.style.animation = 'fadeIn 0.5s'; </script>

4. Input Validation for Animation Parameters

Validators Needed
  • Durations: Must be positive number with 's' or 'ms' unit
  • Colors: Whitelist hex/rgb or use predefined classes
  • Easing: Whitelist known timing functions
  • Transforms: Parse and validate each function
  • CSS variables: Only set custom properties, not standard props
Validation Functions
function sanitizeDuration(val) { const num = parseFloat(val); if (isNaN(num) || num < 0 || num > 10) { return '0.3s'; // safe default } return num + 's'; } function sanitizeColor(val) { const allowed = [ '#0d6efd','#198754','#dc3545', '#ffc107','#0dcaf0' ]; return allowed.includes(val) ? val : '#0d6efd'; } function sanitizeEasing(val) { const allowed = [ 'ease','linear','ease-in', 'ease-out','ease-in-out' ]; return allowed.includes(val) ? val : 'ease'; }

5. CSS Custom Properties — The Safe Way

CSS custom properties (variables) are the safest way to inject dynamic values into CSS. They cannot execute code and are inherently safe from injection attacks.
