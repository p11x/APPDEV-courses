3. Browser Rendering Pipeline

Stage Triggers Cost Example Properties
Layout (Reflow) Geometry changes VERY HIGH width, height, top, left, margin, padding
Paint Visual changes MEDIUM color, background, box-shadow, border-radius
Composite Layer transforms LOW (GPU) transform, opacity, filter
Rule of thumb: Only transform and opacity can be animated without triggering layout or paint. All other properties will trigger at least paint, and many trigger layout.

6. Layout Thrashing Prevention

BAD: Read-Write Loop
// Forces layout on EACH iteration!
items.forEach(el => {
    const height = el.offsetHeight; // READ
    el.style.height = height + 10 + 'px'; // WRITE
});
GOOD: Batch Reads, Then Writes
// Batch all reads first
const heights = items.map(el => el.offsetHeight);
// Then batch all writes
items.forEach((el, i) => {
    el.style.height = heights[i] + 10 + 'px';
});

7. CSS contain & content-visibility

contain: layout style

Tells the browser that changes inside this element don't affect outside layout. Enables rendering optimizations.

This container has contain: layout style. Internal changes won't trigger external reflows.

content-visibility: auto

Skips rendering of off-screen content entirely. Dramatically improves page load performance for long pages.

This element uses content-visibility: auto. If off-screen, it won't be rendered at all.
