Animation Maintenance

Naming conventions, design tokens, modular architecture, and team practices for maintainable animations.

1. Design Tokens (CSS Custom Properties)

All animation values are defined as CSS custom properties in :root. Change once, update everywhere.

TokenValueUsage
--anim-duration-instant0.05sMicro-interactions, focus rings
--anim-duration-fast0.15sHover feedback, button states
--anim-duration-normal0.3sStandard transitions
--anim-duration-slow0.5sPage-level animations
--anim-duration-slower0.8sDramatic entrances
--anim-ease-defaultcubic-bezier(0.25,0.1,0.25,1)General purpose
--anim-ease-bouncecubic-bezier(0.68,-0.55,0.27,1.55)Playful elements
--anim-delay-stagger0.05sStaggered list increments

2. Naming Convention in Action

Pattern: .anim-{type}-{direction}-{variant}

anim-fade-in-up
Delay: 1
anim-slide-in-left
Delay: 2
anim-slide-in-right
Delay: 3
anim-scale-in
Delay: 4

3. Composable Utility Classes

Combine base animation class with delay, duration, and easing utilities.

Example Combinations
.anim-fade-in-up .anim-delay-1 .anim-dur-fast
.anim-fade-in-up .anim-delay-3 .anim-dur-slow .anim-ease-bounce
.anim-scale-in .anim-delay-5 .anim-dur-slower

4. Deprecation Strategy

Deprecation Steps
  1. Add @deprecated comment to old class
  2. Add new class alongside old one
  3. Document migration in changelog
  4. Run codemod to find all usages
  5. Remove old class in next major version
CSS Deprecation Comment
/** * @deprecated 2.0.0 * Use .anim-fade-in instead. * Will be removed in 3.0.0 */ .fade-in-old { animation: anim-fade-in var(--anim-duration-normal) var(--anim-ease-default) forwards; }

5. Recommended File Structure

styles/ ├── _tokens.css /* Design tokens: durations, easings, delays */ ├── _keyframes.css /* All @keyframes definitions */ ├── _animations.css /* Animation utility classes */ ├── _transitions.css /* Transition utility classes */ ├── _reduced-motion.css /* prefers-reduced-motion overrides */ └── main.css /* Imports all partials */ /* _animations.css */ .anim-fade-in { ... } .anim-fade-in-up { ... } .anim-slide-in-left { ... } /* main.css */ @import '_tokens.css'; @import '_keyframes.css'; @import '_animations.css'; @import '_transitions.css'; @import '_reduced-motion.css';

7. Team Collaboration Practices

Code Review Checklist
  • Uses design tokens, not hardcoded values
  • Follows naming convention
  • Includes prefers-reduced-motion
  • Only animates transform/opacity
  • No transition: all
Documentation
  • JSDoc comments on all utilities
  • Storybook stories for each animation
  • Visual regression test baselines
  • Changelog entries
Tools
  • Stylelint for CSS linting
  • Autoprefixer for vendor prefixes
  • PostCSS for CSS transforms
  • Storybook for visual testing
