Accessibility & Security

Understanding how accessibility features intersect with security concerns.

XSS Risks Through ARIA

Vulnerable: Unsanitized aria-label

User input in ARIA attributes can be exploited for XSS:

// DANGEROUS: user input in aria-label
const userInput = '<img src=x onerror=alert(1)>';
button.setAttribute('aria-label', userInput);

// Some screen readers may execute embedded
// scripts when parsing ARIA content
Safe: Sanitized ARIA Values

Always sanitize user input before using in ARIA attributes:

// SAFE: sanitize user input
function sanitize(str) {
  const div = document.createElement('div');
  div.textContent = str;
  return div.textContent;
}

const userInput = getUserInput();
button.setAttribute(
  'aria-label',
  sanitize(userInput)
);

// Or use textContent instead of innerHTML
labelElement.textContent = userInput;

Safe ARIA Implementation

Pattern Risk Mitigation
aria-label with user content XSS if unsanitized Sanitize before setting attribute
aria-labelledby references ID spoofing Ensure referenced elements are trusted
aria-live regions Content injection announced to SR Sanitize live region content
role attribute Role hijacking (button to link) Validate role values against allowed list
Focus management Focus theft/phishing Only manage focus within owned components

Focus-Related Security

Preventing Focus-Based Attacks
Focus Theft Prevention
  • Never steal focus from user-initiated actions without warning
  • Trap focus only in modal dialogs (per WAI-ARIA)
  • Restore focus to trigger element on close
  • Don't auto-focus form fields on pages with sensitive data
Tabnabbing Prevention
// Links opening new tabs need rel="noopener"
<a href="https://external.com"
    target="_blank"
    rel="noopener noreferrer">
  External Link
</a>

// Prevents target page from accessing
// window.opener (phishing attack vector)

Security Best Practices for A11y

Content Security Policy

CSP prevents XSS in ARIA content by restricting script sources.

Input Sanitization

Always sanitize user input before use in ARIA attributes or live regions.

Output Encoding

Encode special characters when rendering user content in accessible text.

HTTPS Only

Screen readers fetch content over network; ensure it's encrypted.

SRI on CDN Resources

Verify integrity of Bootstrap and assistive technology libraries.

Regular Audits

Include ARIA attribute validation in security penetration testing.
