Focus Management

Control keyboard focus for accessible dynamic interfaces.

Focus Trapping

When a modal or drawer opens, focus must be trapped inside it. Tab should cycle through the modal's focusable elements without escaping to the page behind.

Focus Trap Rules
  1. Move focus to the first focusable element when opening
  2. Tab cycles through focusable elements within the trap
  3. Shift+Tab from first element wraps to last
  4. Tab from last element wraps to first
  5. Escape key closes and restores focus
  6. Background elements get aria-hidden="true" and inert
Manual Focus Trap Code
function trapFocus(container) {
  const focusable = container.querySelectorAll(
    'button, [href], input, select, '
    + 'textarea, [tabindex]:not(
      [tabindex="-1"])'
  );
  const first = focusable[0];
  const last = focusable[focusable.length - 1];

  container.addEventListener('keydown', e => {
    if (e.key !== 'Tab') return;
    if (e.shiftKey &&
      document.activeElement === first) {
      e.preventDefault();
      last.focus();
    } else if (!e.shiftKey &&
      document.activeElement === last) {
      e.preventDefault();
      first.focus();
    }
  });
}

Focus Restoration

When a modal/drawer closes, return focus to the element that triggered it. This maintains the user's place in the page.

// Store reference to triggering element
let previouslyFocused = null;

function openModal() {
  // Save current focus
  previouslyFocused = document.activeElement;
  // Show modal and focus first element
  const modal = document.getElementById('myModal');
  modal.classList.add('show');
  modal.querySelector('button, input').focus();
}

function closeModal() {
  // Hide modal
  const modal = document.getElementById('myModal');
  modal.classList.remove('show');
  // Restore focus to trigger
  if (previouslyFocused) previouslyFocused.focus();
}
Note: Bootstrap's Modal component handles focus trapping and restoration automatically. This code shows the underlying pattern for custom components.

Dynamic Content Focus

Move Focus on Update

When content changes dynamically (e.g., form validation), move focus to the error message or new content.

Live Region Announcements

Use aria-live regions when content changes but focus shouldn't move.

// Announce without moving focus
function announce(message) {
  const region = document.getElementById(
    'announcement'
  );
  region.textContent = '';
  // Small delay ensures screen readers
  // detect the change
  setTimeout(() => {
    region.textContent = message;
  }, 100);
}

<div id="announcement"
     aria-live="polite"
     aria-atomic="true"
     class="visually-hidden">
</div>
