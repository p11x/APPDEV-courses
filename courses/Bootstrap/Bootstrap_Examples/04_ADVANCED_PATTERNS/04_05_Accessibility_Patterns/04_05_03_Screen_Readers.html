Screen Reader Optimization

Provide meaningful context and announcements for assistive technology users.

Visually Hidden Content

.visually-hidden

Content hidden visually but read by screen readers. Use for context that sighted users get from visual cues.

<!-- Icon-only button needs text for SR -->
<button class="btn btn-primary">
  <svg aria-hidden="true">...</svg>
  <span class="visually-hidden">Close</span>
</button>

<!-- Table needs caption for SR -->
<table>
  <caption class="visually-hidden">
    Monthly revenue data
  </caption>
  ...
</table>
Hover/inspect to see icon with hidden label
.visually-hidden-focusable

Hidden visually but revealed when focused. Used for "Skip to content" links.

<!-- Skip link at top of page -->
<a href="#main-content"
    class="visually-hidden-focusable">
  Skip to main content
</a>

/* CSS: hidden until focused */
.visually-hidden-focusable:not(:focus) {
  position: absolute;
  width: 1px;
  height: 1px;
  overflow: hidden;
  clip: rect(0,0,0,0);
}

Semantic HTML Benefits

Element Screen Reader Announces Non-Semantic Equivalent Problem
<button> "Button, [label]" <div onclick> is silent
<nav> "Navigation landmark" <div class="nav"> is just a div
<h1>-<h6> "Heading level [n], [text]" <div class="h1"> is just styled text
<ul>/<ol> "List, [n] items" <div> items provide no count
<table> "Table, [n] rows, [m] columns" <div> grid provides no structure
<img alt> "Image, [alt text]" <div style="background"> is silent
<form> "Form landmark" Unlabeled forms are confusing

Screen Reader Testing

NVDA (Windows)

Free, open-source. Most popular for web testing.

  • Insert+Space - Toggle focus/browse
  • H - Next heading
  • D - Next landmark
  • T - Next table
  • F - Next form field
VoiceOver (macOS)

Built into macOS. Great for Safari testing.

  • Cmd+F5 - Toggle VoiceOver
  • Ctrl+Opt+U - Rotor
  • Ctrl+Opt+H - Next heading
  • Ctrl+Opt+Cmd+H - Prev heading
  • Ctrl+Opt+Arrow - Navigate
TalkBack (Android)

Built into Android. Essential for mobile a11y.

  • Swipe right - Next element
  • Swipe left - Previous element
  • Double tap - Activate
  • Swipe down then right - Next heading
  • Two-finger swipe down - Scroll
