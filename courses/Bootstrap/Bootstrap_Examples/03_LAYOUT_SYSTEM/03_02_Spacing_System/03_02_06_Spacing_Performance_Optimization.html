Spacing Performance Optimization

Techniques to minimize CSS output and improve rendering performance when using Bootstrap spacing utilities.

1. Purge Unused Spacing Utilities

Bootstrap generates spacing utilities for all breakpoints and sides. Purge unused ones.

Without Purging
~2000+ spacing utility classes
  • ✗ All breakpoints generated
  • ✗ All sides (t/b/l/r/x/y) for every size
  • ✗ Negative margins included
  • ✗ CSS file bloats significantly
With Purging (PurgeCSS)
~150 used classes only
  • ✓ Only used classes included
  • ✓ Unused breakpoints removed
  • ✓ CSS file shrinks 60-80%
  • ✓ Faster page loads
PurgeCSS Configuration
// postcss.config.js
module.exports = {
  plugins: [
    require('@fullhuman/postcss-purgecss')({
      content: ['./**/*.html', './src/**/*.js'],
      // Safelist classes that might be added dynamically
      safelist: {
        standard: [/^m[trblxy]?-/, /^p[trblxy]?-/, /^gap-/],
        greedy: [/^modal/, /^tooltip/, /^dropdown/]
      }
    })
  ]
}

2. Use CSS Custom Properties

CSS variables enable runtime spacing changes without recompiling.

Define Spacing Tokens
/* Define your spacing tokens once */
:root {
  --space-xs: 0.25rem;   /* 4px */
  --space-sm: 0.5rem;    /* 8px */
  --space-md: 1rem;      /* 16px */
  --space-lg: 1.5rem;    /* 24px */
  --space-xl: 3rem;      /* 48px */
}

/* Custom spacing classes use the variables */
.section-gap { margin-bottom: var(--space-xl); }
.card-pad { padding: var(--space-md); }
.inline-gap { gap: var(--space-sm); }

/* Override per theme or mode */
[data-theme="compact"] {
  --space-md: 0.75rem;
  --space-lg: 1rem;
  --space-xl: 2rem;
}

3. Avoid Excessive Nested Margins

Poor: Margin Stacking
<!-- Each child has its own margin -->
<div class="mb-3">
  <div class="mb-2">
    <div class="mb-1">
      <!-- Accumulated margin: 24px -->
    </div>
  </div>
</div>

Margins accumulate through nesting, creating unpredictable spacing and potential layout shifts.

Better: Gap on Container
<!-- Gap on parent controls all -->
<div class="d-flex flex-column gap-3">
  <div>Item 1</div>
  <div>Item 2</div>
  <div>Item 3</div>
</div>

Gap utilities provide consistent spacing without margin collapse issues or accumulation.

4. Minimize Layout Reflows

Inconsistent spacing causes layout recalculations during rendering.

Performance Tip: Avoid changing margins/padding dynamically on visible elements. Use transform and opacity for animations instead of spacing properties.
Spacing Animation: Do vs Don't
/* DON'T: Animating margin causes reflows */
.element {
  transition: margin 0.3s; /* Triggers layout recalculation */
}
.element:hover {
  margin-top: 20px; /* Expensive! */
}

/* DO: Use transform instead */
.element {
  transition: transform 0.3s; /* GPU accelerated, no reflow */
}
.element:hover {
  transform: translateY(-20px); /* Cheap! */
}

/* DO: Use opacity for fade effects */
.element {
  transition: opacity 0.3s;
}

5. Responsive Spacing Efficiency

All Breakpoints
<div class="
  m-1 m-sm-2 m-md-3
  m-lg-4 m-xl-5 m-xxl-5
  p-1 p-sm-2 p-md-3
  p-lg-4 p-xl-5 p-xxl-5
">
12 classes per element

Generates many CSS rules for breakpoints you may not need.

Strategic Breakpoints
<div class="
  p-3 p-lg-5
  my-4 my-lg-5
">
4 classes per element

Only change spacing where the layout actually shifts. Fewer classes, same result.

7. Custom Build: Include Only What You Need

Selective Sass Import
// Only import spacing utilities you actually use
// custom-bootstrap.scss

// Import only the spacing utilities
@import "bootstrap/scss/functions";
@import "bootstrap/scss/variables";
@import "bootstrap/scss/mixins";

// Import only margin utilities (skip padding if unused)
@import "bootstrap/scss/utilities/api";
@import "bootstrap/scss/utilities/_spacing";

// Or override the spacing utilities map to include fewer values
$utilities: (
  "margin": (
    property: margin,
    class: m,
    values: map-merge($spacers, (auto: auto))
  ),
  "padding": (
    property: padding,
    class: p,
    values: $spacers
  )
);

8. Critical CSS: Inline Spacing for Above-the-Fold

Inline Critical Spacing CSS
<head>
  <!-- Inline critical spacing CSS for immediate render -->
  <style>
    /* Only above-the-fold spacing */
    .container { padding-left: 15px; padding-right: 15px; }
    .hero { padding: 3rem 0; margin-bottom: 2rem; }
    .nav { padding: 1rem; }
  </style>

  <!-- Defer full Bootstrap CSS -->
  <link rel="preload" href="bootstrap.min.css"
        as="style" onload="this.onload=null;this.rel='stylesheet'">
</head>

9. CSS Size Impact by Approach

Full Bootstrap CSS: ~220KB (all spacing utilities)
Custom Build: ~100KB (selected utilities)
Purged Build: ~45KB (only used classes)
Tailored: ~22KB (critical + lazy load)
Best Practice: Use Bootstrap's CDN for development, then switch to a custom/purged build for production.
