Spacing & Accessibility

Proper spacing improves readability, reduces cognitive load, and ensures interactive elements are usable by everyone, including users with motor impairments.

WCAG 2.1 Guideline 1.4.12: Text spacing should be adjustable without loss of content. Line height >= 1.5x font size, paragraph spacing >= 2x font size, letter spacing >= 0.12x font size.

1. Touch Target Spacing (WCAG 2.5.5)

Interactive elements need minimum 44x44px touch targets with adequate spacing.

Poor: Too Small, Too Close

Users with tremors or limited dexterity cannot accurately tap these buttons.

Good: Large Targets, Gap Spacing

gap-3 ensures 16px between buttons. min-width/height ensures 44px touch targets.

2. Text Spacing for Readability

Poor: Cramped Text

This paragraph has line-height of 1, tiny letter spacing, and almost no paragraph spacing. It is extremely difficult to read, especially for users with dyslexia or low vision. The lines run together making it hard to track where you are.

Another paragraph with the same tight spacing. Notice how exhausting this is to read.

Good: Comfortable Text

This paragraph uses Bootstrap's lh-lg class for generous line height and mb-3 for comfortable paragraph spacing. It is much easier to read, especially for users with dyslexia or low vision.

The clear separation between lines and paragraphs helps everyone read more comfortably and efficiently.

Bootstrap Text Spacing Classes
  • .lh-1 - line-height: 1 (tight, for display text only)
  • .lh-sm - line-height: 1.25
  • .lh-base - line-height: 1.5 (WCAG minimum for body text)
  • .lh-lg - line-height: 2 (extra comfortable)

5. Dyslexia-Friendly Spacing

Article Title

This content uses generous line height (lh-lg), comfortable paragraph spacing (mb-4), and slightly increased word spacing. These adjustments significantly help users with dyslexia track text across lines.

Short paragraphs with clear spacing reduce visual stress. Each paragraph is kept to a comfortable length with ample whitespace between sections.

The max-width constraint of 65 characters per line also prevents eye fatigue from tracking extremely long lines of text.

Tip: Let users control their own spacing. Never override user-set spacing preferences with !important.

7. Responsive Spacing for Accessibility

Spacing should increase on larger screens where users are further away.

<!-- Mobile: smaller spacing (close viewing distance) -->
<!-- Desktop: larger spacing (further viewing distance) -->
<button class="btn btn-primary p-2 p-md-3 p-lg-4">
  Responsive Button
</button>

<!-- Text spacing that adapts -->
<p class="mb-3 mb-lg-4 lh-base lh-lg-lg">
  Body text with responsive paragraph and line spacing
</p>

8. Spacing Transitions with Reduced Motion

Respect the prefers-reduced-motion media query for users sensitive to motion.

/* Spacing animation that respects reduced motion */
.expandable {
  padding: 1rem;
  transition: padding 0.3s ease;
}

.expandable:hover {
  padding: 2rem;
}

/* Disable spacing transitions for users who prefer reduced motion */
@media (prefers-reduced-motion: reduce) {
  .expandable {
    transition: none;
  }
}

9. Accessibility Spacing Checklist

Check Requirement Bootstrap Class
☑ Touch targets >= 44x44px btn-lg, py-3, px-4
☑ Gap between interactive elements >= 8px gap-2, gap-3
☑ Line height >= 1.5 for body text lh-base, lh-lg
☑ Paragraph spacing >= 2x font size mb-3, mb-4
☑ Focus indicators visible (space around element) mb-3 on sibling elements
☑ Content grouped with spacing (chunking) mb-4 between groups, mb-2 within
