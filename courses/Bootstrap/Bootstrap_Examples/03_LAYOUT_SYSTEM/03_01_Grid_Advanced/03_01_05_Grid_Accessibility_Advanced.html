1. DOM Order vs Visual Order

Critical: CSS Grid can reorder items visually with grid-column and grid-row, but screen readers and keyboard navigation always follow DOM order. Visual reordering creates a disconnect for assistive technology users.
Bad: Visual reordering (sidebar DOM-first, displayed right):
Sidebar (DOM 1st, visual RIGHT)
Main Content (DOM 2nd, visual LEFT)

Screen readers read sidebar first, but it appears on the right. This is confusing.

Correct: DOM order matches visual order:
Main Content (DOM 1st, visual LEFT)

DOM and visual order match. Screen reader users experience the same flow.

Sidebar (DOM 2nd, visual RIGHT)

2. Keyboard-Accessible Grid Cards

Grid cards that are clickable must be focusable. Use <a> or <button> elements rather than <div> with click handlers. The :focus-visible pseudo-class shows focus only for keyboard navigation.

This is a link element. Press Tab to focus, Enter to activate.
Card 2

Card 3

3. ARIA Roles for Semantic Data Grids

When CSS Grid displays tabular data, add role="grid" to the container, role="row" to row wrappers, and role="columnheader" / role="gridcell" to cells. This enables screen reader table navigation.

5. Font-Size Resilient Grid Items

Users who increase browser font size should see grid items scale properly. Use em/rem units for padding, gaps, and min-heights instead of px.

7. Respecting prefers-reduced-motion

Users with vestibular disorders can set prefers-reduced-motion: reduce in their OS. CSS animations on grid items should respect this preference.

Check Why How
DOM order matches visual order Screen readers follow DOM order Don't use grid-column/row to reorder content
Interactive items are focusable Keyboard users can't click divs Use <a> or <button>
Focus indicators visible Keyboard users need to see focus Use :focus-visible styling
Data grids have ARIA roles Enables table navigation in screen readers Add role="grid", role="row", role="gridcell"
Skip navigation links Keyboard users bypass repeated nav Add skip link to #main-content

Reduced motion support Prevents vestibular discomfort Use @media (prefers-reduced-motion)
Relative units (em/rem) Text scales with user preferences Use rem for font-size, em for spacing
