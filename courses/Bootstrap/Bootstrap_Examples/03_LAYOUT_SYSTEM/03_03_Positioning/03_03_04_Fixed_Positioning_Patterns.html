Fixed Positioning Patterns

Fixed positioning anchors elements to the viewport. They stay in place regardless of scrolling. This navbar uses fixed-top.

Notice: The body has padding-top: 70px to prevent content from hiding behind the fixed navbar.

1. Fixed Positioning Basics

position: fixed positions elements relative to the viewport. They do not move when the page is scrolled.

Bootstrap Fixed Classes
ClassPositionCommon Use
.fixed-topTop of viewportNavigation bars
.fixed-bottomBottom of viewportFooters, action bars
 
<!-- Fixed to top of viewport -->
<nav class="navbar fixed-top">...</nav>

<!-- Fixed to bottom of viewport -->
<div class="fixed-bottom bg-dark text-white text-center py-2">
  Fixed bottom bar
</div>

<!-- Custom fixed positioning -->
<div class="position-fixed" style="top: 50%; right: 20px;">
  Fixed to right side
</div>

2. Fixed Bottom Action Bar

A common pattern for mobile: fixed bottom action bar.

<!-- Fixed bottom action bar -->
<div class="fixed-bottom bg-white border-top shadow-sm py-3">
  <div class="container d-flex justify-content-between align-items-center">
    <span class="fw-bold">$29.99</span>
    <div>
      <button class="btn btn-outline-secondary me-2">Add to Cart</button>
      <button class="btn btn-primary">Buy Now</button>
    </div>
  </div>
</div>

<!-- Remember to add bottom padding to body/content -->
<body style="padding-bottom: 80px;">

4. Scroll-to-Top Button

<!-- Scroll-to-top button -->
<button id="scrollTop"
        class="btn btn-dark position-fixed bottom-0 end-0 m-4 rounded-circle d-none"
        style="width: 50px; height: 50px; z-index: 1040;">
  ↑
</button>

<script>
  // Show/hide scroll-to-top button
  window.addEventListener('scroll', function() {
    const btn = document.getElementById('scrollTop');
    if (window.scrollY > 300) {
      btn.classList.remove('d-none');
    } else {
      btn.classList.add('d-none');
    }
  });

  document.getElementById('scrollTop').addEventListener('click', function() {
    window.scrollTo({ top: 0, behavior: 'smooth' });
  });
</script>

7. Mobile Considerations

iOS Safari Issue: Fixed positioning can be problematic on iOS Safari. Elements may jump when the address bar shows/hides. Use position: sticky or 100dvh units as alternatives. 
/* Safe approach for mobile fixed bottom */
.safe-fixed-bottom {
  position: fixed;
  bottom: 0;
  left: 0;
  right: 0;
  /* Account for iOS safe area */
  padding-bottom: env(safe-area-inset-bottom);
}

/* Alternative: sticky bottom (avoids iOS issues) */
.sticky-bottom-alternative {
  position: sticky;
  bottom: 0;
}
