2. iOS Safari Fixed Position Quirks

Issue: Fixed Elements Jump on Scroll

On iOS Safari, fixed elements can jump when the address bar shows/hides during scroll.

Solutions:
/* Solution 1: Use dvh (dynamic viewport height) */
.full-height-section {
  min-height: 100vh;  /* Fallback */
  min-height: 100dvh; /* Accounts for browser chrome */
}

/* Solution 2: Use sticky instead of fixed where possible */
.navbar {
  position: sticky; /* Not fixed */
  top: 0;
}

/* Solution 3: Safe area insets for notched devices */
.fixed-bottom-bar {
  position: fixed;
  bottom: 0;
  padding-bottom: env(safe-area-inset-bottom);
  /* iOS notch safe area */
}

/* Solution 4: Use -webkit-overflow-scrolling */
.scrollable-container {
  -webkit-overflow-scrolling: touch;
  overflow-y: auto;
}

3. Safari Sticky Positioning Bugs

Known Safari Sticky Issues
Issue 1: Sticky inside overflow:auto/hidden

Safari (and other browsers) disable sticky when an ancestor has overflow other than visible.

/* BROKEN: overflow: auto on parent breaks sticky */
.parent { overflow: auto; }
.child { position: sticky; top: 0; } /* Won't work! */

/* FIX: Remove overflow from parent, use a different scrolling approach */
.parent { overflow: visible; }
.scroll-wrapper { overflow: auto; height: 300px; }
.child { position: sticky; top: 0; } /* Works! */
Issue 2: Table sticky thead
/* Sticky table header - works in modern Safari */
.table-wrapper {
  height: 300px;
  overflow-y: auto;
}
.table thead th {
  position: sticky;
  top: 0;
  background: white;
  z-index: 1;
}

4. RTL Positioning Cross-Browser

Logical vs Physical Properties

Bootstrap uses logical properties (start/end) that work consistently in LTR and RTL across all browsers.

Physical (avoid)Logical (use)Effect in LTREffect in RTL
leftinset-inline-startLeftRight
rightinset-inline-endRightLeft
.start-0.start-0left: 0right: 0
.end-0.end-0right: 0left: 0
 
<!-- Correct: works in both LTR and RTL -->
<div class="position-absolute end-0">End-aligned</div>

<!-- Avoid: breaks in RTL -->
<div class="position-absolute" style="right: 0;">Right-aligned</div>

5. Z-Index Stacking Context

Consistent Stacking Across Browsers

Z-index behavior is consistent across modern browsers, but stacking context creation varies.

/* These all create new stacking contexts (consistent across browsers): */
.new-context {
  position: relative;
  z-index: 1;
  /* OR */
  position: absolute;
  z-index: 1;
  /* OR */
  opacity: 0.99; /* Less than 1 creates context */
  /* OR */
  transform: translateZ(0);
  /* OR */
  will-change: transform;
  /* OR */
  isolation: isolate; /* Explicit stacking context */
}

/* Sticky: 1020 */
/* Modal: 1055 */
/* Tooltip: 1080 */
