Typography Performance Best Practices

1. System Font Stack (Bootstrap Default)

Bootstrap's default font stack requires no font downloads, making it the fastest option.

Bootstrap's Default Font Stack
font-family: system-ui, -apple-system, "Segoe UI", Roboto,
    "Helvetica Neue", "Noto Sans", "Liberation Sans", Arial,
    sans-serif, "Apple Color Emoji", "Segoe UI Emoji",
    "Segoe UI Symbol", "Noto Color Emoji";
System Font (Fast)

This text uses Bootstrap's system font stack. No font files are downloaded. The browser uses the native OS font, resulting in zero network overhead and instant rendering.

  • No HTTP requests
  • No FOUT/FOIT
  • Instant text rendering
  • Native OS look and feel
Custom Web Font (Slower)

This text simulates a custom web font. Custom fonts require downloading font files (WOFF2), adding network overhead and potential rendering delays.

  • Additional HTTP request(s)
  • Potential FOUT/FOIT
  • 100-500KB+ per font family
  • Delayed text rendering

2. Efficient Font Loading

If you must use custom fonts, load them efficiently.

Font Display Strategies
Strategy Behavior Best For
font-display: swap Show fallback immediately, swap when font loads Body text (most common)
font-display: optional Show fallback, only use font if cached Performance-critical pages
font-display: fallback Short block period, then swap if loaded within 3s Balanced approach
font-display: block Invisible text until font loads (FOIT) Icon fonts only
Preloading Critical Fonts
<!-- Preload critical font file -->
<link rel="preload" href="/fonts/my-font.woff2"
      as="font" type="font/woff2" crossorigin>

<!-- Define font with swap display -->
<style>
@font-face {
    font-family: 'MyFont';
    src: url('/fonts/my-font.woff2') format('woff2');
    font-display: swap; /* Show fallback while loading */
}
</style>

3. Minimize Font Variations

Each font weight and style is a separate font file. Fewer variations = faster loading.

Efficient: 2-3 Weights

Normal (400)

Semi-bold (600)

Bold (700)

Only load the weights you actually use. Map all needs to 2-3 weights.

Excessive: 9 Weights

Thin 100

Extra-light 200

Light 300

Normal 400

Medium 500

Semi-bold 600

Bold 700

Extra-bold 800

Black 900

9 weights = 9 font files. Huge download overhead.

4. Prefer Bootstrap Utilities Over Custom CSS

Optimal: Bootstrap Utilities
<!-- Zero additional CSS -->
<h2 class="fw-bold text-primary
    text-uppercase">
  Title
</h2>

Classes already exist in Bootstrap's cached CSS bundle. No additional bytes.

Avoid: Custom CSS
/* Custom CSS adds bytes */
.custom-title {
    font-weight: 700;
    color: #0d6efd;
    text-transform: uppercase;
}

Custom CSS adds ~100+ bytes per rule to stylesheet. Multiply by many elements.

5. Text Rendering Performance

Some CSS properties impact how quickly text renders on screen.

font-display and text-rendering
Property Value Effect
text-rendering optimizeSpeed Faster rendering, less precise kerning
text-rendering geometricPrecision Precise rendering, slower
text-rendering auto Browser decides (recommended)
font-smooth always Smooth font rendering
-webkit-font-smoothing antialiased Smoother on macOS
Tip: text-transform: uppercase (Bootstrap's .text-uppercase) triggers a repaint. Use sparingly on large blocks of text.

6. Avoiding Layout Thrashing

Frequent text content changes can cause expensive reflows.

Bad: Layout Thrashing

Reading and writing DOM text in a loop causes repeated reflows.

// BAD: causes layout thrashing
items.forEach(item => {
    const width = item.offsetWidth;
    item.textContent = `${width}px`;
});
Good: Batch Operations

Batch reads and writes separately to minimize reflows.

// GOOD: batch reads, then writes
const widths = items.map(
    i => i.offsetWidth
);
items.forEach((item, i) => {
    item.textContent = `${widths[i]}px`;
});

Typography Performance Checklist

  • ✓ Use Bootstrap's system font stack when possible
  • ✓ Limit custom font families to 1-2 maximum
  • ✓ Load only needed font weights (2-3 maximum)
  • ✓ Use WOFF2 format (best compression)
  • ✓ Set font-display: swap for body fonts
  • ✓ Preload critical fonts with <link rel="preload">
  • ✓ Prefer Bootstrap utility classes over custom CSS
  • ✓ Avoid text-transform on large text blocks
