Border Accessibility

WCAG Compliance & Best Practices

WCAG Contrast Requirements

When borders are used as the only visual indicator of a UI component (e.g., form inputs), they must meet a 3:1 contrast ratio against adjacent colors (WCAG 2.1 SC 1.4.11 Non-text Contrast).

Focus Indicators with Borders

Interactive elements must have visible focus indicators. Border changes are a common approach.

Semantic vs Decorative Borders

Decorative Borders

Borders used purely for visual styling do NOT need to meet contrast requirements.

  • Card borders
  • Section dividers
  • Visual accents
  • Layout separators

Safe to use any color.

Functional Borders

Borders that convey meaning MUST meet 3:1 contrast ratio:

  • Form input boundaries
  • Error state indicators
  • Required field markers
  • Active/selected state indicators

Must ensure sufficient contrast.

Don't Rely Solely on Color/Borders

WCAG 2.1 SC 1.4.1: Don't use color (including border color) as the ONLY way to convey information. Always provide additional visual cues like icons, text, or patterns.

High Contrast Mode Support

In Windows High Contrast Mode, borders are forced to use system colors. Ensure your layouts don't break when border colors change. Use forced-colors: active media query for adjustments.
