Float Cross-Browser Compatibility

CSS floats have been supported since CSS1 (1996), making them one of the most universally compatible layout mechanisms. This page covers browser-specific quirks, edge cases, and testing strategies.

2. IE Double-Margin Bug (Historical)

Historical note: IE6 doubled the margin on the side of a float (e.g., margin-left on a float: left element). The fix was display: inline on the floated element. Modern browsers do not have this bug.
float-start
margin-right: 1.5rem

This float has a right margin. In modern browsers (Chrome, Firefox, Safari, Edge), the margin renders correctly at the specified value. The double-margin bug is relevant only for IE6 and is handled by Bootstrap's CSS.

3. IE 3-Pixel Text-Jog Bug (Historical)

Historical note: In IE6, a 3px gap appeared between a floated element and the adjacent text when certain conditions were met. This is irrelevant for modern browsers but worth knowing for legacy support.
No margin

In modern browsers, text hugs the float edge with no unexpected gap. In old IE, a 3px jog would appear. Current Bootstrap targets modern browsers and does not include IE-specific hacks.

4. Clearfix Cross-Browser Reliability

Bootstrap's clearfix uses ::after with display: block and clear: both, which is supported in all browsers including IE8+. 

/* Bootstrap's clearfix — works in all modern browsers */
.clearfix::after {
  display: block;
  clear: both;
  content: "";
}

5. RTL (Right-to-Left) Support

Bootstrap 5 uses float-start and float-end as logical names. In RTL mode, float-start maps to float: right.

LTR (Left-to-Right):

float-start (left)

Text wraps on the right side in LTR layouts.

RTL (Right-to-Left):

float-start (right in RTL)

In RTL, float-start maps to the right side. Text wraps on the left.

Bootstrap's logical naming (start/end) ensures float utilities work correctly in both LTR and RTL layouts without code changes.

6. Print Media Behavior

Browsers generally respect floats in print, but page breaks inside floated content can cause unexpected results. Use @media print to control float behavior. 

/* Reset floats for print if needed */
@media print {
  .float-start, .float-end {
    float: none !important;
  }
  /* Or keep floats but ensure page-break-inside: avoid */
  .float-box {
    page-break-inside: avoid;
  }
}
