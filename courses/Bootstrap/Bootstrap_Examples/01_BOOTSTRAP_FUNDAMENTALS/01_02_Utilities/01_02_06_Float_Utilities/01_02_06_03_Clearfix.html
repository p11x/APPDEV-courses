Clearfix

When an element is floated, it is removed from the normal document flow. This causes the parent container to "collapse" — it no longer stretches to contain its floated children. Bootstrap's clearfix class solves this problem.

1. The Problem: Collapsed Container

Both boxes below are floated. Without clearfix, the yellow parent container collapses to zero height because floated elements are out of flow.

2. The Solution: clearfix

Add clearfix to the parent element. It uses the ::after pseudo-element to create a clearing block that forces the parent to expand.

float-start
float-end
Next element This content stays below the container because clearfix contains the floats.

3. How It Works (Under the Hood)

.clearfix::after {
  content: "";
  display: block;
  clear: both;
}

The ::after pseudo-element is inserted as the last child of the container. clear: both tells the browser: "place this element below any floating elements above it." This forces the parent to expand to contain all floated children.

7. Alternative Clearing Methods

MethodHowPros/Cons
clearfix class Add to parent Recommended. Works everywhere.
overflow: auto On parent Creates new BFC. May add scrollbars.
overflow: hidden On parent Creates new BFC. May clip content.
display: flow-root On parent Modern approach. No side effects. IE not supported.
Empty <div style="clear:both"> After floats Non-semantic. Avoid in production.
