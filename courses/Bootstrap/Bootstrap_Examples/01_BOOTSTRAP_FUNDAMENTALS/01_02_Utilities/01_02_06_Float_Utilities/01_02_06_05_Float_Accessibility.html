Float Accessibility

Float utilities affect document flow and visual layout, which can create accessibility issues if not handled carefully. This guide covers best practices for maintaining accessible designs when using floats.

1. Floats Preserve DOM Order

Advantage: Unlike flex-order or flex-row-reverse, float utilities do not change visual reading order. Screen readers and keyboard navigation follow the same sequence as the visual layout.

2. Always Use Clearfix

Without clearfix, content after floated elements can overlap them, creating confusing reading order for screen reader users.

3. Meaningful Alt Text on Floated Images

Images floated for text wrapping must still convey meaning through alt attributes. Decorative images should use alt="".

This image has a descriptive alt attribute that conveys its content to screen reader users. If the image were purely decorative, use alt="" and role="presentation" instead.

4. Color Contrast in Wrapped Text

When text wraps around a floated element with a colored background, ensure the text maintains sufficient contrast ratio (4.5:1 for normal text per WCAG AA).

High contrast

This text is on a light background, maintaining high contrast even as it wraps next to the dark floated box. Always test contrast in wrapped layouts because the narrow text column may make readability harder.

5. Responsive Float Considerations

When using float-md-end to move an element to the right on desktop, the DOM order stays the same — but the visual distance between elements increases. Ensure the content still reads logically at all breakpoints.
Important Notice

On mobile, this notice appears above the text. On medium screens, it floats right. In both cases, the DOM order is unchanged — the notice comes first in the HTML, matching its visual priority on mobile.

6. Focusable Elements in Floated Containers

Buttons and links inside floated elements maintain their DOM tab order. This is generally consistent with visual order, unlike flex reordering.

Press Tab to navigate: focus moves from "Tab 1" (left) to "Tab 2" (right), matching the visual left-to-right reading order.

7. Respect Reduced Motion Preferences

If you animate floated elements, respect prefers-reduced-motion.

Hover me

This box has a subtle scale animation on hover. Users with prefers-reduced-motion: reduce should not see the animation. Add this CSS to respect their preference: 

@media (prefers-reduced-motion: reduce) {
  .animated-float { transition: none; }
}

Accessibility Best Practices Summary

  1. Floats preserve DOM order — safer than flex reordering for screen readers
  2. Always use clearfix — prevents content overlap that confuses AT users
  3. Meaningful alt text — floated images need proper descriptions
  4. Test contrast — wrapped text columns may reduce readability
  5. Verify focus order — tab through floated layouts at every breakpoint
  6. Test with screen readers — NVDA, VoiceOver, and JAWS may interpret layouts differently
  7. Respect reduced motion — disable animations when prefers-reduced-motion is set
