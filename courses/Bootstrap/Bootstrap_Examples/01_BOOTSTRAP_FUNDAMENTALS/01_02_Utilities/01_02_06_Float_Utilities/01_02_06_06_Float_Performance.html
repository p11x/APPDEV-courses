Float Performance

Floats are one of the oldest CSS layout mechanisms. Understanding their rendering characteristics helps you make informed decisions about when to use floats versus flexbox or grid for optimal performance.

1. Float Rendering Pipeline

Browser rendering: Style → Layout → Paint → Composite. Floats trigger Layout (reflow) because surrounding content must recalculate its position relative to the floated element.

When a float is added or removed, the browser must re-layout all adjacent inline content that wraps around it. This makes float operations more expensive than transform or opacity changes, which only affect Composite.

2. Clearfix Overhead

The clearfix pseudo-element forces the browser to calculate the clearing position, which requires measuring all floated children. In deeply nested layouts, this can cascade.

5. Minimize Dynamic Float Changes

Toggling float classes with JavaScript causes layout thrashing. Prefer using CSS media queries for responsive float changes, and avoid modifying floats in animation loops.

Static float

This float is set via CSS class and does not change dynamically. The browser calculates its position once and reuses the result.

Anti-pattern: Don't do element.style.float = 'left' inside a requestAnimationFrame loop. This forces a layout recalc every frame.

6. Avoid Layout Thrashing

Layout thrashing occurs when you alternately read and write layout properties, forcing the browser to recalculate layout repeatedly within a single frame.

Bad — forces reflow each iteration:

items.forEach(item => {
  item.style.float = 'left';          // WRITE
  const width = item.offsetWidth;     // READ (forces reflow)
  item.style.width = (width + 10) + 'px'; // WRITE
});

Good — batch reads, then writes:

const widths = items.map(i => i.offsetWidth);  // READ all
items.forEach((item, i) => {
  item.style.float = 'left';                    // WRITE
  item.style.width = (widths[i] + 10) + 'px';  // WRITE
});

7. CSS Containment

The contain CSS property tells the browser that an element's internal layout is independent, limiting the scope of reflows.
