Visibility Utilities - Mobile First

Designing mobile-first experiences using visibility utilities.

1. Mobile-First Approach

Bootstrap's responsive utilities follow a mobile-first approach: classes apply from the breakpoint upward.

Key rule: .d-sm-none hides the element on sm and above (tablet, desktop). It does NOT hide it on xs (mobile). The element is visible on mobile by default.
ClassMobile (xs)Tablet (sm)Desktop (md+)
.d-none .d-md-blockHiddenHiddenVisible
.d-none .d-sm-blockHiddenVisibleVisible
.d-block .d-sm-noneVisibleHiddenHidden
.d-md-none .d-lg-blockVisibleVisibleVisible

7. Mobile-First Breakpoint Reference

Classxssmmdlgxlxxl
.d-none------
.d-blockBBBBBB
.d-none .d-sm-block-BBBBB
.d-none .d-md-block--BBBB
.d-none .d-lg-block---BBB
.d-block .d-sm-noneB-----
.d-block .d-md-noneBB----

B = Block (visible), - = Hidden

Mobile-first tip: Start by designing for mobile (xs). Then use .d-none .d-md-block to add desktop-only features. Never remove functionality from mobile that desktop users have.
