Visibility Utilities - Cross Browser

How Bootstrap's visibility utilities behave across different browsers and devices.

1. Browser Support Overview

Bootstrap 5 supports all modern browsers. Here is the compatibility matrix for visibility utilities.

Utility CSS Property Chrome Firefox Safari Edge IE 11
.d-none display: none All All All All Yes
.invisible visibility: hidden All All All All Yes
.visible visibility: visible All All All All Yes
.visually-hidden clip/position All All All All Yes
.d-*-none display + media All All All All Partial

2. Mobile Browser Behavior

iOS Safari

  • .d-none works correctly
  • .invisible works correctly
  • .visually-hidden works with VoiceOver
  • Viewport units include address bar height
  • Use dvh for dynamic viewport height

Chrome (Android)

  • .d-none works correctly
  • .invisible works correctly
  • .visually-hidden works with TalkBack
  • Viewport units may differ when keyboard open
  • Responsive breakpoints work as expected

Samsung Internet

  • All visibility utilities work
  • Based on Chromium, same as Chrome
  • Supports all Bootstrap 5 features
  • Responsive breakpoints match Chrome
  • Screen reader: works with TalkBack

3. Known Issues & Workarounds

Problem: .d-contents is not supported in Safari < 15.4.

Workaround: Use .d-block or .d-flex as a fallback, or check for support with @supports.

@supports (display: contents) {
    .d-contents { display: contents !important; }
}

Problem: 100vh includes the browser's address bar, making content taller than the visible area.

Workaround: Use CSS custom property with JavaScript or svh/dvh units in modern browsers.

/* Modern solution */
.vh-100-safe {
    height: 100vh; /* fallback */
    height: 100dvh; /* dynamic viewport height */
}

/* JavaScript fallback */
:root {
    --vh: 1vh;
}
.vh-100-safe {
    height: calc(var(--vh, 1vh) * 100);
}

window.addEventListener('resize', () => {
    document.documentElement.style.setProperty(
        '--vh', window.innerHeight * 0.01 + 'px'
    );
});

Problem: Elements with visibility: hidden can still receive focus in certain edge cases.

Workaround: Add tabindex="-1" to invisible elements that should not be focusable.

<div class="invisible" tabindex="-1">
    This won't receive keyboard focus
</div>

/* Or use d-none if you want full hiding */
<div class="d-none">
    Completely removed from tab order
</div>

Problem: Windows High Contrast Mode may show borders/elements that should be invisible.

Workaround: Use forced-colors: none media query to adjust.

@media (forced-colors: active) {
    .invisible {
        border: none;
        outline: none;
    }
}

Recommendation: Use Bootstrap's standard visibility utilities. They are tested across all supported browsers. Only write custom workarounds for specific issues encountered in your testing.

6. CSS Feature Detection

Use @supports to provide fallbacks for newer CSS features.

/* Progressive enhancement for visibility */

/* Base: works everywhere */
.hidden {
    display: none;
}

/* Enhancement: if content-visibility is supported */
@supports (content-visibility: hidden) {
    .hidden-lazy {
        content-visibility: hidden;
        contain-intrinsic-size: 0 500px;
    }
}

/* Enhancement: if dynamic viewport units are supported */
@supports (height: 100dvh) {
    .vh-100 {
        height: 100dvh;
    }
}
