3. Screen Reader Utilities

Classes for managing content accessibility.

ClassVisuallyScreen ReaderFocusable
.visually-hidden Hidden Readable No
.visually-hidden-focusable Hidden (shown on focus) Readable Yes 
<!-- Visually hidden, readable by screen readers -->
<span class="visually-hidden">Additional context</span>

<!-- Skip navigation link -->
<a href="#main" class="visually-hidden-focusable">Skip to content</a>

<!-- Icon button with accessible label -->
<button>
    <svg aria-hidden="true">...</svg>
    <span class="visually-hidden">Search</span>
</button>

4. Responsive Display

Apply display utilities at specific breakpoints.

BreakpointClass FormatExample
xs (default).d-{value}.d-none
sm (>=576px).d-sm-{value}.d-sm-block
md (>=768px).d-md-{value}.d-md-flex
lg (>=992px).d-lg-{value}.d-lg-grid
xl (>=1200px).d-xl-{value}.d-xl-inline
xxl (>=1400px).d-xxl-{value}.d-xxl-block
Pattern: Use .d-none .d-{breakpoint}-block to show only from a breakpoint upward. Use .d-block .d-{breakpoint}-none to show only below a breakpoint.

6. Common Patterns

Mobile-Only Element

<div class="d-block d-sm-none">Only visible on phones</div>

Desktop-Only Element

<div class="d-none d-md-block">Only visible on tablets and desktops</div>

Accessible Icon Button

<button>
    <svg aria-hidden="true">...</svg>
    <span class="visually-hidden">Delete item</span>
</button>

Skip Navigation

<a href="#main" class="visually-hidden-focusable btn btn-primary">
    Skip to main content
</a>

Responsive Table/Card Toggle

<!-- Table on desktop -->
<table class="d-none d-md-table">...</table>

<!-- Cards on mobile -->
<div class="d-block d-md-none">
    <div class="card">...</div>
</div>

Loading Placeholder

<!-- While loading -->
<div class="placeholder-glow">
    <span class="placeholder col-6"></span>
</div>

<!-- After loading (JavaScript) -->
<script>
    placeholder.classList.add('d-none');
    realContent.classList.remove('d-none');
</script>

Live Region for Screen Readers

<div class="visually-hidden" aria-live="polite" aria-atomic="true">
    <!-- Injected with JavaScript for announcements -->
</div>

7. Decision Tree

Which class should you use?

QuestionAnswerUse
Should the element be completely removed from layout? Yes .d-none
Should the element be hidden but reserve space? Yes .invisible
Should screen readers still read the element? Yes, visually hidden .visually-hidden
Should it appear on keyboard focus? Yes .visually-hidden-focusable
Should it only show on mobile? Yes .d-block .d-sm-none
Should it only show on desktop? Yes .d-none .d-md-block
Should it be hidden when printing? Yes .d-print-none
Should it only appear when printing? Yes .d-none .d-print-block
Is it a decorative element (icon, image)? Yes aria-hidden="true"

8. Common Gotchas

Gotcha #1: .d-none removes from screen readers

.d-none sets display: none, which removes the element from both the visual layout AND the accessibility tree. Use .visually-hidden if screen readers need to access the content.

Gotcha #2: Responsive classes are additive

.d-none .d-md-block means: hidden by default, block on md+. The class does NOT revert at larger breakpoints - it persists upward.

Gotcha #3: .invisible preserves space

.invisible keeps the element in the layout. If you want to remove it from the layout entirely, use .d-none instead.

Gotcha #4: !important overrides

All Bootstrap display utilities use !important. They will override any custom CSS or inline styles.

Gotcha #5: Print classes don't override screen classes

.d-none .d-print-block works: hidden on screen, visible on print. But .d-block .d-print-none is needed to do the opposite (visible on screen, hidden on print).
