Visibility Utilities - Accessibility

Making visibility decisions that work for all users, including those using assistive technologies.

1. Visually Hidden Content (.visually-hidden)

The .visually-hidden class hides elements visually but keeps them in the accessibility tree for screen readers.

ClassVisible?Screen Reader?Takes Space?
.visually-hiddenNoYesNo
.d-noneNoNoNo
.invisibleNoYesYes

2. Accessible Icon-Only Buttons

Icon-only buttons need accessible labels. Use .visually-hidden to provide text for screen readers.

Accessible (Good)

Each button has .visually-hidden text describing its action.

Inaccessible (Bad)

No accessible labels. Screen readers cannot identify these buttons.

3. Skip Navigation Link

A .visually-hidden-focusable link appears only when focused via keyboard (Tab key).

HTML Structure:

<!-- Place as first element in body -->
<a href="#main-content" class="visually-hidden-focusable">
    Skip to main content
</a>

<nav>...</nav>

<main id="main-content">
    <!-- page content -->
</main>

The class .visually-hidden-focusable differs from .visually-hidden in that the element becomes visible when it receives keyboard focus.

4. When to Use Each Class

ScenarioUseReason
Hide decorative element .d-none No meaning, screen readers don't need it
Label for icon button .visually-hidden Screen readers need the label text
Skip navigation link .visually-hidden-focusable Should appear on keyboard focus
Responsive content swap .d-none .d-md-block Completely different content per breakpoint
Alternative text for mobile .visually-hidden .d-md-none + visible alternative Mobile users need the content too
Additional context for screen readers .visually-hidden Provides context not visible on screen
Reserve space while hiding .invisible Prevent layout shift

7. ARIA Attributes and Visibility

Use aria-hidden to control what assistive technologies can access.

AttributeVisible?Screen Reader?Use Case
aria-hidden="true"YesNoDecorative images, icons
aria-hidden="false"YesYesDefault (accessible)
.visually-hiddenNoYesLabels, context
.d-noneNoNoCompletely hidden
Rule of thumb: If a sighted user doesn't need to see it but a screen reader user needs to know about it, use .visually-hidden. If nobody needs it, use .d-none.
