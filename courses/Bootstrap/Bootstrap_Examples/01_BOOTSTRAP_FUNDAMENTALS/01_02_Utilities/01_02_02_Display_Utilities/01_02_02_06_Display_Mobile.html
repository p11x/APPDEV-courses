Mobile-First Display Utilities

Bootstrap is mobile-first. Base display classes apply everywhere; breakpoint classes override from that breakpoint up.

1. Mobile-First Principle

Class Applies To Direction
d-none All sizes Always hidden
d-block All sizes Always visible
d-md-none md, lg, xl, xxl Hidden from 768px up
d-md-block md, lg, xl, xxl Visible from 768px up
d-none d-md-block All Hidden on mobile, visible on tablet+
Key insight: d-none d-md-block means "hidden by default, visible from md up." The base class sets the mobile state; the responsive class overrides it.

7. Mobile Bottom Action Bar

On mobile devices, a fixed bottom bar provides easy thumb access.

Desktop uses inline buttons. Mobile uses a fixed bottom bar.

8. Mobile-First Strategy Summary

  1. Start mobile: Write base classes (no breakpoint) for mobile layout first.
  2. Add breakpoints upward: Use d-md-* to override for larger screens.
  3. Touch targets: Ensure interactive elements are at least 44×44px.
  4. Content priority: Hide secondary content on mobile with d-none d-md-block.
  5. Navigation: Use bottom bars or hamburger menus on mobile, full nav on desktop.
  6. Tables: Convert to card layouts on mobile using d-none d-md-block pattern.
  7. Swipe patterns: Use horizontal scroll containers on mobile.
