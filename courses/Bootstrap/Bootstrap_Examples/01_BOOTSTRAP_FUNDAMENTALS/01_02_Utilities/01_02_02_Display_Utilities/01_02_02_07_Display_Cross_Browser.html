4. IE11 Legacy Fallback (Reference)

Historical Reference Only — Bootstrap 5 does not support IE11.
IE11 flexbox quirks:
  • IE11 requires -ms- prefix for flexbox
  • IE11 has bugs with flex-basis and flex-wrap
  • display: contents is not supported in IE11
  • display: grid is not supported
Vendor-prefix approach (if needed):
/* Full vendor prefix chain */
.element {
    display: -webkit-box;      /* iOS 6-, Safari 3.1-6 */
    display: -webkit-flex;     /* Safari 6.1+, Chrome */
    display: -ms-flexbox;      /* IE 10 */
    display: flex;             /* Standard */
}
Feature detection with @supports:
/* Fallback for browsers without grid */
@supports not (display: grid) {
    .d-grid {
        display: flex;
        flex-wrap: wrap;
    }
}

7. Common Issues & Fixes

Issue Affected Browser Fix
Flex item overflow not constrained Safari Add min-width: 0 to flex children
display: contents breaks accessibility All (bug) Avoid d-contents on semantic elements
Grid fallback for older browsers Safari <12 Use flexbox fallback with @supports not (display:grid)
d-print-* not working Some print contexts Ensure media query specificity is correct
!important conflicts All Bootstrap uses !important on display utils — override with matching specificity

8. Cross-Browser Testing Checklist

  • Test responsive display at all breakpoints in Chrome DevTools
  • Verify d-none, d-block, d-flex, d-grid in Firefox
  • Check flexbox overflow behavior in Safari
  • Test print display (d-print-*) by printing the page
  • Run Lighthouse accessibility audit
  • Verify touch targets on real mobile devices
  • Test with screen readers (NVDA, VoiceOver) for d-none vs visually-hidden
