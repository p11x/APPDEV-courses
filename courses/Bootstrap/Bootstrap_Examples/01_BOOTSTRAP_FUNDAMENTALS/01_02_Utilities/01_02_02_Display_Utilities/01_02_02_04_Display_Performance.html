Display Utilities — Performance Considerations

Understanding the performance implications of display utilities helps you build faster, more efficient web pages.

1. Visibility Techniques Comparison

Method Comparison
Technique Rendered? In Layout? Accessible? Performance
d-none No No No Best — browser skips rendering
visibility: hidden Yes Yes No Moderate — still rendered, space reserved
opacity: 0 Yes Yes Yes Same — fully rendered, can animate
.visually-hidden Yes No Yes Good — accessible, removed from layout

6. CSS Class Overhead

Display Utility Classes Generated by Bootstrap

Bootstrap generates the following display utility classes:

Base (all sizes)
  • d-none
  • d-inline
  • d-block
  • d-inline-block
  • d-flex
  • d-inline-flex
  • d-table
  • d-table-cell
  • d-table-row
  • d-grid
  • d-inline-grid
  • d-contents
Per Breakpoint (×5 breakpoints = 60 additional classes)

Each base class is repeated for sm, md, lg, xl, xxl.

  • d-sm-none ... d-sm-contents
  • d-md-none ... d-md-contents
  • d-lg-none ... d-lg-contents
  • d-xl-none ... d-xl-contents
  • d-xxl-none ... d-xxl-contents
Optimization tip: In production, use tools like PurgeCSS, or Bootstrap's Sass $utilities map to disable unused display utility breakpoints and reduce CSS bundle size.

8. Performance Tips Summary

Tip Technique Benefit
Hide heavy content Start with d-none, show on demand Faster initial paint
Smooth transitions Use opacity + CSS transition No reflow, GPU-accelerated
Batch toggles Toggle parent container instead of children Single reflow instead of N
Reduce CSS size Disable unused display breakpoints in Sass Smaller CSS bundle
Accessible hiding Use .visually-hidden over d-none Screen readers can access
Avoid layout shifts Use d-invisible pattern for skeleton No CLS penalty
