Shadow Utilities - Performance Considerations

Understanding how shadow utilities affect rendering performance.

1. Shadow Rendering Cost

Shadow Render Cost Reason
.shadow-sm Low Small blur radius (2px), minimal computation
.shadow Medium Medium blur radius (1rem), moderate computation
.shadow-lg Medium-High Large blur radius (3rem), more computation
Multiple shadows High Each shadow layer adds to paint cost
Animated shadows Very High Repaint on every animation frame

3. Shadows on Scrolling Content

Scroll Performance Tips

  • Avoid shadows on many scrolling items: A list of 100+ items with shadows can cause scroll jank. Use shadows sparingly in long lists.
  • Use will-change: box-shadow: Hints the browser to optimize shadow rendering. Don't overuse - only for frequently updated elements.
  • Consider CSS containment: contain: paint limits shadow repainting to the element boundary.
  • Simplify shadows during scroll: Use JavaScript to reduce shadow complexity during active scrolling.

5. Optimization Techniques

Do
  • Use .shadow-sm for large lists
  • Use .shadow for cards and containers
  • Use .shadow-lg sparingly for featured items
  • Combine shadows with will-change for animated elements
  • Use contain: paint on shadow containers
  • Remove shadows during scroll with JS
Don't
  • Animate box-shadow directly
  • Apply .shadow-lg to 100+ list items
  • Stack many overlapping shadow layers
  • Use custom shadows with huge blur values
  • Apply shadows to elements that resize frequently
  • Combine shadows with complex CSS filters
