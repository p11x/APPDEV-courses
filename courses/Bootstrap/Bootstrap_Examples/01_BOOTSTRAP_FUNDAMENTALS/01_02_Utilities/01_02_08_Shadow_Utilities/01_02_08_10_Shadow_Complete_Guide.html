5. Do's and Don'ts

Do
  • Use .shadow-sm for most UI elements
  • Use .shadow for primary cards and containers
  • Use .shadow-lg sparingly for emphasis
  • Pair shadows with borders for accessibility
  • Consider performance on long lists
  • Respect prefers-reduced-motion
  • Test on mobile devices
  • Use semantic HTML with shadow styling
Don't
  • Rely solely on shadows for visual distinction
  • Use .shadow-lg on every element
  • Animate box-shadow directly (use transform)
  • Apply heavy shadows to long scrollable lists
  • Use shadows as the only focus indicator
  • Ignore high contrast mode users
  • Stack many shadow layers
  • Combine shadows with complex filters on mobile

Key Takeaways
  1. Three shadow levels: small, regular, large
  2. Shadows are purely visual - always provide borders for accessibility
  3. Use shadow-sm for performance on mobile and long lists
  4. Respect prefers-reduced-motion for animations
  5. Test in high contrast mode
