Close Button Accessibility

Ensuring close buttons are usable by everyone, including users of assistive technologies.

1. ARIA Label - The Most Important Attribute

Required: Every close button MUST have an aria-label attribute describing its action.

Correct Implementation:

aria-label="Close"

More Descriptive Labels:

"Close notification"
"Dismiss alert"
"Remove item from cart"

2. Focus Management

Tab Navigation Test:

Press Tab to navigate to the close button below. It should receive visible focus.

Focus After Dismissal: When a close button dismisses an element, focus should move to a logical place - typically the next sibling or the element that contained the dismissed element.

Focus Trap Example:

After closing the alert below, focus will move here.

This alert will return focus to the paragraph above when closed.

3. Keyboard Accessibility

Native Button Keyboard Support:

Key Action Status
Tab Move focus to the close button Built-in
Enter Activate the close button Built-in
Space Activate the close button Built-in
Escape Close parent modal/overlay JS Handler

Test Keyboard Activation:

Focus the close button using Tab, then press Enter or Space:

Keyboard test container (click X to remove)

4. Screen Reader Announcements

Live Region for Announcements:

Success message that will be announced when dismissed.

When dismissed, "Success message dismissed" will be announced to screen readers via the live region.

5. Touch Target Size (WCAG 2.5.8)

Increasing Touch Target Size:

Default (16px)

p-2 (32px target)

p-3 (44px target)

/* Increase touch target with padding */
.btn-close {
    padding: 0.75rem; /* Increases clickable area */
    /* Visual size stays the same, only hit area grows */
}

6. Color Contrast Requirements

Good Contrast:

Dark close on white background

Good Contrast:

White close on dark background
Warning: Never place the default (dark) close button on a dark background, or the white close button on a light background. This fails WCAG contrast requirements.

7. Complete Accessible Close Button Pattern

<!-- 
    Complete accessible close button:
    1. type="button" - prevents form submission
    2. aria-label - provides accessible name
    3. Sufficient touch target (padding)
    4. Visible focus indicator
    5. Proper contrast ratio
-->
<button 
    type="button" 
    class="btn-close p-2"
    aria-label="Dismiss this notification"
    data-bs-dismiss="alert"
></button>
