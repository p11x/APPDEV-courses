Sizing Utilities - Accessibility

Ensuring sizing decisions work for all users, including those with disabilities.

1. Minimum Touch Target Size (WCAG 2.5.5)

Interactive elements should be at least 44x44 CSS pixels for touch accessibility.

Accessible Touch Targets

All buttons are at least 44x44px. Meets WCAG 2.5.5 Level AAA.

Inaccessible Touch Targets

Buttons are too small. Difficult for users with motor impairments.

2. Text Width for Readability

Constrain text width for optimal readability. Lines should be 50-75 characters wide.

Good Line Length

Reading is easier when lines of text are not too wide. The optimal line length is between 50 and 75 characters. This paragraph demonstrates a comfortable reading width.

max-width: 65ch constrains to ~65 characters.

Poor Line Length

This paragraph has no width constraint. On a wide screen, lines can stretch across the entire viewport making them very difficult to read. Users lose their place when scanning back to the start of the next line. This is especially problematic for users with dyslexia or cognitive disabilities.

Full-width text is hard to scan and causes eye fatigue.

3. Zoom and Scaling (WCAG 1.4.4)

Use relative units (%, em, rem) for sizing. Avoid fixed pixel widths that break at high zoom levels.

Flexible Width Container

This container uses .w-100 (percentage-based width), so it scales properly when the user zooms in to 200% or more.

Tip: Never set maximum-scale=1 or user-scalable=no in the viewport meta tag.

Never do this: <meta name="viewport" content="..., maximum-scale=1, user-scalable=no">
This prevents users from zooming, violating WCAG 1.4.4.

4. High Contrast Mode Compatibility

Sizing utilities work fine in high contrast mode since they only affect dimensions, not colors.

This box uses .w-50 and .border-3. The width and border thickness are preserved in high contrast mode.

Tip: Avoid using box-shadow as a visual indicator since it may disappear in high contrast mode. Use borders instead.
