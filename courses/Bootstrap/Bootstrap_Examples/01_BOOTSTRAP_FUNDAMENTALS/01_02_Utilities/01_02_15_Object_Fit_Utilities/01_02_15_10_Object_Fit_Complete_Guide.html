4. Responsive Patterns

/* Responsive height */
.hero { height: 200px; }
@media (min-width: 768px) { .hero { height: 400px; } }
@media (min-width: 992px) { .hero { height: 500px; } }

/* Responsive object-fit value */
<img class="object-fit-contain d-md-none" src="...">
<img class="object-fit-cover d-none d-md-block" src="...">

5. Best Practices

  • 1 Always set explicit width AND height on the container
  • 2 Use overflow: hidden on the container
  • 3 Include meaningful alt text for cropped images
  • 4 Use loading="lazy" for below-fold images
  • 5 Serve appropriately sized images (srcset)
  • 6 Use cover for fill, contain for logos
  • 7 Test on real devices with different aspect ratios

6. Common Mistakes

Mistake 1: Forgetting to set width/height on the image element. Always add w-100 h-100 or explicit dimensions.
Mistake 2: Using object-fit without overflow:hidden on the container, causing image to bleed out.
Mistake 3: Using fill when you mean cover. Fill distorts; cover preserves aspect ratio.
