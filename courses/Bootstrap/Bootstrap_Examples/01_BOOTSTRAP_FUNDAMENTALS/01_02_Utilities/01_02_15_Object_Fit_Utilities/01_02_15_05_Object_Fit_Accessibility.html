Object Fit Accessibility

Ensuring images with object-fit remain accessible to all users.

1. Alt Text is Critical

When object-fit crops or hides parts of an image, the alt text must describe the intended meaning, not just what's visible.

Best Practice: Describe the image's purpose, not its dimensions or cropping.

2. Decorative Images

For purely decorative images, use an empty alt="" and role="presentation".

<!-- Decorative image: empty alt -->
<img src="background.jpg" class="object-fit-cover" alt="" role="presentation">

3. Text Over Images

When text is overlaid on object-fit images, ensure sufficient contrast:

Text with overlay

High contrast ensures readability

WCAG 2.1: Text over images requires a contrast ratio of at least 4.5:1 for normal text.

4. Clickable/Focusable Images

Images that are links or buttons must be focusable and have descriptive labels.

<!-- Clickable image with descriptive alt -->
<a href="/gallery">
    <img src="preview.jpg" class="object-fit-cover" 
         alt="Click to view full gallery of sunset photos">
</a>

5. Respecting Reduced Motion

If using CSS transitions on object-fit images, respect prefers-reduced-motion:

@media (prefers-reduced-motion: reduce) {
    .transition-zoom {
        transition: none !important;
    }
}

6. Screen Reader Considerations

  • Object-fit doesn't affect screen reader behavior - they read the alt text regardless
  • Cropped content is still announced via the alt attribute
  • Use aria-describedby for complex images that need longer descriptions
  • Consider figure and figcaption for images with captions
