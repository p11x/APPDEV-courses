Object Fit Performance

Optimizing images when using object-fit for better web performance.

1. Serve Appropriately Sized Images

Even though object-fit: cover crops the image, the browser still downloads the full file. Use appropriately sized images.

Performance Tip: Don't use a 4000px image for a 400px container. The browser downloads all pixels even if they're cropped.

Bad: 4000px image in 400px container

Downloads ~3MB unnecessarily

Good: 400px image in 400px container

Downloads ~50KB - 60x smaller!

2. Lazy Loading

Use loading="lazy" to defer loading of off-screen images:

<img src="image.jpg" 
     class="w-100 h-100 object-fit-cover" 
     loading="lazy"
     alt="Lazy loaded image">

3. Responsive Images with srcset

Serve different image sizes based on viewport width:

<img src="image-800.jpg"
     srcset="image-400.jpg 400w, image-800.jpg 800w, image-1200.jpg 1200w"
     sizes="(max-width: 576px) 400px, (max-width: 992px) 800px, 1200px"
     class="w-100 h-100 object-fit-cover"
     loading="lazy"
     alt="...">

4. Modern Image Formats

Use WebP or AVIF formats with JPEG fallback:

<picture>
    <source srcset="image.avif" type="image/avif">
    <source srcset="image.webp" type="image/webp">
    <img src="image.jpg" class="w-100 h-100 object-fit-cover" alt="...">
</picture>

5. CSS contain Property

Use contain: layout paint style on the container to isolate rendering:

.image-container {
    contain: layout paint style;
    /* Browser won't recalculate layout for siblings */
}

6. Performance Checklist

  • ✓ Serve images at the display size (or 2x for retina)
  • ✓ Use loading="lazy" for below-fold images
  • ✓ Use srcset for responsive image serving
  • ✓ Use modern formats (WebP, AVIF) with fallbacks
  • ✓ Add width and height attributes for CLS prevention
  • ✓ Use CSS contain on containers
  • ✓ Compress images before uploading
