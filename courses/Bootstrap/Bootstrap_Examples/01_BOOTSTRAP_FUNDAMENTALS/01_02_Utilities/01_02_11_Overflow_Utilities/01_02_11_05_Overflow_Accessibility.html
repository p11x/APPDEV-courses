Overflow Accessibility

WCAG Compliance & Screen Reader Best Practices

WCAG 2.1 Guidelines for Overflow

  • 1.4.10 Reflow: Content must reflow at 400% zoom without horizontal scrolling (except for data tables, code, etc.)
  • 2.1.1 Keyboard: All scrollable content must be keyboard accessible
  • 2.4.7 Focus Visible: Focus indicators must be visible within scrollable regions
  • 1.4.12 Text Spacing: Content must remain readable with adjusted text spacing

Keyboard Accessible Scrolling

Accessibility Best Practices Summary

Practice Why Implementation
Add tabindex to scroll regions Enables keyboard focus tabindex="0"
Label scroll regions Screen readers identify purpose role="region" aria-label="..."
Visible focus indicators Keyboard users see current focus Custom :focus styles
Use aria-hidden for decorative overflow Prevents screen reader confusion aria-hidden="true"
Avoid overflow-hidden on important content Hidden content may be inaccessible Use overflow-auto instead
Test with keyboard only Verify all content is reachable Tab through entire page
