3. Fixed Position Browser Quirks

Known Fixed Position Issues

  • iOS Safari Address Bar: Fixed elements at bottom can shift when the address bar appears/disappears. Solution: Use position: sticky or JavaScript to detect viewport changes.
  • Mobile Virtual Keyboard: Fixed elements may move when the on-screen keyboard opens. Solution: Use visualViewport API to adjust positioning.
  • Transform Ancestors: If a parent has transform, position: fixed behaves like position: absolute. Solution: Avoid transform on ancestors of fixed elements.
  • Filter Ancestors: If a parent has filter, fixed positioning is relative to that parent, not the viewport. Solution: Move fixed elements outside filtered containers.

5. Safari-Specific Considerations

Safari Compatibility Notes

  • Sticky in overflow:auto: Safari handles position: sticky inside overflow: auto containers correctly in modern versions. Older Safari (pre-13) had issues with nested sticky elements.
  • Fixed in overflow:hidden: Safari clips position: fixed elements inside ancestors with overflow: hidden. This is actually spec-compliant behavior - other browsers do the same.
  • 100vh on iOS: Safari's 100vh includes the address bar area, which can cause fixed elements to be partially hidden. Use 100dvh (dynamic viewport height) or JavaScript for accurate sizing.
  • -webkit-overflow-scrolling: Safari uses -webkit-overflow-scrolling: touch for momentum scrolling. Modern Safari supports standard overflow: auto with smooth scrolling.
