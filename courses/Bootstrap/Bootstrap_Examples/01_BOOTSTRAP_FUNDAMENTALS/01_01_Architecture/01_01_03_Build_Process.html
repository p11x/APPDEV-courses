Build Process Overview

Understanding the toolchain and pipeline that transforms Bootstrap's SASS source into optimized CSS and JavaScript for production deployment.

Build Pipeline

1Source Files

Bootstrap's source code lives in the scss/ directory. JavaScript source is in js/src/. These files use modern syntax (SASS, ES modules) that browsers cannot execute directly.

  • _variables.scss - Configuration
  • bootstrap.scss - Entry point
  • js/src/*.js - JS components
2SASS Compilation

The Dart SASS compiler processes all .scss files, resolving @import / @use statements, evaluating variables, expanding mixins, and flattening nesting.

  • Resolves all imports
  • Evaluates variables
  • Expands mixins
  • Flattens nesting
3PostCSS Processing

PostCSS is a tool for transforming CSS with JavaScript plugins. Bootstrap uses Autoprefixer to add vendor prefixes automatically based on browser support targets.

// Input:
display: flex;
// Output (with Autoprefixer):
display: -webkit-box;
display: -ms-flexbox;
display: flex;
4Minification

Minification removes whitespace, comments, and shortens property names where safe. This reduces file size for faster downloads. Bootstrap's CSS goes from ~250KB to ~50KB minified.

  • Removes whitespace
  • Removes comments
  • Optimizes values
  • ~80% size reduction

Build Tools

Dart SASS

The official SASS compiler. Bootstrap uses Dart SASS (not the deprecated Node-SASS) for compilation.

$ npm install sass
+ sass@1.77.0

Dart SASS is written in Dart but distributed as a JavaScript package for Node.js. It's the primary implementation and receives new SASS features first.

PostCSS + Autoprefixer

PostCSS processes CSS with plugins. Autoprefixer automatically adds vendor prefixes based on your browser support targets.

$ npm install postcss autoprefixer
+ postcss@8.4.38
+ autoprefixer@10.4.19

Reads .browserslistrc to know which browsers to support, then adds only necessary prefixes.

npm Scripts

Bootstrap uses npm scripts in package.json to orchestrate the build pipeline. No Grunt or Gulp needed.

$ npm run css
Compiling SASS...
Running PostCSS...
Minifying CSS...
Done: dist/css/bootstrap.min.css

Simple, cross-platform, no extra dependencies beyond Node.js.

Configuration Files

package.json

The npm configuration file. Defines dependencies, scripts, and project metadata. 

{
  "name": "bootstrap",
  "version": "5.3.3",
  "scripts": {
    "css": "npm-run-all css-compile css-prefix css-minify",
    "css-compile": "sass --style expanded --source-map ...",
    "css-prefix": "postcss --replace ...",
    "css-minify": "cleancss --level 1 ..."
  },
  "devDependencies": {
    "sass": "^1.77.0",
    "postcss": "^8.4.38",
    "autoprefixer": "^10.4.19",
    "clean-css-cli": "^5.6.3"
  }
}
.browserslistrc

Defines which browsers the project targets. Autoprefixer and other tools read this to determine what CSS transformations to apply. 

# Bootstrap's browser support targets
# https://browsersl.ist/

>= 0.5%
last 2 major versions
not dead
Chrome >= 60
Firefox >= 60
Firefox ESR
iOS >= 12
Safari >= 12
not Explorer <= 11
not op_mini all

These targets mean: support browsers with at least 0.5% global usage, the last 2 major versions of each browser, and specific minimum versions for key browsers.

postcss.config.js

PostCSS plugin configuration. Defines which transformations to apply to compiled CSS. 

module.exports = {
  plugins: [
    // Adds vendor prefixes automatically
    require('autoprefixer')({
      overrideBrowserslist: [
        '>= 0.5%',
        'last 2 major versions'
      ]
    }),
    // Optional: discard comments
    // require('postcss-discard-comments')()
  ]
};
.browserslistrc Query Examples

Different query formats you can use:

Query Meaning
> 1% Browsers with >1% global usage
last 2 versions Last 2 versions of each browser
not dead Exclude browsers without official support
defaults Common sensible defaults
cover 99.5% Cover 99.5% of all users

NPM Build Commands

Bootstrap Build Commands

Commands available in Bootstrap's package.json:

# Clone Bootstrap and install dependencies
$ git clone https://github.com/twbs/bootstrap.git
$ cd bootstrap
$ npm install
# Build all CSS
$ npm run css
# Build all JavaScript
$ npm run js
# Build everything (CSS + JS)
$ npm run build
# Watch for changes during development
$ npm run css-watch
# Run the documentation site locally
$ npm run docs-serve
