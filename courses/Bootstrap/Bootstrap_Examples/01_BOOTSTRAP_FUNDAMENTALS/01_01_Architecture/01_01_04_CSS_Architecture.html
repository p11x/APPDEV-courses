CSS Architecture Patterns

Understanding how Bootstrap organizes its CSS into logical layers, manages specificity, and uses consistent naming conventions.

CSS Layers

Bootstrap organizes its CSS into 5 distinct layers, each building upon the previous. This layered architecture ensures predictable cascading behavior.

Layer 1: Reboot (CSS Reset)

Normalizes browser defaults. Sets box-sizing: border-box on all elements. Removes margins, normalizes headings, forms, tables.

Layer 2: Type & Root (Base Styles)

Sets root font-size, body defaults, link styles, heading styles. Defines CSS custom properties on :root.

Layer 3: Layout (Grid & Containers)

Container, row, column classes. Flex utilities, display utilities. The grid system lives in this layer.

Layer 4: Components (Cards, Buttons, Forms, etc.)

All Bootstrap components. Each has a base class (e.g., .btn) and modifier classes (e.g., .btn-primary).

Layer 5: Utilities (Override Everything)

Highest specificity. !important used on some utilities. Classes like .d-flex, .text-center, .bg-primary.

Why order matters: Because CSS follows the "last rule wins" principle (when specificity is equal), utilities must come last to be able to override component styles. Reboot must come first to establish a consistent baseline across browsers.
SASS Import Order

bootstrap.scss imports files in layer order:

// Layer 1: Foundation
@import "reboot";
// Layer 2: Base
@import "root";
@import "type";
// Layer 3: Layout
@import "containers";
@import "grid";
@import "helpers";
// Layer 4: Components
@import "buttons";
@import "forms";
@import "card";
@import "modal";
@import "navbar";
// Layer 5: Utilities (last!)
@import "utilities/api";
@import "utilities/display";
@import "utilities/flex";

Specificity

Specificity Hierarchy

CSS specificity is calculated as a 4-part value: (a,b,c,d)

Selector Type Weight Example
Inline styles 1,0,0,0 style="color:red"
ID selector 0,1,0,0 #myId
Class / Attribute / Pseudo-class 0,0,1,0 .btn-primary
Element / Pseudo-element 0,0,0,1 div, ::before
Universal selector 0,0,0,0 *
!important overrides all specificity. Bootstrap uses it sparingly, mainly on utility classes to ensure they always win.
Bootstrap's Specificity Strategy

Bootstrap keeps component specificity low (single class selectors) so you can easily override with your own class-based selectors.

// Bootstrap uses single-class selectors
.btn { ... } /* 0,0,1,0 */
.card { ... } /* 0,0,1,0 */
// Compound selectors for variants
.btn-primary { ... } /* 0,0,1,0 */
.card-body { ... } /* 0,0,1,0 */
// Utilities use !important to win
.text-center { text-align: center !important; }
.d-flex { display: flex !important; }
Overriding Bootstrap

To override Bootstrap, use equal or higher specificity:

// Same specificity - wins by order
.btn-primary { background-color: purple; }
// Higher specificity
.my-theme .btn-primary { ... }
// Best: Use Bootstrap's variables
$primary: purple; // Before importing

Naming Conventions

Bootstrap's Naming Pattern

Bootstrap uses a component-modifier pattern with hyphens as delimiters. This is not BEM, but serves the same purpose.

component - sub-component - modifier
nav
card
btn
-item
-body
-group
-primary
-lg
-active
Examples
  • .nav - Component
  • .nav-item - Sub-component
  • .nav-link - Sub-component
  • .nav-tabs - Variant
  • .nav-pills - Variant
  • .btn - Component
  • .btn-primary - Color variant
  • .btn-lg - Size variant
  • .btn-group - Wrapper
  • .btn-outline-primary - Outline + color
Naming in Action

BEM vs Bootstrap

BEM uses __ for elements and -- for modifiers. Bootstrap uses hyphens for everything.

BEM Bootstrap
.card .card
.card__header .card-header
.card--dark .card-dark
.btn--primary--lg .btn-primary.btn-lg

Customization Strategies

Override CSS

Add your own CSS after Bootstrap. Use equal or higher specificity to override default styles.

/* After Bootstrap CSS */
.btn-primary {
  background: purple;
}
Easy Limited
SASS Variables

Override Bootstrap's !default variables before importing Bootstrap's SASS. Changes propagate everywhere.

// Before importing Bootstrap
$primary: #8b5cf6;
$border-radius: 0.5rem;
$enable-gradients: true;
@import "bootstrap";
Medium Recommended
Selective Imports

Import only the Bootstrap parts you need. Reduces CSS bundle size significantly.

// Only what you need
@import "functions";
@import "variables";
@import "mixins";
@import "reboot";
@import "grid";
@import "buttons";
// Skip: modals, carousels, etc.
