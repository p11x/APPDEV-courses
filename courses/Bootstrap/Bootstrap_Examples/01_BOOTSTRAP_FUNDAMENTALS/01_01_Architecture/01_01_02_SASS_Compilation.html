SASS Compilation

Understanding how Bootstrap uses SASS (Syntactically Awesome Style Sheets) to generate its CSS. SASS adds powerful features like variables, nesting, mixins, and partials to standard CSS.

SASS Variables

What Are SASS Variables?

SASS variables store values that can be reused throughout your stylesheet. They use the $ prefix. Bootstrap's _variables.scss file contains over 1,500 variables that control every aspect of the framework.

// Bootstrap's primary color variable
$primary: #0d6efd !default;
// Spacing scale variable
$spacer: 1rem !default;
// Border radius variable
$border-radius: 0.375rem !default;
// Using variables in CSS rules
.btn-primary {
  background-color: $primary;
  border-radius: $border-radius;
}
Key Concept: The !default flag allows you to override Bootstrap's default values. Set your variables before importing Bootstrap's SASS to customize the framework.
Variable Output Example

After SASS compilation, variable references are replaced with their actual values:

CSS Output: border-radius: 0.5rem;
border: 2px solid #0d6efd;
background-color: #f8f9fa;

The compiled CSS contains the final values, not the variable names. Browsers don't understand SASS variables - they only see plain CSS.

SASS Nesting

Without Nesting (Plain CSS)
.nav {
  display: flex;
  list-style: none;
}
.nav .nav-item {
  margin-right: 1rem;
}
.nav .nav-item .nav-link {
  color: #6c757d;
}
.nav .nav-item .nav-link:hover {
  color: #0d6efd;
}

Repetitive parent selectors increase file size and make maintenance harder. Changing the parent name requires updating every selector.

With Nesting (SASS)
.nav {
  display: flex;
  list-style: none;
  .nav-item {
    margin-right: 1rem;
    .nav-link {
      color: #6c757d;
      &:hover {
        color: #0d6efd;
      }
    }
  }
}

The & (ampersand) compiles to the parent selector. &:hover becomes .nav-link:hover.

SASS Mixins

What Are Mixins?

Mixins let you define reusable groups of CSS declarations. Think of them as functions for CSS. You define them once and include them wherever needed using @include.

// Define a mixin
@mixin flex-center {
  display: flex;
  justify-content: center;
  align-items: center;
}
// Use it with @include
.hero-banner {
  @include flex-center;
  height: 100vh;
}
Mixins With Parameters

Mixins can accept parameters for flexibility:

// Mixin with default parameter
@mixin border-radius($radius: 0.25rem) {
  border-radius: $radius;
  -webkit-border-radius: $radius;
}
// Usage
.card { @include border-radius(0.5rem); }
.badge { @include border-radius(50rem); }
Bootstrap's Actual Mixins
// From _mixins.scss
@mixin media-breakpoint-up($name) {
  @media (min-width: breakpoint-min($name)) {
    @content;
  }
}
// Used to create responsive classes
@include media-breakpoint-up(md) {
  .col-md-6 { width: 50%; }
}

SASS Partials & Imports

Bootstrap's File Structure

Bootstrap's SASS source is organized into many partial files. The main entry point (bootstrap.scss) imports all partials in a specific order. This modular structure makes the codebase maintainable.

scss/
├── bootstrap.scss ← Main entry point
├── _variables.scss ← All default values
├── _variables-dark.scss
├── _mixins.scss ← Reusable patterns
├── _root.scss
├── _reboot.scss ← CSS reset
├── _type.scss
├── _buttons.scss
├── _forms.scss
├── _navbar.scss
├── _card.scss
├── _modal.scss
├── _utilities/
│  ├── _api.scss
│  ├── _display.scss
│  └── _flex.scss
└── bootstrap-grid.scss ← Grid-only build
The Import Chain

Bootstrap's main file imports partials in a specific order. Order matters because later files depend on earlier ones.

// bootstrap.scss (simplified)
@import "functions"; // Must be first
@import "variables"; // Uses functions
@import "mixins"; // Uses variables
@import "root"; // CSS custom properties
@import "reboot"; // CSS reset
@import "type"; // Typography
@import "buttons"; // Components
@import "card";
@import "modal";
// Modern SASS uses @use instead of @import
@use "variables" as *;
SASS Color Functions

SASS can modify colors programmatically:

$primary (#0d6efd)
darken($primary, 15%) (#084298)
lighten($primary, 30%) (#6ea8fe)
rgba($primary, 0.5)
@use vs @import

@import is deprecated in modern SASS. Use @use and @forward instead.

@import "variables";
@use "variables" as *;
@use "variables" as v;
// v.$primary or just $primary

Compilation Workflow

1. Write SCSS

Create .scss files using variables, nesting, mixins, and imports. Override Bootstrap defaults in your own variables file.

2. Compile

Run the SASS compiler (sass input.scss output.css) or use a build tool like Webpack, Vite, or Parcel with SASS plugins.

3. Serve CSS

The compiled .css file contains plain CSS that browsers understand. No SASS syntax remains. Optionally minify for production.

Common SASS Commands
# Install SASS globally
npm install -g sass
# Compile once
sass src/scss/main.scss dist/css/main.css
# Watch for changes (auto-recompile)
sass --watch src/scss:dist/css
# Compile with compression
sass src/scss/main.scss dist/css/main.css --style compressed
# Using npm scripts in package.json
"scripts": {
  "sass:build": "sass src/scss:dist/css --style compressed",
  "sass:watch": "sass --watch src/scss:dist/css"
}
