Custom Build Configurations

Learn how to create custom Bootstrap builds by overriding variables, importing only what you need, and configuring feature flags.

Variable Overrides

The Override Pattern

Every Bootstrap variable uses the !default flag. This means "only set this if it hasn't been defined yet." By defining variables before importing Bootstrap, you override the defaults.

// custom.scss - Your customization file
// 1. Override variables FIRST
$primary: #8b5cf6;
$secondary: #64748b;
$success: #10b981;
$border-radius: 0.75rem;
$enable-shadows: true;
$enable-gradients: true;
// 2. Then import Bootstrap
@import "bootstrap/scss/bootstrap";
Order matters! Variables must be defined before the @import statement. Bootstrap checks if each variable is already set before applying its default.

Feature Flags

Enable/Disable Features

Bootstrap uses $enable-* variables as feature flags. Set them to false to exclude related CSS.

Selective Import Build

Import only the parts of Bootstrap you need. Each @import adds specific CSS output.

// Minimal Bootstrap build
// Required foundation
@import "bootstrap/scss/functions";
@import "bootstrap/scss/variables";
@import "bootstrap/scss/mixins";
// CSS Reset
@import "bootstrap/scss/reboot";
// Grid system only
@import "bootstrap/scss/containers";
@import "bootstrap/scss/grid";
// Only the components you need
@import "bootstrap/scss/buttons";
@import "bootstrap/scss/forms";
// Skip: modals, carousels, tooltips,
// dropdowns, popovers, etc.

Quick Custom Build
# 1. Install Bootstrap SASS
npm install bootstrap
# 2. Create custom.scss
# 3. Override + Import
# 4. Compile
sass custom.scss custom.css
# 5. Minify
sass custom.scss custom.min.css \
  --style compressed
A typical custom build saves 60-80% of the CSS file size compared to the full Bootstrap build.
