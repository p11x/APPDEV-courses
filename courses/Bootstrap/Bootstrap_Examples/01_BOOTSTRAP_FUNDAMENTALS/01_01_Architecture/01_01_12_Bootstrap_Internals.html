CSS Internals

CSS Custom Properties

Bootstrap 5.2+ defines CSS custom properties (variables) on :root. These enable runtime theming without recompiling SASS.

/* Bootstrap defines these on :root */
:root {
  --bs-blue: #0d6efd;
  --bs-indigo: #6610f2;
  --bs-purple: #6f42c1;
  --bs-primary: #0d6efd;
  --bs-success: #198754;
  --bs-body-font-family: system-ui, ...;
  --bs-body-bg: #fff;
}
Runtime theming: Override CSS custom properties without recompiling SASS: :root { --bs-primary: #8b5cf6; }
Reboot (CSS Reset)

Bootstrap's reboot.scss normalizes browser defaults. It's based on Normalize.css but goes further with opinionated resets.

/* Key reboot rules */
/* Border-box on everything */
*, *::before, *::after {
  box-sizing: border-box;
}
/* Consistent font rendering */
body {
  margin: 0;
  font-family: var(--bs-body-font-family);
  line-height: 1.5;
}
/* Remove default list styles */
ul, ol { list-style: none; }
/* Consistent table spacing */
table { border-collapse: collapse; }
Utility API

Bootstrap 5's utility classes are generated programmatically using a SASS API. This is defined in scss/utilities/_api.scss and uses maps and loops.

// scss/_utilities.scss
$utilities: (
  "display": (
    property: display,
    class: d,
    values: (
      none: none,
      inline: inline,
      block: block,
      flex: flex,
    )
  ),
)
// Generated CSS output:
.d-none { display: none !important; }
.d-inline { display: inline !important; }
.d-block { display: block !important; }
.d-flex { display: flex !important; }
// Responsive variants:
@media (min-width: 768px) {
  .d-md-none { ... }
  .d-md-flex { ... }
}

JavaScript Internals

Class Hierarchy

All Bootstrap JS components extend a BaseComponent class. This provides common functionality: initialization, disposal, and event handling.

// BaseComponent - foundation for all components
class BaseComponent {
  constructor(element, config) {
    this._element = element;
    this._config = config;
  }
  dispose() { ... } // Cleanup
}
// All components extend BaseComponent
class Modal extends BaseComponent { ... }
class Dropdown extends BaseComponent { ... }
class Tooltip extends BaseComponent { ... }
class Carousel extends BaseComponent { ... }
class Collapse extends BaseComponent { ... }
class Tab extends BaseComponent { ... }
Event System

Bootstrap uses native DOM events with a consistent naming pattern. Each component fires events at specific lifecycle points.

// Event naming pattern: action.bs.component
// Modal events:
'show.bs.modal' // Before show
'shown.bs.modal' // After show
'hide.bs.modal' // Before hide
'hidden.bs.modal' // After hide
'hidePrevented.bs.modal' // Backdrop click with static
// Listening for events:
document.getElementById('myModal')
  .addEventListener('show.bs.modal', (event) => {
    console.log('Modal is about to show');
    // event.target = the modal element
    // event.relatedTarget = trigger element
  });
Component Lifecycle
Initialization

Constructor reads config from data attributes, merges with default options

Data API Discovery

data-bs-toggle="modal" triggers automatic initialization via the Data API

Event Binding

Component binds click, keydown, and other event handlers to the DOM

User Interaction

Events fire, show/hide/toggle methods are called

Disposal

.dispose() removes event listeners, clears data, frees memory

BaseComponent Deep Dive

Static Methods

BaseComponent provides static methods on each component class for accessing instances from DOM elements.

// Get existing instance
const modal = bootstrap.Modal
  .getInstance(element);
// Get or create instance
const modal = bootstrap.Modal
  .getOrCreateInstance(element);
// jQuery interface (if jQuery loaded)
// $(element).modal('show')
// Default config (shared across instances)
Modal.Default = {
  backdrop: true,
  focus: true,
  keyboard: true
};
Data API

The Data API automatically initializes components from HTML attributes. No JavaScript required for basic usage.

<!-- Data API triggers this -->
<button data-bs-toggle="modal"
  data-bs-target="#myModal">
  Open Modal
</button>
// This automatically runs:
document.querySelectorAll(
  '[data-bs-toggle="modal"]'
).forEach(el => {
  el.addEventListener('click', () => {
    const target = el.getAttribute(
      'data-bs-target');
    new Modal(
      document.querySelector(target)
    ).show();
  });
});
Internal Helper Modules

Bootstrap's JS uses several internal utility modules that components rely on:

EventHandler

Manages event listeners with namespace support. Enables one-line event delegation.

Manipulator

Reads/writes data-bs-* attributes and CSS custom properties from elements.

SelectorEngine

DOM query helpers. Supports :scope pseudo-class for scoped queries.

Backdrop

Manages modal/offcanvas backdrops. Handles click-to-dismiss behavior.
