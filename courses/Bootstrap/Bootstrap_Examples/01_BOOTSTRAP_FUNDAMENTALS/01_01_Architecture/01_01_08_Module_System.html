Module System Patterns

Understanding JavaScript module formats: ES Modules, CommonJS, AMD, and UMD. How Bootstrap ships in multiple formats for maximum compatibility.

Module Formats

ESM
ES Modules

The native JavaScript module system. Uses import/export. Supported natively in modern browsers and Node.js.

import { Modal } from 'bootstrap';
export const myPlugin = ...;
  • Tree-shakeable
  • Async loading
  • Static analysis
  • No IE support
CJS
CommonJS

The Node.js module system. Uses require()/module.exports. Synchronous loading, not natively supported in browsers.

const { Modal } = require('bootstrap');
module.exports = { myPlugin };
  • Node.js native
  • Simple syntax
  • Not tree-shakeable
  • Synchronous only
AMD
AMD

Asynchronous Module Definition. Uses define(). Designed for browsers. Used by RequireJS.

define(['bootstrap'], function(bs) {
  return new bs.Modal(...);
});
  • Async loading
  • Browser-native
  • Complex syntax
  • Declining usage
UMD
UMD

Universal Module Definition. Works everywhere: AMD, CommonJS, and global (window). A compatibility wrapper.

// Works as:
// AMD define()
// CommonJS require()
// window.bootstrap
  • Universal
  • CDN-friendly
  • Larger size
  • Not tree-shakeable

Bootstrap's Distribution Files

dist/js/ File Guide

Bootstrap's dist/js/ directory contains pre-built JavaScript in multiple formats. Choose the right file for your setup.

File Format Popper.js Use Case
bootstrap.js UMD CDN / legacy bundlers
bootstrap.min.js UMD (min) CDN production
bootstrap.bundle.js UMD CDN with tooltips/popovers
bootstrap.bundle.min.js UMD (min) CDN production (recommended)
bootstrap.esm.js ESM Modern bundlers (Webpack/Vite)
bootstrap.esm.min.js ESM (min) Production bundler builds
Recommendation: Use .bundle.min.js for CDN/script-tag usage. Use .esm.min.js for bundler-based projects with tree shaking.
Which File to Use?
Using a CDN?
bootstrap.bundle.min.js
Using Webpack/Vite/Rollup?
import { ... } from 'bootstrap' (ESM)
Using Node.js server-side?
require('bootstrap') (CJS)
Need tooltips/popovers?
Use *.bundle.* (includes Popper.js)

Importing Bootstrap

ES Modules (Bundlers)

For Webpack, Vite, Rollup, or native ESM:

// Import specific components
import { Modal, Tooltip, Dropdown } from 'bootstrap';
// Import all (no tree shaking)
import * as bootstrap from 'bootstrap';
// Import CSS (if bundler supports it)
import 'bootstrap/dist/css/bootstrap.min.css';
// Import SASS (for customization)
import './custom-bootstrap.scss';
// Initialize a modal
const modal = new Modal(
  document.getElementById('myModal')
);
CDN Script Tag

For simple HTML pages without a build step:

<!-- CSS in head -->
<link
  href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css"
  rel="stylesheet">
<!-- JS at end of body -->
<script
  src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js">
</script>
// Access via global
const modal = new bootstrap.Modal(el);
How Node.js Resolves Modules

When you write import { Modal } from 'bootstrap', Node.js:

Looks for node_modules/bootstrap/

Reads package.json "exports" field

Matches "import" condition for ESM

Loads dist/js/bootstrap.esm.js
