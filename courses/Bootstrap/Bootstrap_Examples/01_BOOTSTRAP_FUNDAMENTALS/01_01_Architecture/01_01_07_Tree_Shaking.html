Tree Shaking

Eliminating unused code from your bundle. Tree shaking removes dead code to reduce file sizes and improve load performance.

What Is Tree Shaking?

The Concept

Tree shaking is a term for dead code elimination in JavaScript and CSS. Imagine your codebase as a tree. The live (used) code are the leaves that get "shaken off" the tree, leaving only what you need.

Dependency Tree Visualization
bootstrap
buttons forms grid carousel modal popover
Used = included in bundle | Unused = removed by tree shaking
Key requirement: Tree shaking works best with ES modules (import/export). CommonJS (require/module.exports) is harder to tree-shake because of its dynamic nature.
Benefits
  • Faster Loading

    Smaller bundles download faster

  • Less Parsing

    Browser processes less CSS/JS

  • Lower Memory

    Less code in browser memory

  • Smaller Attack Surface

    Less code = fewer vulnerabilities

CSS Tree Shaking

Method 1: Selective SASS Imports

Only import the SASS partials you actually use. This is the simplest form of CSS tree shaking.

// Only import what you need
@import "bootstrap/scss/functions";
@import "bootstrap/scss/variables";
@import "bootstrap/scss/mixins";
@import "bootstrap/scss/reboot";
@import "bootstrap/scss/grid";
@import "bootstrap/scss/buttons";
// Everything else is excluded
// No: modals, carousels, tooltips...
Method 2: PurgeCSS

PurgeCSS scans your HTML/JS files and removes CSS rules whose selectors are never used in your markup.

// postcss.config.js
module.exports = {
  plugins: [
    require('@fullhuman/postcss-purgecss')({
      content: [
        './**/*.html',
        './js/**/*.js'
      ],
      safelist: {
        standard: [/^modal/, /^tooltip/]
      }
    })
  ]
}

JavaScript Tree Shaking

ES Module Imports

Bootstrap 5 provides ES module builds that enable tree shaking. Import only the components you need.

// Import only what you use
import { Tooltip } from 'bootstrap';
// This is tree-shakeable
// Bundler sees only Tooltip is imported
// Modal, Carousel, etc. are excluded
Do NOT import the full bundle if you want tree shaking:
import 'bootstrap'; // imports everything!

Tree Shaking Tools

Webpack

Uses sideEffects in package.json to identify safe-to-remove modules. Set mode: 'production' to enable tree shaking.

// webpack.config.js
module.exports = {
  mode: 'production',
  optimization: {
    usedExports: true
  }
}
Vite

Built-in tree shaking via Rollup. Just use ES module imports and Vite handles the rest automatically.

// vite.config.js
import { defineConfig }
from 'vite';
export default defineConfig({
  // Tree shaking is
  // enabled by default
  build: {
    minify: 'terser'
  }
});
PurgeCSS

Specifically for CSS. Scans your content files and removes unused CSS selectors. Can reduce Bootstrap CSS by 70-90%.

# CLI usage
purgecss \
  --css bootstrap.css \
  --content **/*.html \
  --output dist/css/
# npm package
npm install \
  @fullhuman/postcss-purgecss
