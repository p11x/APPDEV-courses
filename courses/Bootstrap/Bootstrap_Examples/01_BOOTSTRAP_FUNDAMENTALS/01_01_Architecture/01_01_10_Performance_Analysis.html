Core Web Vitals

Google's Core Web Vitals measure real-world user experience. Bootstrap can affect all three metrics.

Metric Measures Good Bootstrap Impact
LCP
Largest Contentful Paint 		How fast the main content loads < 2.5s Large CSS files delay rendering. Use <link rel="preload"> for critical CSS.
FID / INP
Interaction to Next Paint 		How fast the page responds to input < 200ms Large JS bundles block the main thread. Use async/defer or load only needed components.
CLS
Cumulative Layout Shift 		Visual stability (elements moving) < 0.1 Web fonts and lazy-loaded content can cause shifts. Set explicit dimensions on images and containers.

Optimization Strategies

1. Critical CSS

Inline critical above-the-fold CSS to eliminate render-blocking requests.

<!-- Inline critical CSS -->
<style>
  /* Only navbar, hero, first section */
  .navbar { ... }
  .hero { ... }
</style>
<!-- Load rest asynchronously -->
<link rel="preload"
  href="bootstrap.min.css"
  as="style"
  onload="this.rel='stylesheet'">
2. Async/Defer JS

Prevent JavaScript from blocking HTML parsing.

<!-- Blocks parsing (avoid) -->
<script src="bootstrap.bundle.js"></script>
<!-- Async: loads in parallel, executes ASAP -->
<script async
  src="bootstrap.bundle.js"></script>
<!-- Defer: loads in parallel, executes after HTML -->
<script defer
  src="bootstrap.bundle.js"></script>
<!-- Best: at end of body -->
</body>
<script
  src="bootstrap.bundle.js"></script>
3. CDN Caching

Leverage browser caching and CDN edge servers.

// CDN with SRI (recommended)
<link
  href="https://cdn.jsdelivr.net/npm/"
    "bootstrap@5.3.3/dist/css/"
    "bootstrap.min.css"
  integrity="sha384-..."
  crossorigin="anonymous">
// Benefits:
// - Cached across sites using same CDN
// - Served from nearest edge server
// - HTTP/2 multiplexing
4. Compression

Enable gzip or Brotli compression on your server.

// Nginx config
gzip on;
gzip_types text/css application/javascript;
gzip_min_length 1000;
// Size comparison
bootstrap.min.css: 50 KB
gzip: ~12 KB (76% smaller)
Brotli: ~10 KB (80% smaller)

Continuous Monitoring

Bundle Size Tracking

Track your CSS and JS bundle sizes over time. Set budgets to prevent regressions.

# Install bundlesize
npm install --save-dev bundlesize
// package.json
"bundlesize": [{
  "path": "./dist/css/bootstrap.min.css",
  "maxSize": "55 kB"
}]
Performance Budget

Set performance budgets in Lighthouse CI to catch regressions.

// lighthouserc.js
module.exports = {
  ci: {
    assert: {
      assertions: {
        "resource-summary:stylesheet:size":
          ["error", {maxNumericValue: 60000}],
        "resource-summary:script:size":
          ["error", {maxNumericValue: 85000}],
        "first-contentful-paint":
          ["warn", {maxNumericValue: 2000}]
      }
    }
  }
};
