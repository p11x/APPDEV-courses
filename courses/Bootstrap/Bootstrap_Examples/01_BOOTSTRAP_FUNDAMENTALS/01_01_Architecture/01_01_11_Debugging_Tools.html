Key DevTools Panels

Inspect and modify HTML and CSS in real-time.

  • Click the picker to select elements on page
  • View computed styles, box model, and CSS rules
  • Toggle CSS properties on/off with checkboxes
  • Edit HTML directly by double-clicking elements
  • See which Bootstrap classes apply which styles

View logs, errors, and execute JavaScript.

// Check Bootstrap version
console.log(bootstrap.Tooltip.VERSION);
// Test if Bootstrap is loaded
console.log(typeof bootstrap); // "object"
// List all Bootstrap components
console.log(Object.keys(bootstrap));

Monitor all network requests and their timing.

  • See Bootstrap CSS/JS file sizes and load times
  • Filter by type: CSS, JS, XHR, etc.
  • Check CDN response headers and caching
  • Identify slow or failed requests
  • Throttle network to simulate slow connections

Shows how much CSS/JS is actually used on the page.

  • Open with Ctrl+Shift+P → "Show Coverage"
  • Red = unused bytes, Blue = used bytes
  • Click a file to see line-by-line usage
  • Typically 60-80% of Bootstrap CSS is unused per page

Common CSS Issues

Problem: Your styles don't override Bootstrap.
Fix: Ensure your CSS loads after Bootstrap. Use equal or higher specificity.
// Bad: loads before Bootstrap
// Good: loads after Bootstrap
<link href="bootstrap.css">
<link href="custom.css">

Problem: col-md-6 doesn't activate.
Fix: Check viewport meta tag is present. Resize browser to test breakpoints. Bootstrap breakpoints: sm=576, md=768, lg=992, xl=1200, xxl=1400.

Problem: Gaps or margins you didn't set.
Fix: Check for g-* (gutters) on rows, mb-* on elements, and default margins on <p>, <h*> tags. Use m-0 or p-0 to remove.

JavaScript Debugging

Component Instance Debugging

Access Bootstrap component instances from the console to inspect their state.

// Get a modal's instance
const modalEl = document.
  querySelector('#exampleModal');
const modal = bootstrap.Modal
  .getInstance(modalEl);
// Check its options
modal._config // {backdrop: true, ...}
// Check if it's shown
modalEl.classList
  .contains('show');
// Listen for events
modalEl.addEventListener(
  'show.bs.modal',
  e => console.log(e)
);

Common JavaScript Issues
Problem Cause Solution
Modal won't open JS loaded before DOM Move <script> to end of <body>
Tooltip not showing Missing Popper.js Use bootstrap.bundle.min.js (includes Popper)
Dropdown not working Bootstrap JS not loaded Check console for errors, verify CDN URL
Carousel won't auto-play Missing data-bs-ride Add data-bs-ride="carousel" to the container
Tab content not switching Wrong data-bs-target Verify target ID matches the panel ID

Debugging Tips

Isolate the Problem

Create a minimal reproduction. Strip your page down to just the Bootstrap component that's broken. This helps identify if the issue is Bootstrap or your custom code.

Check the Console

JavaScript errors appear in the console. Look for red error messages. Warnings often indicate deprecated APIs or missing configurations.

Use Breakpoints

In the Sources panel, click line numbers to set breakpoints. Step through code with F10 (step over) and F11 (step into) to trace execution.
