Grid Maintenance Guide

Long-term maintenance patterns, naming conventions, refactoring strategies, and upgrade planning for Bootstrap grids.

1. File Organization

Organize grid-related code into logical, maintainable file structures.

Recommended Sass Structure
src/
├── scss/
│ ├── abstracts/
│ │ ├── _variables.scss # Grid overrides
│ │ ├── _mixins.scss # Custom grid mixins
│ │ └── _functions.scss # Grid helpers
│ ├── layout/
│ │ ├── _grid.scss # Custom grid styles
│ │ ├── _container.scss # Container overrides
│ │ ├── _header.scss
│ │ ├── _sidebar.scss
│ │ └── _footer.scss
│ ├── components/
│ │ ├── _card-grid.scss # Card grid patterns
│ │ ├── _dashboard.scss # Dashboard grid
│ │ └── _gallery.scss # Image gallery grid
│ └── main.scss # Import all
main.scss Import Order
// 1. Bootstrap functions (required first)
@import "bootstrap/scss/functions";

// 2. Custom variables (overrides)
@import "abstracts/variables";

// 3. Bootstrap variables
@import "bootstrap/scss/variables";

// 4. Bootstrap & custom mixins
@import "bootstrap/scss/mixins";
@import "abstracts/mixins";

// 5. Bootstrap grid
@import "bootstrap/scss/grid";

// 6. Custom layout styles
@import "layout/grid";
@import "layout/header";
@import "layout/sidebar";

2. Naming Conventions

Use consistent, descriptive naming for custom grid classes and Sass variables.

Custom Grid Class Naming
/* Layout-level grid classes */
.l-dashboard { } /* Layout: dashboard */
.l-content-grid { } /* Layout: content grid */

/* Component grid classes */
.c-card-grid { } /* Component: card grid */
.c-product-grid { } /* Component: product grid */

/* Responsive modifiers */
.l-dashboard--mobile { } /* Mobile variant */
.l-dashboard--desktop { } /* Desktop variant */

/* State modifiers */
.c-card-grid--loading { } /* Loading state */
.c-card-grid--empty { } /* Empty state */
Variable Naming
// Grid configuration variables
$grid-columns: 12;
$grid-gutter-width: 1.5rem;

// Project-specific grid variables
$dashboard-sidebar-width: 280px;
$dashboard-header-height: 64px;
$card-grid-min-width: 300px;
$content-max-width: 1200px;

// Breakpoint aliases
$bp-mobile: 576px;
$bp-tablet: 768px;
$bp-desktop: 992px;
$bp-wide: 1200px;

// Spacing scale (consistent gutters)
$gutter-sm: 0.5rem;
$gutter-md: 1rem;
$gutter-lg: 1.5rem;
$gutter-xl: 2rem;

3. Anti-Patterns to Refactor

Common grid anti-patterns and their maintainable replacements.

/* Hardcoded pixel values */
.dashboard {
  padding: 23px;
  margin: 17px;
  width: calc(100% - 312px);
}
/* Use Bootstrap variables and spacing scale */
.dashboard {
  @extend .p-4; /* 1.5rem */
  @extend .mb-3; /* 1rem */
  width: calc(100% - var(--sidebar-width));
}
/* Overly nested grid selectors */
.dashboard .main .content
  .row .col-md-8 .widget
  .row .col-md-6 .card
  .card-body {
    font-size: 14px;
  }
/* Flat, maintainable selectors */
.widget-card {
  font-size: 14px;
}

/* Or use Bootstrap utilities */
<div class="col-md-6">
  <div class="card fs-6">...</div>
</div>
<!-- Inline styles are unmaintainable -->
<div class="row"
   style="display:flex; flex-wrap:wrap;
         margin:0 -15px; padding:20px">
<!-- Use Bootstrap utility classes -->
<div class="row gx-3 p-3">
  <div class="col-md-6">...</div>
</div>

<!-- Or semantic custom classes -->
<div class="dashboard-grid">...</div>

4. Reusable Grid Patterns

Create Sass mixins for frequently-used grid patterns.

/** * Auto-fill grid: responsive columns without media queries. * @param {length} $min-width - Minimum column width * @param {length} $gap - Gap between items */ @mixin auto-grid($min-width: 250px, $gap: 1rem) {   display: grid;   grid-template-columns: repeat(auto-fill, minmax($min-width, 1fr));   gap: $gap; } /** * Responsive dashboard layout. * Sidebar collapses on mobile. */ @mixin dashboard-layout($sidebar-width: 280px) {   display: grid;   grid-template-columns: $sidebar-width 1fr;   grid-template-rows: auto 1fr;   min-height: 100vh;   @media (max-width: 767px) {     grid-template-columns: 1fr;   } } /** * Equal-height card grid using Bootstrap classes. */ @mixin card-grid($columns: 3) {   @extend .row;   @extend .g-4;   > * {     @extend .col-md-#{12 / $columns};     @extend .mb-4;   } }

5. Bootstrap Version Upgrade Strategy

Plan for Bootstrap upgrades by isolating grid customizations.

Step 1: Audit
  • List all grid class usages
  • Find deprecated classes
  • Check custom Sass overrides
  • Test in all breakpoints
Step 2: Migrate
  • Update CDN links (or npm package)
  • Rename changed classes
  • Update Sass variable names
  • Run visual regression tests
Step 3: Validate
  • Cross-browser testing
  • Responsive layout verification
  • Accessibility audit
  • Performance benchmarking
Common Migration Changes
Bootstrap 4 Bootstrap 5 Notes
