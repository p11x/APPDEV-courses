Grid Accessibility Patterns

Making Bootstrap grid layouts accessible to all users.

1. Use Semantic HTML Elements

Use <main>, <aside>, <nav>, <section> instead of generic <div>s for grid containers.

<aside>
Sidebar
<main>
Primary content
<!-- Good: Semantic -->
<div class="row">
    <aside class="col-lg-3">Sidebar</aside>
    <main class="col-lg-9">Content</main>
</div>

<!-- Avoid: All divs -->
<div class="row">
    <div class="col-lg-3">Sidebar</div>
    <div class="col-lg-9">Content</div>
</div>

2. Logical Tab Order

Grid's visual order should match the DOM order. Avoid using .order-* to drastically reorder content.

Warning: Changing visual order with .order-* creates a disconnect between what sighted users see and what keyboard/screen reader users experience.
1. First in DOM (tab first)
2. Second in DOM (tab second)
3. Third in DOM (tab third)

3. ARIA Landmarks in Grid Layouts

Use ARIA roles and landmarks to help screen readers understand your layout.

<div class="row">
    <nav class="col-lg-3" aria-label="Sidebar navigation">
        <!-- Sidebar nav -->
    </nav>
    <main class="col-lg-6" role="main">
        <!-- Main content -->
    </main>
    <aside class="col-lg-3" role="complementary" aria-label="Related content">
        <!-- Sidebar -->
    </aside>
</div>

4. Skip Navigation Links

Add skip links for keyboard users to bypass navigation in grid layouts.

<!-- Skip link (visually hidden until focused) -->
<a href="#main-content" class="visually-hidden-focusable">
    Skip to main content
</a>

<div class="row">
    <nav class="col-lg-3">...</nav>
    <main class="col-lg-9" id="main-content">
        Main content starts here
    </main>
</div>

5. Responsive Content Reordering

When reordering on mobile, ensure the new order still makes logical sense.

Main Content (appears first on mobile via DOM order)
Sidebar (appears second on mobile, first on desktop)
Best Practice: Only reorder between 1-2 positions. Major reordering confuses keyboard users.

6. Focus Management in Grid Navigation

When grid items can be filtered or rearranged dynamically, manage focus appropriately.

// When filtering grid items
function filterGrid(category) {
    // Filter items...
    
    // Move focus to the first visible item
    var firstVisible = document.querySelector('.grid-item:not(.d-none) a');
    if (firstVisible) {
        firstVisible.focus();
        // Announce change to screen readers
        announceToSR(category + ' items shown');
    }
}

8. Grid Accessibility Checklist

  • ✓ Use semantic HTML (main, aside, nav, section)
  • ✓ Maintain logical DOM order
  • ✓ Avoid drastic visual reordering with .order-*
  • ✓ Include ARIA landmarks and labels
  • ✓ Add skip navigation links
  • ✓ Manage focus after dynamic grid changes
  • ✓ Test with keyboard navigation (Tab, Enter, Escape)
  • ✓ Test with screen readers (NVDA, VoiceOver)
