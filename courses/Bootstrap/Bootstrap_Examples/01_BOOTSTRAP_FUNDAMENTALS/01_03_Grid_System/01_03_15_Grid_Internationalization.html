Grid Internationalization

RTL layouts, locale-aware spacing, multi-language content, and i18n patterns for Bootstrap grids.

Bootstrap 5 RTL: Bootstrap 5 has built-in RTL support. Use bootstrap.rtl.min.css and set dir="rtl" on the HTML element.

1. RTL Grid Layout

When dir="rtl" is set, Bootstrap grid columns flow from right to left. All flexbox utilities (justify-content-start/end) also reverse automatically.

2. Logical Properties for i18n

CSS logical properties (margin-inline-start, padding-inline-end) adapt to the document's writing direction automatically.

Physical Properties (Fixed)
margin-left: 1rem;

Always applies to the physical left side, regardless of text direction. Breaks in RTL layouts.

Logical Properties (i18n Safe)
margin-inline-start: 1rem;

Applies to the "start" side of the inline axis. In LTR: left. In RTL: right. Automatically adapts.

4. Locale-Aware Number Formatting

Numbers are formatted differently across cultures. Use Intl.NumberFormat for proper localization.

English (US)

1,234,567.89

Comma separator, period decimal
German

1.234.567,89

Period separator, comma decimal
French

1 234 567,89

Space separator, comma decimal
Arabic

١٬٢٣٤٬٥٦٧٫٨٩

Arabic-Indic numerals
// Format number for different locales
new Intl.NumberFormat('en-US').format(1234567.89); // "1,234,567.89"
new Intl.NumberFormat('de-DE').format(1234567.89); // "1.234.567,89"
new Intl.NumberFormat('ar-SA').format(1234567.89); // "١٬٢٣٤٬٥٦٧٫٨٩"

5. RTL-Aware Icon Placement

Icons that indicate direction (arrows, carets) must flip in RTL. Bootstrap provides .ms-auto and .me-auto for this purpose.

LTR Buttons
RTL Buttons
Key Point: In RTL, .ms-* (margin-start) applies to the right, and .me-* (margin-end) applies to the left. Bootstrap handles this automatically.

7. i18n Grid Best Practices

DO
  • Use logical CSS properties (margin-inline-*)
  • Use Bootstrap's RTL CSS for RTL languages
  • Design flexible grid widths for text expansion
  • Use em/rem units for spacing
  • Set lang and dir attributes on HTML element
  • Use Intl API for number/date formatting
  • Test with longest expected translations
DON'T
  • Don't use fixed pixel widths for text containers
  • Don't hardcode left/right in CSS
  • Don't assume left-to-right reading direction
  • Don't use directional icons without flipping
  • Don't concatenate translated strings
  • Don't use text in images (can't be translated)
  • Don't assume English text length as baseline
