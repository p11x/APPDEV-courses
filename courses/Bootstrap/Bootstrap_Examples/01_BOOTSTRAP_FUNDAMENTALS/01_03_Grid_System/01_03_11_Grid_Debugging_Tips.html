2. Common Issues and Solutions

Cause: Using col-12 on all breakpoints.
Solution: Add col-md-6 or similar for larger screens.

Cause: Columns exceed 12 units total.
Solution: Ensure col-* values in a row sum to 12 or less.

Cause: Missing .g-* class or .row parent.
Solution: Add gutters with .g-3 or similar on the row.

Cause: Fixed-width content wider than column.
Solution: Add .overflow-auto or .text-break to the column.

Cause: Offset class needs breakpoint prefix.
Solution: Use .offset-md-3 instead of .offset-3 for responsive offsets.

3. CSS Debug Stylesheet

/* Add this CSS to debug your grid */
.debug-mode .row {
    outline: 2px dashed red;
    outline-offset: 2px;
    position: relative;
}
.debug-mode .row::after {
    content: ".row";
    position: absolute;
    top: 0; left: 0;
    background: red;
    color: white;
    font-size: 10px;
    padding: 2px 6px;
}
.debug-mode [class*="col-"] {
    outline: 2px solid blue;
    outline-offset: -2px;
}
.debug-mode [class*="col-"]::after {
    content: attr(class);
    background: blue;
    color: white;
    font-size: 9px;
    padding: 1px 4px;
}

4. Browser DevTools Tips

  • Chrome/Firefox: Right-click an element and select "Inspect" to see applied grid classes
  • Flexbox overlay: In Chrome DevTools, click the flex badge next to elements to visualize the flexbox
  • Responsive mode: Press F12, then Ctrl+Shift+M to test different breakpoints
  • Computed styles: Check the "Computed" tab to see final CSS values
  • Layout panel: Chrome DevTools has a dedicated Layout panel showing grid/flex info

5. Grid Validation Checklist

  • 1 All columns are inside a .row
  • 2 All rows are inside a .container
  • 3 Column widths sum to 12 or less per row
  • 4 Responsive classes use mobile-first approach
  • 5 No extra elements between .row and .col
  • 6 Gutters are applied to .row, not .col
