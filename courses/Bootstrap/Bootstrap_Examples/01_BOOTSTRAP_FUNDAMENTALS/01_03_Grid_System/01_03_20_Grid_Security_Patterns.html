Grid Security Patterns

Security best practices for dynamic Bootstrap grids: XSS prevention, CSP compliance, and safe content generation.

1. XSS Prevention in Dynamic Grids

When generating grid content dynamically, always sanitize user input to prevent Cross-Site Scripting (XSS) attacks.

✗ UNSAFE (DO NOT USE)
// DANGEROUS: innerHTML with user data
const userInput = '<img src=x onerror="alert(1)">';
container.innerHTML = `
  <div class="col">
    ${userInput}
  </div>
`;

// XSS executes! User input contains malicious HTML
✓ SAFE (USE THIS)
// SAFE: Use textContent + DOM API
const userInput = '<img src=x onerror="alert(1)">';
const col = document.createElement('div');
col.className = 'col';
col.textContent = userInput; // Escaped!
container.appendChild(col);

// Output: &lt;img src=x onerror=...&gt;
Safe Dynamic Grid Generation
/** * Safely create a Bootstrap grid item with user content. * Uses DOM API instead of innerHTML to prevent XSS. * * @param {Object} data - User data object * @returns {HTMLElement} - Safe grid column element */ function createSafeGridItem(data) {   // Create elements using DOM API (safe from XSS)   const col = document.createElement('div');   col.className = 'col-md-4 mb-3';   const card = document.createElement('div');   card.className = 'card h-100';   const body = document.createElement('div');   body.className = 'card-body';   const title = document.createElement('h5');   title.className = 'card-title';   title.textContent = data.title; // SAFE: textContent escapes HTML   const text = document.createElement('p');   text.className = 'card-text';   text.textContent = data.description; // SAFE   body.appendChild(title);   body.appendChild(text);   card.appendChild(body);   col.appendChild(card);   return col; }

2. Content Security Policy Compliance

CSP restricts resource loading to prevent XSS. Bootstrap 5 is CSP-compatible if you avoid inline styles and scripts.

CSP Header Example
Content-Security-Policy:
  default-src 'self';
  script-src 'self'
    https://cdn.jsdelivr.net;
  style-src 'self'
    https://cdn.jsdelivr.net;
  img-src 'self' data: https:;
  font-src 'self'
    https://cdn.jsdelivr.net;
CSP Best Practices
  • No inline styles
    Move style="..." to external CSS files
  • No inline scripts
    Move onclick="..." to external JS files
  • Use nonce for inline scripts
    <script nonce="random123">...</script>
  • Use SRI hashes for CDN resources
    integrity="sha384-..." attribute on <link> and <script>
CSP-Safe CDN Loading with SRI:
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" integrity="sha384-..." crossorigin="anonymous">

3. Input Sanitization for Grid Content

When user input must be rendered as HTML (e.g., rich text), use a sanitizer library to strip dangerous elements.

DOMPurify Sanitization
import DOMPurify from 'dompurify';

// Sanitize user HTML before inserting into grid
const userHTML = userInput;
const safeHTML = DOMPurify.sanitize(userHTML, {
  // Only allow specific tags
  ALLOWED_TAGS: ['p', 'strong', 'em', 'a'],
  // Only allow specific attributes
  ALLOWED_ATTR: ['href', 'class'],
  // Prevent data: URIs in href
  ALLOW_DATA_ATTR: false
});

// Now safe to use with innerHTML
col.innerHTML = safeHTML;
Escaping HTML Entities (Vanilla)
/** * Escape HTML entities to prevent XSS. * Use when you can't load a sanitizer library. * * @param {string} str - Potentially dangerous string * @returns {string} - Safe string with entities escaped */ function escapeHTML(str) {   const div = document.createElement('div');   div.textContent = str;   return div.innerHTML; } // Usage: const safe = escapeHTML(userInput); // "<script>alert(1)</script>" → // "&lt;script&gt;alert(1)&lt;/script&gt;"

4. Subresource Integrity (SRI)

Use the integrity attribute on CDN resources to verify they haven't been modified by a man-in-the-middle attack.

Bootstrap 5.3.3 SRI Hashes
<!-- CSS with integrity hash -->
<link
  href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css"
  rel="stylesheet"
  integrity="sha384-QWTKZyjpPEjISv5WaRU9OFeRpok6YcnS/1Vx6wgc68mFRAS7I+s3GRznyrA2f27"
  crossorigin="anonymous">

<!-- JS with integrity hash -->
<script
  src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"
  integrity="sha384-YvpcrYf0tY3lHB60NNkmXc5s9fDVZLESaAA55NDzOxhy9GkcIdslK1eN7N6jIeHz"
  crossorigin="anonymous"></script>
Without SRI: If the CDN is compromised, malicious code could be injected into your page. SRI ensures the browser only executes the exact file you intended.

5. Safe Template Patterns

Patterns for safely rendering dynamic grid content in different frameworks.

React (Safe by Default)
// React auto-escapes JSX content
function SafeGrid({ data }) {
  return (
    <div className="row">
      {data.map(item => (
        <div className="col"
          key={item.id}>
          {item.title}
        </div>
      ))}
    </div>
  );
}
// Title is auto-escaped!
Vue (Safe by Default)
<!-- Vue auto-escapes {{ }} -->
<template>
  <div class="row">
    <div
      v-for="item in items"
      :key="item.id"
      class="col">
      {{ item.title }}
    </div>
  </div>
</template>
<!-- Title is auto-escaped! -->
Angular (Safe by Default)
<!-- Angular auto-escapes {{ }} -->
<div class="row">
  <div
    *ngFor="let item of items"
    class="col">
    {{ item.title }}
  </div>
</div>

<!-- For HTML: [innerHTML]="sanitized"
Use DomSanitizer -->

6. Grid Security Checklist

✓ Use textContent instead of innerHTML for user data
✓ Enable CSP with strict directives
✓ Use SRI hashes on all CDN resources
✓ Sanitize HTML with DOMPurify if needed
✓ Use HTTPS for all external resources
✗ Don't use innerHTML with user-provided strings
✗ Don't eval() dynamic grid configurations
✗ Don't trust URL params for grid content
✗ Don't use inline event handlers (onclick, onerror)
✗ Don't skip SRI verification on CDN resources
