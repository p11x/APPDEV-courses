CSS Cascade Layers

@layer for specificity management

1. Layer Priority Order

Styles in later layers override earlier layers. Unlayered styles always win.

2. Specificity Within Layers

Inside a layer, normal specificity applies. Between layers, order wins.

Styled by high-priority layer (declared after low-priority). Even if low-priority had #id specificity, high-priority wins.
/* Cascade layers solve the common problem of overriding a CSS */ /* framework (like Bootstrap) without using !important. */

3. Framework Override Pattern

Layer Order Strategy
/* Declare layer order */
@layer reset, framework, components, utilities;

/* Framework styles (e.g., Bootstrap) */
@layer framework {
    .btn { background: blue; }
}

/* Your component overrides */
@layer components {
    .btn { background: green; }
    /* Overrides framework without !important */
}

/* Utility overrides */
@layer utilities {
    .btn-urgent { background: red; }
    /* Highest priority */
}

4. Nested Layers

@layer framework { @layer layout { ... } } creates sub-layers.

Nested layer demo: @layer framework.layout overrides @layer framework.theme

5. @import with Layers

/* Import Bootstrap into a framework layer */
@import url('bootstrap.css') layer(framework);

/* Import your overrides into a higher layer */
@import url('overrides.css') layer(utilities);

/* Or use anonymous layers for isolation */
@import url('third-party.css') layer();

/* Anonymous layers can't be extended later */
