CSS Animation Worklet

High-performance animations on a dedicated thread

1. Why Animation Worklet?

Problem: Main Thread Jank

CSS animations and requestAnimationFrame run on the main thread. When JavaScript blocks the main thread (heavy computation, layout), animations stutter and drop frames.

Solution: Animation Worklet

The Animation Worklet runs on a separate thread. Animations continue smoothly even when the main thread is busy. Similar to how CSS transitions on compositor properties already work.

3. Animation Worklet Code Example

// Step 1: Define the animator class in a worklet
// (in a separate file: animator.js)
class SpringAnimator {
    constructor(options) {
        this.currentTime = options.currentTime;
    }

    animate(target, timing) {
        // Spring physics calculation
        const progress = timing.progress;
        const spring = 1 - Math.cos(progress * Math.PI * 4)
                     * Math.exp(-progress * 3);
        target.style.transform = `scale(${spring})`;
    }
}

// Step 2: Register the animator
CSS.registerAnimator('spring-bounce', SpringAnimator);

// Step 3: Use in CSS (hypothetical syntax)
/*
.my-element {
    animation: spring-bounce 1s linear;
    animation-timeline: scroll();
}
*/

// Step 4: Load the worklet
if ('animationWorklet' in CSS) {
    CSS.animationWorklet.addModule('animator.js');
}
