PWA Deployment

Deploy your Progressive Web App to production with proper hosting, CI/CD pipelines, and multi-platform distribution.

📦 Deployment Platforms

Netlify
# netlify.toml
[build]
  publish = "dist"
  command = "npm run build"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200

[[headers]]
  for = "/sw.js"
  [headers.values]
    Cache-Control = "no-cache"
    Service-Worker-Allowed = "/"

# Deploy
netlify deploy --prod
Vercel
// vercel.json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ],
  "headers": [
    {
      "source": "/sw.js",
      "headers": [
        { "key": "Cache-Control", "value": "no-cache" },
        { "key": "Service-Worker-Allowed", "value": "/" }
      ]
    },
    {
      "source": "/manifest.json",
      "headers": [
        { "key": "Content-Type", "value": "application/manifest+json" }
      ]
    }
  ]
}
Firebase Hosting
// firebase.json
{
  "hosting": {
    "public": "dist",
    "rewrites": [{
      "source": "**",
      "destination": "/index.html"
    }],
    "headers": [{
      "source": "/sw.js",
      "headers": [{
        "key": "Cache-Control",
        "value": "no-cache"
      }]
    }]
  }
}

# Deploy
firebase deploy --only hosting

🛠 CI/CD Pipeline

GitHub Actions PWA Deployment
# .github/workflows/deploy.yml
name: Build and Deploy PWA

on:
  push:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      # Checkout code
      - uses: actions/checkout@v4

      # Setup Node.js
      - uses: actions/setup-node@v4
        with:
          node-version: '20'

      # Install dependencies
      - run: npm ci

      # Build the app
      - run: npm run build

      # Run Lighthouse PWA audit
      - name: Lighthouse PWA Audit
        uses: treosh/lighthouse-ci-action@v10
        with:
          configPath: ./lighthouserc.json

      # Generate service worker (if using Workbox)
      - run: npx workbox generateSW workbox-config.js

      # Deploy to Netlify
      - name: Deploy to Netlify
        uses: nwtgck/actions-netlify@v2
        with:
          publish-dir: './dist'
          production-deploy: true
        env:
          NETLIFY_AUTH_TOKEN: ${{ secrets.NETLIFY_AUTH_TOKEN }}
          NETLIFY_SITE_ID: ${{ secrets.NETLIFY_SITE_ID }}

🛠 Hosting Requirements

Requirement Why Solution
HTTPS Service workers require secure context All major hosts provide free SSL
SW no-cache header Browsers cache SW aggressively Cache-Control: no-cache
SW scope SW controls pages within its scope Service-Worker-Allowed: /
Correct MIME types Manifest and SW need proper types application/manifest+json
CORS headers Cross-origin resources need CORS Access-Control-Allow-Origin: *
