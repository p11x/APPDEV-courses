Caching Strategies

Master the art of caching in Progressive Web Apps. Choose the right strategy for each type of resource.

📋 Core Caching Strategies

📦 Cache-First (Cache Falling Back to Network)

Flow: Check cache first. If found, return it. If not, fetch from network, cache the response, then return it.

Best for: Static assets (CSS, JS, images, fonts) that rarely change.

// Cache-First Strategy
const cacheFirst = async (request) => {
    // Step 1: Check cache
    const cachedResponse = await caches.match(request);
    if (cachedResponse) {
        return cachedResponse;
    }

    // Step 2: Fetch from network
    try {
        const networkResponse = await fetch(request);
        // Step 3: Cache for next time
        const cache = await caches.open('dynamic-v1');
        cache.put(request, networkResponse.clone());
        return networkResponse;
    } catch (error) {
        // Step 4: Offline fallback
        return caches.match('/offline.html');
    }
};
🌐 Network-First (Network Falling Back to Cache)

Flow: Try network first. If successful, update cache and return. If offline, return cached version.

Best for: Dynamic content that needs freshness (API responses, news, data).

// Network-First Strategy
const networkFirst = async (request) => {
    try {
        // Step 1: Try network
        const networkResponse = await fetch(request);
        // Step 2: Update cache
        const cache = await caches.open('dynamic-v1');
        cache.put(request, networkResponse.clone());
        return networkResponse;
    } catch (error) {
        // Step 3: Fall back to cache
        const cachedResponse = await caches.match(request);
        if (cachedResponse) {
            return cachedResponse;
        }
        // Step 4: Offline fallback
        return caches.match('/offline.html');
    }
};
⚡ Stale-While-Revalidate

Flow: Serve cached version immediately, then fetch from network in background to update cache for next visit.

Best for: Content where speed matters more than absolute freshness.

// Stale-While-Revalidate Strategy
const staleWhileRevalidate = async (request) => {
    // Step 1: Open cache
    const cache = await caches.open('dynamic-v1');

    // Step 2: Get cached response
    const cachedResponse = await cache.match(request);

    // Step 3: Fetch in background (don't await)
    const fetchPromise = fetch(request).then(
        networkResponse => {
            cache.put(request, networkResponse.clone());
            return networkResponse;
        }
    );

    // Step 4: Return cached immediately, or wait for network
    return cachedResponse || fetchPromise;
};
❌ Network-Only

Flow: Always fetch from network, never use cache. Fails completely when offline.

Best for: Real-time data that must never be stale (live prices, chat).

// Network-Only Strategy
const networkOnly = async (request) => {
    try {
        return await fetch(request);
    } catch (error) {
        // Return error response when offline
        return new Response(
            JSON.stringify({ error: 'Offline' }),
            {
                status: 503,
                headers: { 'Content-Type': 'application/json' }
            }
        );
    }
};

⚙ Runtime Caching Configuration

Workbox-Style Runtime Caching Config
// Runtime caching configuration (Workbox-compatible pattern)
// Different strategies for different resource types

const runtimeCacheConfig = [
    // Cache Google Fonts stylesheets
    {
        urlPattern: /^https:\/\/fonts\.googleapis\.com/,
        handler: 'StaleWhileRevalidate',
        options: {
            cacheName: 'google-fonts-stylesheets',
        }
    },
    // Cache Google Fonts webfonts
    {
        urlPattern: /^https:\/\/fonts\.gstatic\.com/,
        handler: 'CacheFirst',
        options: {
            cacheName: 'google-fonts-webfonts',
            expiration: {
                maxEntries: 30,
                maxAgeSeconds: 60 * 60 * 24 * 365 // 1 year
            }
        }
    },
    // Cache images with CacheFirst
    {
        urlPattern: /\.(?:png|jpg|jpeg|svg|gif|webp)$/,
        handler: 'CacheFirst',
        options: {
            cacheName: 'images',
            expiration: {
                maxEntries: 60,
                maxAgeSeconds: 60 * 60 * 24 * 30 // 30 days
            }
        }
    },
    // API calls with NetworkFirst
    {
        urlPattern: /\/api\//,
        handler: 'NetworkFirst',
        options: {
            cacheName: 'api-responses',
            networkTimeoutSeconds: 3, // Fall back to cache after 3s
            expiration: {
                maxEntries: 50,
                maxAgeSeconds: 60 * 60 * 24 // 1 day
            }
        }
    },
    // CSS and JS with StaleWhileRevalidate
    {
        urlPattern: /\.(?:css|js)$/,
        handler: 'StaleWhileRevalidate',
        options: {
            cacheName: 'static-resources',
        }
    }
];

🕑 Cache Expiration & Cleanup

Cache Expiration Plugin
// Custom cache expiration implementation
// Limits cache size and entry age

class CacheExpirationPlugin {
    constructor(config) {
        this.maxEntries = config.maxEntries || 50;
        this.maxAgeSeconds = config.maxAgeSeconds || 86400; // 1 day
    }

    // Clean up expired and excess entries
    async cleanup(cacheName) {
        const cache = await caches.open(cacheName);
        const requests = await cache.keys();
        const now = Date.now();

        // Get all entries with timestamps
        const entries = await Promise.all(
            requests.map(async request => {
                const response = await cache.match(request);
                const dateHeader = response.headers.get('sw-fetched-on');
                return {
                    request,
                    timestamp: dateHeader ? parseInt(dateHeader) : 0
                };
            })
        );

        // Delete expired entries
        for (const entry of entries) {
            const age = (now - entry.timestamp) / 1000;
            if (age > this.maxAgeSeconds) {
                await cache.delete(entry.request);
            }
        }

        // Delete oldest entries if over maxEntries
        const remaining = await cache.keys();
        if (remaining.length > this.maxEntries) {
            const sorted = entries.sort((a, b) => a.timestamp - b.timestamp);
            const toDelete = sorted.slice(0, remaining.length - this.maxEntries);
            for (const entry of toDelete) {
                await cache.delete(entry.request);
            }
        }
    }
}

// Usage: clean up periodically
const expiration = new CacheExpirationPlugin({
    maxEntries: 50,
    maxAgeSeconds: 7 * 24 * 60 * 60 // 7 days
});
expiration.cleanup('dynamic-cache');

📊 Strategy Comparison

Strategy Speed Freshness Offline Best For
Cache-First Fastest May be stale Full support Static assets, icons, fonts
Network-First Depends on network Always fresh Cache fallback API data, user content
Stale-While-Revalidate Fast Updates in background Cache fallback Images, non-critical data
Network-Only Slowest Always fresh No support Real-time, critical operations
Cache-Only Fastest Frozen Full support Precached app shell
💡 Rule of Thumb: Use Cache-First for static assets, Network-First for API data, Stale-While-Revalidate for images, and Network-Only for real-time features.
