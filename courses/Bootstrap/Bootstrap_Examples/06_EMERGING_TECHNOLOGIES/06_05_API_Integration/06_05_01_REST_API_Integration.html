REST API Concepts

GET Read

Retrieves resource(s) from the server. Should be safe and idempotent (no side effects; same result every time).

GET /posts
GET /posts/1
POST Create

Sends data to the server to create a new resource. The server typically responds with 201 Created and the new resource.

POST /posts
Body: { title, body }
PUT Update

Replaces an entire resource with the provided data. PUT is idempotent; sending the same request twice produces the same result.

PUT /posts/1
Body: { id, title, body }
DELETE Delete

Removes the specified resource from the server. Typically returns 200 OK or 204 No Content on success.

DELETE /posts/1

Common HTTP Status Codes

Code Meaning When Used
200OKSuccessful GET, PUT, DELETE
201CreatedSuccessful POST (resource created)
204No ContentSuccessful DELETE (nothing to return)
400Bad RequestClient sent invalid data
401UnauthorizedAuthentication required
403ForbiddenClient lacks permission
404Not FoundResource does not exist
500Internal Server ErrorServer-side failure
