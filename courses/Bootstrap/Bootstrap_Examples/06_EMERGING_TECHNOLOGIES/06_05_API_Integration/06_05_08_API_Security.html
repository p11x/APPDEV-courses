1 — CORS (Cross-Origin Resource Sharing)

How CORS Works
  1. Simple request: The browser sends the request with an Origin header. The server replies with Access-Control-Allow-Origin. If the origin is allowed, the browser exposes the response to JavaScript.
  2. Preflight request: For non-simple requests (e.g., PUT, custom headers), the browser first sends an OPTIONS request. The server must respond with appropriate Access-Control-* headers before the actual request proceeds.
  3. Credentials: When cookies or authorization headers are involved, the server must set Access-Control-Allow-Credentials: true and specify an explicit origin (not *).

2 — Content Security Policy (CSP)

Recommended CSP Header

Deploy this as an HTTP response header named Content-Security-Policy. Adjust sources to match your own domains and CDN dependencies. 

default-src      'self';
script-src       'self' https://cdn.jsdelivr.net;
style-src        'self' https://cdn.jsdelivr.net 'unsafe-inline';
img-src          'self' data: https:;
font-src         'self' https://cdn.jsdelivr.net;
connect-src      'self' https://api.example.com;
frame-ancestors  'none';
base-uri         'self';
form-action      'self';

5 — SQL Injection Prevention

Vulnerable Pattern

String concatenation lets an attacker break out of the quoted context and inject arbitrary SQL.

-- User input: '; DROP TABLE users; --

query = "SELECT * FROM users
  WHERE username = '" + userInput + "'"

-- Resulting query:
SELECT * FROM users
WHERE username = ''; DROP TABLE users; --'
Risk: Attacker can read, modify, or delete any data in the database.
Safe Pattern (Parameterized)

Placeholders ensure user input is always treated as data, never as executable SQL.

-- User input: '; DROP TABLE users; --

query = "SELECT * FROM users
  WHERE username = $1"
params = [userInput]

-- The DB driver treats $1 as a literal
-- string value, neutralising the attack.
Result: The injected characters are safely escaped; the query returns zero rows instead of dropping the table.

6 — HTTPS / TLS

HSTS Header

The Strict-Transport-Security header forces browsers to use HTTPS for the specified duration. 

Strict-Transport-Security: max-age=63072000; includeSubDomains; preload
  • max-age – seconds to remember (2 years).
  • includeSubDomains – apply to all sub-domains.
  • preload – opt into browser built-in list.
TLS Best Practices
  • Redirect HTTP → HTTPS (301)
  • Enable HSTS with long max-age
  • Use TLS 1.2 or 1.3 only
  • Disable weak cipher suites
  • Use forward secrecy (ECDHE)
  • Keep certificates renewed
  • Use OCSP stapling
  • Pin certificates if appropriate
