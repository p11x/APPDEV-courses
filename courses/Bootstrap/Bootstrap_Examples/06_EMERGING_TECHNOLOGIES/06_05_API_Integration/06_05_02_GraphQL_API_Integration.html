01. GraphQL Query Structure & Syntax

Query

A Query in GraphQL is used to read or fetch data. It is analogous to a GET request in REST, but far more flexible. Clients specify exactly which fields they need, reducing over-fetching.

# Simple query query { launches(limit: 3) { mission_name launch_date_utc } }
Mutation

A Mutation is used to write or modify data on the server. This includes creating, updating, and deleting resources. Mutations can also return data after the operation.

# Mutation to insert data mutation { insert_users( objects: {name: "Alice"} ) { returning { id name } } }
Variables

Variables allow you to parameterize queries dynamically instead of string interpolation. Variables are passed separately from the query in the request body, preventing injection attacks.

# Query with variables query GetLaunch( $limit: Int ) { launches(limit: $limit) { mission_name } } # Variables JSON: {"limit": 5}
REST vs GraphQL Comparison
Feature REST GraphQL
Endpoints Multiple URLs (/users, /posts) Single endpoint (/graphql)
Data Fetching Fixed data structure per endpoint Client specifies exact fields needed
Over-fetching Common - returns all fields Eliminated - request only what you need
Under-fetching Common - multiple requests needed Single request for nested data
Type System Optional (OpenAPI/Swagger) Built-in strong type system

02. Fetching Data with GraphQL

How GraphQL fetch() Works

Unlike REST's GET requests, GraphQL uses POST for all operations. The query is sent in the request body as JSON:

/* * fetch() call structure for GraphQL: * * 1. URL: The single GraphQL endpoint * 2. Method: Always POST * 3. Headers: Content-Type must be application/json * - Add Authorization header if API requires it * 4. Body: JSON.stringify() with query + variables */ const response = await fetch( 'https://spacex-production.up.railway.app/', { method: 'POST', headers: { 'Content-Type': 'application/json' }, // Body contains the GraphQL query body: JSON.stringify({ query: `{ company { name } }` }) } ); const { data } = await response.json();

03. Mutations

Mutation Concepts

While queries only read data, mutations modify it. GraphQL mutations can return data after execution, allowing you to confirm changes.

Key Differences from Queries:
  • Sequential execution: Fields in a mutation run one after another
  • Side effects: Mutations modify server state
  • Return values: Often return the modified object
  • Input types: Use input objects for complex arguments
// Mutation with variables const mutation = ` mutation InsertUser($name: String!, $email: String!) { insert_users(objects: [{ name: $name, email: $email }]) { returning { id name email created_at } affected_rows } } `; const variables = { name: "John Doe", email: "john@example.com" }; fetch(endpoint, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ query: mutation, variables }) });
