Modals: Accessibility Patterns

Focus management, ARIA attributes, keyboard navigation, and screen reader support

1. ARIA Attributes Reference

Proper ARIA attributes ensure screen readers understand the modal structure.

Accessible Modal Title

This modal demonstrates all required ARIA attributes:

  • aria-labelledby - Points to the modal title
  • aria-describedby - Points to this description
  • aria-hidden="true" - Hides modal from screen readers when closed
  • tabindex="-1" - Makes modal focusable but not in tab order
  • role="dialog" - Identifies this as a dialog
  • aria-modal="true" - Indicates modal behavior

3. Keyboard Navigation

Modals support keyboard navigation out of the box.

Key Action Notes
Escape Close modal Disabled with data-bs-keyboard="false"
Tab Next focusable element Wraps at end
Shift + Tab Previous focusable element Wraps at beginning
Enter Activate focused button When button is focused
