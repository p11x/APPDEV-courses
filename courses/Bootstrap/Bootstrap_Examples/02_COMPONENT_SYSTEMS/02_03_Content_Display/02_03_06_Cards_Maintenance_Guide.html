Cards Maintenance Guide

Patterns for writing maintainable, scalable card components that are easy to update and extend across large codebases.

1. CSS Custom Properties (Variables)

Standard Themed Card
Consistent Styling

CSS custom properties ensure all themed cards share the same border-radius, shadows, and transitions.

Another Themed Card
Single Source of Truth

Change the custom properties in one place and all themed cards update automatically. No hunting through SCSS files.

Third Themed Card
Theme Switching

Override variables at the data-theme attribute level to support dark mode or custom themes.

2. Consistent Naming Conventions

BEM Naming for Custom Card Classes
Block__Element--Modifier Pattern
/* Block: the card component */
.card-custom { }

/* Element: parts of the card */
.card-custom__header { }
.card-custom__body { }
.card-custom__footer { }
.card-custom__image { }

/* Modifier: variations */
.card-custom--featured { }
.card-custom--compact { }
.card-custom--dark { }

/* Usage in HTML:
<div class="card-custom card-custom--featured">
  <div class="card-custom__header">...</div>
  <div class="card-custom__body">...</div>
</div>
*/
Data Attribute Conventions
/* Semantic data attributes */
[data-card-type="product"] { }
[data-card-type="profile"] { }
[data-card-type="article"] { }

/* State attributes */
[data-card-state="loading"] { }
[data-card-state="error"] { }
[data-card-state="empty"] { }

/* Usage:
<div class="card"
     data-card-type="product"
     data-card-state="loaded">
*/

3. Component Composition Patterns

Composable Card

6. Maintenance Checklist

Code Quality
  • ✓ CSS custom properties for theming
  • ✓ Consistent class naming (BEM/utility)
  • ✓ No magic numbers in CSS
  • ✓ Semantic HTML structure
  • ✓ Proper use of Bootstrap utilities
Accessibility
  • ✓ ARIA labels on interactive cards
  • ✓ Keyboard navigation support
  • ✓ Color contrast compliance
  • ✓ Alt text on all images
  • ✓ Focus indicators visible
Performance
  • ✓ Lazy loading on images
  • ✓ Explicit image dimensions
  • ✓ No layout shifts (CLS)
  • ✓ Minimal custom CSS
  • ✓ Efficient selectors
