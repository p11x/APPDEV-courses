4. Grid vs Flexbox for Card Layouts

Layout Approach Comparison
Bootstrap Grid (Flexbox-based)
<!-- Bootstrap grid for cards -->
<div class="row row-cols-1 row-cols-md-3 g-4">
  <div class="col">
    <div class="card h-100">...</div>
  </div>
</div>

/* Browser support: 98%+
   Pros: Equal height cards (h-100),
   gutters with g-*, responsive row-cols */
/* Cons: Cards may not align in
   irregular content grids */
CSS Grid (Alternative)
<!-- CSS Grid for masonry layouts -->
<div class="card-grid">
  <div class="card">...</div>
</div>

.card-grid {
  display: grid;
  grid-template-columns:
    repeat(auto-fill, minmax(300px, 1fr));
  gap: 1rem;
}
/* Browser support: 97%+
   Pros: True masonry, denser packing
   Cons: No built-in Bootstrap utilities */

5. Safari-Specific Fixes

Safari Flex Overflow

Safari has a known issue where flex items can overflow their container. Adding min-height: 0 to flex children or overflow: hidden to the container fixes this.

Fix: min-height: 0 on flex children
Safari Image Lazy Loading

Safari 15.4+ supports loading="lazy" natively. Older versions need an IntersectionObserver polyfill.

Note: Safari <15.4 requires polyfill for native lazy loading

6. Feature Detection & Fallbacks

@supports Queries
/* Progressive enhancement */
.card-media {
  aspect-ratio: 16/9;
}

@supports not (aspect-ratio: 1/1) {
  .card-media {
    padding-bottom: 56.25%;
    position: relative;
  }
}
Feature Detection (JS)
// Check for IntersectionObserver
if ('IntersectionObserver' in window) {
  // Use native lazy loading
  const observer = new IntersectionObserver(...);
} else {
  // Fallback: load all images
  document.querySelectorAll('[data-src]')
    .forEach(img => {
      img.src = img.dataset.src;
    });
}
CSS Custom Props Fallback
/* Fallback for no custom props */
.card-themed {
  border-radius: 0.5rem;
  border-radius: var(
    --card-radius,
    0.5rem
  );
}
/* Browser ignores second rule
   if it doesn't support vars */

7. Cross-Browser Testing Checklist

Visual Consistency
  • ☐ Border radius renders correctly
  • ☐ Box shadows are consistent
  • ☐ Images display at correct aspect ratio
  • ☐ Text rendering is readable
  • ☐ Focus indicators are visible
Layout
  • ☐ Grid columns align properly
  • ☐ Equal height cards in rows
  • ☐ Gutters consistent across browsers
  • ☐ Responsive breakpoints work
  • ☐ No overflow or horizontal scroll
Interaction
  • ☐ Hover states work (non-touch)
  • ☐ Touch targets 48px+ (mobile)
  • ☐ Keyboard navigation works
  • ☐ Animations are smooth (60fps)
  • ☐ No flash of unstyled content
