Form Security Best Practices

CSRF protection, input sanitization, autocomplete attributes, and secure form patterns.

1. CSRF Protection

CSRF tokens verify that form submissions originate from your own site, not from malicious third-party pages.

CSRF Protection Layers:
  1. CSRF Token: Hidden field with unique token (above)
  2. SameSite Cookies: Set SameSite=Strict or SameSite=Lax
  3. Origin/Referer Check: Verify request came from your domain
  4. Double Submit: Token in cookie AND form field, must match

7. Content Security Policy

Recommended CSP Headers for Forms:
Content-Security-Policy:
  default-src 'self';
  form-action 'self';
  script-src 'self' 'nonce-{random}';
  style-src 'self' 'unsafe-inline';
  img-src 'self' data:;
  • form-action 'self': Prevents form from submitting to external domains
  • script-src 'nonce': Only allows scripts with matching nonce (prevents injected scripts)
  • default-src 'self': Blocks loading resources from unauthorized domains
  • upgrade-insecure-requests: Automatically upgrades HTTP to HTTPS

8. Honeypot Anti-Spam

A hidden field that humans leave empty but bots fill out. Server rejects submissions with filled honeypot.
