12. Accessibility Checklist

  • ✅ Every input has a <label> with matching for/id
  • ✅ Grouped controls use <fieldset> + <legend>
  • ✅ Error messages use aria-describedby
  • ✅ Required fields have aria-required="true"
  • ✅ Invalid fields have aria-invalid="true"
  • ✅ Form is keyboard navigable (Tab, Shift+Tab, Enter, Space)
  • ✅ Color is not the only indicator (text + icons)
  • ✅ Icon-only buttons have aria-label or .visually-hidden text
  • ✅ Dynamic content uses aria-live regions

13. Security Checklist

  • ✅ Use method="POST" for sensitive data
  • ✅ Include CSRF tokens
  • ✅ Set autocomplete correctly (new-password, current-password)
  • ✅ Set spellcheck="false" on sensitive fields
  • ✅ Use inputmode for appropriate mobile keyboards
  • ✅ Validate on server (never trust client-only validation)
  • ✅ Sanitize all inputs to prevent XSS
  • ✅ Use parameterized queries to prevent SQL injection
  • ✅ Set proper CSP headers including form-action

14. Performance Checklist

  • ✅ Debounce input validation (300-500ms delay)
  • ✅ Use event delegation for large forms
  • ✅ Batch DOM updates with DocumentFragment
  • ✅ Lazy-load non-visible form sections
  • ✅ Use requestAnimationFrame for visual updates
  • ✅ Minimize forced reflows during validation
