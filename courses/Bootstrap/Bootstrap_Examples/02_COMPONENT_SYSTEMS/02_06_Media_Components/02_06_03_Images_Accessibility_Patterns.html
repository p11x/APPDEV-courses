Images: Accessibility Patterns

Alt text best practices, ARIA roles, lazy loading, and screen reader support

1. Alt Text Best Practices

Good Alt Text
alt="A golden retriever playing fetch
in a sunny park with green grass"
Bad Alt Text
alt="image"          // Too vague
alt="photo.jpg"      // Filename not helpful
alt=""               // Missing entirely
alt="dog dog dog"    // Keyword stuffing
Alt Text Guidelines
Image TypeAlt Text ApproachExample
Informative image Describe the content alt="Bar chart showing Q3 revenue of $2.5M"
Decorative image Empty alt (alt="") alt="" - Screen readers skip it
Functional image (button/link) Describe the function alt="Search" on a search icon
Text in image Include the text content alt="Sale: 50% off all items"
Complex image (chart) Summary + nearby description alt="Sales trend chart" + detailed text below

2. Decorative vs Informative Images

Decorative Image (skipped by screen readers)
This icon is decorative and provides no additional information. 
<img src="icon.png" alt="" role="presentation">
Informative Image (announced by screen readers)
This icon conveys important meaning. 
<img src="warning.png" alt="Warning icon indicating caution">

3. ARIA Roles for Images

Accessible SVG Image
<svg role="img" aria-labelledby="svgTitle svgDesc">
  <title id="svgTitle">Company Logo</title>
  <desc id="svgDesc">Description here</desc>
</svg>
CSS Background Image with ARIA
<div role="img" aria-label="Mountain landscape at sunset"></div>

4. Accessible Image Links

Image as a Link
<a href="/product/123">
  <img src="headphones.jpg" alt="View product details for Wireless Headphones">
</a>
Icon Button with Label
<button aria-label="Delete item">
  <svg>...</svg>
</button>

5. Lazy Loading Accessibility

Scroll down to see lazy-loaded images appear.

<img src="photo.jpg"
     loading="lazy"
     alt="Descriptive alt text"
     width="800"
     height="400">

/* Always include width/height to prevent layout shift
   which is an accessibility and UX issue */
