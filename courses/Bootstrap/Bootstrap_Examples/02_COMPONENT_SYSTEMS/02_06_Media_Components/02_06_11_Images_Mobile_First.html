Images: Mobile First

Mobile image optimization, touch targets, viewport-aware sizing

1. Mobile-First Responsive Images

Start with mobile size and use srcset to serve larger images on bigger screens.

<!-- Mobile-first: smallest default, progressively enhanced -->
<img src="photo-400.jpg"
     srcset="photo-400.jpg 400w,
             photo-800.jpg 800w,
             photo-1200.jpg 1200w"
     sizes="(max-width: 576px) 100vw,
            (max-width: 992px) 75vw,
            50vw"
     class="img-fluid"
     alt="Responsive image"
     width="1200" height="800"
     loading="lazy">

This image uses mobile-first responsive sizing. On mobile, a 400px image loads. On desktop, the 800px version loads instead.

4. Mobile Performance Tips

Key Considerations
  • Smaller base images: Start at 400px for mobile, not 1200px
  • Lazy load everything below fold: Mobile users scroll more
  • Use loading="lazy": Native browser support
  • Set dimensions: Prevents layout shift on slow connections
  • WebP first: 25-35% savings on mobile data
  • Limit gallery images: Use pagination or virtual scrolling
