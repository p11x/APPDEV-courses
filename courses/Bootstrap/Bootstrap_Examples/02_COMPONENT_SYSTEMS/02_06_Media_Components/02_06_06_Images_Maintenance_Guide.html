Images: Maintenance Guide

Naming conventions, organization, fallbacks, and maintainable patterns

1. Naming Conventions

Recommended File Naming Pattern
/* Pattern: [category]-[descriptor]-[size].[ext] */

hero-background-1920x600.webp
product-widget-thumb-300x300.jpg
team-john-doe-avatar-128x128.webp
icon-arrow-right-24.svg
blog-post-title-image-800x400.webp

/* Folder structure: */
/assets
  /images
    /hero        - Hero/banner images
    /products    - Product photos
    /team        - Team member photos
    /icons       - SVG icons
    /blog        - Blog post images
    /backgrounds - Background patterns

2. Fallback Patterns

Broken Image Placeholder
<img src="product.jpg"
     alt="Product photo"
     onerror="this.onerror=null; this.src='placeholder.jpg';">
Picture Element Fallback Chain
<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <!-- Final fallback always loads if nothing else works -->
  <img src="image.jpg" alt="Description">
</picture>

3. CSS Background vs <img>

Use CaseElementReason
Content image <img> SEO, accessibility, semantic meaning
Decorative background CSS background No alt text needed, purely visual
Hero with text overlay CSS background Responsive sizing, overlay control
Product photo <img> Important content, needs alt text
Pattern/texture CSS background Decorative, repeating, no meaning
