Images: Internationalization

RTL support, locale-specific images, translated alt text, and cultural considerations

1. RTL (Right-to-Left) Image Handling

Some directional images need to be mirrored for RTL layouts.

LTR (Default)

Text flows around the image on the left side in left-to-right layouts.

RTL (Arabic/Hebrew)

Text flows around the image on the right side in right-to-left layouts.

Mirroring Directional Icons
/* CSS for RTL icon mirroring */
[dir="rtl"] .icon-arrow-right {
    transform: scaleX(-1);
}

/* Or use separate images per locale */
<img src="/icons/arrow-{{locale}}.svg" alt="...">

// i18n image pattern with data attributes
<img src="team-meeting.webp"
     data-i18n-alt="images.teamMeeting"
     alt="Team meeting in a conference room">

// JavaScript to update alt text based on locale
document.querySelectorAll('[data-i18n-alt]').forEach(img => {
    const key = img.getAttribute('data-i18n-alt');
    img.alt = i18n.t(key); // Using i18next or similar
});

3. Locale-Specific Images

<picture>
  <source srcset="/images/hero-jp.webp" media="(lang: ja)">
  <source srcset="/images/hero-ar.webp" media="(lang: ar)">
  <source srcset="/images/hero-es.webp" media="(lang: es)">
  <img src="/images/hero-en.webp" alt="Welcome banner">
</picture>

// Or with JavaScript
const locale = document.documentElement.lang;
const heroImg = document.getElementById('hero');
heroImg.src = `/images/hero-${locale}.webp`;
heroImg.alt = translations[locale].heroAlt;
