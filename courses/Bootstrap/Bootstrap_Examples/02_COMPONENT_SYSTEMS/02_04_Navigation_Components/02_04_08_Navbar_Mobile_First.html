Mobile First Navigation Patterns

8.1 Touch Target Sizing

Mobile navigation must meet minimum touch target sizes:

  • Apple HIG: 44x44 points minimum
  • Material Design: 48x48 dp minimum
  • WCAG 2.5.8: 24x24 CSS pixels minimum (Level AA)
/* Mobile touch-friendly sizing */
@media (max-width: 991.98px) {
    .navbar .nav-link {
        padding: 0.75rem 1rem;
        min-height: 44px;         /* Apple minimum */
    }
    .navbar .dropdown-item {
        padding: 0.75rem 1rem;
        min-height: 44px;
    }
    .navbar-toggler {
        min-width: 44px;
        min-height: 44px;
    }
}

8.4 Mobile Navigation Best Practices

PracticeWhy
Use hamburger icon (three lines) Universally recognized mobile menu icon
44px minimum touch targets Prevents mis-taps on finger navigation
Full-width collapsed menu items Easier to tap than narrow centered items
Close menu on outside tap Bootstrap does this automatically
Keep mobile nav shallow (2 levels max) Deep nesting is confusing on small screens
Show current page indicator Users need to know where they are
Use offcanvas for complex menus More space than collapse dropdown
Test on real devices Emulators don't catch all touch issues
