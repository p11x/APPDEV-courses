Navbar Accessibility Patterns

3.1 Skip Link Implementation

Press Tab to reveal the "Skip to main content" link at the top of the page.

  • Skip links must be the first focusable element in the DOM
  • Hidden with position: absolute; top: -100% until focused
  • Target must use tabindex="-1" for programmatic focus
  • Required by WCAG 2.1 SC 2.4.1 (Bypass Blocks - Level A)
<!-- HTML -->
<a href="#main-content" class="skip-link">Skip to main content</a>

/* CSS */
.skip-link { position: absolute; top: -100%; }
.skip-link:focus { top: 0; }

/* Target */
<main id="main-content" tabindex="-1">

3.2 ARIA Attributes Reference

Attribute Element Purpose
aria-label <nav> Provides accessible name when multiple nav elements exist
aria-current="page" <a> Indicates the current page in navigation
aria-expanded <button> Indicates open/closed state of collapsible content
aria-controls <button> References the ID of the controlled element
aria-haspopup <a> Indicates the element opens a popup menu
aria-hidden SVG icons Hides decorative icons from screen readers
.visually-hidden <span> Visually hides content but keeps it accessible to screen readers

3.4 Keyboard Navigation Patterns

Key Action Context
Tab Move focus to next focusable element All navigation elements
Shift+Tab Move focus to previous element All navigation elements
Enter / Space Activate link or toggle dropdown Links, buttons, dropdown toggles
Escape Close dropdown or collapse Open dropdowns, collapse panels
Arrow Down Move to next dropdown item Open dropdown menus
Arrow Up Move to previous dropdown item Open dropdown menus
