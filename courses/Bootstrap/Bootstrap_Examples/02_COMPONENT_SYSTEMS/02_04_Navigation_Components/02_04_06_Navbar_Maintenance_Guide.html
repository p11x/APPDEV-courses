Navbar Maintenance Guide

6.1 Naming Conventions

Use consistent, predictable naming for IDs and classes:

Good Practice
/* Consistent ID naming */
id="mainNav"
id="mainNavCollapse"
id="mainNavDropdown-products"

/* BEM-style custom classes */
.navbar-custom__brand
.navbar-custom__link
.navbar-custom__link--active
.navbar-custom__dropdown-menu
Bad Practice
/* Inconsistent naming */
id="nav1"
id="collapseMenu"
id="dd2"

/* Conflicting class names */
.nav-item-custom
.custom-nav-link
.my-dropdown

6.3 Version Management Checklist

TaskFrequencyNotes
Update Bootstrap CDN version Quarterly Test breaking changes in staging first
Audit deprecated classes Each Bootstrap release Check migration guide
Review custom CSS overrides Monthly Remove unused overrides
Test across target browsers Each deployment Include mobile viewports
Validate ARIA attributes Each change to navbar Use axe-core or Lighthouse
Check link rot Monthly Verify all href values

6.5 Common Refactoring Patterns

Extract Reusable Components:
/* Instead of duplicating navbar markup, use server-side includes or components */

/* NAVBAR.PARTIAL (EJS/Handlebars/Vue/React) */
<%- include('partials/navbar', { 
    brand: 'MyApp',
    links: [
        { text: 'Home', href: '/', active: true },
        { text: 'About', href: '/about' },
        { text: 'Contact', href: '/contact' }
    ],
    theme: 'dark'
}) %>
Centralize Configuration:
// navbar-config.js
const navbarConfig = {
    brand: { text: 'MyApp', href: '/' },
    breakpoint: 'lg',      // navbar-expand-lg
    theme: 'dark',         // navbar-dark bg-dark
    sticky: true,          // sticky-top
    links: [
        { text: 'Home', href: '/', active: true },
        { text: 'Products', href: '/products', dropdown: [
            { text: 'Software', href: '/products/software' },
            { text: 'Hardware', href: '/products/hardware' }
        ]},
        { text: 'About', href: '/about' }
    ]
};
