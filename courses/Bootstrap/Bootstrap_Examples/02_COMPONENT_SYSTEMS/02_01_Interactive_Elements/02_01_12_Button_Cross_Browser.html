2. Feature Detection

// CSS feature detection
if (CSS.supports('gap', '1rem')) {
    // Use flex gap
} else {
    // Use margin fallback
}

// :has() detection
if (CSS.supports('selector(:has(*))')) {
    // Native :has() available
} else {
    // JS fallback
}

3. Focus Style Fallback

Tab through these buttons. Focus ring appears on keyboard navigation only (with fallback). 

/* Modern: only keyboard focus */
.btn:focus-visible { outline: 3px solid blue; }
/* Fallback: remove mouse-click focus */
.btn:focus:not(:focus-visible) { outline: none; }

4. CSS Gap Fallback

/* Margin fallback */
.btn-group-gap > .btn { margin-right: 0.5rem; }
/* Enhanced: use gap when supported */
@supports (gap: 0.5rem) {
    .btn-group-gap { gap: 0.5rem; }
    .btn-group-gap > .btn { margin-right: 0; }
}

5. Appearance Reset

Native <input type="button"> with appearance reset: 

input.btn-appearance-none {
    -webkit-appearance: none;
    -moz-appearance: none;
    appearance: none;
    /* Then restyle freely */
}

6. Backdrop Filter Fallback

Glass buttons with backdrop-filter fallback: 

.btn-blur-backdrop {
    background: rgba(13, 110, 253, 0.8);
    backdrop-filter: blur(10px);
    -webkit-backdrop-filter: blur(10px);
}
@supports not (backdrop-filter: blur(10px)) {
    .btn-blur-backdrop { background: #0d6efd; }
}

7. :has() Selector Fallback

// Detect :has() support
if (CSS.supports('selector(:has(*))')) {
    // Native CSS :has() works
} else {
    // JS fallback: add class when child is active
    parent.classList.toggle('has-active-child',
        parent.querySelector('.active') !== null
    );
}
