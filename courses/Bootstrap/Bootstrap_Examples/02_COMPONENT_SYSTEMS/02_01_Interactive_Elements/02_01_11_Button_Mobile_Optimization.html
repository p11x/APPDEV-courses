Button Mobile Optimization

Touch targets, mobile UX patterns, tap feedback, and responsive button sizing.

1. Touch Target Sizes

WCAG 2.5.8 requires minimum 24x24px targets. Best practice is 44x44px (Apple) or 48x48px (Material).

Minimum (44x44px) - WCAG Recommended
Comfortable (48x48px) - Material Design
Large (56x56px) - Primary Actions

2. Tap Feedback Effects

Provide immediate visual feedback on tap. Use -webkit-tap-highlight-color: transparent to remove default browser flash.

5. Sticky Action Bar

Fixed bottom bar pattern for mobile forms. Respects safe area insets on notched devices. 

.mobile-action-bar {
    position: fixed;
    bottom: 0;
    padding-bottom: max(12px, env(safe-area-inset-bottom));
}

.btn-safe-area {
    padding-left: max(16px, env(safe-area-inset-left));
    padding-right: max(16px, env(safe-area-inset-right));
}

7. Touch Target Spacing

WCAG 2.5.8: Ensure 8px spacing between touch targets to prevent accidental activation.

Good: 8px+ gap
Bad: No gap (hard to tap accurately)
