Button Testing Strategies

Unit tests, visual regression tests, integration tests, and accessibility audits.

1. Testable Button Patterns (data-testid)

Use data-testid attributes for stable test selectors that don't break when classes or structure change. 

// Test selector pattern (Jest/Cypress/Playwright)
// Preferred: data-testid (stable, semantic)
cy.get('[data-testid="btn-submit"]').click();
document.querySelector('[data-testid="btn-submit"]');

// Avoid: class-based selectors (break easily)
cy.get('.btn.btn-primary').click(); // Fragile!

2. Unit Test Examples (Runnable)

// Jest-style unit test examples
describe('Counter Button', () => {
  test('starts at 0', () => {
    expect(getCounterValue()).toBe(0);
  });

  test('increments on click', () => {
    clickButton('#counterBtn');
    expect(getCounterValue()).toBe(1);
  });

  test('resets to 0', () => {
    clickButton('#counterBtn');
    clickButton('#resetBtn');
    expect(getCounterValue()).toBe(0);
  });

  test('is disabled during processing', () => {
    startProcessing();
    expect(isDisabled('#counterBtn')).toBe(true);
  });
});
