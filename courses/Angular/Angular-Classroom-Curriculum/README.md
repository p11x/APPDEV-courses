# Angular Classroom Curriculum

Welcome to the comprehensive Angular learning curriculum designed for classroom instruction. This repository contains a complete teaching repository taking students from zero Angular knowledge to building professional-grade Single Page Applications.

## Curriculum Structure

| Level | Weeks | Focus Area |
|-------|-------|------------|
| [01_Beginner_Fundamentals](./01_Beginner_Fundamentals/) | 1-4 | Core concepts, components, directives, pipes |
| [02_Intermediate_Development](./02_Intermediate_Development/) | 5-9 | Services, routing, HTTP, forms, RxJS |
| [03_Advanced_Professional](./03_Advanced_Professional/) | 10-14 | State management, testing, performance, deployment |
| [04_Capstone_Project](./04_Capstone_Project/) | 15-16 | Full-stack application development |

## Prerequisites

Before starting this curriculum, ensure you have:
- Basic knowledge of HTML, CSS, and JavaScript
- Understanding of object-oriented programming concepts
- Familiarity with web development concepts (HTTP, REST APIs)
- Node.js 18+ installed on your machine
- A modern code editor (VS Code recommended)

## Technology Stack

- **Framework**: Angular 17+ (Standalone Components, Signals, Control Flow)
- **Language**: TypeScript 5.x
- **Testing**: Jasmine/Karma, Jest
- **UI Library**: Angular Material
- **State Management**: NgRx (optional for advanced)
- **Build Tool**: Angular CLI with esbuild

## How to Use This Curriculum

1. **Lecture Mode**: Follow the conceptual explanations and code examples for classroom teaching
2. **Lab Mode**: Complete hands-on exercises and mini-projects at the end of each topic
3. **Self-Study**: Use the suggested reading materials and best practices for independent learning

## Course Duration

- **Total**: 16 weeks (48-60 lecture hours + 48-60 lab hours)
- **Weekly Structure**: 
  - 2-3 lecture sessions (1.5 hours each)
  - 2-3 lab sessions (1.5 hours each)

## Angular Version

This curriculum uses **Angular 17+** with modern features:
- Standalone Components (default)
- Signals for reactive state
- Control flow syntax (@if, @for, @switch)
- Lazy loading routes
- Functional guards and interceptors
