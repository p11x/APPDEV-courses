# GitOps Principles

## Overview

GitOps applies DevOps best practices to cluster management. This guide covers the four core principles.

## Core Concepts

### Four Principles

1. **Declarative**: All infrastructure as code
2. **Versioned**: Immutable, versioned in Git
3. **Automated**: Pull-based reconciliation
4. **Continuous**: Continuous reconciliation

## Quick Reference

| Principle | Description |
|-----------|-------------|
| Declarative | Code defines desired state |
| Versioned | Git as source of truth |
| Automated | Auto-sync changes |
| Continuous | Always converging |
