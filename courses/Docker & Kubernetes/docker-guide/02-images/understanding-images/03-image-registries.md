# Image Registries

## Overview

An image registry is a storage and distribution system for Docker images. Registries are the "app stores" of the container world, providing a central place to store, share, and manage container images. Understanding registries is essential for deploying applications in production and collaborating with teams.

## Prerequisites

- Understanding of Docker images (previous sections)
- Basic command-line knowledge
- Familiarity with pulling and pushing images

## Core Concepts

### What is a Registry?

A Docker registry is a service that stores container images. When you run `docker pull nginx`, you're pulling the nginx image from Docker Hub, which is Docker's default public registry. Registries organize images into repositories and provide versioning through tags.

A registry consists of:
- **Repository**: A named collection of related images (e.g., nginx/nginx-ingress)
- **Tag**: A specific version of an image (e.g., 1.25.0, latest, alpine)
- **Manifest**: JSON file describing the image layers and configuration
- **API**: HTTP endpoints for pushing, pulling, and managing images

### Types of Registries

**Public Registries** - Anyone can pull images:
- Docker Hub (hub.docker.com) - Default and largest public registry
- GitHub Container Registry (ghcr.io) - Integrated with GitHub
- Quay (quay.io) - Red Hat's registry with security features

**Private Registries** - Access controlled:
- Docker Registry (self-hosted)
- Amazon Elastic Container Registry (ECR)
- Google Container Registry (GCR) / Artifact Registry
- Azure Container Registry (ACR)
- Harbor (open-source, enterprise-grade)

### Authentication

Most private registries require authentication. Docker supports several authentication methods:

- **Username/Password**: Basic authentication using docker login
- **Token**: Generated by CI/CD systems for automation
- **IAM/RBAC**: Cloud provider identity integration (AWS IAM, GCP IAM, Azure AD)

### Registry URL Structure

Registry URLs follow a specific format:

```
registry.example.com:port/organization/image-name:tag
```

Examples:
- `docker.io/library/nginx:latest` - Docker Hub library image
- `docker.io/myuser/myapp:v1.0` - Docker Hub user repository
- `123456789012.dkr.ecr.us-east-1.amazonaws.com/myapp:latest` - AWS ECR
- `gcr.io/my-project/my-image:latest` - Google Container Registry

## Step-by-Step Examples

### Working with Docker Hub

```bash
# Search for images on Docker Hub
# This queries the Docker Hub registry
docker search nginx
# Shows: NAME, DESCRIPTION, STARS, OFFICIAL, AUTOMATED

# Pull an image from Docker Hub
docker pull nginx:1.25-alpine

# Pull a specific architecture variant
docker pull --platform linux/arm64 nginx:1.25-alpine

# Push to Docker Hub (requires account)
# First, tag for your namespace
docker tag nginx:1.25-alpine your-username/nginx:1.25-alpine

# Login to Docker Hub
docker login -u your-username
# Enter password when prompted

# Push your image
docker push your-username/nginx:1.25-alpine
```

### Using Private Registries

```bash
# Login to a private registry
docker login myregistry.example.com -u username
# Enter password or token

# Pull from private registry
docker pull myregistry.example.com/myapp:v1.0

# Tag for your private registry
docker tag myapp:latest myregistry.example.com/myapp:latest

# Push to private registry
docker push myregistry.example.com/myapp:latest

# Logout when done
docker logout myregistry.example.com
```

### Setting Up Local Registry

For testing or CI/CD, run a local registry:

```bash
# Start a local registry container
# -d = detached mode (background)
# -p 5000:5000 = map host port 5000 to container port 5000
# --name registry = friendly name for the container
# registry:2 = image to run (Docker Registry version 2)
docker run -d \
  --name registry \
  -p 5000:5000 \
  --restart=always \
  registry:2

# Push to local registry
docker pull alpine:latest
docker tag alpine:latest localhost:5000/alpine:latest
docker push localhost:5000/alpine:latest

# Pull from local registry
docker pull localhost:5000/alpine:latest

# List images in local registry
curl http://localhost:5000/v2/_catalog
# Returns: {"repositories":["alpine"]}

# Check tags for an image
curl http://localhost:5000/v2/alpine/tags/list
# Returns: {"name":"alpine","tags":["latest"]}
```

### Using Amazon ECR

```bash
# Install AWS CLI (if not present)
# aws --version

# Login to ECR (get login password)
aws ecr get-login-password --region us-east-1 | \
  docker login --username AWS --password-stdin 123456789012.dkr.ecr.us-east-1.amazonaws.com

# Pull an image
docker pull 123456789012.dkr.ecr.us-east-1.amazonaws.com/myapp:latest

# Push an image
docker tag myapp:latest 123456789012.dkr.ecr.us-east-1.amazonaws.com/myapp:latest
docker push 123456789012.dkr.ecr.us-east-1.amazonaws.com/myapp:latest
```

### Using Google Container Registry

```bash
# Configure Docker to use gcloud credentials
gcloud auth configure-docker

# Or use GCP service account
gcloud auth activate-service-account --key-file=/path/to/key.json
gcloud auth configure-docker

# Push images
docker tag myapp gcr.io/my-project/myapp:latest
docker push gcr.io/my-project/myapp:latest

# Pull images
docker pull gcr.io/my-project/myapp:latest
```

## Common Mistakes

- **Not using specific tags**: Always specify tags in production. Using :latest can lead to unpredictable deployments.
- **Leaving credentials in images**: Never bake credentials into images. Use environment variables or secrets at runtime.
- **Ignoring image size**: Large images are slower to pull and deploy. Use Alpine-based or slim variants.
- **Not scanning for vulnerabilities**: Public images may contain vulnerabilities. Use docker scout or other scanning tools.
- **Forgetting to logout**: Leaving authenticated sessions can be a security risk, especially on shared machines.
- **Assuming public images are safe**: Even official images can have vulnerabilities. Always verify and scan.

## Quick Reference

| Registry | URL Format | Use Case |
|----------|------------|----------|
| Docker Hub | docker.io/username/image:tag | Public images, OSS projects |
| AWS ECR | account.dkr.ecr.region.amazonaws.com | AWS deployments |
| GCR | gcr.io/project/image | GCP deployments |
| ACR | registry.azurecr.io | Azure deployments |
| Local | localhost:5000 | Testing, CI/CD |

| Command | Description |
|---------|-------------|
| `docker login` | Authenticate to a registry |
| `docker logout` | Remove authentication |
| `docker search` | Search for images |
| `docker pull` | Download an image |
| `docker push` | Upload an image |
