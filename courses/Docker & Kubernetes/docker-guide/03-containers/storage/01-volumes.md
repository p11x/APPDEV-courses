# Volumes

## Overview

Volumes are the preferred mechanism for persisting data generated by Docker containers. Unlike container filesystem data, volumes exist independently of container lifecycles, making them essential for databases, stateful applications, and any data that must survive container restarts and recreation. Volumes are stored on the host filesystem and can be shared between containers.

## Prerequisites

- Understanding of container lifecycle
- Basic command-line knowledge
- Understanding of filesystem concepts

## Core Concepts

### What are Volumes?

Volumes are directories stored outside the container's writable layer:

- Persist after container stops/removed
- Can be shared between containers
- Managed by Docker (not in container filesystem)
- Independent of container lifecycle

### Types of Volumes

1. **Named volumes**: Docker-managed, with a friendly name
2. **Anonymous volumes**: Temporary, for single containers
3. **Host volumes (bind mounts)**: Map host directories

### Volume Lifecycle

- Created when first used by a container
- Persists until explicitly deleted
- Can be backed up, moved, or shared

## Step-by-Step Examples

### Creating and Using Named Volumes

```bash
# Create a named volume
# docker volume create makes a managed volume
docker volume create my-data

# Run container with volume
# -v volumeName:containerPath mounts volume
docker run -d -v my-data:/app/data nginx

# Check volume exists
docker volume ls

# Inspect volume
docker volume inspect my-data
```

### Using Volumes with Databases

```bash
# PostgreSQL with persistent data
docker run -d \
  --name postgres \
  -e POSTGRES_PASSWORD=mysecretpassword \
  -v postgres-data:/var/lib/postgresql/data \
  postgres:15-alpine

# Data persists after container stop
docker stop postgres
docker rm postgres

# Recreate with same volume - data is intact
docker run -d \
  --name postgres \
  -e POSTGRES_PASSWORD=mysecretpassword \
  -v postgres-data:/var/lib/postgresql/data \
  postgres:15-alpine
```

### Sharing Volumes Between Containers

```bash
# Create volume
docker volume create shared-data

# First container writes to volume
docker run -d -v shared-data:/data --name writer alpine sh -c 'echo "Hello" > /data/file.txt'

# Second container reads from same volume
docker run -d -v shared-data:/data --name reader alpine cat /data/file.txt

# Both containers share the same data
```

### Backup and Restore

```bash
# Backup volume
docker run --rm -v my-data:/data -v $(pwd):/backup alpine tar czf /backup/backup.tar.gz -C /data .

# Restore volume
docker volume create restored-data
docker run --rm -v restored-data:/data -v $(pwd):/backup alpine tar xzf /backup/backup.tar.gz -C /data
```

### Volume Drivers

```bash
# Create volume with specific driver
docker volume create \
  --driver local \
  --opt type=none \
  --opt o=bind \
  --opt device=/path/on/host \
  my-bind-volume
```

### Docker Compose with Volumes

```yaml
# docker-compose.yml
version: "3.8"

services:
  db:
    image: postgres:15
    volumes:
      - db-data:/var/lib/postgresql/data
    environment:
      POSTGRES_PASSWORD: secret

  redis:
    image: redis:alpine
    volumes:
      - redis-data:/data

volumes:
  db-data:
  redis-data:
```

## Common Mistakes

- **Not using volumes for persistence**: Data in container filesystem is lost on recreate.
- **Forgetting volume cleanup**: Unused volumes consume disk space.
- **Permission issues**: Container user may not have write permission.
- **Mixing up volume types**: Don't confuse volumes with bind mounts.
- **Not backing up**: Important data should be backed up.

## Quick Reference

| Command | Description |
|---------|-------------|
| `docker volume create` | Create named volume |
| `docker volume ls` | List volumes |
| `docker volume inspect` | View details |
| `docker volume rm` | Remove volume |
| `docker volume prune` | Remove unused |
| `-v name:/path` | Mount volume |

| Volume Type | Use Case |
|-------------|----------|
| Named volumes | Persistent application data |
| Anonymous volumes | Temporary data |
| Bind mounts | Host file access |
