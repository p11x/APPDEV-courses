# Association Rules Mining

## Introduction

Association Rules Mining represents a fundamental unsupervised learning technique for discovering interesting relationships between variables in large datasets. Originally developed for market basket analysis, the method identifies frequent patterns, correlations, and associations among sets of items. The classic example is the discovery that customers who buy diapers often also buy beer—a counterintuitive finding that revolutionized retail analytics.

The algorithm operates by analyzing transaction data to find items that frequently co-occur. It outputs rules in the form "if X then Y" with metrics quantifying the rule's strength. Key metrics include support (how frequently the items appear together), confidence (how often the rule is true), and lift (how much the items are associated compared to random chance). These metrics help identify actionable patterns worth pursuing.

The business value of association rules extends across industries. In banking, it identifies product bundles that customers commonly use together, enabling cross-selling strategies. It discovers unusual transaction patterns that may indicate fraudulent activity. In healthcare, it finds patterns in patient diagnoses, treatments, and outcomes that inform clinical decision-making. The algorithm also supports genomic analysis, web usage mining, and network intrusion detection.

## Fundamentals

### Frequent Pattern Mining

The foundation of association rules mining is finding itemsets that appear frequently in the data. An itemset is a collection of one or more items, and its support is the proportion of transactions containing that itemset. A minimum support threshold filters out rare itemsets, focusing analysis on patterns that occur often enough to be actionable.

The Apriori algorithm efficiently finds frequent itemsets through a level-wise approach. It starts with single items, identifies those meeting minimum support, then builds larger itemsets only from those frequent smaller items. This "anti-monotonic" property—any superset of an infrequent itemset must also be infrequent—dramatically reduces the search space.

The algorithm generates candidates of size k from frequent itemsets of size k-1. Each candidate is checked against the database for support. This process continues until no new frequent itemsets are found. The computational complexity depends on data size and pattern density, but the Apriori property ensures efficiency.

### Rule Generation and Metrics

Once frequent itemsets are identified, association rules are generated by splitting each itemset into antecedent and consequent parts. A rule "A → B" means that if A is in a transaction, then B is also likely to be in that transaction. The rule's strength is measured by several metrics.

Support measures how frequently the rule applies in the database: support(A → B) = transactions containing both A and B / total transactions. This identifies rules that are broadly applicable. Confidence measures the conditional probability: confidence(A → B) = support(A,B) / support(A). This identifies rules that are reliable when triggered. Lift measures the association strength: lift(A → B) = confidence(A → B) / support(B). Lift greater than 1 indicates positive association; less than 1 indicates negative association.

Additional metrics include leverage (difference from expected co-occurrence under independence), conviction (inverse of the proportion of times the rule makes an error), and cosine (geometric mean of supports). Each metric emphasizes different aspects of rule quality.

### FP-Growth Algorithm

The FP-Growth algorithm provides an efficient alternative to Apriori that avoids candidate generation. It builds a compressed data structure (FP-tree) that stores transaction information, then extracts frequent itemsets directly from this tree through recursive projection. This approach is typically faster than Apriori for dense datasets.

The algorithm first scans the database to identify frequent items, sorted by frequency. Transactions are compressed into a prefix tree where common prefixes are shared. The tree captures all transaction information while dramatically reducing storage requirements. Frequent patterns are extracted by mining this tree bottom-up.

FP-Growth's advantage grows with data density. For sparse transaction data, Apriori may still be competitive. For dense data with many frequent itemsets, FP-Growth provides substantial speedups. The algorithm is particularly effective when the same items appear frequently across transactions.

## Implementation with Scikit-Learn

### Basic Association Rules Implementation

```python
import numpy as np
import pandas as pd
from sklearn.datasets import load_iris
from mlxtend.preprocessing import TransactionEncoder
from mlxtend.frequent_patterns import apriori, association_rules
import warnings
warnings.filterwarnings('ignore')

print("=" * 70)
print("ASSOCIATION RULES MINING - BASIC IMPLEMENTATION")
print("=" * 70)

np.random.seed(42)
n_transactions = 1000

transactions = []
items_pool = ['milk', 'bread', 'butter', 'eggs', 'cheese', 
              'yogurt', 'cream', 'jam', 'cereal', 'coffee',
              'tea', 'juice', 'fruit', 'vegetables', 'meat',
              'fish', 'chicken', 'rice', 'pasta', 'soup']

for _ in range(n_transactions):
    n_items = np.random.poisson(6)
    transaction = np.random.choice(items_pool, size=n_items, replace=False)
    transactions.append(list(transaction))

print(f"\nTransaction Dataset")
print(f"Number of transactions: {n_transactions}")
print(f"Sample transactions:")
for i in range(3):
    print(f"  {i+1}: {transactions[i]}")

te = TransactionEncoder()
te_array = te.fit_transform(transactions)
df = pd.DataFrame(te_array, columns=te.columns_)

frequent_itemsets = apriori(df, min_support=0.05, use_colnames=True)

print(f"\n{'='*50}")
print("FREQUENT ITEMSETS")
print(f"{'='*50}")
print(f"Number of frequent itemsets: {len(frequent_itemsets)}")
print(f"\nTop itemsets by support:")
frequent_itemsets_sorted = frequent_itemsets.sort_values('support', ascending=False)
print(frequent_itemsets_sorted.head(10).to_string(index=False))

rules = association_rules(frequent_itemsets, metric="confidence", min_threshold=0.5)

print(f"\n{'='*50}")
print("ASSOCIATION RULES")
print(f"{'='*50}")
print(f"Number of rules: {len(rules)}")

rules_sorted = rules.sort_values('lift', ascending=False)
print(f"\nTop rules by lift:")
for idx, row in rules_sorted.head(5).iterrows():
    antecedent = ', '.join(list(row['antecedents']))
    consequent = ', '.join(list(row['consequents']))
    print(f"  {antecedent} → {consequent}")
    print(f"    Support: {row['support']:.3f}, Confidence: {row['confidence']:.3f}, Lift: {row['lift']:.3f}")
```

### Healthcare Application: Diagnosis Patterns

```python
print("=" * 70)
print("HEALTHCARE APPLICATION - DIAGNOSIS PATTERNS")
print("=" * 70)

np.random.seed(42)
n_patients = 1500

symptoms = [
    'fever', 'cough', 'fatigue', 'headache', 'nausea',
    'chest_pain', 'shortness_breath', 'abdominal_pain',
    'joint_pain', 'rash', 'dizziness', 'sore_throat'
]

diagnoses = [
    'viral_infection', 'bacterial_infection', 'allergy',
    'migraine', 'gastroenteritis', 'respiratory_infection',
    'arthritis', 'hypertension', 'diabetes'
]

patient_conditions = []
for _ in range(n_patients):
    conditions = []
    
    if np.random.random() < 0.45:
        conditions.append('fever')
    if np.random.random() < 0.35:
        conditions.append('cough')
    if np.random.random() < 0.40:
        conditions.append('fatigue')
    
    if np.random.random() < 0.25:
        conditions.append('headache')
    if conditions and np.random.random() < 0.30:
        conditions.append('nausea')
    
    if np.random.random() < 0.20:
        conditions.append('chest_pain')
    if np.random.random() < 0.18:
        conditions.append('shortness_breath')
    
    viral_pattern = conditions.copy()
    if 'fever' in viral_pattern and 'cough' in viral_pattern:
        if np.random.random() < 0.65:
            conditions.append('viral_infection')
    
    bacterial_pattern = conditions.copy()
    if 'fever' in bacterial_pattern and 'sore_throat' in bacterial_pattern:
        conditions.append('sore_throat')
    if 'fever' in bacterial_pattern:
        if np.random.random() < 0.35:
            conditions.append('bacterial_infection')
    
    if 'headache' in conditions and 'nausea' not in conditions:
        if np.random.random() < 0.40:
            conditions.append('migraine')
    
    if 'abdominal_pain' in conditions or 'nausea' in conditions:
        if np.random.random() < 0.30:
            conditions.append('gastroenteritis')
    
    if 'joint_pain' in conditions and 'rash' not in conditions:
        if np.random.random() < 0.25:
            conditions.append('arthritis')
    
    patient_conditions.append(conditions)

print(f"\nPatient Diagnosis Dataset")
print(f"Number of patients: {n_patients}")
print(f"Symptoms tracked: {len(symptoms)}")

te = TransactionEncoder()
te_array = te.fit_transform(patient_conditions)
df = pd.DataFrame(te_array, columns=te.columns_)

frequent_itemsets = apriori(df, min_support=0.06, use_colnames=True)

print(f"\n{'='*50}")
print("SYMPTOM COMBINATIONS")
print(f"{'='*50}")
frequent_sorted = frequent_itemsets.sort_values('support', ascending=False)
print(frequent_sorted.head(10).to_string(index=False))

rules = association_rules(frequent_itemsets, metric="confidence", min_threshold=0.4)

print(f"\n{'='*50}")
print("DIAGNOSIS ASSOCIATION RULES")
print(f"{'='*50}")

symptom_rules = rules[rules['consequents'].apply(lambda x: 'infection' in str(x) or 'migraine' in str(x) or 'arthritis' in str(x))]
symptom_rules = symptom_rules.sort_values('confidence', ascending=False)

for idx, row in symptom_rules.head(8).iterrows():
    ant = ', '.join(sorted(list(row['antecedents']), key=lambda x: (not 'pain' in str(x), not 'fever' in str(x), str(x))))
    cons = ', '.join(sorted(list(row['consequents'])))
    print(f"\n{ant} → {cons}")
    print(f"  Support: {row['support']:.3f}, Confidence: {row['confidence']:.3f}, Lift: {row['lift']:.2f}")
```

## Applications

### Banking Applications

Product affinity analysis identifies which banking products customers commonly hold together. This enables cross-selling strategies that recommend products based on existing holdings. High-lift rules indicate strong associations worth promoting.

Service usage patterns discover how customers use banking channels. Understanding which services cluster together informs product development and marketing. Digital channel adoption patterns guide technology investments.

Fraud pattern detection uses association rules to identify unusual transaction combinations that may indicate fraud. Rules with unexpected item combinations can flag suspicious activity for investigation.

### Healthcare Applications

Symptom pattern analysis identifies frequently co-occurring symptoms that may indicate specific conditions. This supports differential diagnosis by highlighting symptom combinations associated with particular diagnoses.

Treatment pattern mining discovers which treatments are commonly used together. This informs clinical protocol development and identifies potential drug interactions or synergies.

Comorbidity analysis identifies diseases that frequently occur together in patients. This supports population health management and informs care coordination for patients with multiple conditions.

### Interestingness Metrics

```python
print("=" * 70)
print("MULTIPLE METRIC ANALYSIS")
print("=" * 70)

rules_analysis = rules.copy()
rules_analysis['leverage'] = rules_analysis['support'] - \
    rules_analysis['antecedent support'] * rules_analysis['consequent support']
rules_analysis['conviction'] = (1 - rules_analysis['consequent support']) / \
    (1 - rules_analysis['confidence'])

rules_sorted = rules_analysis.sort_values('conviction', ascending=False)
```

## Conclusion

Association Rules Mining provides valuable insight into item co-occurrence patterns across diverse domains. The algorithm's ability to discover surprising relationships that might not be apparent through traditional analysis makes it particularly valuable for exploratory analysis and hypothesis generation.

Key considerations include appropriate support threshold selection (lower values find more patterns but increase computational cost), understanding that rules with high lift may have limited practical application due to low support, and the need for domain expertise to interpret discovered patterns meaningfully.

For banking applications, association rules enable product affinity analysis and cross-selling strategies. For healthcare, they support symptom pattern analysis and diagnosis association. The algorithm's business value in market basket analysis and beyond demonstrates its fundamental importance in data mining.
