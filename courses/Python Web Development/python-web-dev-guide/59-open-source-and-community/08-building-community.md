# Building Community

## What You'll Learn

- Creating welcoming environments
- Organizing meetups and events
- Mentoring contributors

## Prerequisites

- Completed `07-maintaining-open-source-projects.md`

## Creating Welcoming Spaces

A welcoming community attracts more contributors:

1. **Clear documentation** - Easy to get started
2. **Friendly tone** - Be kind in all communications
3. **Helpful responses** - Answer questions patiently
4. **Recognize contributions** - Credit contributors

## Code of Conduct

Every project should have a code of conduct:

```markdown
# Code of Conduct

## Our Pledge

We pledge to make participation in our community a harassment-free experience.

## Our Standards

- Be respectful to others
- Use welcoming language
- Accept constructive criticism gracefully

## Enforcement

Instances of abusive behavior may be reported to [contact email].
```

## Mentoring

Ways to mentor new contributors:

- Write good first issues
- Guide through their first PR
- Answer questions in forums
- Review their code patiently

## Summary

- Welcoming communities grow stronger
- Set clear expectations with code of conduct
- Mentor new contributors
