# Code Review Process

## What You'll Learn

- How code review works
- Giving and receiving feedback
- Handling review comments

## Prerequisites

- Completed `05-understanding-contribution-guidelines.md`

## What to Expect

Code review is where maintainers examine your code before merging:

1. **Automated checks** - CI runs tests, linting
2. **Maintainer review** - Someone reads your code
3. **Feedback** - You may need to make changes
4. **Approval** - Changes are approved
5. **Merge** - Code is merged

## Giving Good Feedback

When reviewing others' code:

- Be constructive, not critical
- Ask questions instead of making demands
- Suggest improvements, don't just point out problems
- Praise good solutions

Example feedback:

```
Good approach here! One suggestion: 
consider using a dict comprehension instead of a loop 
for better performance.
```

## Receiving Feedback

When your code is reviewed:

- Don't take feedback personally
- Ask clarifying questions
- Make requested changes
- Thank reviewers for their time

## Handling Conflicts

If you disagree with feedback:

1. Explain your reasoning
2. Provide evidence
3. Be willing to compromise
4. Accept final decisions

## Summary

- Code review is collaborative
- Give constructive feedback
- Accept feedback graciously
