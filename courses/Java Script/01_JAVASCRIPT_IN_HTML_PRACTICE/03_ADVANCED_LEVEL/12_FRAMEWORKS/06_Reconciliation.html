06 - Reconciliation

What is Reconciliation?

Reconciliation is the algorithm React uses to diff one tree with another to determine which parts need to be changed. It's the process of comparing the Virtual DOM and determining the minimum number of operations to update the real DOM.

Diffing Algorithm

// React's reconciliation rules:
1. Different types → Different components → Recreate
2. Same type → Update props, keep component same
3. Keys → Help identify stable elements

// Example: List reordering
const oldChildren = [
    { key: 'a', value: 'Apple' },
    { key: 'b', value: 'Banana' }
];

const newChildren = [
    { key: 'b', value: 'Banana' },
    { key: 'a', value: 'Apple' }
];
// Without keys: both items replaced
// With keys: just reordered (efficient!)
