05 - Virtual DOM

What is Virtual DOM?

The Virtual DOM is a lightweight JavaScript representation of the real DOM. It's a virtual representation of the UI kept in memory and synced with the real DOM through a process called reconciliation.

Virtual DOM vs Real DOM

// Real DOM - Direct manipulation (slow)
document.getElementById('app').innerHTML = '
New Content
';

// Virtual DOM - Lightweight JS objects (fast)
const vNode = {
    type: 'div',
    props: { children: ['New Content'] }
};

// React uses virtual DOM for efficient updates

How Virtual DOM Works

class VirtualNode {
    constructor(type, props, children) {
        this.type = type;
        this.props = props;
        this.children = children;
    }
}

// Step 1: Create virtual DOM tree
const vDOM = new VirtualNode('div', { class: 'container' }, [
    new VirtualNode('h1', {}, ['Title']),
    new VirtualNode('p', {}, ['Description'])
]);

// Step 2: Render to real DOM
function render(vNode) {
    const element = document.createElement(vNode.type);
    if (vNode.props) {
        Object.assign(element, vNode.props);
    }
    if (vNode.children) {
        vNode.children.forEach(child => {
            if (typeof child === 'string') {
                element.appendChild(document.createTextNode(child));
            } else {
                element.appendChild(render(child));
            }
        });
    }
    return element;
}
