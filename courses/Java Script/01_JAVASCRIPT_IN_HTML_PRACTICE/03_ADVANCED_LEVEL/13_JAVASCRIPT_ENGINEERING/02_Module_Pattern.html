JavaScript Module Patterns

1. IIFE (Immediately Invoked Function Expression)

A function that executes immediately upon definition, creating private scope.

// IIFE Pattern - creates private scope
const Counter = (() => {
    let count = 0; // private variable
    
    return {
        increment() { return ++count; },
        decrement() { return --count; },
        getCount() { return count; }
    };
})(); // Immediately invoked

console.log(Counter.increment()); // 1
console.log(Counter.increment()); // 2
console.log(Counter.getCount()); // 2
// count is private - cannot access directly

2. Revealing Module Pattern

IIFE variant that returns an object revealing only certain methods.

// Revealing Module Pattern
const UserManager = (() => {
    const users = [];
    
    function validateEmail(email) {
        return email.includes('@');
    }
    
    return {
        addUser(name, email) {
            if (!validateEmail(email)) {
                return { success: false, message: 'Invalid email' };
            }
            users.push({ name, email });
            return { success: true, message: 'User added' };
        },
        getUsers() {
            return users.map(u => u.name);
        }
    };
})();

3. ES Modules Pattern

Native JavaScript modules using import/export syntax.

// ES Module syntax (requires separate files with type="module")
// file: math.js
// export function add(a, b) { return a + b; }

// file: main.js
// import { add } from './math.js';
// console.log(add(2, 3)); // 5

// Inline demo simulating ES modules
const MathModule = (() => {
    const add = (a, b) => a + b;
    const subtract = (a, b) => a - b;
    const multiply = (a, b) => a * b;
    
    return { add, subtract, multiply };
})();
