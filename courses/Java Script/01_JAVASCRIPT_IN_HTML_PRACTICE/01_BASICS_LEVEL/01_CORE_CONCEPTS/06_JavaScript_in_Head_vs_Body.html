JavaScript in Head vs Body

Understanding where to place JavaScript affects when it runs and what it can access.

JavaScript in <head>

Script placed in the <head> section

Executes BEFORE the body content is parsed and rendered.

  • Executes early in page load
  • Cannot access body elements yet (they don't exist)
  • May block page rendering
  • Use window.onload or defer for DOM access

JavaScript in <body>

Script placed at the end of <body>

Executes AFTER the body content has been parsed.

  • Can access all DOM elements
  • Doesn't block page rendering
  • Page content loads first (better user experience)
  • Recommended for most cases

Page Load Timeline:

1. Browser starts parsing HTML
2. Executes scripts in <head> ⚠️
3. Parses <body> content
4. Executes scripts at end of <body> ✓
5. Page fully loaded
Best Practice: Place scripts at the end of the body tag for better performance and easier DOM access.
