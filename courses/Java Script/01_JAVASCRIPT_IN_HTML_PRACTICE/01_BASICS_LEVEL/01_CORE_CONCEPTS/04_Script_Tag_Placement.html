Script Tag Placement

Where and how you place the <script> tag affects page loading and execution.

1. Inline Script (in body)

Placing script inside <body>

Scripts are executed in order as the page parses. Best for small scripts that need to run after HTML is parsed.

<script>console.log("Inline script"); </script>

2. External Script (separate file)

Linking to an external .js file

Better for reusability and caching. Browser can cache the JS file.

<script src="script.js"></script>

3. Deferred Script (defers execution)

Using defer attribute

Script downloads in parallel but executes AFTER HTML parsing is complete, in order.

<script defer src="script.js"></script>

4. Async Script (non-blocking)

Using async attribute

Script downloads in parallel and executes as soon as ready. Order not guaranteed!

<script async src="script.js"></script>
