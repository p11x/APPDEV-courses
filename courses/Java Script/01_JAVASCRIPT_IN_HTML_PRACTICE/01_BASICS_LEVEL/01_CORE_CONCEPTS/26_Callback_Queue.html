Callback Queue (Task Queue)

The callback queue (also called task queue or macrotask queue) stores callbacks waiting to be executed.

How It Works:

1. Queue - Callbacks are added as events complete
2. Wait - Event loop checks if stack is empty
3. Execute - Move first callback to call stack
