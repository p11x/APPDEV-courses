Immutability

Immutability means data cannot be changed after it's created. Instead, new data is created.

Immutability Concepts

// Don't modify original, create new
var newArray = [...oldArray, newItem];
var newObject = { ...oldObject, newProp: value };

Examples

Example 1: Mutability Problem

var arr = [1, 2, 3];
var arrCopy = arr;

arr.push(4);
document.getElementById("output1").innerHTML = 
    "arr: " + arr.join(", ") + "<br>" +
    "arrCopy: " + arrCopy.join(", ") + "<br>" +
    "(Both changed - mutability issue!)";

Example 2: const Doesn't Make Objects Immutable

const person = { name: "John", age: 25 };
person.age = 30; // This works!
person.city = "NYC"; // This also works!

document.getElementById("output2").innerHTML = 
    "Name: " + person.name + ", Age: " + person.age + ", City: " + person.city + "<br>" +
    "(const prevents reassignment, not modification!)";

Example 3: Object.freeze()

const frozen = Object.freeze({ value: 10 });
try {
    frozen.value = 20;
    document.getElementById("output3").innerHTML = "Value: " + frozen.value;
} catch(e) {
    document.getElementById("output3").innerHTML = "Cannot modify frozen object!";
}

Example 4: Immutable Patterns

var original = [1, 2, 3];
var added = [...original, 4];
var filtered = original.filter(n => n > 1);
var doubled = original.map(n => n * 2);

document.getElementById("output4").innerHTML = 
    "Original: " + original.join(", ") + "<br>" +
    "Added: " + added.join(", ") + "<br>" +
    "Filtered: " + filtered.join(", ") + "<br>" +
    "Doubled: " + doubled.join(", ");

Example 5: Immutable Object Update

var user = { name: "John", age: 25, city: "NYC" };

function updateUserAge(user, newAge) {
    return { ...user, age: newAge };
}

function updateUserCity(user, newCity) {
    return { ...user, city: newCity };
}

var updated = updateUserAge(updateUserCity(user, "LA"), 30);

document.getElementById("output5").innerHTML = 
    "Original: " + user.name + ", " + user.age + ", " + user.city + "<br>" +
    "Updated: " + updated.name + ", " + updated.age + ", " + updated.city;

Key Points

  • Immutability: don't change data, create new data
  • Use spread operator (...)
  • const doesn't make objects immutable
  • Object.freeze() prevents modifications
