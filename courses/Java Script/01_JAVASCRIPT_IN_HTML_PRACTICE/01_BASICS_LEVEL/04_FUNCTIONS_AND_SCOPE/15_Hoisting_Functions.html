Hoisting - Functions

Hoisting moves function declarations to the top of their scope before code execution.

Function Declaration Hoisting

// These can be called before their declaration
sayHello(); // Works!
function sayHello() { ... }

Examples

Example 1: Function Declaration Hoisting

// Call function before declaration
var result = add(5, 3);
document.getElementById("output1").innerHTML = "5 + 3 = " + result;

// Function declaration - hoisted to top
function add(a, b) {
    return a + b;
}

Example 2: Function Expression Hoisting

// This will cause an error - can't call before assignment
try {
    var result = multiply(4, 5);
    document.getElementById("output2").innerHTML = result;
} catch(e) {
    document.getElementById("output2").innerHTML = "Error: " + e.message;
}

var multiply = function(a, b) {
    return a * b;
};

Example 3: Declaration vs Expression

// Declaration - fully hoisted
var addResult = sayHi("John");
document.getElementById("output3").innerHTML = addHiResult + " - " + addResult;

function sayHi(name) {
    return "Hi, " + name;
}

// Expression - only variable hoisted
var sayHi = function(name) {
    return "Hello, " + name;
};

var addHiResult = sayHi("Jane");
document.getElementById("output3").innerHTML = addHiResult + " - " + addResult;

Example 4: Multiple Function Declarations

// Call in any order - all hoisted
var area = calculateArea(5, 3);
var perim = calculatePerimeter(5, 3);

document.getElementById("output4").innerHTML = "Area: " + area + ", Perimeter: " + perim;

function calculateArea(w, h) { return w * h; }
function calculatePerimeter(w, h) { return 2 * (w + h); }

Example 5: Hoisting Order

// Function declarations take precedence
var result = test(5);
document.getElementById("output5").innerHTML = result;

function test(x) {
    return "Function: " + x;
}

var test = "String";

document.getElementById("output5").innerHTML += "<br>After variable: " + typeof test;

Key Points

  • Function declarations are fully hoisted - can be called before definition
  • Function expressions are only partially hoisted - variable is hoisted, but assignment is not
  • Function declarations take precedence over variable declarations
  • It's best practice to declare functions before using them
