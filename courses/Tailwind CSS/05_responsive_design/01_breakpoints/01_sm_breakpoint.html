SM Breakpoint (640px)

Understanding the SM Breakpoint

The sm breakpoint in Tailwind CSS targets screens with a minimum width of 640px. This is the second-smallest breakpoint and is commonly used for tablet devices in portrait mode and larger smartphones.

Tailwind CSS Default Breakpoints

  • sm: 640px (tablets)
  • md: 768px (small laptops)
  • lg: 1024px (desktops)
  • xl: 1280px (large desktops)
  • 2xl: 1536px (extra large screens)

Standard Example: Basic SM Breakpoint Usage

<!-- Base classes apply to all sizes -->
<div class="w-full bg-red-500">
    Visible on all devices
</div>

<!-- sm: prefix applies at 640px and above -->
<div class="w-full sm:bg-blue-500">
    Changes from red to blue at sm breakpoint
</div>

<!-- Mobile-first: base is mobile, sm: enhances for larger screens -->
<div class="text-sm sm:text-base lg:text-lg">
    Font size adjusts at each breakpoint
</div>

Output Statement:

On devices narrower than 640px: element shows red background, small text.
On devices 640px and wider: element shows blue background, base text size.

Real-World Example 1: Card Component with SM Breakpoint

<!-- Product card that adapts at sm breakpoint -->
<div class="bg-white rounded-lg shadow-md overflow-hidden">
    <!-- Image: full width on mobile, fixed height on sm+ -->
    <img src="product.jpg" 
         class="w-full h-48 object-cover" 
         alt="Product">
    
    <div class="p-4">
        <!-- Title: smaller on mobile, larger on sm+ -->
        <h3 class="text-lg sm:text-xl font-bold text-gray-800">
            Product Title
        </h3>
        
        <!-- Description text -->
        <p class="text-gray-600 text-sm sm:text-base mt-2">
            Product description text that adapts to screen size
        </p>
        
        <!-- Price and button container -->
        <div class="flex items-center justify-between mt-4">
            <span class="text-2xl font-bold text-green-600">$99</span>
            <button class="bg-blue-600 hover:bg-blue-700 text-white 
                          px-4 py-2 rounded-lg text-sm sm:px-6 sm:py-2.5
                          transition-colors duration-200">
                Add to Cart
            </button>
        </div>
    </div>
</div>

Real-World Example 2: Navigation with SM Breakpoint

<!-- Responsive navigation bar -->
<nav class="bg-white shadow-md">
    <div class="max-w-7xl mx-auto px-4">
        <div class="flex justify-between items-center h-16">
            <!-- Logo -->
            <div class="flex-shrink-0 flex items-center">
                <span class="text-xl font-bold text-blue-600">Logo</span>
            </div>
            
            <!-- Mobile menu button (hidden on sm+) -->
            <div class="sm:hidden">
                <button class="text-gray-600 hover:text-gray-900 p-2">
                    ☰
                </button>
            </div>

            <div class="hidden sm:flex sm:space-x-8">

                    Home
                </a>
                <a href="#" class="text-gray-700 hover:text-blue-600

                    About
                </a>
                <a href="#" class="text-gray-700 hover:text-blue-600 
                                   px-3 py-2 text-sm font-medium">

Output Statement:

Below 640px: Shows only logo and hamburger menu button. Navigation links are hidden.
At 640px and above: Hamburger menu disappears, navigation links become visible in a horizontal row.

Edge Cases and Considerations

Browser Support

All modern browsers support media queries. Tailwind's sm: prefix compiles to @media (min-width: 640px) which has excellent cross-browser support including IE11 (with PostCSS setup).

Common Pitfall: Using Wrong Prefix

Remember: sm: applies at 640px and ABOVE. If you want styles only for mobile (below 640px), use base classes without prefix and override with sm: for larger screens.

Performance Note

Tailwind only generates CSS for breakpoints you actually use. Using sm: classes won't bloat your CSS if you're not using them in your HTML.

Accessibility Considerations

  • Ensure touch targets are at least 44x44 pixels on mobile (use appropriate padding)
  • Use sm: to increase font sizes on larger screens for better readability
  • Maintain adequate color contrast at all breakpoints
  • Test with actual devices, not just browser dev tools
  • Consider users who zoom their browser - use relative units (rem/em) not pixels
