Responsive Display Utilities

Understanding Display Properties

Tailwind CSS provides responsive display utilities that control how elements are displayed on different screen sizes. The display property determines the type of box an element generates, affecting layout behavior and visibility.

Key Display Classes

  • block, inline-block, inline
  • flex, inline-flex
  • grid, inline-grid
  • table, table-row, table-cell
  • hidden - Completely hides element (display: none)
  • contents - Removes wrapper, exposes children

Standard Example: Show/Hide Navigation

<!-- Mobile-first: hidden on mobile, visible on larger screens -->
<nav>
    <!-- Always visible on mobile -->
    <div class="md:hidden">
        <button class="p-2 bg-blue-600 text-white rounded">☰ Menu</button>
    </div>
    
    <!-- Hidden on mobile, visible on md+ (768px) -->
    <div class="hidden md:flex space-x-4">
        <a href="#" class="text-gray-700 hover:text-blue-600">Home</a>
        <a href="#" class="text-gray-700 hover:text-blue-600">About</a>
        <a href="#" class="text-gray-700 hover:text-blue-600">Contact</a>
    </div>
</nav>

<!-- Responsive block/inline -->
<div class="bg-yellow-100 p-4">
    <span class="block md:inline bg-yellow-200 p-2 m-1">
        Block on mobile, inline on desktop
    </span>
    <span class="block md:inline bg-yellow-200 p-2 m-1">
        Another span
    </span>
</div>

Output Statement:

Below 768px: Hamburger menu visible, horizontal nav hidden, spans stack vertically.
At 768px and above: Hamburger hidden, horizontal nav visible, spans inline.

Display Property Reference Table

Class CSS Description
block display: block Element takes full width
inline-block display: inline-block Inline but with block properties
inline display: inline Inline with other elements
flex display: flex Flexbox container
inline-flex display: inline-flex Inline flex container
grid display: grid CSS Grid container
hidden display: none Completely hidden

Common Responsive Display Patterns

Pattern 1: Mobile Menu

.hidden.md:flex → Show nav on desktop, hide on mobile

Pattern 2: Side-by-Side Forms

.grid-cols-1.md:grid-cols-2 → Stack on mobile, side-by-side on desktop

Pattern 3: Show Extra Content

.hidden.lg:block → Only show extra content on large screens

Pattern 4: Sticky Headers

sticky top-0 → Works with any display type for persistent navigation

Accessibility Considerations

  • When hiding navigation with hidden, ensure users can still access content via other means
  • Use aria-hidden="true" for purely decorative elements
  • Don't hide essential content that users need to complete tasks
  • Consider using visible/invisible instead of hidden for screen readers
  • Test with actual screen readers to verify hidden content is truly inaccessible
