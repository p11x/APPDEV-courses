Class Not Found Error

Understanding the Issue

The "class not found" error occurs when Tailwind CSS cannot detect a class you've used in your HTML. This typically happens during the build process when the content configuration is incorrect or when classes are generated dynamically.

Common Causes:

  • Incorrect content configuration in tailwind.config.js
  • Classes used in string concatenation or template literals
  • Classes in dynamically loaded content
  • Missing file extensions in content array
  • PurgeCSS removing unused classes incorrectly

Standard Example: The Problem

Problem: Dynamic Class Generation

// This pattern doesn't work - class is never detected
function getButtonClass(variant) {
    return 'btn-' + variant; // "btn-primary" never found
}

// In your HTML
<button class="btn-primary">Click Me</button>
// Tailwind sees "btn-primary" literal but can't trace it

Solution: Use safelist

// tailwind.config.js
module.exports = {
    content: ['./src/**/*.{html,js}'],
    safelist: [
        'btn-primary',
        'btn-secondary', 
        'btn-danger',
        'text-red-500',
        'bg-blue-600'
    ],
    theme: {},
    plugins: []
}

Real-World Example 1: Template Literal Issue

// PROBLEM: String interpolation breaks class detection
const color = 'blue';
const size = 'lg';

// This WON'T work - dynamically built class strings
const buttonClass = `bg-${color}-${size}-500`;
// Result: "bg-blue-lg-500" - Tailwind can't predict this!

// In HTML element directly
<div class="bg-{{ color }}-600"></div>
// Template engines: class won't be detected

// SOLUTION 1: Use arbitrary values
<div class="bg-[#1a1a1a]"></div>

// SOLUTION 2: Use style attribute for dynamic colors
<div :style="{ backgroundColor: userColor }"></div>

// SOLUTION 3: Map to predefined classes
const colorMap = {
    primary: 'bg-blue-600',
    secondary: 'bg-gray-600', 
    danger: 'bg-red-600'
};
// Then use: class="bg-blue-600" (literal string)

Output:

Without safelist: Class "btn-primary" not found in content

With safelist: Class properly included in CSS

Real-World Example 2: Content Path Configuration

// tailwind.config.js - INCORRECT
module.exports = {
    content: ['./src/**/*'],      // Missing file extension!
    // Tailwind needs explicit extensions for some build tools
    theme: {},
    plugins: []
}

// tailwind.config.js - CORRECT
module.exports = {
    content: [
        './src/**/*.{html,js,jsx,ts,tsx,vue,svelte}',
        './views/**/*.ejs',
        './components/*.{js,ts}'
    ],
    theme: {
        extend: {}
    },
    plugins: []
}

// Real-world: Next.js configuration
// tailwind.config.js
module.exports = {
    content: [
        './pages/**/*.{js,ts,jsx,tsx,mdx}',
        './components/**/*.{js,ts,jsx,tsx,mdx}',
        './app/**/*.{js,ts,jsx,tsx,mdx}',
        './src/**/*.{js,ts,jsx,tsx,mdx}',
    ],
    theme: {
        extend: {
            colors: {
                brand: '#ff6b6b'
            }
        }
    },
    plugins: []
}

Output:

Error: Cannot find tailwind classes - make sure content paths are correct

Success: Tailwind generates all found classes

Edge Cases & Considerations

Edge Cases

  • • Classes in conditional comments
  • • Classes in imported components
  • • Classes in JSON data files
  • • Classes generated at runtime
  • • Third-party component libraries

Performance

  • • Safelist increases CSS size
  • • Only safelist truly dynamic classes
  • • Consider PurgeCSS alternatives
  • • Use CDN for development only

Accessibility

  • • Dynamic classes may break focus styles
  • • Ensure color contrast maintained
  • • Test with screen readers

Cross-Browser

  • • Works in all modern browsers
  • • Use prefix if IE11 needed
  • • Test in target browsers

Quick Fix Commands

# Debug what Tailwind is seeing
npx tailwindcss --content --debug

# Build with verbose output  
npx tailwindcss -i ./input.css -o ./output.css --verbose

# Check for missing classes in production build
npm run build 2>&1 | grep -i "warn"

# Enable all classes for debugging (not production!)
# In tailwind.config.js:
module.exports = {
    corePlugins: {
        preflight: false  // Debug mode
    }
}
