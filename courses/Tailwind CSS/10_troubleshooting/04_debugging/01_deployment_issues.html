Deployment Issues

Understanding the Issue

Deployment issues occur when Tailwind CSS works locally but fails in production environments. These can be related to build processes, file paths, server configuration, caching, or CDN issues.

Common Causes:

  • Build process not running correctly
  • Static file paths incorrect
  • Server not serving static files
  • Cache not clearing after deploy
  • Environment variables not set

Standard Example: The Problem

Problem: Build Not Running

/* Problem: Deploying without building */

/* Local development works because: */
npx tailwindcss -i input.css -o output.css --watch
/* Runs continuously, watches for changes */

/* Production server expects static CSS! */
/* But deploy only uploads source files */

/* package.json - missing build script */
{
    "scripts": {
        "dev": "vite",
        "start": "vite preview"
        /* Missing: "build": "vite build" */
    }
}

/* Result: No output.css in production! */
<link rel="stylesheet" href="output.css">
/* 404 - Not Found */

Solution: Proper Build Pipeline

/* package.json - with proper scripts */
{
    "scripts": {
        "dev": "tailwindcss -i input.css -o output.css --watch",
        "build": "tailwindcss -i input.css -o output.css --minify",
        "start": "serve dist"
    }
}

/* Build steps for deployment: */
npm run build
/* Runs: npx tailwindcss -i input.css -o output.css --minify */

/* Deploy should include: */
/* dist/
   ├── index.html
   ├── output.css    <- Generated CSS
   ├── output.css.map
   └── js/
*/

/* Server config (Vercel example) */
/* vercel.json */
{
    "buildCommand": "npm run build",
    "outputDirectory": "dist"
}

Real-World Example 1: CDN Path Issues

/* PROBLEM: CDN not loading in production */

/* Development works: */
<script src="https://cdn.tailwindcss.com"></script>

/* But some servers block external scripts! */

/* SOLUTION 1: Self-hosted CSS */
npm install -D tailwindcss postcss autoprefixer

npx tailwindcss -i input.css -o dist/output.css

/* In production HTML: */
<link rel="stylesheet" href="/output.css">

/* SOLUTION 2: Use proper CDN with config */
<script src="https://cdn.tailwindcss.com?plugins=forms,typography"></script>
<script>
    tailwind.config = {
        content: ['./src/**/*.{html,js}'],
        theme: { extend: {} },
        plugins: []
    }
</script>

/* SOLUTION 3: Check Content Security Policy */
/* If using CSP, add cdn.tailwindcss.com to script-src */

Output:

CDN blocked: Styles not loading in production

Self-hosted: Works reliably everywhere

Real-World Example 2: Path & Caching

/* PROBLEM: Wrong paths or stale cache */

/* Wrong path in HTML */
<link rel="stylesheet" href="css/output.css">
/* But file is at: /output.css */

/* Correct path for static hosting */
<link rel="stylesheet" href="/output.css">
<!-- From root of domain */

/* If in subfolder: */
<link rel="stylesheet" href="/myapp/output.css">

/* PROBLEM: Browser caches old CSS */
/* User sees: white page, no styles */

/* SOLUTION: Cache busting */
<link rel="stylesheet" href="/output.css?v=1.2.3">

/* Or use content hash (recommended) */
/* Output filename: output.a1b2c3d4.css */
<link rel="stylesheet" href="/output.a1b2c3d4.css">

/* Server: Add cache headers */
Cache-Control: public, max-age=31536000, immutable

Output:

Wrong path: 404 error, no styles

Stale cache: Old styles persist after update

With versioning: Always fresh, cached properly

Edge Cases & Considerations

Edge Cases

  • Subdirectory deployments
  • CDN origins need special config
  • Serverless functions don't serve static
  • Docker builds need different steps

Performance

  • Minify CSS in production
  • Use content hash for caching
  • Consider CDN for assets

Platforms

  • Vercel: auto-detects
  • Netlify: needs build command
  • AWS S3: upload dist folder

Tools

  • Vite, Webpack, Parcel
  • Next.js, Nuxt have built-in
  • Check platform docs

Quick Fix Commands

# Build for production
npm run build
# or: npx tailwindcss -i input.css -o output.css --minify

# Check output file exists
ls -la output.css

# Test locally with serve
npx serve dist

# Clear all cache
# Browser: Hard refresh (Ctrl+Shift+R)
# CDN: Purge cache if using one

# Verify file in production
curl -I https://yoursite.com/output.css
