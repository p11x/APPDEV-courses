Debugging Techniques

Understanding Debugging

Debugging Tailwind CSS issues requires systematic approach using browser DevTools, build configuration checks, and understanding how Tailwind generates CSS. This guide covers essential debugging techniques.

Debugging Tools:

  • Browser DevTools (Elements, Console, Network)
  • Tailwind CLI with --verbose flag
  • Content path verification
  • Generated CSS inspection
  • Build process monitoring

Standard Example: Debug Flow

Step 1: Identify the Problem

/* Check if styles are applied */
getComputedStyle(element).backgroundColor
// Returns: "rgb(59, 130, 246)" = bg-blue-500 applied

/* Check what CSS exists */
getComputedStyle(element).cssText
// Shows all applied styles

/* Check class in HTML */
element.className
// Shows all classes applied to element

/* Check computed values */
getComputedStyle(document.querySelector('.flex'))
// display: "flex" - class is working!

Step 2: Use DevTools

/* 1. Inspect Element (Right-click → Inspect) */
/* 2. Look at Styles panel - shows applied CSS */

/* 6. Look at Computed tab for final values */

/* 7. Check Network tab */
/* Is CSS file loading? (Status 200 vs 404) */

/* 8. Check Console tab */

Real-World Example 1: Build Debugging

/* Run build with verbose output */
npx tailwindcss -i input.css -o output.css --verbose

/* Output shows: */
/*
warn - No utility classes found in content files
   Make sure content includes all template files
warn - Using default values for missing config
*/

/* Debug content paths */
npx tailwindcss --content --debug

/* Output shows all files being scanned: */
/*
Content files:
  ./src/index.html
  ./src/components/button.html
  ...
*/

/* Check if specific class is detected */
npx tailwindcss -i input.css -o output.css --verbose 2>&1 | grep "bg-blue-500"

/* Verify output CSS exists */
ls -la output.css

/* Check first 100 lines of output */
head -100 output.css

Output:

Verbose shows content scanning, class detection

Debug flags reveal hidden issues

Real-World Example 2: Browser Debugging

/* In Browser Console - Tailwind global */
console.log(tailwind.config)
// Shows your config object

console.log(tailwind)
// Shows: { config, matchUtilities, ... }

/* Debug specific element */
const el = document.querySelector('.my-element');
const styles = getComputedStyle(el);

// Check if class is applied
el.classList.contains('bg-blue-500');
// true/false

/* Check if CSS file loaded */
const links = document.querySelectorAll('link');
links.forEach(link => console.log(link.href));

/* Force Tailwind to re-run (if using JIT) */
__TAILWIND__.refresh();

/* Debug with custom utility */
@layer utilities {
    .debug-border {
        outline: 1px solid red !important;
    }
}
<div class="debug-border"></div>

Output:

Console reveals Tailwind config and state

Element inspection shows actual CSS

Common Debugging Commands

CLI Commands

  • npx tailwindcss --verbose
  • npx tailwindcss --debug
  • npx tailwindcss --content
  • npm run build 2>&1

Browser Console

  • console.log(tailwind)
  • console.log(tailwind.config)
  • getComputedStyle(el)

Node.js

  • node -e "require('./tailwind.config.js')"
  • npm ls tailwindcss
  • npm ls postcss

File Inspection

  • grep "bg-blue-500" output.css
  • head output.css
  • wc -l output.css

Quick Debugging Checklist

/* Check these in order: */

1. HTML class is correct?
   class="bg-blue-500" not "bg-blue-500 "

2. CSS file is in HTML?
   <link rel="stylesheet" href="output.css">

3. CSS file has content?
   output.css size > 0

4. Class in CSS file?
   grep "bg-blue-500" output.css

5. Config has correct paths?
   content: ['./src/**/*.{html,js}']

6. Build runs without error?
   npm run build

7. No conflicting styles?
   DevTools shows overridden?

8. Cache cleared?
   Hard refresh (Ctrl+Shift+R)
