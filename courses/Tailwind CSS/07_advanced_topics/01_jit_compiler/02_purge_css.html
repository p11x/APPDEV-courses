Purge CSS and JIT Compiler

Understanding Purge CSS

Purge CSS is a tool that removes unused CSS from your project. In the context of Tailwind CSS, purge CSS was essential in older versions (pre-JIT) to remove all the unused utility classes from the generated CSS file, dramatically reducing file size. With JIT, purge happens automatically during compilation.

How Purge Works with JIT

The JIT compiler inherently performs purge functionality. It scans your content files (HTML, templates, JavaScript, etc.) and only generates CSS for classes that are actually found in your content. This automatic purge is more efficient than traditional post-processing purge because it happens at generation time rather than as a separate processing step.

Content Configuration Options

module.exports = {
  content: [
    // Specific files
    "./src/index.html",
    
    // All files with specific extensions
    "./src/**/*.{html,js,jsx,ts,tsx,vue,svelte}",
    
    // Using globs
    "./src/components/**/*",
    
    // Safelist for dynamic classes
  ],
  safelist: [
    'bg-red-500',
    'px-4', 
  ],
}

Edge Cases and Considerations

  • Dynamic class names from template literals may not be detected
  • Classes used conditionally via JavaScript need safelist
  • Content must include all file types that contain classes
  • Third-party libraries may need custom content paths

Performance Considerations

  • Content scanning adds some build time but is worthwhile for output size
  • Use specific content patterns rather than scanning entire node_modules
  • Proper content configuration is critical for correct purge results

Accessibility Notes

Proper purge doesn't affect accessibility - it only removes unused CSS rules. All classes found in your HTML and JavaScript files are preserved. Ensure your content files include all templates that render in the browser.

Cross-Browser Notes

Purge CSS works identically across all platforms and browsers. The output is standard CSS that works in all modern browsers.
