Negative Spacing in Tailwind CSS

Understanding Negative Margins

Negative margins pull elements in the opposite direction of positive margins, allowing overlap and creative layouts.

Note: Negative margins only work with margin (m-), not padding (p-).

Standard Example: Negative Margin

<!-- Negative margin on all sides -->
<div class="-m-4">
    <!-- -16px margin pulling in all directions -->
</div>

<!-- Negative vertical margin -->
<div class="-my-8">
    <!-- -32px top and bottom -->
</div>

<!-- Specific negative sides -->
<div class="-mt-4 -mb-8">
    <!-- Negative top and bottom -->
</div>
-m-4 (negative margin)
-mt-8 (negative top)

Real-World Example 1: Overlapping Cards

<!-- Cards that overlap each other -->
<div class="flex flex-col">
    <div class="bg-white p-6 shadow-lg rounded-t-lg z-10">
        Card on top
    </div>
    <div class="bg-white p-6 shadow-lg -mt-8 rounded-lg">
        Card overlapping above with -mt-8
    </div>
</div>
Card on top
Overlapping with -mt-4

Real-World Example 2: Image Pull Request

<!-- Pull image up into header area -->
<div class="bg-white rounded-lg shadow-lg p-6">
    <h2 class="text-xl font-bold mb-4">Blog Post Title</h2>
    <img 
        class="w-full -mt-8 mb-4 rounded-lg shadow" 
        src="feature.jpg" 
        alt="Feature"
    >
    <p>Content starts below image...</p>
</div>

Output Statement

  • Use -m-1 through -m-96 for negative margins
  • Combine with z-index for proper layering
  • Negative margins can cause unexpected overlaps - use carefully
  • Only margin (not padding) supports negative values

Edge Cases

  • Flexbox children don't collapse margins - need to handle differently
  • Negative margins on grid items can cause layout overflow
  • Use with caution in responsive designs
