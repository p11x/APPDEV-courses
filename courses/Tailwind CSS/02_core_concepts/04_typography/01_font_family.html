Font Family in Tailwind CSS

Default Font Families

Tailwind uses the system font stack by default, which provides platform-native typography. You can also use sans-serif, serif, or monospace explicitly.

Standard Example: Setting Font Family

<!-- Default sans-serif -->
<p class="font-sans">
    Main content text with sans-serif
</p>

<!-- Serif for headings or articles -->
<h1 class="font-serif">
    Article Title
</h1>

<!-- Monospace for code -->
<code class="font-mono">
    codeSnippet = "inline code"
</code>

Real-World Example 1: Blog Typography

<article class="max-w-2xl mx-auto">
    <h1 class="font-serif text-4xl font-bold mb-4">
        Blog Post Title
    </h1>
    <p class="font-sans text-lg text-gray-700 leading-relaxed">
        Article content uses sans-serif for readability.
    </p>
</article>

Real-World Example 2: Code Display

<pre class="bg-gray-900 p-4 rounded font-mono text-sm">
function hello() {
    console.log("Hello World");
}
</pre>
function hello() {
    console.log("Hello World");
}

Output Statement

  • font-sans - system sans-serif stack (default)
  • font-serif - system serif stack
  • font-mono - system monospace stack
  • Font family is inherited by child elements
