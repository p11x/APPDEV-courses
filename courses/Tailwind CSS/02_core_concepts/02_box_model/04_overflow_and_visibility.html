Standard Example: Overflow Control

<!-- Scrollable container -->
<div class="h-48 overflow-y-auto">
    <!--Vertical scroll only -->
    Long content here...
</div>

<!-- Hide overflow -->
<div class="h-32 overflow-hidden">
    <!-- Masks excess content -->
    Extra content hidden
</div>

<!-- Auto scroll (shows when needed) -->
<div class="h-40 overflow-auto">
    <!-- Shows scrollbar only if needed -->
    Content...
</div>

Visibility Control

<!-- Hide element visually but keep in DOM -->
<div class="invisible">
    <!-- Takes up space but not visible -->
</div>

<!-- Completely hide element -->
<div class="hidden">
    <!-- Removes from layout entirely -->
</div>

Real-World Example 2: Conditional Visibility

<!-- Hide on mobile, show on desktop -->
<div class="hidden md:block">
    Desktop only content
</div>

<!-- Responsive overflow -->
<div class="overflow-x-auto md:overflow-visible">
    <!-- Scroll on mobile, wrap on desktop -->
</div>

Output Statement

  • overflow-auto shows scrollbars only when content overflows
  • overflow-scroll shows scrollbars always
  • overflow-hidden clips content at container boundary
  • hidden removes element completely from DOM
  • invisible hides but preserves layout space

Edge Cases

  • hidden requires block/inline-block reset in flex containers
  • Use sr-only for accessible hidden content
  • overflow-auto on flex items needs flex-shrink-0

Accessibility

  • sr-only class hides from view but keeps for screen readers
  • Ensure scrollable content is keyboard accessible
  • Don't use hidden on focusable elements without warning
