Performance-First Architecture

Performance optimization is no longer optional, with Core Web Vitals influencing CSS architecture decisions.

REAL-WORLD EXAMPLE: Performance Optimization

/* Performance-First CSS Patterns

Core Web Vitals Optimizations:
┌─────────────────────────────────────────────────────────────┐
│                    LCP OPTIMIZATION                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. Critical CSS Inlining                                   │
│     - Inline above-fold styles                              │
│     - Defer non-critical styles                             │
│                                                              │
│  2. Font Loading Optimization                               │
│     - Use font-display: swap;                              │
│     - Preload critical fonts                                 │
│                                                              │
│  3. Image Optimization                                     │
│     - Use modern formats (WebP, AVIF)                       │
│     - Proper sizing with srcset                           │

Islands Architecture Integration

Islands architecture changes how CSS is delivered, with implications for utility-first approaches.

STANDARD EXAMPLE: Islands-Ready CSS

/* Islands Architecture CSS Design

Traditional vs Islands Approach:

Traditional SPA:
┌─────────────────────────────────────┐
│     ┌─────────────────────────┐      │
│     │     Single Page App     │      │
│     │   ┌─────────────────┐  │      │
│     │   │ All CSS bundled │  │      │
│     │   └─────────────────┘  │      │
│     └─────────────────────────┘      │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│     Islands Architecture            │
│     ┌─────────┐ ┌─────────┐       │
│     │ Island1│ │ Island2 │         │
│     │ Global │ │ Per-Island         │
│     │  CSS   │ │  CSS   │          │
│     └─────────┘ └─────────┘       │
└─────────────────────────────────────┘

CSS Strategy for Islands:
/* Shared utilities - load once */
.utilities-shared {
  display: flex;
  padding: 1rem;
}

/* Island-specific - scoped CSS */
.island-menu {
  /* Only loads for menu island */
  background: white;
}
