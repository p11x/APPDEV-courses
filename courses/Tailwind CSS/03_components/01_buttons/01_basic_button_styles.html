Basic Button Styles

Buttons are essential interactive elements in web interfaces. In Tailwind CSS, buttons can be styled using utility classes for padding, colors, borders, rounded corners, and text properties. The base button styles provide a foundation that can be customized for different use cases.

Key Concepts:

  • Padding: Use px- (horizontal) and py- (vertical) utilities like px-4 py-2
  • Background Color: bg-blue-600, bg-green-500, bg-red-500, etc.
  • Text Color: text-white for dark backgrounds, text-gray- for light backgrounds
  • Border Radius: rounded, rounded-lg, rounded-full for different corner styles
  • Font Weight: font-semibold or font-bold for button text
  • Cursor: cursor-pointer to indicate clickability
  • Transition: transition for smooth hover effects

Standard Example: Basic Button Variants

The following example demonstrates basic button styles with different colors and sizes.

<!-- Primary Button (Blue) -->
<button class="px-4 py-2 bg-blue-600 text-white font-semibold rounded">
    Primary Button
</button>

<!-- Secondary Button (Gray) -->
<button class="px-4 py-2 bg-gray-200 text-gray-800 font-semibold rounded">
    Secondary Button
</button>

<!-- Success Button (Green) -->
<button class="px-4 py-2 bg-green-600 text-white font-semibold rounded">
    Success Button
</button>

<!-- Danger Button (Red) -->
<button class="px-4 py-2 bg-red-600 text-white font-semibold rounded">
    Danger Button
</button>

<!-- Outline Button -->
<button class="px-4 py-2 border-2 border-blue-600 text-blue-600 font-semibold rounded">
    Outline Button
</button>

Edge Cases & Considerations

Accessibility Considerations:

  • Always include focus states (focus:ring-) for keyboard navigation
  • Use sufficient color contrast (WCAG requires 4.5:1 ratio for normal text)
  • Include aria-label for icon-only buttons
  • Ensure buttons are at least 44x44px for touch targets
  • Use descriptive button text, not just "Click Here"

Performance Considerations:

  • Use transition utilities efficiently - avoid excessive properties
  • Consider will-change for complex animations
  • Use transform scale instead of width/height for hover effects
  • Optimize gradient usage as they can be performance-heavy

Cross-Browser Notes:

  • Some border-radius values may render differently across browsers
  • Gradient syntax is widely supported but test in Safari
  • Focus ring may appear differently in Firefox
  • Use -webkit-appearance: none for consistent styling

Output Statement

  • Hover states with smooth transitions
  • Focus states for keyboard accessibility
  • Buttons with proper padding, colors, and border radius
