Loading States

Loading states inform users that an operation is in progress and they should wait. Common patterns include spinners, pulsing buttons, skeleton loaders, and text changes. Proper loading states improve user experience by setting expectations.

Loading Patterns:

  • Spinner: Rotating icon indicating activity
  • Pulse: Subtle animation showing pending state
  • Skeleton: Placeholder content during loading
  • Text Change: Button text changes to indicate loading
  • Progress: Linear or circular progress indicator

Standard Example: Loading Button Patterns

<!-- Spinner Button -->
<button class="
    px-6 py-3 bg-blue-600 text-white font-semibold rounded-lg
    disabled:opacity-75 disabled:cursor-wait
">
    <svg class="animate-spin -ml-1 mr-3 h-5 w-5" fill="none" viewBox="0 0 24 24">
        <circle class="opacity-25" cx="12" cy="12" r="10" stroke="currentColor" stroke-width="4"></circle>
        <path class="opacity-75" fill="currentColor" d="M4 12a8 8 0 018-8V0C5.373 0 0 5.373 0 12h4z"></path>
    </svg>
    Processing...
</button>

<!-- Pulse Button -->
<button class="
    px-6 py-3 bg-green-600 text-white font-semibold rounded-lg
    animate-pulse
">
    Saving...
</button>

<!-- Skeleton Button -->
<button class="
    px-6 py-3 bg-gray-200 rounded-lg
    animate-pulse
">
    <span class="inline-block w-24 h-4 bg-gray-300 rounded"></span>
</button>

Real-World Example 1: Form Submit with Loading

Complete example of a form submission button with loading state.

<button 
    id="submitBtn"
    type="submit"
    class="
        relative px-6 py-3 
        bg-indigo-600 text-white font-bold rounded-lg
        hover:bg-indigo-700
        focus:outline-none focus:ring-2 focus:ring-indigo-500 focus:ring-offset-2
        disabled:opacity-75 disabled:cursor-wait
        transition-all duration-200
    "
    onclick="this.classList.add('opacity-75'); this.querySelector('.spinner').classList.remove('hidden'); this.querySelector('.btn-text').textContent='Submitting...'"
>
    <span class="btn-text">Submit Form</span>
    <span class="spinner hidden absolute right-3 top-3">
        <svg class="animate-spin h-5 w-5" fill="none" viewBox="0 0 24 24">
            <circle class="opacity-25" cx="12" cy="12" r="10" stroke="white" stroke-width="4"></circle>
            <path class="opacity-75" fill="white" d="M4 12a8 8 0 018-8V0C5.373 0 0 5.373 0 12h4z"></path>
        </svg>
    </span>
</button>

Edge Cases & Considerations

  • Always provide feedback within 100ms of user action
  • Include aria-busy="true" on loading elements for accessibility
  • Prevent double-click submissions with disabled state
  • Show estimated time for long operations when possible
  • Allow cancellation of loading operations

Output Statement

After implementing loading states, you will have:

  • Spinner buttons for short operations
  • Pulse animations for pending states
  • Progress bars for long operations
  • Accessible loading indicators
