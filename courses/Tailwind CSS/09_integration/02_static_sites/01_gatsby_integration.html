Gatsby + Tailwind CSS Integration

Overview

Gatsby is a static site generator that produces highly optimized static websites. Tailwind CSS integrates with Gatsby through PostCSS and the gatsby-plugin-postcss. Gatsby's GraphQL data layer and component system work seamlessly with Tailwind's utility-first approach. The gatsby-plugin-purgecss helps reduce CSS bundle size by removing unused styles.

Key Point: Gatsby + Tailwind produces extremely fast static websites. Use gatsby-plugin-postcss for Tailwind integration and gatsby-plugin-purgecss for optimal CSS sizes.

Standard Example: Gatsby Setup

Setting up Tailwind CSS with Gatsby:

// Install dependencies

npm install -D tailwindcss postcss autoprefixer gatsby-plugin-postcss

npm install -D gatsby-plugin-purgecss

 

// gatsby-config.js

module.exports = {

plugins: [

{

resolve: 'gatsby-plugin-postcss',

options: {

tailwindCssOptimization: {

removeDuplicates: false,

removeWatchedFiles: false

}

}

},

'gatsby-plugin-purgecss'

]

}

 

// tailwind.config.js

module.exports = {

content: [

"./src/**/*.{js,jsx,ts,tsx}",

"./gatsby-config.{js,ts}"

],

theme: { extend: {} },

plugins: []

}

Real-World Example 1: Gatsby Layout Component

A responsive layout component with Tailwind:

// src/components/layout.js

import

React from 'react'

import

{ useStaticQuery, graphql } from 'gatsby'

 

const

Layout

= ({ children }) => {

const data = useStaticQuery(graphql`

query {

site {

siteMetadata {

title

}

}

}

`)

 

return (

<div className="min-h-screen flex flex-col bg-gray-50">

<header className="bg-white shadow sticky top-0 z-10">

<div className="max-w-7xl mx-auto px-4 py-4">

<h1 className="text-2xl font-bold text-gray-800">

{data.site.siteMetadata.title}

</h1>

</div>

</header>

<main className="flex-grow max-w-7xl mx-auto px-4 py-8 w-full">

{children}

</main>

<footer className="bg-gray-800 text-white py-6">

<div className="max-w-7xl mx-auto px-4 text-center">

<p>© {new Date().getFullYear()} My Site</p>

</div>

</footer>

</div>

)

}

 

export

default

Layout

Real-World Example 2: Blog Index Page

A blog index page querying Markdown posts:

// src/pages/index.js

import

React from 'react'

import

{ graphql, Link } from 'gatsby'

import Layout from '../components/layout'

 

const

IndexPage

= ({ data }) => {

const posts = data.allMarkdownRemark.nodes

 

return (

<Layout>

<div className="grid gap-6 md:grid-cols-2 lg:grid-cols-3">

{posts.map(post => (

<article key={post.id} className="bg-white rounded-lg shadow-md overflow-hidden hover:shadow-lg transition-shadow">

<div className="p-6">

<h2 className="text-xl font-semibold text-gray-800 mb-2">

<Link to={`/blog/${post.frontmatter.slug}`}

className="hover:text-blue-600"

>{post.frontmatter.title}</Link>

</h2>

<p className="text-gray-500 text-sm mb-2">

{post.frontmatter.date}

</p>

<p className="text-gray-600">

{post.excerpt}

</p>

</div>

</article>

))}

</div>

</Layout>

)

}

 

export

const

query

= graphql`

query {

allMarkdownRemark(sort: { frontmatter: { date: DESC } }) {

nodes {

id

excerpt(pruneLength: 160)

frontmatter {

title

date(formatString: "MMMM DD, YYYY")

slug

}

}

}

}

`

Edge Cases & Considerations

  • PurgeCSS False Positives: May remove dynamically generated classes; use purgecssWhitelist.
  • Image Processing: Use gatsby-plugin-image with Tailwind for optimized images.
  • MDX Support: MDX works with Tailwind through @mdx-js/react.
  • Head API: Use Gatsby Head API for SEO metadata.

Performance Considerations

  • Static Generation: Gatsby creates static HTML for maximum performance.
  • PurgeCSS: Dramatically reduces CSS size (often 90%+ reduction).
  • Code Splitting: Automatic per-route code splitting.
  • Prefetching: Gatsby prefetches links for instant page loads.

Accessibility Notes

  • Skip Links: Include skip navigation links.
  • Color Contrast: Ensure sufficient contrast ratios in theme.
  • Focus States: Add visible focus indicators.
  • Reduced Motion: Use prefers-reduced-motion queries.

Cross-Browser Compatibility

  • Static Output: Works across all browsers that support static HTML.
  • Autoprefixer: Handles vendor prefixes automatically.
  • Testing: Test rendered HTML output for consistency.
  • Progressive Enhancement: Works without JavaScript.
