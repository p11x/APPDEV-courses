Hugo + Tailwind CSS Integration

Overview

Hugo is the fastest static site generator, built in Go. Tailwind CSS integrates with Hugo through PostCSS/Pipeline or by directly linking the Tailwind CLI. Hugo's asset processing pipeline can handle Tailwind compilation by integrating with PostCSS. Hugo's templating system works well with Tailwind's utility classes.

Key Point: Hugo is the fastest option for static sites. Tailwind integration uses Hugo's asset pipeline with PostCSS for efficient build processing.

Standard Example: Hugo Setup

Setting up Tailwind CSS with Hugo's asset pipeline:

// Install Tailwind CLI

npm install -D tailwindcss

 

// tailwind.config.js

module.exports = {

content: ["./content/**/*.html", "./layouts/**/*.html"],

theme: { extend: {} },

plugins: []

};

 

// assets/css/tailwind.css

@tailwind

base;

@tailwind

components;

@tailwind

utilities;

 

// hugo.toml

[pipeline]

enableJS = true

 

// layouts/_default/baseof.html

<{{

resources.Concat "css" | fingerprint }}

<link rel="stylesheet">

->

 

// OR use resources.ExecuteAsTemplate

<style>

{{- $style := resources.Get "css/tailwind.css" | postCSS }}

{{ $style.Content | safeCSS }}

</style>

Edge Cases & Considerations

  • Watch Mode: Use Hugo server with --reload for live updates.
  • Pipes API: Hugo 0.55+ has improved asset pipeline with PostCSS.
  • Shortcodes: Use Tailwind classes in Hugo shortcodes.
  • _taxonomy: Handle categories and tags with Tailwind styling.

Performance Considerations

  • Build Speed: Hugo is the fastest static site generator (Go-based).
  • Fingerprinting: Use Hugo's fingerprint for cache busting.
  • Minification: Hugo can minify HTML/CSS automatically.
  • Asset Bundling: Combine multiple CSS files with resources.Concat.

Accessibility Notes

  • Navigation: Include skip-to-content links.
  • ARIA: Use proper ARIA attributes in Hugo templates.
  • Focus: Ensure visible focus indicators.
  • Reduced Motion: Use prefers-reduced-motion in CSS.

Cross-Browser Notes

  • Static Output: Works across all modern browsers.
  • No JS Required: Works progressively without JavaScript.
  • HTML Output: Test final HTML for consistency.
  • Testing: Verify across major browsers.
