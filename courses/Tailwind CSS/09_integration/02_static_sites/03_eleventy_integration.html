Eleventy (11ty) + Tailwind CSS Integration

Overview

Eleventy (11ty) is a simpler static site generator that transforms content with templating languages. Tailwind CSS integrates with Eleventy through PostCSS and npm scripts. Eleventy has no built-in asset pipeline, so you integrate directly with npm scripts or use a plugin like @11ty/eleventy-plugin-syntaxhighlight alongside Tailwind.

Key Point: Eleventy's simplicity makes it easy to integrate with Tailwind. Just run Tailwind as an npm script during the build process - no special plugins needed.

Standard Example: Eleventy Setup

Setting up Tailwind CSS with Eleventy build process:

// Install dependencies

npm init -y

npm install -D @11ty/eleventy tailwindcss postcss autoprefixer

 

// package.json

"scripts": {

"start": "eleventy --serve & npx tailwindcss -i ./src/css/input.css -o ./_site/css/style.css --watch",

"build": "eleventy && npx tailwindcss -i ./src/css/input.css -o ./_site/css/style.css --minify"

}

 

// tailwind.config.js

module.exports = {

content: ["./src/**/*.njk", "./src/**/*.md", "./_includes/**/*.njk"],

theme: { extend: {} },

plugins: []

};

 

// src/css/input.css

@tailwind

base;

@tailwind

components;

@tailwind

utilities;

Edge Cases & Considerations

  • Watch Mode: Run Tailwind and Eleventy concurrently using npm-run-all.
  • Passthrough Copies: Use Eleventy's passthrough copy for static assets.
  • Collections: Create collections for organizing content types.
  • Pagination: Use Eleventy pagination with Tailwind styles.

Performance Considerations

  • Fast Builds: Eleventy is extremely fast for static generation.
  • No Client JS: Outputs pure static HTML by default.
  • Code Folding: Use Eleventy watch targets for efficient rebuilds.
  • Caching: Leverage file system caching.

Accessibility Notes

  • Semantic HTML: Use semantic elements with Tailwind.
  • Skip Links: Include skip-to-content links.
  • Focus Styles: Add visible focus indicators.
  • Color Contrast: Ensure WCAG AA compliance.

Cross-Browser Compatibility

  • Static HTML: Works across all browsers.
  • No JavaScript Required: Progressive enhancement supported.
  • Offline Capable: Works without network.
  • Testing: Test HTML output across browsers.
