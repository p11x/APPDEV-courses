Ember.js + Tailwind CSS Integration

Overview

Ember.js is a productive, battle-tested JavaScript framework for building modern web applications. It emphasizes convention over configuration and provides a mature ecosystem with strong backward compatibility. Tailwind CSS integrates with Ember through PostCSS addon or ember-cli-postcss. Ember's component system works well with Tailwind's utility classes for building sophisticated UIs.

Key Point: Ember's convention-over-configuration approach pairs well with Tailwind. Use ember-cli-postcss for seamless integration with the Ember build pipeline.

Real-World Example 2: Todo List Component

A todo component using Ember's tracked properties:

<!-- app/components/todo-list.hbs -->

<div class="max-w-md mx-auto p-4">

<h2 class="text-2xl font-bold mb-4 text-gray-800">Todo List</h2>

<form {{on "submit" this.addTodo}} class="flex gap-2 mb-4">

<input

type="text"

value={{this.newTodo}}

{{on "input" (fn this.updateNewTodo event.target.value)}}

placeholder="Add new todo..."

class="flex-1 p-2 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500"

/>

<button type="submit" class="px-4 py-2 bg-blue-600 text-white rounded-lg hover:bg-blue-700 transition-colors">Add</button>

</form>

<ul class="space-y-2">

{{#each this.todos as |todo|}}

<li class="flex items-center gap-3 p-3 bg-gray-50 rounded-lg">

<input

type="checkbox"

checked={{todo.done}}

{{on "change" (fn this.toggleTodo todo.id)}}

class="w-5 h-5 accent-blue-600"

/>

<span class="flex-1 {{if todo.done 'line-through text-gray-400' 'text-gray-700'}}">{{todo.text}}</span>

<button {{on "click" (fn this.removeTodo todo.id)}} class="text-red-600 hover:text-red-800 font-bold">×</button>

</li>

{{/each}}

</ul>

</div>

 

<!-- app/components/todo-list.js -->

import Component from '@glimmer/component';

import { tracked } from '@glimmer/tracking';

import { action } from '@ember/object';

 

export default class TodoListComponent extends Component {

@tracked todos = [

{ id: 1, text: 'Learn Ember', done: false }

];

@tracked newTodo = '';

 

@action updateNewTodo(value) {

this.newTodo = value;

}

 

@action addTodo(e) {

e.preventDefault();

if (!this.newTodo.trim()) return;

this.todos = [...this.todos, {

id: Date.now(),

text: this.newTodo,

done: false

}];

this.newTodo = '';

}

 

@action toggleTodo(id) {

this.todos = this.todos.map(t =>

t.id === id ? { ...t, done: !t.done } : t

);

}

 

@action removeTodo(id) {

this.todos = this.todos.filter(t => t.id !== id);

}

}

Edge Cases & Considerations

  • Classic Components: Use classic component syntax for older addons.
  • Ember Data: Integrate with Ember Data for API handling.
  • Ember Octane: Use modern Octane patterns with Tailwind.
  • Addons: Check addon compatibility before using Tailwind plugins.

Performance Considerations

  • Code Splitting: Automatic route-based code splitting.
  • Glimmer VM: Efficient rendering engine.
  • Tree Shaking: Ember's optimizer removes dead code.
  • Lazy Loading: Load routes on demand.

Accessibility Notes

  • Focus Management: Use route lifecycle for focus management.
  • ARIA: Ember's aria attributes work with Tailwind.
  • Screen Readers: Test with screen readers.
  • Keyboard Nav: Ensure all interactive elements are accessible.

Cross-Browser Notes

  • Browser Support: Ember supports modern browsers.
  • Auto-polyfilling: Uses broccoli-env to polyfill features.
  • Testing: Test across major browsers.
  • Legacy: Use Ember 3.x for IE11 support.
