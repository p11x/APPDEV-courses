Svelte + Tailwind CSS Integration

Overview

Svelte compiles components into vanilla JavaScript with no virtual DOM overhead. Tailwind CSS integrates with Svelte through PostCSS configuration. Svelte's reactive nature pairs excellently with Tailwind's utility classes for building highly interactive user interfaces. The svelte-preprocess package handles Tailwind integration seamlessly.

Key Point: Svelte + Tailwind produces the smallest bundles. There's no virtual DOM, so Tailwind classes work directly with Svelte's compiled output.

Standard Example: SvelteKit Setup

Setting up Tailwind with SvelteKit (recommended):

// Create SvelteKit project

npm create svelte@latest my-app

cd my-app && npm install

 

// Install Tailwind

npx svelte-add@latest tailwindcss

 

// tailwind.config.js

module.exports = {

content: ['./src/**/*.{html,js,svelte,ts}'],

theme: { extend: {} },

plugins: []

};

 

// src/app.css

@tailwind

base;

@tailwind

components;

@tailwind

utilities;

 

// src/routes/+layout.svelte

<script>

import '../app.css';

</script>

<slot />

Real-World Example 1: Todo Component

A reactive todo component with Tailwind classes:

<!-- src/lib/Todo.svelte -->

<script>

let todos = [

{ id: 1, text: 'Learn Svelte', done: false }

];

let newTodo = '';

 

function

addTodo

() {

if (!newTodo.trim()) return;

todos = [...todos, {

id: Date.now(),

text: newTodo,

done: false

}];

newTodo = '';

}

 

function

removeTodo

(id) {

todos = todos.filter(t => t.id !== id);

}

 

function

toggleTodo

(id) {

todos = todos.map(t =>

t.id === id ? { ...t, done: !t.done } : t

);

}

</script>

 

<div class="max-w-md mx-auto p-4">

<h2 class="text-2xl font-bold mb-4 text-gray-800">Todo List</h2>

<form on:submit|preventDefault={addTodo} class="flex gap-2 mb-4">

<input

bind:value={newTodo}

type="text"

placeholder="Add new todo..."

class="flex-1 p-2 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500"

/>

<button type="submit" class="px-4 py-2 bg-blue-600 text-white rounded-lg hover:bg-blue-700 transition-colors">Add</button>

</form>

<ul class="space-y-2">

{#each todos as todo (todo.id)}

<li class="flex items-center gap-3 p-3 bg-gray-50 rounded-lg group">

<input

type="checkbox"

checked={todo.done}

on:change={() => toggleTodo(todo.id)}

class="w-5 h-5 accent-blue-600"

/>

<span class="flex-1 {todo.done ? 'line-through text-gray-400' : 'text-gray-700'}">{todo.text}</span>

<button on:click={() => removeTodo(todo.id)} class="text-red-600 opacity-0 group-hover:opacity-100 hover:text-red-800 transition-opacity">×</button>

</li>

{/each}

</ul>

</div>

Real-World Example 2: Card Component

A reusable card component with slots:

<!-- src/lib/Card.svelte -->

<script>

export let variant = 'default';

export let hoverable = false;

</script>

 

<div

class="p-6 bg-white rounded-lg border {variantClasses[variant]} {hoverable ? 'hover:shadow-lg transition-shadow' : 'shadow-md'}"

>

<slot />

</div>

 

<script context="module">

const variantClasses = {

default: 'border-gray-200',

primary: 'border-blue-200',

success: 'border-green-200',

danger: 'border-red-200'

};

</script>

 

<!-- Usage: -->

<Card variant="primary" hoverable>

<h3 class="text-lg font-semibold">Title</h3>

<p>Content goes here...</p>

</Card>

Edge Cases & Considerations

  • Reactive Classes: Use template literals for dynamic Tailwind classes.
  • Scoped Styles: Svelte scoped styles can coexist with Tailwind.
  • Transitions: Combine Svelte transitions with Tailwind animations.
  • Stores: Use Svelte stores with Tailwind reactive state classes.

Performance Considerations

  • No Virtual DOM: Svelte compiles to vanilla JS - smallest bundles.
  • Reactive Updates: Updates only change what's needed.
  • Code Splitting: SvelteKit provides automatic code splitting.
  • SSR Support: Server-side rendering with SvelteKit.

Accessibility Notes

  • Focus Management: Use Svelte's focus handling with Tailwind.
  • Keyboard Navigation: Add focus-visible classes for interactive elements.
  • ARIA: Bind ARIA attributes in Svelte components.
  • Reduced Motion: Use transition fly with reduced motion.

Cross-Browser Compatibility

  • Compilation Target: Configure Svelte for ES targets.
  • Modern Browsers: Works on all modern browsers.
  • Polyfills: Add polyfills for legacy support.
  • Testing: Test across browser versions.
