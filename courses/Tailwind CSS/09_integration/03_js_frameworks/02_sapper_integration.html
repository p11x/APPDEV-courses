Sapper + Tailwind CSS Integration

Overview

Sapper is a framework for building web applications of all sizes, with a beautiful development experience and flexible filesystem-based routing. Built on Svelte, it provides server-side rendering, code-splitting, and optimized production builds. Tailwind integrates with Sapper through PostCSS configuration and the standard rollup build process. Note: Sapper is now deprecated in favor of SvelteKit, but these integrations still work for existing projects.

Key Point: Sapper builds on Svelte's compile-time approach. Tailwind classes integrate through rollup-postcss. For new projects, prefer SvelteKit.

Standard Example: Sapper Setup

Setting up Tailwind CSS with Sapper:

// Install dependencies

npx degit sveltejs/sapper-template#rollup my-app

cd my-app && npm install

npm install -D tailwindcss postcss autoprefixer

 

// tailwind.config.js

module.exports = {

content: ['./src/**/*.svelte', './src/**/*.html'],

theme: { extend: {} },

plugins: []

};

 

// postcss.config.js

module.exports = {

plugins: {

tailwindcss: {},

autoprefixer: {}

}

};

 

// src/routes/_layout.svelte

<script>

import '../../global.css';

</script>

<slot></slot>

 

// rollup.config.js - add postcss to plugins

import postcss from 'rollup-plugin-postcss'

 

css: ({

input: 'src/global.css',

plugins: [postcss({ ... })]

})

Edge Cases & Considerations

  • Migration: Consider migrating to SvelteKit for active development.
  • Preloading: Use data-sveltekit-preload-data for fast navigation.
  • Server Hooks: Use hooks.js for server-side logic.
  • Error Pages: Create error.svelte for custom error handling.

Performance Considerations

  • Code Splitting: Automatic per-route code splitting.
  • SSR: Server-side rendering for SEO and performance.
  • Preloading: Prefetch resources on hover.
  • Service Worker: Cache assets for offline use.

Accessibility Notes

  • Focus: Manage focus after navigation.
  • Skip Links: Add skip-to-content links.
  • Reduced Motion: Respect motion preferences.
  • ARIA Live: Use aria-live for dynamic content.

Cross-Browser Notes

  • SSR Fallback: Works without JavaScript (basic HTML).
  • Modern Browsers: Requires ES2015+ support.
  • Testing: Test SSR output separately.
  • Note: SvelteKit is recommended for new projects.
