Vue.js + Tailwind CSS Integration

Overview

Vue.js integrates beautifully with Tailwind CSS through Vue CLI, Vite, or manual PostCSS configuration. Vue's single-file component (.vue) system works seamlessly with Tailwind's utility classes, and the Vue ecosystem has official Tailwind plugins for enhanced developer experience.

Key Point: Vue 3 with Vite provides the fastest development experience. The @tailwindcss/forms and @tailwindcss/typography plugins are particularly useful for Vue projects.

Standard Example: Vite + Vue Setup

Setting up Tailwind with Vue 3 using Vite:

// Install dependencies

npm install -D tailwindcss postcss autoprefixer

npx tailwindcss init -p

 

// tailwind.config.js

module.exports = {

content: [

"./index.html"

"./src/**/*.{vue,js,ts,jsx,tsx}"

],

theme: {

extend: {}

},

plugins: [],

};

 

// postcss.config.js

module.exports = {

plugins: {

tailwindcss: {},

autoprefixer: {},

}

};

 

// src/assets/main.css

@tailwind

base;

@tailwind

components;

@tailwind

utilities;

Real-World Example 1: Vue Component with Tailwind

A complete Vue 3 component using Tailwind CSS with composition API:

<!-- src/components/Card.vue -->

<script setup>

import { ref, computed } from 'vue'

 

const

props

= defineProps({

title: {

type: String,

required: true

},

variant: {

type: String,

default: 'default'

}

})

 

const

variantClasses

= computed(() => {

const classes = {

default: 'border-gray-200',

primary: 'border-blue-200 bg-blue-50',

success: 'border-green-200 bg-green-50',

danger: 'border-red-200 bg-red-50'

}

return classes[props.variant] || classes.default

})

</script>

 

<template>

<div

:class="['p-6 rounded-lg border-2', variantClasses]"

>

<h3 class="text-lg font-semibold text-gray-800 mb-2">

{{ title }}

</h3>

<slot></slot>

</div>

</template>

Real-World Example 2: Todo List Component

A responsive todo list with Vue 3 composition API and Tailwind:

<!-- src/components/TodoList.vue -->

<script setup>

import { ref } from 'vue'

 

const

newTodo

= ref('')

const

todos

= ref([

{ id: 1, text: 'Learn Vue', done: false }

])

 

const

addTodo

= () => {

if (!newTodo.value.trim()) return

todos.value.push({

id: Date.now(),

text: newTodo.value,

done: false

})

newTodo.value = ''

}

 

const

removeTodo

= (id) => {

todos.value = todos.value.filter(t => t.id !== id)

}

</script>

 

<template>

<div class="max-w-md mx-auto p-4">

<h2 class="text-2xl font-bold mb-4">Todo List</h2>

<form @submit.prevent="addTodo" class="flex gap-2 mb-4">

<input

v-model="newTodo"

type="text"

placeholder="Add new todo..."

class="flex-1 p-2 border rounded-lg"

/>

<button type="submit" class="px-4 py-2 bg-blue-600 text-white rounded-lg hover:bg-blue-700">Add</button>

</form>

<ul class="space-y-2">

<li v-for="todo in todos" :key="todo.id" class="flex items-center gap-3 p-3 bg-gray-50 rounded-lg">

<input type="checkbox" v-model="todo.done" class="w-5 h-5" />

<span :class="{'line-through text-gray-400': todo.done}" class="flex-1">{{ todo.text }}</span>

<button @click="removeTodo(todo.id)" class="text-red-600 hover:text-red-800">×</button>

</li>

</ul>

</div>

</template>

Edge Cases & Considerations

  • Reactive Classes: Use computed properties to map reactive data to Tailwind classes in Vue templates.
  • Scoped Styles: Tailwind utilities in scoped styles work fine but may conflict; use deep selectors if needed.
  • Vue Transitions: Combine Tailwind with Vue's transition classes for smooth animations.
  • Component Libraries: Many Vuetify/Element+ components can be styled with Tailwind using the utility classes.

Performance Considerations

  • Vite Dev Server: Provides instant hot module replacement for fast development iterations.
  • Lazy Loading: Use Vue's defineAsyncComponent for route-based code splitting.
  • Tree Shaking: Ensure JIT mode is enabled for optimal unused class removal.
  • Production Build: Use npm run build for minified, optimized output.

Accessibility Notes

  • Keyboard Navigation: Use focus-visible classes for interactive elements (buttons, inputs).
  • ARIA Labels: Add proper aria-label attributes alongside Tailwind classes.
  • Color Contrast: Ensure sufficient contrast ratio between text and background colors.
  • Screen Reader Text: Use sr-only class for hidden text that should be announced.

Cross-Browser Compatibility

  • Vue 3 Compatibility: Requires modern browsers with ES2015+ support.
  • Autoprefixer: PostCSS with autoprefixer handles vendor prefixes automatically.
  • CSS Custom Properties: Tailwind uses CSS variables widely; requires modern browsers.
  • Testing: Test across Chrome, Firefox, Safari, Edge for consistent rendering.
