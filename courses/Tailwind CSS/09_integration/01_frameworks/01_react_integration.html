React + Tailwind CSS Integration

Overview

Integrating Tailwind CSS with React involves several approaches: Create React App, Vite, or manual setup. The most modern approach uses PostCSS configuration or the Tailwind CLI with your build tool.

Key Point: React + Tailwind works seamlessly with any React build setup. The key is proper PostCSS configuration and import statements in your CSS files.

Edge Cases & Considerations

  • Dynamic Classes: Tailwind cannot detect classes built at runtime. Use template literals carefully or map classes to object keys.
  • JSS/CSS-in-JS: When using styled-components, configure Tailwind to work with emotion or use tailwind-styled-components.
  • PurgeCSS Conflicts: Ensure content paths in tailwind.config.js include all component files (JSX, TSX).
  • Hot Module Replacement: Vite handles HMR well; Create React App may need restarts for config changes.

Performance Considerations

  • Production Build: Always use production mode for optimized output (smaller CSS bundle).
  • JIT Mode: Just-In-Time compiler generates only used classes, dramatically reducing CSS size.
  • Code Splitting: Use dynamic imports for components that aren't needed initially.
  • Tree Shaking: Modern bundlers tree-shake unused classes with proper PostCSS setup.

Accessibility Notes

  • Focus States: Always include focus ring classes (focus:ring-2, focus:ring-offset-2) for keyboard navigation.
  • Color Contrast: Ensure text colors meet WCAG AA standards (4.5:1 ratio minimum).
  • Reduced Motion: Use motion-reduced classes for users with motion sensitivity preferences.
  • Screen Readers: Use sr-only class for text that should only be visible to assistive technology.

Cross-Browser Compatibility

  • Browser Support: Tailwind v3+ requires modern browsers with CSS custom properties support.
  • Autoprefixer: PostCSS with autoprefixer handles vendor prefixes automatically.
  • IE11: For legacy browser support, configure PostCSS with additional polyfills or use Tailwind v2.
  • Flexbox/Grid: Both are well-supported; Tailwind utilities handle fallbacks gracefully.
