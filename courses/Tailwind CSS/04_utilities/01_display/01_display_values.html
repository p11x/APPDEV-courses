Display Values in Tailwind CSS

Understanding the Display Property

The display property determines how an element is rendered in the document flow. Tailwind CSS provides utility classes for all standard CSS display values including block, inline, inline-block, flex, grid, table, and none.

Real-World Example 1: Navigation Menu


<nav class="flex justify-between items-center bg-gray-800 p-4">
    <div class="text-white font-bold">Logo</div>
    <ul class="flex gap-4">
        <li class="text-gray-300">Home</li>
        <li class="text-gray-300">About</li>
        <li class="text-gray-300">Contact</li>
    </ul>
</nav>

Output: A horizontal navigation bar with logo on left, menu items on right using flexbox.

Real-World Example 2: Card Grid Layout


<div class="grid grid-cols-3 gap-4 p-4">
    <div class="bg-white shadow p-4 rounded">Card 1</div>
    <div class="bg-white shadow p-4 rounded">Card 2</div>
    <div class="bg-white shadow p-4 rounded">Card 3</div>
</div>

Output: A 3-column grid of cards with consistent spacing.

Display Utility Classes Reference

Class CSS Property Description
block display: block Element generates a block box
inline display: inline Element generates inline box
inline-block display: inline-block Inline-level block container
flex display: flex Flexbox container
grid display: grid CSS Grid container
table display: table Table layout container
hidden display: none Element is completely hidden

Edge Cases & Considerations

Performance Notes:

  • Using display: none removes elements from the rendering tree entirely
  • Avoid频繁 toggling display properties as it triggers reflow
  • Prefer using CSS transforms for animations over display property changes

Accessibility Considerations:

  • Elements with display: none are hidden from assistive technologies
  • Use visibility: hidden to hide visually but keep available for screen readers
  • Ensure focusable elements are never hidden using display: none

Cross-Browser Notes:

  • All display utilities have excellent cross-browser support
  • CSS Grid requires IE11 prefix (-ms-grid) for full support
  • Table display values work consistently across all modern browsers
