Border Colors in Tailwind CSS

Understanding Border Colors

Border color utilities control the color of element borders. The border-* classes in Tailwind follow the same color palette system as background colors, with shades ranging from 50 to 950.

Standard Example: Border Color Classes


<div class="border-2 border-gray-300 p-4 rounded mb-4">border-gray-300</div>
<div class="border-2 border-gray-500 p-4 rounded mb-4">border-gray-500</div>
<div class="border-2 border-blue-500 p-4 rounded mb-4">border-blue-500</div>
<div class="border-2 border-red-500 p-4 rounded mb-4">border-red-500</div>
<div class="border-2 border-green-500 p-4 rounded mb-4">border-green-500</div>


<div class="border border-gray-200 p-4 rounded mb-4">border-gray-200 (subtle)</div>
<div class="border border-gray-100 p-4 rounded mb-4">border-gray-100 (very light)</div>


<div class="border-2 border-gray-800 p-4 rounded mb-4">border-gray-800 (dark)</div>
<div class="border-2 border-gray-900 p-4 rounded mb-4">border-gray-900 (darkest)</div>

Output: Elements with various border colors.

Border Color Utility Classes

Class Usage
border-{color}Standard border (1px default)
border-{color}-{shade}Full palette (50-950)
border-transparentTransparent border
border-currentInherits currentColor
border-x-{color}X-axis (left+right)
border-y-{color}Y-axis (top+bottom)
border-t-{color}Top border only
border-b-{color}Bottom border only

Accessibility & Best Practices

Focus States:

  • Use focus:border-{color} for keyboard focus indication
  • Pair border with ring for better visibility
  • Ensure 3:1 contrast ratio for focus indicators

Interactive Elements:

  • Hover darkens border by one shade
  • Focus uses brighter shade (500)
  • Error states use red-500 or red-600

Border Widths:

  • border (1px), border-2 (2px), border-4 (4px)
  • Use 2px for emphasis/buttons
  • Use border-b for dividers
