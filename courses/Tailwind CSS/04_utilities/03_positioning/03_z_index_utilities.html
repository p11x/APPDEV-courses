Z-Index Utilities in Tailwind CSS

Understanding Z-Index

The z-index property controls the stacking order of positioned elements along the z-axis. Higher values appear in front of lower values. Only works on positioned elements (relative, absolute, fixed, sticky).

Standard Example: Z-Index Classes


<div class="relative z-0 bg-blue-200 p-2">z-0 (default)</div>


<div class="relative z-auto bg-green-200 p-2">z-auto</div>


<div class="relative -z-10 bg-yellow-200 p-2">-z-10 (behind)</div>


<div class="relative z-10 bg-red-200 p-2">z-10</div>
<div class="relative z-20 bg-purple-200 p-2">z-20 (in front of z-10)</div>
<div class="relative z-30 bg-green-200 p-2">z-30 (in front of z-20)</div>


<div class="relative z-40 bg-blue-200 p-2">z-40 (highest visible)</div>

Output: Elements stack in order based on z-index value.

Real-World Example 1: Modal Stacking

<!-- Base content below modal -->
<div class="bg-gray-100 p-4">Content...</div>

<!-- Modal backdrop - z-40 -->
<div class="fixed inset-0 bg-black bg-opacity-50 z-40">
    <!-- Modal content - z-50, higher than backdrop -->
    <div class="relative bg-white rounded-lg shadow-xl z-50 max-w-md mx-auto mt-20 p-6">
        <h2 class="text-xl font-bold">Modal Content</h2>
    </div>
</div>

Output: Modal backdrop at z-40, content at z-50 ensures content above overlay.

Real-World Example 2: Navigation Dropdown

<!-- Navigation container - z-30 -->
<nav class="sticky top-0 bg-white shadow z-30">
    <div class="relative inline-block">
        <button class="px-4 py-2 bg-gray-800 text-white">Menu</button>
        
        <!-- Dropdown - z-40, appears above nav -->
        <div class="absolute right-0 mt-2 w-48 bg-white shadow-lg rounded z-40">
            <ul class="py-2">
                <li class="px-4 py-2 hover:bg-gray-100">Option</li>
            </ul>
        </div>
    </div>
</nav>

Output: Dropdown appears above other content with higher z-index.

Edge Cases & Considerations

Stacking Context:

  • position + z-index creates new stacking context
  • transform, opacity, filter also create stacking context
  • Child z-index is limited by parent stacking context

Best Practices:

  • Use z-40 for backdrop, z-50 for modal content
  • Space z-index values to allow for insertions
  • Avoid using very high values (9999)

Common Issues:

  • Child not appearing above parent - parent has lower stacking context
  • z-index not working - element not positioned
  • Negative z-index may not work in IE without position
