Top, Right, Bottom, Left Utilities in Tailwind CSS

Understanding Position Offsets

Top, right, bottom, and left utilities specify the offset position for positioned elements. They work with absolute, fixed, and relative (for initial position) positioning methods.

Standard Example: Position Offset Classes


<div class="absolute top-0 right-0 bg-red-200 p-2">top-0 right-0</div>
<div class="absolute bottom-0 left-0 bg-red-200 p-2">bottom-0 left-0</div>
<div class="absolute top-0 left-0 bg-red-200 p-2">top-0 left-0</div>
<div class="absolute bottom-0 right-0 bg-red-200 p-2">bottom-0 right-0</div>


<div class="absolute inset-0 bg-blue-200">inset-0 (all sides)</div>


<div class="absolute inset-x-0 bottom-0 bg-green-200">inset-x-0 bottom-0</div>
<div class="absolute inset-y-0 right-0 bg-purple-200">inset-y-0 right-0</div>

Output: Elements positioned at corners and edges using offset utilities.

Real-World Example 1: Floating Action Button

<div class="relative min-h-64 bg-gray-100">
    <!-- FAB positioned bottom-right with offset -->
    <button class="fixed bottom-6 right-6 
                    w-14 h-14 bg-blue-600 text-white 
                    rounded-full shadow-lg hover:shadow-xl 
                    flex items-center justify-center
                    transition-shadow">
        <svg class="w-6 h-6" fill="none" viewBox="0 0 24 24" stroke="currentColor">
            <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M12"/>
        </svg>
    </button>
</div>

Output: Fixed position FAB at bottom-right of screen.

Real-World Example 2: Dropdown Menu Positioning

<div class="relative inline-block">
    <button class="px-4 py-2 bg-gray-800 text-white rounded">Dropdown</button>
    
    <!-- Dropdown menu - positioned below trigger -->
    <div class="absolute top-full left-0 mt-2
                    w-48 bg-white shadow-lg rounded
                    border opacity-0 hidden
                    group-hover:opacity-100 group-hover:block">
        <ul class="py-2">
            <li class="px-4 py-2 hover:bg-gray-100">Option 1</li>
            <li class="px-4 py-2 hover:bg-gray-100">Option 2</li>
            <li class="px-4 py-2 hover:bg-gray-100">Option 3</li>
        </ul>
    </div>
</div>

Output: Dropdown positioned below the button triggering it.

Edge Cases & Considerations

Centering Techniques:

  • top-1/2 + transform-translate-y-1/2 for vertical centering
  • inset-0 + margin-auto for perfect centering
  • top-left-full + flex for centering

Common Mistakes:

  • Forgetting to set position on parent (defaults to static)
  • Using negative values incorrectly
  • Not considering content overflow

Mobile Considerations:

  • Consider safe-area-inset for notched devices
  • bottom may be too close to screen edge on mobile
  • Test fixed elements with mobile browser chrome
