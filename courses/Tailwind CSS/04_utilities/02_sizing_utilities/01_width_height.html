Width & Height Utilities in Tailwind CSS

Understanding Width & Height

Width and height utilities control element dimensions. Tailwind provides extensive sizing options including fixed sizes (pixels), fluid sizes (percentages), viewport units, and intrinsic sizing keywords.

Standard Example: Basic Width & Height


<div class="w-16 h-16 bg-blue-200">64px × 64px</div>


<div class="w-1/2 h-auto bg-green-200">50% Width</div>


<div class="w-screen h-16 bg-purple-200">100vw Width</div>


<div class="min-w-full max-w-xs h-12 bg-red-200">Min Full, Max XS</div>

Output: Elements sized at 64x64px, 50% width, full viewport, and constrained width.

Real-World Example 1: Avatar with Fixed Size

<!-- Profile avatar component -->
<img src="avatar.jpg" 
     alt="User Avatar" 
     class="w-12 h-12 rounded-full object-cover border-2 border-white shadow"/>

<!-- Small avatar variant -->
<img src="avatar.jpg" 
     alt="User Avatar" 
     class="w-8 h-8 rounded-full object-cover"/>

<!-- Large avatar variant -->
<img src="avatar.jpg" 
     alt="User Avatar" 
     class="w-24 h-24 rounded-full object-cover shadow-lg"/>

Output: Circular avatars at 32px (sm), 48px (default), and 96px (lg) sizes.

Real-World Example 2: Full-Width Hero Section

<!-- Hero with full viewport height -->
<section class="w-full h-screen bg-gradient-to-r from-blue-600 to-purple-600 flex items-center justify-center">
    <div class="text-center text-white">
        <h1 class="text-4xl font-bold">Welcome</h1>
        <p class="mt-4">Full-screen hero section</p>
    </div>
</section>

<!-- Content container with max-width -->
<div class="w-full max-w-4xl mx-auto px-4">
    <p>Content centered with max width</p>
</div>

Output: Full viewport hero section with centered content container.

Edge Cases & Considerations

Performance Notes:

  • Percentage heights require parent to have explicit height
  • Avoid using fixed pixel sizes for responsive layouts
  • Use h-screen with care - may cause issues on mobile browsers

Common Issues:

  • w-full fails if parent doesn't have explicit width
  • Percentage height (h-full) requires parent with h-* set
  • Content overflows when child exceeds parent bounds - use overflow utilities

Accessibility:

  • Ensure text remains readable at all sizes
  • Touch targets should be minimum 44x44px on mobile
  • Consider text scaling when using fixed widths
