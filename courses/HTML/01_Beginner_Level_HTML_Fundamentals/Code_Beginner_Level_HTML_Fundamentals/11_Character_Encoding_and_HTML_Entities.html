Complete Character Encoding Guide

Character Encoding & HTML Entities

Complete guide to displaying special characters correctly

1. Character Encoding

Character encoding specifies how characters are stored and displayed. UTF-8 is the standard encoding for modern web pages.

Setting UTF-8 Encoding:

Why UTF-8?

  • Supports all Unicode characters
  • Works with all modern browsers
  • Required for multi-language websites
  • Default encoding for HTML5

2. HTML Entities

HTML entities are codes used to represent special characters that might otherwise be interpreted as HTML.

Entity Syntax:

&entity_name;
OR
&#entity_number;

Examples:

The non-breaking space ( ) prevents line breaks between words.

  • Names: Mr. Smith

7. Best Practices

Do's:

  • Always use UTF-8 encoding ()
  • Use named entities for common characters
  • Escape <, >, and & in HTML code
  • Use   for keeping text together

Don'ts:

  • Don't use numeric entities when named ones exist
  • Don't forget to close entities with semicolon
  • Don't overuse   for spacing (use CSS instead)
