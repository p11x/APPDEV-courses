## Theoretical Background

Cross-validation is a resampling method used to evaluate machine learning models on limited data. It estimates how well a model will generalize to independent datasets. The key idea is to repeatedly split the data into training and validation sets to obtain reliable performance estimates.

### Why Cross-Validation Matters

1. **有限数据**: When data is limited, a single train-test split may not represent the true model performance
2. **减少过拟合风险**: Using all data for both training and testing leads to overly optimistic estimates
3. **模型选择**: Cross-validation helps select the best model or hyperparameters

### How K-Fold Cross-Validation Works

1. **分割数据**: Randomly divide the dataset into k equal-sized folds (or subsets)
2. **迭代训练**: For each iteration i from 1 to k:
   - Use fold i as the validation set
   - Use the remaining k-1 folds as the training set
3. **计算指标**: Calculate the performance metric (e.g., accuracy) on the validation set
4. **平均结果**: Average the performance metrics across all k iterations

#### The Process

```
Fold 1:    [val]    [train] [train] [train]
Fold 2:    [train] [val]    [train] [train]
Fold 3:    [train] [train] [val]    [train]
Fold 4:    [train] [train] [train] [val]
Fold 5:    [train] [train] [train] [train] [val]
```

### Advantages of K-Fold CV

- **Reduced Variance**: Each sample is used for validation exactly once, reducing variance in performance estimates
- **Efficient Use of Data**: All data is used for both training and validation
- **Better Estimate**: Provides more reliable estimate of model performance than single split

### Disadvantages of K-Fold CV

- **Computationally Expensive**: Must train the model k times
- **Not Suitable for Time Series**: Temporal dependencies are ignored
- **Can Be Biased**: When class distribution is uneven

### Variations

#### Leave-One-Out Cross-Validation (LOOCV)
- Special case where k = n (number of samples)
- Uses n-1 samples for training, 1 sample for validation
- Maximum training data each iteration
- Very computationally expensive for large datasets

#### Repeated K-Fold
- Run k-fold multiple times with different random splits
- Reduces variance in estimates
- More stable performance estimates

### Step 2: Basic K-Fold Cross-Validation

# Load the Iris dataset
data(iris)

# Set seed for reproducibility
set.seed(42)

# Define training control
# Method: k-fold cross-validation
# Number: 10 folds
train_control <- trainControl(method = "cv", number = 10)

# Train the model
# Using glm (logistic regression) for classification
model_cv <- train(Species ~ ., 
                 data = iris,
                 method = "glm",
                 family = "binomial",
                 trControl = train_control)

### Example 2: Hyperparameter Tuning for SVM

This example shows hyperparameter tuning with k-fold CV.

```r
# Create synthetic data
set.seed(456)
n <- 200

data_tune <- data.frame(
  x1 = rnorm(n, mean = 0),
  x2 = rnorm(n, mean = 0),
  y = factor(ifelse(rnorm(n) > 0, "A", "B"))
)

# Add non-linear separation
data_tune$x1[data_tune$y == "B"] <- data_tune$x1[data_tune$y == "B"] + 2
data_tune$x2[data_tune$y == "B"] <- data_tune$x2[data_tune$y == "B"] + 2

# Set seed
set.seed(42)

# Define control
ctrl <- trainControl(method = "cv", number = 5)

# Define tune grid
tune_grid <- expand.grid(
  C = c(0.1, 1, 10, 100),
  sigma = c(0.01, 0.1, 1)
)

# Train SVM with tuning
svm_tune <- train(y ~ ., 
                  data = data_tune,
                  method = "svmRadial",
                  preProcess = c("center", "scale"),
                  trControl = ctrl,
                  tuneGrid = tune_grid)

# View best parameters
print(svm_tune$bestTune)
#    C sigma
# 7  10   0.1

# Plot tuning results
plot(svm_tune)
```

## Best Practices and Common Pitfalls

### Best Practices

1. **Use 5 or 10 Folds**: Standard choices that balance bias and variance
2. **Set Random Seeds**: Always set seeds for reproducibility
3. **Preprocess Inside CV**: Any preprocessing should be done inside the CV loop to avoid data leakage
4. **Use Same Folds for Comparison**: When comparing models, use same folds for fair comparison
5. **Consider Stratified Sampling**: For classification with imbalanced classes

### Common Pitfalls

1. **Data Leakage**: Applying preprocessing (like scaling) to entire dataset before CV
2. **Ignoring Class Imbalance**: Not using stratified sampling for imbalanced data
3. **Too Few Folds**: Too few folds leads to high variance in estimates
4. **Too Many Folds**: Too many folds leads to high bias and longer computation
5. **Feature Selection in CV**: Must do feature selection inside CV to avoid leakage

## Performance Considerations

### Choosing K

| K Value | Bias | Variance | Computation Time |
|---------|------|----------|------------------|
| 2 | High | Low | Very Fast |
| 5 | Medium | Medium | Fast |
| 10 | Low | Medium | Moderate |
| n (LOOCV) | Very Low | High | Slow |

### Computational Complexity

- Training time: O(k × training_time_per_fold)
- Memory: O(k × data_size)
- For large datasets, consider smaller k or sampling

## Related Concepts

- **Stratified K-Fold**: Maintains class distribution in each fold
- **Bootstrap Validation**: Uses random sampling with replacement
- **Repeated Cross-Validation**: Multiple rounds of CV for stability
- **Nested CV**: CV inside CV for model selection and evaluation
