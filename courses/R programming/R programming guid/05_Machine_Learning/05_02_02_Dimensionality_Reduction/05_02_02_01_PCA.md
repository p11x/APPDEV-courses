## Theoretical Background

Principal Component Analysis (PCA) is one of the most widely used dimensionality reduction techniques. It transforms the data to a new coordinate system defined by the principal components, which are orthogonal directions of maximum variance in the data.

### The Core Idea

PCA finds new axes (principal components) that:
1. Capture the maximum variance in the data
2. Are orthogonal (perpendicular) to each other
3. Are linear combinations of the original features

### Mathematical Foundation

#### Covariance Matrix

The covariance matrix captures relationships between features:
$$Cov(X, Y) = \frac{\sum_{i=1}^{n}(x_i - \bar{x})(y_i - \bar{y})}{n-1}$$

For p features, the covariance matrix is p × p.

#### Eigenvalue Decomposition

PCA finds eigenvalues and eigenvectors of the covariance matrix:
- **Eigenvalues**: Amount of variance captured by each component
- **Eigenvectors**: Direction of each principal component

The eigenvectors form the new axes, and eigenvalues tell us how important each axis is.

#### Steps in PCA

1. **Standardize Data**: Subtract mean, divide by standard deviation
2. **Compute Covariance Matrix**: Calculate covariance of all feature pairs
3. **Eigenvalue Decomposition**: Find eigenvalues and eigenvectors
4. **Sort by Eigenvalues**: Order components by variance explained
5. **Project Data**: Transform data onto new axes

### Why PCA Works

The key insight is that data often has redundancy - features are correlated. By finding directions of maximum variance, PCA can:
- Reduce dimensionality with minimal information loss
- Remove correlated features
- Focus on the most important directions

### When to Use PCA

- **High-dimensional data**: When you have many features
- **Correlated features**: When features are linearly correlated
- **Visualization**: When you need to visualize in 2D or 3D
- **Preprocessing**: Before algorithms that don't handle high dimensions well

## Step-by-Step Implementation

### Step 1: Install and Load Required Packages

```r
install.packages("factoextra")
install.packages("stats")
install.packages("ggplot2")

library(factoextra)
library(stats)
library(ggplot2)
```

### Step 2: Prepare Data

```r
# Use the iris dataset
data(iris)

# Select numerical features only
features <- iris[, -5]

# Check data
head(features)
dim(features)

# Scale the data (critical for PCA)
features_scaled <- scale(features)

# Verify scaling
colMeans(features_scaled)
apply(features_scaled, 2, sd)
```

### Step 3: Perform PCA

```r
# Method 1: Using prcomp
pca_result <- prcomp(features_scaled)

# View summary
summary(pca_result)
# Importance of components:
#                           PC1    PC2    PC3    PC4
# Standard deviation     1.706  0.960 0.383  0.144
# Proportion of Variance 0.730 0.229 0.037 0.005
# Cumulative Proportion  0.730 0.958 0.994 1.000

# View rotation (eigenvectors)
pca_result$rotation
#           PC1         PC2        PC3        PC4
# Sepal.Length  0.521 -0.377  0.720  0.261
# Sepal.Width  -0.269 -0.925 -0.244  0.124
# ...

# View standard deviations
pca_result$sdev

# Method 2: Using princomp
pca_result2 <- princomp(features_scaled)
print(pca_result2)
```

### Step 4: Interpret Results

```r
# Eigenvalues (variance of each PC)
eigenvalues <- pca_result$sdev^2
print(eigenvalues)

# Proportion of variance explained
prop_variance <- eigenvalues / sum(eigenvalues)
print(prop_variance)

# Cumulative variance
cum_variance <- cumsum(prop_variance)
print(cum_variance)

# Data frame for plotting
var_df <- data.frame(
  PC = 1:4,
  Eigenvalue = eigenvalues,
  Proportion = prop_variance,
  Cumulative = cum_variance
)
print(var_df)
```

### Step 5: Visualize Results

```r
# Scree plot
fviz_eig(pca_result,

plot(prop_variance, type = "b",
```

## Code Examples

### Example 1: PCA for Data Visualization

This example demonstrates using PCA to visualize high-dimensional data.

```r
# Use wine dataset
data(wine, package = "rattle")
wine_df <- wine

# Select numerical features
wine_features <- wine_df[, -1]  # Remove Type column

# Scale features
wine_scaled <- scale(wine_features)

# Perform PCA
wine_pca <- prcomp(wine_scaled)

# Variance explained
var_explained <- wine_pca$sdev^2 / sum(wine_pca$sdev^2)

# Plot first 2 PCs colored by wine type
wine_scores <- wine_pca$x

plot_df <- data.frame(
  PC1 = wine_scores[, 1],
  PC2 = wine_scores[, 2],
  Type = wine_df$Type
)

ggplot(plot_df, aes(x = PC1, y = PC2, color = factor(Type))) +
  geom_point(alpha = 0.7) +
  ggtitle("Wine Data - PCA Visualization") +
  labs(color = "Wine Type") +
  theme_minimal()

# How many PCs to explain 90% variance?
ninety_pct <- which(cumsum(var_explained) >= 0.9)[1]
cat("Components for 90% variance:", ninety_pct, "\n")
```

### Example 2: PCA for Preprocessing

This example shows using PCA as preprocessing for classification.

```r
# Use Pima Indians Diabetes data
data(PimaIndiansDiabetes, package = "mlbench")
pima <- PimaIndiansDiabetes
colnames(pima)[9] <- "Outcome"

# Split features and target
pima_features <- pima[, -9]

# Scale and PCA
pima_scaled <- scale(pima_features)
pima_pca <- prcomp(pima_scaled)

# Keep first 3 PCs
pima_pca_data <- pima_pca$x[, 1:3]

# Combine with target
pima_train <- data.frame(pima_pca_data, Outcome = pima$Outcome)

# Split data
set.seed(42)
train_idx <- sample(1:nrow(pima_train), 0.7 * nrow(pima_train))
train_pima <- pima_train[train_idx, ]
test_pima <- pima_train[-train_idx, ]

# Train classifier
library(caret)
model <- train(Outcome ~ ., data = train_pima, method = "glm", 
               family = "binomial")

# Predictions and evaluation
predictions <- predict(model, test_pima)
cm <- confusionMatrix(predictions, test_pima$Outcome)
print(cm)

# Compare with using original features
model_orig <- train(Outcome ~ ., data = pima[train_idx, ], method = "glm",
                    family = "binomial")
pred_orig <- predict(model_orig, pima[-train_idx, ])
cm_orig <- confusionMatrix(pred_orig, pima[-train_idx, ]$Outcome)

cat("\nAccuracy with PCA:", cm$overall["Accuracy"], "\n")
cat("Accuracy without PCA:", cm_orig$overall["Accuracy"], "\n")
```

## Best Practices and Common Pitfalls

### Best Practices

1. **Always Scale Data**: PCA is variance-based; features with larger scales dominate
2. **Check Variance Explained**: Use scree plot to decide number of components
3. **Interpret Components**: Look at loadings to understand what each PC captures
4. **Consider Correlation**: If variables aren't correlated, PCA won't help much

### Common Pitfalls

1. **Not Scaling Data**: Features with large ranges dominate
2. **Keeping Too Many Components**: Includes noise
3. **Keeping Too Few**: Loses important information
4. **Interpreting PC Loadings Incorrectly**: Correlation ≠ causation
5. **Ignoring Outliers**: Can distort PCA results

## Performance Considerations

### How Many Components to Keep

| Method | Description |
|--------|-------------|
| Kaiser | Keep eigenvalues > 1 |
| Elbow | Keep where scree plot flattens |
| 80-90% | Keep enough to explain 80-90% variance |
| Cross-validation | Optimal for prediction tasks |

### Computational Complexity

- **Time**: O(p³) for eigenvalue decomposition where p = features
- **Space**: O(n × p) for data, O(p²) for covariance
- For n >> p, can use SVD for faster computation

## Related Concepts

- **SVD**: Singular value decomposition, related to PCA
- **Factor Analysis**: Similar but with different assumptions
- **t-SNE**: Non-linear dimensionality reduction for visualization
- **UMAP**: Non-linear, faster than t-SNE
