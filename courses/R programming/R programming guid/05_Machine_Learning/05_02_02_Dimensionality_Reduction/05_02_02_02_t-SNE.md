## Theoretical Background

t-SNE (t-Distributed Stochastic Neighbor Embedding) is a non-linear dimensionality reduction technique particularly effective for visualizing high-dimensional data in 2 or 3 dimensions. Unlike PCA, which preserves global structure, t-SNE focuses on preserving local neighborhood relationships.

### The Core Idea

t-SNE converts distances between points in high-dimensional space to probabilities, then finds a low-dimensional representation that preserves these probabilities.

### How t-SNE Works

#### Step 1: Define Neighborhood Probabilities (High-Dimensional)

For each point i, calculate similarity to point j:

$$p_{j|i} = \frac{\exp(-||x_i - x_j||^2 / 2\sigma_i^2)}{\sum_{k \neq i} \exp(-||x_i - x_k||^2 / 2\sigma_i^2)}$$

The variance σ_i is chosen so that the perplexity matches a specified value (similar to number of neighbors).

#### Step 2: Define Joint Probabilities

Combine symmetric pairs:

$$p_{ij} = \frac{p_{j|i} + p_{i|j}}{2n}$$

#### Step 3: Define Low-Dimensional Similararities

Map to low-dimensional space (typically 2D) and calculate:

$$q_{ij} = \frac{(1 + ||y_i - y_j||^2)^{-1}}{\sum_{k \neq l} (1 + ||y_k - y_l||^2)^{-1}}$$

#### Step 4: Minimize Divergence

Minimize Kullback-Leibler divergence between P and Q:

$$KL(P || Q) = \sum_{i,j} p_{ij} \log \frac{p_{ij}}{q_{ij}}$$

Use gradient descent to find optimal y positions.

### Why t-SNE Works

1. **Student-t Distribution**: Uses heavy-tailed distribution in low-dim space to spread clusters
2. **Local Preservation**: Preserves neighborhood relationships
3. **Non-linear**: Can discover complex structures PCA cannot

### Key Hyperparameters

1. **Perplexity**: Balance between local and global aspects (typically 5-50)
2. **Learning Rate**: Speed of gradient descent (typically 10-1000)
3. **Iterations**: Number of gradient descent steps (typically 1000)
4. **Theta**: Speed/accuracy trade-off for large datasets

## Step-by-Step Implementation

### Step 1: Install and Load Required Packages

```r
install.packages("Rtsne")
install.packages("ggplot2")
install.packages("factoextra")

library(Rtsne)
library(ggplot2)
library(factoextra)
```

### Step 2: Prepare Data

```r
# Use iris dataset
data(iris)

# Get features
features <- iris[, -5]

# Scale data (important for distance-based methods)
features_scaled <- scale(features)

# Remove any NaN values
features_scaled <- na.omit(features_scaled)

dim(features_scaled)
```

### Step 3: Apply t-SNE

```r
# Basic t-SNE
set.seed(42)
tsne_result <- Rtsne(features_scaled, 
                     dims = 2,           # Output dimensions
                     perplexity = 30,    # Number of neighbors
                     theta = 0.5,       # Speed/accuracy trade-off
                     max_iter = 1000,   # Maximum iterations
                     verbose = TRUE)

# View result
print(tsne_result)
# $Y: The 2D embedding coordinates
# $iter: Number of iterations run
# $stress: KL divergence

# Get embeddings
head(tsne_result$Y)
```

### Step 4: Visualize t-SNE Results

```r
# Create data frame for plotting
tsne_df <- data.frame(
  Dim1 = tsne_result$Y[, 1],
  Dim2 = tsne_result$Y[, 2],
  Species = iris$Species
)

# Plot
ggplot(tsne_df, aes(x = Dim1, y = Dim2, color = Species)) +
  geom_point() +
  ggtitle("t-SNE Visualization of Iris Data") +
  theme_minimal()

# Using factoextra
fviz_tsne(tsne_result,
          habillage = iris$Species,
          addEllipses = TRUE,
          main = "t-SNE Visualization")
```

### Step 5: Tune Perplexity

```r
# Try different perplexity values
set.seed(42)
perplexities <- c(5, 10, 30, 50)

par(mfrow = c(2, 2))

for(p in perplexities) {
  tsne_p <- Rtsne(features_scaled, dims = 2, perplexity = p, verbose = FALSE)
  plot(tsne_p$Y, main = paste("Perplexity =", p), 
       col = as.numeric(iris$Species), pch = 19, cex = 0.8)
}
```

### Step 6: Compare with PCA

```r
# Perform PCA first
pca_result <- prcomp(features_scaled)

# Create comparison plot
par(mfrow = c(1, 2))

# PCA
pca_df <- data.frame(
  PC1 = pca_result$x[, 1],
  PC2 = pca_result$x[, 2],

plot(pca_df$PC1, pca_df$PC2, col = as.numeric(pca_df$Species),
```

## Code Examples

### Example 1: MNIST Digit Recognition

This example shows t-SNE on image data.

```r
# Create synthetic digit data (simplified)
set.seed(123)
n <- 300

# Create 8x8 pixel images of 3 "digits"
digit_data <- list()

for(digit in 1:3) {
  base_pattern <- matrix(0, 8, 8)
  # Create pattern based on digit
  if(digit == 1) {
    base_pattern[3:6, ] <- 1  # vertical line
  } else if(digit == 2) {
    base_pattern[, 3:6] <- 1  # horizontal line
  } else {
    base_pattern[3:6, 3:6] <- 1  # square
  }
  
  # Add noise
  patterns <- replicate(n/3, {
    base_pattern + rnorm(64, 0, 0.3)
  })
  digit_data[[digit]] <- t(patterns)
}

# Combine
digit_matrix <- do.call(rbind, digit_data)
digit_labels <- rep(c(1, 2, 3), each = n/3)

# Apply t-SNE
set.seed(42)
tsne_digits <- Rtsne(digit_matrix, dims = 2, perplexity = 30, verbose = FALSE)

# Plot
plot_df <- data.frame(
  Dim1 = tsne_digits$Y[, 1],
  Dim2 = tsne_digits$Y[, 2],
  digit = factor(digit_labels)
)

ggplot(plot_df, aes(x = Dim1, y = Dim2, color = digit)) +
  geom_point() +
  ggtitle("t-SNE of Digit Patterns") +
  theme_minimal()
```

### Example 2: Single Cell RNA-seq Visualization

This example demonstrates t-SNE for single-cell data.

```r
# Create synthetic single-cell data
set.seed(456)
n_cells <- 200
n_genes <- 100

# Different cell types with different gene expression patterns
cell_data <- list()

# Cell type 1: marker genes 1-20 highly expressed
type1 <- matrix(rnorm(n_cells/3 * n_genes, 0, 1), n_cells/3, n_genes)
type1[, 1:20] <- type1[, 1:20] + 3

# Cell type 2: marker genes 21-40 highly expressed  
type2 <- matrix(rnorm(n_cells/3 * n_genes, 0, 1), n_cells/3, n_genes)
type2[, 21:40] <- type2[, 21:40] + 3

# Cell type 3: marker genes 41-60 highly expressed
type3 <- matrix(rnorm(n_cells/3 * n_genes, 0, 1), n_cells/3, n_genes)
type3[, 41:60] <- type3[, 41:60] + 3

sc_data <- rbind(type1, type2, type3)
cell_type <- rep(c("Type A", "Type B", "Type C"), each = n_cells/3)

# Scale
sc_scaled <- scale(sc_data)

# Apply t-SNE
set.seed(42)
tsne_sc <- Rtsne(sc_scaled, dims = 2, perplexity = 30, verbose = FALSE)

# Plot
plot_df <- data.frame(
  Dim1 = tsne_sc$Y[, 1],
  Dim2 = tsne_sc$Y[, 2],
  CellType = cell_type
)

ggplot(plot_df, aes(x = Dim1, y = Dim2, color = CellType)) +
  geom_point(alpha = 0.7) +
  ggtitle("t-SNE of Single-Cell Data") +
  theme_minimal()
```

## Best Practices and Common Pitfalls

### Best Practices

1. **Perplexity Around 30**: Works well for most datasets
2. **Set Seed**: t-SNE has stochastic elements
3. **Use Scaled Data**: Distance-based, so scale features
4. **Large Perplexity for Large Data**: More neighbors to consider
5. **Multiple Runs**: Sometimes different runs reveal different structures

### Common Pitfalls

1. **Wrong Perplexity**: Too small = fragmentation, too large = squeezing
2. **Random Interpretation**: t-SNE distances aren't absolute
3. **Global Structure Lost**: Only preserves local neighborhoods
4. **Not Reproducible**: Different runs can give different results
5. **Slow for Large Data**: Consider using theta parameter

## Performance Considerations

### Hyperparameter Guidelines

| Parameter | Small Value | Large Value |
|-----------|-------------|-------------|
| Perplexity | Fragmented clusters | Squeezed clusters |
| Learning Rate | Slow convergence | May not converge |
| Iterations | May not converge | No improvement |

### Computational Complexity

- **Time**: O(n² log n) to O(n²) per iteration
- **Space**: O(n × original_dim + n × target_dim)
- For very large data, consider using FASTER or simpler methods

### When to Use t-SNE

- **Visualization**: When you need 2D/3D visualization
- **Cluster Exploration**: When discovering clusters is the goal
- **Complex Structures**: When PCA fails to show structure
- **Medium Datasets**: n = 100 to 100,000

### When NOT to Use t-SNE

- **Very Large Data**: Too slow
- **Prediction Tasks**: t-SNE doesn't generalize
- **Preserving Global Structure**: Use PCA or UMAP
- **Reproducibility Concerns**: Results vary

## Related Concepts

- **UMAP**: Faster, preserves both local and global structure
- **PCA**: Linear, preserves global structure
- **LargeVis**: Similar to t-SNE but faster
- **Autoencoders**: Non-linear dimensionality reduction
