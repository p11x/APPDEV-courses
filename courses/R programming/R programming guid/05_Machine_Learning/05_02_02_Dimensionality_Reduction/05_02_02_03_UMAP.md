## Theoretical Background

UMAP (Uniform Manifold Approximation and Projection) is a non-linear dimensionality reduction technique that excels at preserving both local and global structure. It's faster than t-SNE and often produces better results, making it increasingly popular for single-cell analysis and other complex data visualization.

### The Core Idea

UMAP uses concepts from manifold theory and topological data analysis to construct a high-dimensional graph and then optimize a low-dimensional representation that preserves this structure.

### How UMAP Works

#### Step 1: Construct Fuzzy Simplicial Set

1. Find k nearest neighbors for each point
2. Create weighted edges based on distance
3. Use fuzzy logic to create membership weights

#### Step 2: Optimize Low-Dimensional Embedding

1. Initialize random positions in low dimensions
2. Minimize cross-entropy between high-dim and low-dim graphs
3. Use stochastic gradient descent

### Key Advantages of UMAP

1. **Faster than t-SNE**: Typically 10-100x faster
2. **Preserves Global Structure**: Better than t-SNE at maintaining global relationships
3. **Better Scalability**: Works well on larger datasets
4. **Reproducible**: Less stochastic than t-SNE with proper seeding

### Key Hyperparameters

1. **n_neighbors**: Balance between local vs. global structure (5-50 typical)
2. **min_dist**: How tightly points can be packed (0.05-0.5 typical)
3. **metric**: Distance metric to use (euclidean, manhattan, cosine)
4. **n_components**: Number of output dimensions (2-3 for visualization)

## Step-by-Step Implementation

### Step 1: Install and Load Required Packages

```r
install.packages("umap")
install.packages("ggplot2")
install.packages("factoextra")

library(umap)
library(ggplot2)
library(factoextra)
```

### Step 2: Prepare Data

```r
# Use iris dataset
data(iris)

# Get numerical features
features <- iris[, -5]

# Scale data (UMAP is distance-based)
features_scaled <- scale(features)

# Verify no missing values
sum(is.na(features_scaled))
```

### Step 3: Apply UMAP

```r
# Default UMAP
set.seed(42)
umap_result <- umap(features_scaled)

# View result
print(umap_result)
# $layout: The embedding coordinates
# $config: Configuration used

# Get coordinates
head(umap_result$layout)
```

### Step 4: Visualize Results

```r
# Create data frame for plotting
umap_df <- data.frame(
  Dim1 = umap_result$layout[, 1],
  Dim2 = umap_result$layout[, 2],
  Species = iris$Species
)

# Plot
ggplot(umap_df, aes(x = Dim1, y = Dim2, color = Species)) +
  geom_point() +
  ggtitle("UMAP Visualization of Iris Data") +
  theme_minimal()

# Using factoextra
fviz_umap(umap_result,
          habillage = iris$Species,
          addEllipses = TRUE,
          main = "UMAP Visualization")
```

### Step 5: Tune Hyperparameters

```r
# Vary n_neighbors
set.seed(42)
neighbors <- c(5, 15, 30, 50)

par(mfrow = c(2, 2))

for(n in neighbors) {
  umap_n <- umap(features_scaled, n_neighbors = n, verbose = FALSE)
  plot(umap_n$layout, main = paste("n_neighbors =", n),
       col = as.numeric(iris$Species), pch = 19, cex = 0.8)
}

# Vary min_dist
par(mfrow = c(2, 2))
set.seed(42)

for(m in c(0.05, 0.1, 0.3, 0.5)) {
  umap_m <- umap(features_scaled, min_dist = m, verbose = FALSE)
  plot(umap_m$layout, main = paste("min_dist =", m),
       col = as.numeric(iris$Species), pch = 19, cex = 0.8)
}
```

## Code Examples

### Example 1: Single Cell RNA-seq Data

This example shows UMAP for single-cell analysis.

```r
# Create synthetic single-cell data
set.seed(789)
n_cells <- 500
n_genes <- 200

# Three cell types with different signatures
type1 <- matrix(rnorm(n_cells/3 * n_genes), n_cells/3, n_genes)
type1[, 1:30] <- type1[, 1:30] + 4

type2 <- matrix(rnorm(n_cells/3 * n_genes), n_cells/3, n_genes)
type2[, 31:60] <- type2[, 31:60] + 4

type3 <- matrix(rnorm(n_cells/3 * n_genes), n_cells/3, n_genes)
type3[, 61:90] <- type3[, 61:90] + 4

# Combine
sc_data <- rbind(type1, type2, type3)
cell_type <- rep(c("Cell Type A", "Cell Type B", "Cell Type C"), each = n_cells/3)

# Scale
sc_scaled <- scale(sc_data)

# Apply UMAP
set.seed(42)
umap_sc <- umap(sc_scaled, n_neighbors = 30, min_dist = 0.3, verbose = FALSE)

# Plot
plot_df <- data.frame(
  Dim1 = umap_sc$layout[, 1],
  Dim2 = umap_sc$layout[, 2],
  CellType = cell_type
)

ggplot(plot_df, aes(x = Dim1, y = Dim2, color = CellType)) +
  geom_point(alpha = 0.6) +
  ggtitle("UMAP of Single-Cell Data") +
  theme_minimal()
```

### Example 2: UMAP as Preprocessing

This example demonstrates using UMAP as a feature transformation.

```r
# Load Pima Indians data
data(PimaIndiansDiabetes, package = "mlbench")
pima <- PimaIndiansDiabetes
colnames(pima)[9] <- "Outcome"

# Split features and target
pima_features <- pima[, -9]

# Scale
pima_scaled <- scale(pima_features)

# Apply UMAP for dimensionality reduction
set.seed(42)
umap_pima <- umap(pima_scaled, n_components = 3, verbose = FALSE)

# Get UMAP features
umap_features <- umap_pima$layout

# Combine with target
pima_umap <- data.frame(umap_features, Outcome = pima$Outcome)

# Split
set.seed(42)
train_idx <- sample(1:nrow(pima_umap), 0.7 * nrow(pima_umap))
train_pima <- pima_umap[train_idx, ]
test_pima <- pima_umap[-train_idx, ]

# Train classifier
library(caret)
model_umap <- train(Outcome ~ ., data = train_pima, method = "glm", 
                   family = "binomial")

# Evaluate
pred_umap <- predict(model_umap, test_pima)
cm_umap <- confusionMatrix(pred_umap, test_pima$Outcome)

cat("Accuracy with UMAP features:", cm_umap$overall["Accuracy"], "\n")

# Compare with original features
model_orig <- train(Outcome ~ ., data = pima[train_idx, ], method = "glm",
                    family = "binomial")
pred_orig <- predict(model_orig, pima[-train_idx, ])
cm_orig <- confusionMatrix(pred_orig, pima[-train_idx, ]$Outcome)

cat("Accuracy with original features:", cm_orig$overall["Accuracy"], "\n")
```

## Best Practices and Common Pitfalls

### Best Practices

1. **n_neighbors**: 15-30 works well for most datasets
2. **min_dist**: 0.1 for distinct clusters, 0.3 for tighter clusters
3. **Scale Data**: Always scale before UMAP
4. **Set Seed**: For reproducibility

### Common Pitfalls

1. **No Scaling**: Distances are affected by different scales
2. **Wrong n_neighbors**: Too small = fragmented, too large = ignores local structure
3. **min_dist Too Small**: Points overlap too much
4. **Ignoring Metric**: Different metrics work better for different data types

## Performance Considerations

### UMAP vs t-SNE Comparison

| Aspect | UMAP | t-SNE |
|--------|------|-------|
| Speed | Faster | Slower |
| Global Structure | Better preserved | Local only |
| Reproducibility | More consistent | Varies |
| Memory | More efficient | Memory intensive |
| Parameters | Fewer to tune | More critical |

### When to Use UMAP

- **Large Datasets**: When t-SNE is too slow
- **Global Structure**: When preserving both local and global is important
- **Cluster Analysis**: For identifying cell types, etc.
- **Feature Reduction**: As preprocessing for ML

### When NOT to Use UMAP

- **Very Small Data**: May not have enough neighbors
- **Linear Relationships**: PCA is faster and sufficient
- **Deterministic Results Needed**: Consider other methods

## Related Concepts

- **t-SNE**: Similar but slower, local structure only
- **PCA**: Linear, global structure preserved
- **TrimAP**: Topological approach
- **LargeVis**: Alternative to t-SNE
