# Documentation in R

This chapter covers documenting R code using roxygen2, README files, and vignettes.

## 1. roxygen2 Documentation

roxygen2 generates documentation from special comment blocks in source code.

### Basic roxygen2 Tags

```r
#' Calculate Mean Value
#'
#' Computes the arithmetic mean of a numeric vector.
#'
#' @param x A numeric vector
#' @param na.rm Logical; if TRUE, remove NA values
#' @return The mean value as a numeric scalar
#' @examples
#' calculate_mean(c(1, 2, 3, NA), na.rm = TRUE)
calculate_mean <- function(x, na.rm = FALSE) {
  if (!is.numeric(x)) {
    stop("x must be numeric")
  }
  mean(x, na.rm = na.rm)
}
```

### Common roxygen2 Tags

```r
#' Title (one line, no period)
#'
#' @param param_name Description of parameter
#' @param another_param Another parameter
#'
#' @return What the function returns
#'
#' @examples
#' # Basic usage
#' my_function(1, 2)
#'
#' # With named arguments
#' my_function(x = 1, y = 2)
#'
#' @export Makes function available to users
#' @importFrom Imports from other packages
#' @rawNamespace Export without NAMESPACE
```

### Documenting a Complete Function

```r
#' Filter Data by Condition
#'
#' Subsets a data frame to include only rows matching 
#' the specified condition.
#'
#' @param data A data frame to filter
#' @param condition A logical expression in terms of 
#'   column names (e.g., "value > 0")
#' @return A filtered data frame
#' @export
#' @importFrom dplyr filter
#' @examples
#' df <- data.frame(x = 1:5, y = c(TRUE, FALSE, TRUE, FALSE, TRUE))
#' filter_condition(df, "y == TRUE")
filter_condition <- function(data, condition) {
  if (!is.data.frame(data)) {
    stop("data must be a data frame")
  }
  dplyr::filter(data, eval(parse(text = condition)))
}
```

### Documenting a Dataset

```r
#' Example Dataset: Customer Orders
#'
#' A sample dataset containing customer order information.
#'
#' @format A data frame with 100 rows and 5 columns:
#' \describe{
#'   \item{order_id}{Unique order identifier}
#'   \item{customer_id}{Customer identifier}
#'   \item{order_date}{Date of order}
#'   \item{total}{Order total in dollars}
#'   \item{status}{Order status (pending, complete, cancelled)}
#' }
#' @source Generated for example purposes
data("customer_orders")
```

### Documenting S3 Methods

```r
#' Print Method for Custom Class
#'
#' @param x An object of class my_class
#' @param ... Additional arguments (passed to NextMethod)
#' @method print my_class
#' @export
print.my_class <- function(x, ...) {
  cat("Custom class with value:", x$value, "\n")
}
```

## 2. Building Documentation

### Generating Documentation Files

```r
# Install roxygen2
install.packages("roxygen2")
library(roxygen2)

# Document a package
# In your package directory:
roxygen2::roxygenise()

# Or from command line:
# R CMD roxygen2 mypackage

# This creates/manupdates:
# - man/*.Rd files (in older approach)
# - Creates DESCRIPTION from roxygen comments (new approach)
```

### Package Description

```r
# DESCRIPTION file
Package: mypackage
Type: Package
Title: What the Package Does
Version: 0.1.0
Authors@R: person("First", "Last", email = "email@domain.com", role = c("aut", "cre"))
Description: A longer description of what the package does.
License: MIT
Encoding: UTF-8
LazyData: true
Imports: 
    dplyr,
    ggplot2
Suggests:
    testthat,
    knitr
RoxygenNote: 7.2.3
```

### NAMESPACE File

```r
# NAMESPACE (auto-generated)
# Generated by roxygen2
export("my_function")
export("another_function")
exportMethods("print")
importFrom(dplyr, filter, mutate, select)
importFrom(ggplot2, ggplot, aes, geom_point)
```

## 4. Vignettes

### Creating Vignettes

```r
# Create using RStudio
# File > New File > R Markdown > From Template > Package Vignette

# vignette.Rmd
---
title: "Getting Started"
author: "Author Name"
date: "`r Sys.Date()`"
output: rmarkdown::html_vignette
vignette: >
  %\VignetteIndexEntry{Getting Started}
  %\VignetteEngine{rmarkdown::render}
  %\Sweave_driver{gtable:: SweaveDriver}
---

## 5. Code Comments vs Documentation

### What to Document

```r
# Good: explains why (not in roxygen)
# Using log transformation because data spans 
# multiple orders of magnitude
data$log_value <- log(data$value)

# Good: complex algorithm reference
# Implements Kalman filter as described in
# Smith et al. (2020), equation 12

# Document in code: tricky edge cases
# Note: handles empty data frames correctly
# Edge case: returns NA for single-element vectors
```

### When NOT to Comment

```r
# Unnecessary comments
x <- x + 1  # Add 1 to x

# Obvious code
df <- select(df, x, y, z)  # Select columns
```

### Inline Documentation

```r
# Helper: computes standard error
# Returns NA when n < 2
compute_se <- function(x) {
  if (length(x) < 2) return(NA_real_)
  sd(x) / sqrt(length(x))
}
```
