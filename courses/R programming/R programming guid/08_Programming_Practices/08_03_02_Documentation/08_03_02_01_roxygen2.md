# roxygen2 Documentation

## Learning Objectives

- Understand roxygen2 package
- Learn roxygen2 syntax
- Create proper documentation
- Generate Rd files

## Theory

roxygen2 allows documenting R packages from special comments in source code. It generates the Rd file format used by R for help pages. Modern R packages use roxygen2 instead of manually writing documentation.

## Step-by-Step Guide

### Basic roxygen2 Comments

```r
#' Title (one line, sentence case)
#' 
#' Description (longer description).
#' More details here.
#'
#' @param x Description of parameter
#' @return Description of return value
#' @examples
#' mean(c(1, 2, 3))
#' @export
my_function <- function(x) {
  mean(x)
}
```

### Documenting Functions

```r
#' Calculate summary statistics
#'
#' @description
#' Calculates mean, median, and standard deviation.
#'
#' @param x A numeric vector
#' @param na.rm Remove NA values (default FALSE)
#' @return A data frame with statistics
#' @examples
#' summarize_data(c(1, 2, 3, NA), na.rm = TRUE)
#' @export
summarize_data <- function(x, na.rm = FALSE) {
  data.frame(
    mean = mean(x, na.rm = na.rm),
    median = median(x, na.rm = na.rm),
    sd = sd(x, na.rm = na.rm)
  )
}
```

### Tag Reference

```r
#' @param Name and description
#' @return What function returns
#' @examples Runable examples
#' @export Export function in NAMESPACE
#' @importFrom Import specific function
#' @import Import package
#' @keywords Keywords
#' @note Additional notes
#' @references References
#' @author Author information
#' @seealso See also
```

## Code Examples

### Documenting a Dataset

```r
#' \describe{
#'   \item{patient_id}{Patient identifier}
#'   \item{age}{Patient age}
#'   \item{outcome}{Treatment outcome}
#' }
#' @source Simulated data
patient_data <- data.frame(...)
```

### Documenting S3 Methods

```r
#' Print method for summary results
#'
#' @param x A summary_object
#' @param ... Additional arguments
#' @method print summary
#' @export
print.summary_object <- function(x, ...) {
  print(x$results)
}
```

### Using @importFrom

```r
#' @importFrom dplyr filter mutate select
#' @importFrom ggplot2 ggplot aes geom_bar

# The above generates imports in NAMESPACE
```

### Multi-Page Documentation

```r
#' @rdname (redefines name for linking)
#' @aliases (additional aliases)

# Multiple functions documented together
#' @rdname my_functions
#' @export
function1 <- function() {}

#' @rdname my_functions
#' @export  
function2 <- function() {}
```

## Best Practices

1. **Document All Exported Functions**: Everything exported should have docs.

2. **Run Examples**: Use roxygenise() to regenerate, then test examples.

3. **Use @export**: Only document functions that should be exported.

4. **Keep Title Short**: One line, not a complete sentence.

5. **Test Documentation**: Run ?function after build.
