# RMySQL: MySQL Database Connections

## Learning Objectives

- Connect to MySQL databases from R
- Execute SQL queries
- Read and write data to MySQL tables
- Handle connection pooling
- Manage database transactions

## Theory

RMySQL is an R interface to MySQL databases. It uses DBI (Database Interface) as a consistent interface for database operations. The package requires a MySQL server to be running and accessible.

Key functions: dbConnect() to establish connection, dbSendQuery() to execute queries, dbFetch() to retrieve results, and dbWriteTable() to write data. Connection parameters include host, port, username, password, and database name.

## Step-by-Step

1. Install and load RMySQL package
2. Create connection using dbConnect()
3. Execute queries with dbSendQuery()
4. Fetch results with dbFetch()
5. Write data with dbWriteTable()
6. Close connection with dbDisconnect()

## Code Examples

### Basic Connection

```r
cat("===== MYSQL CONNECTION =====\n\n")

library(RMySQL)
library(DBI)

# Connection parameters (adjust for your setup)
# Note: This requires a running MySQL server
con <- tryCatch({
  dbConnect(
    MySQL(),
    host = "localhost",
    port = 3306,
    user = "root",
    password = "",
    dbname = "test"
  )
}, error = function(e) {
  cat("Note: MySQL connection not available\n")
  cat("Error:", conditionMessage(e), "\n")
  NULL
})

if (!is.null(con)) {
  cat("Connected to MySQL\n")
  
  # List tables
  tables <- dbListTables(con)
  cat("Tables in database:", paste(tables, collapse = ", "), "\n")
  
  # Disconnect
  dbDisconnect(con)
  cat("Disconnected\n")
}
```

### Executing Queries

```r
cat("\n===== QUERY EXECUTION =====\n\n")

# Example query execution (without live database)
cat("MySQL Query Examples:\n\n")

# Standard query structure
cat("1. Simple SELECT:\n")
cat("query <- 'SELECT * FROM users WHERE active = 1'\n")
cat("result <- dbFetch(dbSendQuery(con, query))\n\n")

cat("2. Parameterized query (using dbExecute):\n")
cat("dbExecute(con, 'UPDATE users SET status = ? WHERE id = ?', \n")

cat("3. Multiple statements:\n")
cat("dbSendStatement(con, 'SET @total = 0')\n")

```

### Using DBI Interface

```r
cat("\n===== DBI INTERFACE =====\n\n")

cat("DBI provides consistent interface:\n\n")

cat("1. Connection:\n")
cat("con <- dbConnect(MySQL(), ...)\n\n")

cat("2. Queries:\n")
cat("rs <- dbSendQuery(con, 'SELECT * FROM table')\n")
cat("data <- dbFetch(rs, n = 100)  # Limit to 100 rows\n\n")

cat("3. Clear result:\n")
cat("dbClearResult(rs)\n\n")

cat("4. Check for errors:\n")
cat("# dbHasCompleted(rs)\n\n")

cat("5. Transaction:\n")
cat("# dbBegin(con)\n")
cat("# dbCommit(con)\n")
cat("# dbRollback(con)\n")
```

## Real-World Example: Data Pipeline

```r
# Real-world: ETL from CSV to MySQL
cat("===== DATA PIPELINE =====\n\n")

# This would be a typical pipeline
cat("ETL Pipeline Example:\n\n")

cat("# 1. Read from source\n")
cat("# source_data <- read.csv('sales.csv')\n\n")

cat("# 2. Transform data\n")
cat("# cleaned <- source_data %>%\n")
cat("#   mutate(date = as.Date(date),\n")
cat("#          amount = as.numeric(amount))\n\n")

cat("# 3. Write to MySQL\n")
cat("# con <- dbConnect(MySQL(), host = 'dbserver', ...)\n")
cat("# dbWriteTable(con, 'sales', cleaned,\n")
cat("#              overwrite = TRUE,\n")
cat("#              row.names = FALSE)\n")
cat("# dbDisconnect(con)\n\n")

cat("# 4. Read aggregated data back\n")
cat("# con <- dbConnect(MySQL(), host = 'dbserver', ...)\n")
cat("# agg <- dbGetQuery(con, '\n")
cat("#   SELECT DATE(date) as day, SUM(amount) as total\n")
cat("#   FROM sales GROUP BY DATE(date)\n")
cat("# ')\n")
cat("# dbDisconnect(con)\n")
```

## Best Practices

1. Always close connections in finally block or with on.exit()
2. Use connection pooling for frequent queries
3. Parameterize queries to prevent SQL injection
4. Use dbGetQuery() for simple queries that return results
5. Clear results after fetching with dbClearResult()
6. Handle connection errors gracefully
