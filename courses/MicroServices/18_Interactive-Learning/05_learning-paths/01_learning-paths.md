# Learning Paths for Microservices

## Overview

Learning paths provide structured curricula for acquiring microservices skills. They guide learners through a series of courses, labs, and assessments in a logical progression.

## Learning Paths

### Path 1: Microservices Developer

**Duration**: 8 weeks
**Prerequisites**: Basic programming experience

**Weeks 1-2: Fundamentals**
- Microservices architecture
- Domain-driven design basics
- API design principles

**Weeks 3-4: Implementation**
- Building REST APIs
- Database per service
- Service communication

**Weeks 5-6: Operations**
- Docker & Kubernetes
- CI/CD pipelines
- Monitoring & logging

**Weeks 7-8: Advanced**
- Security patterns
- Resilience patterns
- Real-world project

### Path 2: Platform Engineer

**Duration**: 6 weeks
**Prerequisites**: Kubernetes experience

**Focus Areas**:
- Service mesh
- Platform engineering
- GitOps
- Observability

### Path 3: Microservices Architect

**Duration**: 10 weeks
**Prerequisites**: Production microservices experience

**Focus Areas**:
- Architecture patterns
- Technology selection
- Migration strategies
- Performance optimization

## Assessment

Each path includes:
- Knowledge checks
- Hands-on labs
- Final project
- Certificate of completion
