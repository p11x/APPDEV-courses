# Microservices Glossary

## Overview

A comprehensive glossary of terms used in microservices architecture. This reference helps developers understand and communicate using consistent terminology.

## Key Terms

### A

**API Gateway**: A server that acts as an entry point for a set of microservices, handling request routing, composition, and protocol translation.

**Asynchronous Communication**: Communication where the client doesn't wait for a response, using message queues or event streams.

### B

**Bounded Context**: A boundary within which a particular domain model applies, defining explicit ownership of domain concepts.

**Bulkhead Pattern**: A resilience pattern that isolates resources to prevent failures from cascading.

### C

**Circuit Breaker**: A pattern that prevents repeated calls to a failing service, allowing it time to recover.

**Container Orchestration**: Automated management of containerized applications, including deployment, scaling, and networking.

### D

**Database per Service**: Each microservice owns its data store, with no shared database.

**Domain-Driven Design (DDD)**: An approach to software design that focuses on modeling the business domain.

### E

**Event Sourcing**: Storing the state of a business entity as a sequence of events rather than current state.

### F

**Fallback**: Alternative response when a service call fails.

### M

**Microservice**: An independently deployable service that owns a specific business capability.

### S

**Saga Pattern**: A sequence of local transactions where each transaction updates data within a single service.

**Service Mesh**: Infrastructure layer handling service-to-service communication.
