# Microservices Terminology

## Overview

Essential terminology for understanding and working with microservices architecture.

## Core Concepts

### Service
An independently deployable binary or container that provides a specific business capability.

### Service Boundary
The interface defining what functionality resides within a service and how other services interact with it.

### Service Catalog
A registry of available services with metadata for discovery.

### Service Discovery
The process of locating service instances dynamically.

### Service Mesh
Infrastructure layer providing service-to-service communication with cross-cutting concerns.

### Sidecar Pattern
Deploying auxiliary components alongside the main service container.

## Data Terms

### Event Sourcing
Storing state changes as a sequence of events.

### CQRS
Command Query Responsibility Segregation - separating read and write operations.

### Saga
Pattern for managing distributed transactions through a series of local transactions.

## Operational Terms

### Blue-Green Deployment
Running two identical environments and switching traffic between them.

### Canary Release
Gradually rolling out changes to a subset of users.

### GitOps
Using Git for infrastructure and application state management.
