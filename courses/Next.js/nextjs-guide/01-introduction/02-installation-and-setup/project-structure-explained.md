# Project Structure Explained

## What You'll Learn
- What each folder and file in a Next.js project does
- Why the App Router uses this particular structure
- How to organize your own files within this structure

## Prerequisites
- A created Next.js project (from the previous page)
- Basic understanding of files and folders

## Concept Explained Simply

When you first open a Next.js project, it can feel overwhelming. There are dozens of files with strange names like `layout.tsx`, `page.tsx`, and `next.config.ts`. Let's break it down so it makes sense.

Think of your Next.js project like a house. Each room (folder) has a specific purpose, and certain files have special powers that Next.js recognizes automatically. Just like how a front door has a specific meaning to visitors, certain filenames have special meaning to Next.js.

The most important concept is the `src/app` folder — this is where all your pages and layouts live. Each file in this folder becomes a route in your website automatically. It's like magic: create a file called `about.tsx` and suddenly you have an `/about` page!

## Key Folders Explained

### `src/app/` — The Heart of Your App

This is where you'll spend 90% of your time. Every file here becomes part of your website's URL.

| File/Folder | Purpose |
|-------------|---------|
| `page.tsx` | The main page (the "index" or home page) |
| `layout.tsx` | A wrapper that wraps every page (shared UI) |
| `globals.css` | Global styles that apply to everything |
| `not-found.tsx` | The 404 error page |
| `loading.tsx` | A loading screen while data fetches |
| `error.tsx` | An error boundary for catching crashes |

### `src/app/folder-name/` — Creating Routes

Creating a folder inside `app` creates a new URL path:

```
src/app/about/page.tsx     →  /about
src/app/contact/page.tsx   →  /contact
src/app/blog/page.tsx      →  /blog
```

### `public/` — Static Assets

Files here are served directly without processing. Put images, icons, and other static files here.

```
public/              →  serves at root (/)
public/logo.png      →  /logo.png
public/images/       →  /images/
```

### Configuration Files

| File | Purpose |
|------|---------|
| `package.json` | Lists your dependencies and scripts |
| `next.config.ts` | Configure Next.js behavior |
| `tsconfig.json` | TypeScript configuration |
| `tailwind.config.ts` | Tailwind CSS configuration |
| `.env.local` | Environment variables (secrets) |

## Complete Code Example

Here's what a typical project structure looks like after building a few pages:

```
my-first-app/
├── src/
│   ├── app/
│   │   ├── layout.tsx          ← wraps all pages
│   │   ├── page.tsx            ← home page (/)
│   │   ├── globals.css
│   │   ├── about/
│   │   │   └── page.tsx        ← /about
│   │   ├── blog/
│   │   │   ├── page.tsx        ← /blog (list)
│   │   │   └── [slug]/
│   │   │       └── page.tsx    ← /blog/any-slug
│   │   └── api/
│   │       └── hello/
│   │           └── route.ts    ← /api/hello
│   └── components/
│       ├── Header.tsx
│       └── Footer.tsx
├── public/
│   ├── images/
│   └── favicon.ico
├── package.json
├── tsconfig.json
├── next.config.ts
└── tailwind.config.ts
```

Let's look at the root layout file:

```typescript
// src/app/layout.tsx
import type { Metadata } from "next";
import { Inter } from "next/font/google";
import "./globals.css";

const inter = Inter({ subsets: ["latin"] });

export const metadata: Metadata = {
  title: "My First App",
  description: "Generated by create-next-app",
};

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body className={inter.className}>{children}</body>
    </html>
  );
}
```

## Line-by-Line Breakdown

| Line / Block | What It Does | Why It Matters |
|---|---|---|
| `import type { Metadata }` | Imports TypeScript types for metadata | Used for SEO - page titles, descriptions |
| `import { Inter } from "next/font/google"` | Imports Google Font | Optimizes font loading automatically |
| `import "./globals.css"` | Imports global CSS | Applies styles to all pages |
| `const inter = Inter({ subsets: ["latin"] })` | Configures the font | Sets up optimized font loading |
| `export const metadata` | Defines page metadata | Controls SEO and social sharing |
| `export default function RootLayout` | The root layout component | Wraps every page in your app |
| `<html lang="en">` | Sets HTML language | Important for accessibility |
| `{children}` | Renders the current page | This is where your page content appears |

## Common Mistakes

### Mistake #1: Putting Everything in the Root

Don't put all your components and utilities in the `app` folder. Use a separate `components` folder:

```
✓ src/components/Header.tsx
✗ src/app/Header.tsx (not a page, won't work)
```

### Mistake #2: Confusing page.tsx with layout.tsx

- `page.tsx` = the actual content of a route
- `layout.tsx` = a wrapper around pages (navigation, footer, etc.)

### Mistake #3: Using the Pages Router

In Next.js 14/15, always use the App Router. Never create a `pages/` folder at the root level.

## Summary

- The `src/app` folder is where all your routes live
- Special files like `page.tsx`, `layout.tsx`, `error.tsx` have built-in meaning
- Folders inside `app` create URL paths automatically
- Put reusable components in a separate `components` folder
- `public/` is for static files like images
