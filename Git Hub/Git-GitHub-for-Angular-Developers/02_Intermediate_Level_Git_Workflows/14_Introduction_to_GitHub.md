# Introduction to GitHub

## Topic Title
Understanding GitHub - The World's Largest Code Hosting Platform

## Concept Explanation

GitHub is a cloud-based platform built on top of Git that provides additional features for collaboration, project management, and code hosting. It has become the de facto standard for hosting open-source and private software projects.

### What is GitHub?

GitHub is a web-based platform for version control using Git. It offers:
- Cloud hosting for Git repositories
- Collaboration tools (issues, pull requests)
- Project management features
- Code review tools
- CI/CD integration
- Social networking for developers

### History of GitHub

- **2005**: Git was created by Linus Torvalds
- **2008**: GitHub was founded
- **2011**: GitHub reached 1 million repositories
- **2018**: Microsoft acquired GitHub
- **2020**: GitHub reached 50 million developers
- **Today**: Over 100 million repositories

### Why GitHub is Important for Angular Developers

1. **Open Source Angular**: Angular itself is hosted on GitHub
2. **Libraries & Packages**: Most Angular libraries are on GitHub
3. **Community**: Huge Angular community and resources
4. **CI/CD**: GitHub Actions for Angular deployments
5. **Portfolio**: Showcase your Angular projects

## GitHub Features Overview

### 1. Repositories

Repositories (repos) are where your project code lives:

```
github.com/username/repository
```

Repository types:
- **Public**: Anyone can see
- **Private**: Only authorized users can see
- **Forked**: Copy of another user's repository

### 2. Branches

GitHub provides visual branch management:

- View all branches
- Compare branches
- Create pull requests

### 3. Commits

Visual commit history:

- See who made changes
- View what changed
- Track when changes were made

### 4. Pull Requests

The heart of collaboration:

- Propose changes
- Request code review
- Discuss changes
- Merge approved changes

### 5. Issues

Project tracking:

- Bug reports
- Feature requests
- Tasks
- Discussions

### 6. Actions

CI/CD automation:

- Automated testing
- Build deployments
- Custom workflows

## GitHub Terminology

### Common Terms

| Term | Definition |
|------|------------|
| Repository | Project's code storage |
| Fork | Personal copy of someone's repo |
| Clone | Local copy of a repo |
| Branch | Parallel version of code |
| Pull Request | Propose changes to merge |
| Issue | Track bugs/tasks |
| Merge | Combine branch changes |
| Commit | Saved change |
| Push | Upload to remote |
| Pull | Download from remote |
| README | Project description |
| .gitignore | Files to ignore |

## Getting Started with GitHub

### Step 1: Create Account

1. Go to github.com
2. Click "Sign up"
3. Choose username (professional!)
4. Verify email

### Step 2: Explore

- Browse popular repositories
- Star (bookmark) interesting projects
- Follow developers

### Step 3: Create Repository

1. Click "+" → "New repository"
2. Choose name
3. Add description
4. Choose public/private
5. Add README
6. Click "Create repository"
